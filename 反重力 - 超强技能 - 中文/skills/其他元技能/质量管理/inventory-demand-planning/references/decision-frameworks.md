# 决策框架 — 库存需求规划

本参考文档提供多门店零售商库存需求规划的详细决策逻辑、优化模型、方法选择树和分段方法论。当智能体需要制定或推荐精细化规划决策时，按需加载此文档。

所有阈值、公式和成本假设均反映美国多门店零售运营场景，管理数百个 SKU，涵盖食品杂货、一般商品、季节性和促销品类。

---

## 1. 预测方法选择树

### 1.1 主选择算法

目标是为每个 SKU 匹配在时间外验证集上 WMAPE 最小的预测方法。实际上，大多数组织无法对数百个商品逐一进行模型优化。更可行的做法是将商品按需求模式原型分类，然后按原型分配方法。

#### 步骤1 — 分类需求模式

在最近 52 周的去促销需求数据上计算以下统计量（计算前移除促销提升期）：

| 统计量 | 公式 | 用途 |
|---|---|---|
| **变异系数（CV）** | σ_demand / μ_demand | 衡量需求变异性 |
| **平均需求间隔（ADI）** | 总期数 / 非零需求期数 | 衡量间歇性 |
| **趋势强度** | 26 周滚动需求线性回归的 R² | 衡量方向性运动 |
| **季节性强度** | 滞后 52（周度）或滞后 12（月度）的自相关 | 衡量重复性季节模式 |
| **零需求比率** | 零需求期数 / 总期数 | 衡量需求缺失频率 |

#### 步骤2 — 映射到需求原型

| 原型 | CV | ADI | 趋势 R² | 季节性 AC | 零比率 | 示例 |
|---|---|---|---|---|---|---|
| **平稳型** | < 0.5 | 1.0–1.1 | < 0.3 | < 0.3 | < 5% | 牛奶、面包、纸巾 |
| **趋势型** | < 0.7 | 1.0–1.2 | ≥ 0.3 | < 0.3 | < 10% | 增长中的品牌、衰退的老品 |
| **季节型** | 0.3–1.0 | 1.0–1.3 | 任意 | ≥ 0.3 | < 15% | 防晒霜、节日装饰、烤炉 |
| **趋势季节型** | 0.4–1.2 | 1.0–1.3 | ≥ 0.3 | ≥ 0.3 | < 15% | 增长中的季节品类 |
| **波动型** | ≥ 0.7 | 1.0–1.5 | < 0.3 | < 0.3 | < 30% | 时尚配饰、新奇商品 |
| **间歇型** | 任意 | ≥ 1.5 | 任意 | 任意 | ≥ 30% | 备件、特殊食材 |
| **块状型** | ≥ 1.0 | ≥ 1.5 | 任意 | 任意 | ≥ 30% | 零星订购的批量批发商品 |

#### 步骤3 — 分配预测方法

| 原型 | 主要方法 | 参数 | 备选方法 |
|---|---|---|---|
| **平稳型** | 加权移动平均（4–8 周窗口，近期加权） | 权重：4 周为 0.4/0.3/0.2/0.1 | 单指数平滑（α = 0.15–0.25） |
| **趋势型** | Holt 双指数平滑 | α = 0.2–0.4, β = 0.05–0.15 | 26 周滚动线性回归 |
| **季节型** | Holt-Winters（幅度稳定用加法，幅度增长用乘法） | α = 0.1–0.3, β = 0.01–0.05, γ = 0.1–0.3, 周期 = 52 周 | STL 分解 + 对残差做 SES |
| **趋势季节型** | Holt-Winters（乘法） | α = 0.2–0.4, β = 0.05–0.15, γ = 0.15–0.3 | X-13ARIMA-SEATS |
| **波动型** | 阻尼趋势指数平滑 | α = 0.2–0.4, β = 0.05, φ = 0.8–0.95 | 3 种方法集成（取中位数） |
| **间歇型** | Croston 方法或 SBA | α_demand = 0.1–0.2, α_interval = 0.1–0.2 | Bootstrap 模拟（1000 次抽样） |
| **块状型** | SBA（Syntetos-Boylan 近似） | 与 Croston 相同但含偏差校正 | 先聚合到月度再拆分 |

### 1.2 模型切换规则

不要因为单周表现差就切换方法。模型需要时间来证明或否定自己。

| 条件 | 操作 | 最短观察期 |
|---|---|---|
| WMAPE 在验证集上比当前方法改善 > 10% | 切换到候选方法 | 8 周并行测试 |
| 跟踪信号连续 2 期超过 ±4 | 触发模型审查；先重新估算参数 | 2 期（周） |
| 跟踪信号 1 期超过 ±6 | 立即模型审查；可能原型已变化 | 1 期 |
| 需求模式原型发生变化（如平稳→趋势） | 从步骤 1 重新运行选择算法 | 季度原型重评估 |
| 新品从类比数据过渡到自有历史 | 当自有数据 ≥ 12 周且自有 WMAPE < 类比 WMAPE 时切换 | 12 周 |
| 检测到促销后基线污染 | 排除促销期重新拟合基线模型 | 立即 |

### 1.3 参数优化协议

对于指数平滑方法，使用网格搜索在时间序列交叉验证（滚动起点、1 步超前预测、26+ 起点）上优化参数。

**网格搜索范围：**

| 参数 | 范围 | 步长 | 约束 |
|---|---|---|---|
| α（水平） | 0.05–0.50 | 0.05 | — |
| β（趋势） | 0.01–0.20 | 0.01 | β ≤ α |
| γ（季节） | 0.05–0.40 | 0.05 | — |
| φ（阻尼） | 0.80–0.98 | 0.02 | 仅用于阻尼方法 |

**优化指标：** 最小化验证集上的 WMAPE。如果两组参数的 WMAPE 差距在 1 个百分点以内，选择 α 较低（更平滑）的那组以提高稳定性。

**过拟合防护：** 如果优化后的模型在验证集上的 WMAPE 比拟合数据好 > 5 个百分点，模型可能过拟合。增大平滑度（降低 α）直到差距缩小到 < 3 个百分点。

## 2. 安全库存优化模型

### 2.1 标准安全库存（正态需求、固定前置时间）

当需求近似正态分布且前置时间一致时：

```
SS = Z × σ_d × √(LT)
```

其中：
- Z = 目标服务水平对应的 z 分数（见下方查找表）
- σ_d = 每期需求标准差（使用与 LT 相同的期度）
- LT = 前置时间（以期为单位）

**Z 分数查找表：**

| 服务水平 | Z 分数 | 典型应用 |
|---|---|---|
| 85.0% | 1.04 | CZ 商品 — 最低投入 |
| 90.0% | 1.28 | C 类商品、非关键 B 类商品 |
| 92.0% | 1.41 | 中等安全网 |
| 95.0% | 1.65 | A/B 类商品标准目标 |
| 97.5% | 1.96 | AX 商品 — 高价值、可预测 |
| 99.0% | 2.33 | 关键商品 — 缺货成本远超持有成本 |
| 99.5% | 2.58 | 生命安全或合同义务商品 |
| 99.9% | 3.09 | 极少有理由使用 — 极端持有成本 |

### 2.2 含前置时间变异性的安全库存

当前置时间不确定时（前置时间 CV > 0.15）：

```
SS = Z × √(LT_avg × σ_d² + d_avg² × σ_LT²)
```

其中：
- LT_avg = 平均前置时间（以期为单位）
- σ_LT = 前置时间标准差（以期为单位）
- d_avg = 每期平均需求

**实务提示：** 许多规划师低估了前置时间变异性，因为他们仅衡量"供应商发货日到配送中心收货"，而未考虑收货延迟、质量留置或周末/节假日死时间。应从 PO 下达到"可供销售"来衡量前置时间 — 这才是运营上相关的指标。

### 2.3 含审查周期的安全库存

对于定期审查系统（每 R 期审查一次）：

```
SS = Z × σ_d × √(LT + R)
```

审查周期增加了风险暴露时间 — 在两次审查之间，你无法对需求变化做出反应。对于前置时间 2 周的商品，每周审查（R=1）需要 √3 周的安全库存。同一商品月度审查（R=4）需要 √6 周的安全库存 — 多出 41%。

### 2.4 间歇需求的安全库存

当需求有大量零值期时，正态分布公式失效。应改用经验（bootstrap）安全库存：

1. 收集最近 52 期的需求数据（包含零值）。
2. 从历史需求中以有放回随机抽样方式生成 10,000 个长度为（LT + R）的 bootstrap 样本。
3. 计算每个 bootstrap 样本的总和（= 前置时间 + 审查期的模拟需求）。
4. 安全库存 = 模拟需求总量的（服务水平）百分位数减去模拟需求总量均值。

**示例：** 对于 95% 服务水平，安全库存 = bootstrap 需求的 P95 − bootstrap 需求均值。这捕捉了参数公式无法捕捉的偏态、零膨胀分布。

### 2.5 新品安全库存（无历史数据）

当商品自有需求历史 < 8 周时：

1. 识别 3–5 个类比商品，匹配标准：品类、价格点（±20%）、品牌层级（全国品牌/自有品牌）、包装规格和目标人群。
2. 计算类比商品的平均 σ_d 和 CV。
3. 对类比 σ_d 施加 1.25× 的"新品不确定性溢价"。
4. 使用膨胀后的 σ_d 代入标准公式：`SS = Z × (1.25 × σ_d_analog) × √(LT + R)`。
5. 每 2 周，将自有历史 σ_d 与类比 σ_d 混合。到第 8 周使用 70% 自有历史 + 30% 类比。到第 12 周使用 100% 自有历史。

### 2.6 安全库存成本优化

朴素做法是设定服务水平目标然后计算 SS。高级做法是优化持有成本与缺货成本之间的权衡：

```
Optimal SL = 1 − (H / (H + S × D/Q))
```

其中：
- H = 每期每单位持有成本
- S = 每单位缺货成本（损失利润 + 客户商誉 + 替代成本）
- D = 每期需求
- Q = 订货量

对于大多零售商，A 类商品的缺货成本为单位利润的 3–5 倍（包括损失客流和替代效应），推算最优服务水平为 96–98%。C 类商品的缺货成本约等于单位利润，最优服务水平为 88–92%。

## 3. 促销规划框架

### 3.1 促销提升估算方法

促销提升始终相对于基线预测（即不考虑促销时本应生成的预测）计算。将促销历史污染基线是零售业系统性预测误差的头号来源。

#### 步骤1 — 建立清洁基线

在拟合基线模型前，从需求历史中剥离促销期。若以下任一条件生效，将该周标记为"促销"：
- 临时降价（TPR）超过正常价格 5%
- 在传单、数字广告或端架展示中出现
- 买一赠一或多买优惠
- 在其他品类中的交叉促销

剥离后，使用仅基于非促销期拟合的预测模型对空缺进行插值。这创建了"如果按正常价格销售会怎样"的基线。

#### 步骤2 — 计算历史提升

对于该 SKU 的每个历史促销事件：

```
Lift Ratio = Actual Promo-Period Sales / Baseline Forecast for Promo Period
```

提升比为 2.5 意味着促销驱动了 2.5 倍的基线销量（150% 增量）。

按促销机制整理提升比：

| 机制 | 典型提升范围 | 关键驱动因素 |
|---|---|---|
| 仅 TPR（5–15% 折扣） | 1.15–1.40 | 折扣深度、品类弹性 |
| TPR（15–30% 折扣） | 1.40–2.00 | 更深折扣带来更强烈反应 |
| TPR + 展示 | 1.80–2.50 | 展示位置（端架 > 翼架 > 线内） |
| TPR + 传单广告 | 2.00–3.00 | 传单覆盖面和版面（封面 > 内页） |
| TPR + 展示 + 传单 | 2.50–4.00 | 全面支持 — 即"A 级"促销 |
| 买一赠一 | 2.50–5.00 | 感知价值驱动高反应但大量提前购买 |
| 限时抢购 / 亏本引流品 | 3.00–6.00+ | 引流工具；提升因活动差异极大 |

#### 步骤3 — 将提升应用于当前预测

```
Promo Forecast = Baseline Forecast × Lift Ratio
```

当多种促销机制组合时，不要将各自提升相乘。应使用历史数据中的组合机制提升或上表中的数据。交互效应是次可加的（单展示 = 1.5× 且单传单 = 1.8× 不代表展示 + 传单 = 2.7×；通常为 2.0–2.5×）。

#### 步骤4 — 建模促销后低谷

```
Post-Promo Demand = Baseline × (1 − Dip Factor × Decay)
```

按产品类型的默认低谷因子：

| 产品类型 | 低谷因子（占增量提升的%） | 低谷持续时间 | 衰减模式 |
|---|---|---|---|
| **常温储藏食品** | 40–60% | 2–4 周 | 60/30/10（第1/2/3周） |
| **易腐/冷藏品** | 10–20% | 0–1 周 | 立即恢复 |
| **家庭消耗品** | 30–50% | 2–3 周 | 50/35/15 |
| **个人护理** | 25–40% | 2–3 周 | 50/30/20 |
| **季节性** | 15–30% | 1–2 周 | 70/30 |
| **可选品/一般商品** | 10–25% | 1–2 周 | 70/30 |

### 3.2 蚕食效应估算

当 SKU A 被促销时，可替代的 SKU B 会损失销量。蚕食率为：

```
Cannibalization Rate = ΔB_down / ΔA_up
```

其中 ΔA_up 是 A 的增量提升，ΔB_down 是同期 B 的销量损失。

**无交叉弹性数据时的默认估算：**

| 替代程度 | 蚕食率 | 示例 |
|---|---|---|
| 直接替代（同品牌不同规格） | 25–40% | 12 盎司促销，16 盎司损失 |
| 近似替代（不同品牌同细分） | 15–25% | 全国品牌促销，自有品牌损失 |
| 中度替代（同品类不同细分） | 5–15% | 高端促销，价值层受影响 |
| 弱替代（相邻品类） | 0–5% | 薯片促销，饼干略微受影响 |

**重要提示：** 蚕食效应在整个品类中是双向的。构建品类级促销计划时，需汇总所有替代品的蚕食效应来计算真实的品类级提升（这始终低于单品级提升）。

### 3.3 提前购买与囤货行为

深度折扣促销会导致客户提前于消费计划购买。提前购买量是从未来期间拉来的需求，而非增量品类需求。

**提前购买估算：**

```
Forward-Buy Volume = Incremental Lift × Forward-Buy Factor
```

| 促销深度 | 产品保质期 | 提前购买因子 |
|---|---|---|
| 10–20% 折扣 | < 2 周（易腐） | 0.05–0.10 |
| 10–20% 折扣 | 2–12 周 | 0.10–0.20 |
| 10–20% 折扣 | > 12 周（常温） | 0.20–0.35 |
| 20–35% 折扣 | < 2 周 | 0.05–0.15 |
| 20–35% 折扣 | 2–12 周 | 0.20–0.35 |
| 20–35% 折扣 | > 12 周 | 0.35–0.50 |
| 35–50% 折扣 | < 2 周 | 0.10–0.20 |
| 35–50% 折扣 | 2–12 周 | 0.30–0.45 |
| 35–50% 折扣 | > 12 周 | 0.50–0.70 |
| 买一赠一 / > 50% | 任意 | 0.50–0.80 |

提前购买因子告诉你增量提升中有多少来自囤货而非真实消费增长。这直接用于促销后低谷计算 — 低谷约等于提前购买量分摊到其消费周期内。

### 3.4 促销日历规划

规划年度促销日历时，应用以下规则：

1. **最小促销间隔：** 同一 SKU 两次促销之间至少间隔 4 周。更短的间隔会训练客户只在促销时购买，侵蚀品牌资产和基线周转率。
2. **最大促销频率：** 任何单一 SKU 每年不超过 13 周（25%）。超过此限，"促销价"会在消费者心中成为参考价。
3. **季节性对齐：** 在季节建设期（前 40%）促销季节商品，而非在高峰期或衰退期。在高峰期促销浪费了本就会发生的销量。在衰退期促销是降价清仓，不是促销。
4. **跨品类协调：** 避免同时促销近似替代品。替代品之间的促销至少错开 2 周，以避免自我蚕食。
5. **供应商资金对齐：** 将促销时间与供应商贸易资金可用期匹配。许多 CPG 制造商按日历季度运营 — 季度末未承诺的资金将过期。在每季度第 8–12 周规划关键促销，此时供应商有动力花掉剩余资金。

## 4. ABC/XYZ 分段方法论

### 4.1 ABC 分类（价值）

ABC 分类按财务贡献对 SKU 进行分段。该分类驱动在预测投入、安全库存、审查频率和管理关注上的差异化投资。

#### 分类程序

1. **选择价值指标。** 优先顺序如下：
   - 毛利贡献（最佳 — 将投入聚焦于利润而非收入）
   - 收入（当利润数据不可用时可接受）
   - 件数（仅用于仓库空间规划，不用于财务投入）

2. **计算每个活跃 SKU 的滚动 52 周价值。**

3. **按价值指标降序排列。**

4. **计算累计价值百分比** 并分类：

| 等级 | 累计价值百分比 | 典型 SKU 占比 | 描述 |
|---|---|---|---|
| A | 0–80% | 10–20% | 驱动业务的高价值商品 |
| B | 80–95% | 20–30% | 提供品类广度的中价值商品 |
| C | 95–100% | 50–70% | 单个影响极小的长尾商品 |

5. **例外覆盖：**
   - 新品（< 13 周）在拥有足够历史之前，自动比数据建议的等级高一级。计算结果为 C 的新品视为 B。
   - 有合同义务（陈列承诺、供应商协议）的商品，无论当前销售速度如何，最低归为 B。
   - 被商品管理部门标记为战略性商品（如引流商品、竞争比价商品）最低归为 A。

#### 重分类计划

每季度运行 ABC 重分类。季度之间，商品仅在跨越阈值 > 50% 时才重分类（例如商品必须贡献超过 A/B 分界线的 120% 才能在季中从 B 升为 A）。这防止了等级边界处的振荡。

### 4.2 XYZ 分类（可预测性）

XYZ 分类按需求预测难度对 SKU 进行分段。它驱动预测方法选择和安全库存策略的差异化。

#### 分类程序

1. **计算每个 SKU 在滚动 52 周内的去季节化、去促销需求。** 移除季节指数和促销提升期，使变异性指标反映真实的需求不确定性，而非计划性变动。

2. **计算变异系数（CV）：**
   ```
   CV = σ_demand / μ_demand
   ```
   使用去季节化、去促销后的需求序列。

3. **分类：**

| 等级 | CV 范围 | 描述 | 预测难度 |
|---|---|---|---|
| X | < 0.5 | 高度可预测 — 需求在均值附近波动很小 | 低 — 简单方法即可 |
| Y | 0.5–1.0 | 中度可预测 — 有明显变异性 | 中 — 需要好的模型和监控 |
| Z | > 1.0 | 波动/块状 — 需求高度可变或间歇 | 高 — 任何模型都难以高度准确 |

4. **用 ADI（平均需求间隔）补充：** ADI > 2.0（即需求发生频率低于每隔一期）的商品应直接归为 Z，无论 CV 如何，因为间歇性本身带来的预测难度是 CV 无法单独捕捉的。

### 4.3 组合 ABC/XYZ 策略矩阵

| 分段 | 预测方法 | 安全库存 | 审查频率 | 补货方式 | 管理关注 |
|---|---|---|---|---|---|
| **AX** | 指数平滑（自动化） | Z = 1.96（97.5%） | 每周 | 自动化并设异常告警 | 月度审查 |
| **AY** | Holt-Winters 或因果模型 | Z = 1.65（95%） | 每周 | 自动化并由规划师审查 | 双周审查 |
| **AZ** | 集成模型或人工覆盖 | Z = 1.41–1.65（92–95%） | 每周 | 规划师管理；永不完全自动化 | 每周审查 |
| **BX** | 移动平均（自动化） | Z = 1.65（95%） | 双周 | 自动化 | 月度审查 |
| **BY** | 指数平滑（自动化） | Z = 1.65（95%） | 双周 | 自动化并设异常告警 | 月度审查 |
| **BZ** | Croston 或阻尼趋势 | Z = 1.28（90%） | 双周 | 半自动化并需规划师审批 | 月度审查 |
| **CX** | 简单移动平均 | Z = 1.28（90%） | 每月 | 自动化 | 季度审查 |
| **CY** | 简单移动平均 | Z = 1.28（90%） | 每月 | 自动化 | 季度审查 |
| **CZ** | Croston 或无 | Z = 1.04（85%） | 每月 | 手动或最小/最大 | 季度 — 停售候选 |

### 4.4 迁移追踪

每季度追踪 SKU 在各分段之间的移动。需关注的关键迁移模式：

| 迁移 | 信号 | 操作 |
|---|---|---|
| A → B | 收入或利润下降 | 调查：是品类萎缩、竞争损失还是品类问题？ |
| B → A | 收入或利润增长 | 升级预测方法和审查频率。验证安全库存。 |
| X → Y 或 Z | 需求变得更不可预测 | 检查需求模式是否发生结构性变化。审查预测模型拟合度。增加安全库存。 |
| Z → X 或 Y | 需求趋于稳定 | 可能简化预测模型。审查安全库存是否可以缩减。 |
| 任意 → CZ | 低价值 + 波动 | 强烈的停售候选。执行慢动品淘汰决策。 |

## 5. 供应商管理决策逻辑

### 5.1 供应商层级分类

根据年采购额、战略重要性和供应风险特征将供应商分为不同层级：

| 层级 | 标准 | 数量（典型） | 审查频率 |
|---|---|---|---|
| **战略级** | 按支出排名前 5，或 A 类商品唯一供应商 | 3–8 | 月度评分卡、季度业务回顾 |
| **优选级** | 按支出排名前 20，多个 A/B 类商品 | 10–25 | 季度评分卡 |
| **合格级** | 所有其余活跃供应商 | 30–100+ | 年度审查 |
| **观察级** | 正在整改中的供应商 | 不定期 | 周度监控、月度审查 |

### 5.2 供应商评分卡指标

每季度对每个供应商按 0–100 分在以下维度进行评分：

| 维度 | 权重 | 指标 | 目标 | 计算方式 |
|---|---|---|---|---|
| **准时交付** | 30% | 在约定窗口内（±1 天）交付的 PO 占比 | > 95% | 分数 = (实际% / 95%) × 100，上限 100 |
| **满足率** | 25% | 实际发货的订单件数占比 | > 97% | 分数 = (实际% / 97%) × 100，上限 100 |
| **前置时间一致性** | 20% | 实际前置时间与声明前置时间的 CV | CV < 0.15 | 分数 = max(0, 100 − (CV − 0.15) × 500) |
| **质量** | 15% | 收货通过质检的件数占比 | > 99% | 分数 = (实际% / 99%) × 100，上限 100 |
| **响应速度** | 10% | 咨询/问题的平均响应时间（小时） | < 24 小时 | 分数 = max(0, 100 − (平均小时数 − 24) × 2) |

**综合评分阈值：**

| 评分范围 | 评级 | 操作 |
|---|---|---|
| 90–100 | 优秀 | 考虑增加采购量、给予优选条款 |
| 75–89 | 良好 | 正常运营，无需操作 |
| 60–74 | 需改进 | 发出整改要求；90 天改进计划 |
| < 60 | 不可接受 | 立即升级；开始认证替代供应商 |

### 5.3 供应商前置时间管理

前置时间管理是需求规划师降低库存投资的最未充分利用的工具。将所有供应商的前置时间缩短 1 天，可减少 5–8% 的总安全库存。

**前置时间分解：**

| 环节 | 典型范围 | 规划师影响力 |
|---|---|---|
| 供应商订单处理 | 1–3 天 | 低 — 供应商内部流程 |
| 生产/拣货 | 2–10 天 | 中 — 协商优先层级 |
| 供应商发货准备 | 1–2 天 | 低 |
| 运输时间 | 1–14 天 | 中 — 承运商选择、运输方式 |
| 收货与上架 | 1–3 天 | 高 — 内部流程改进 |
| 质量留置（如适用） | 0–5 天 | 高 — 精简质检流程 |

**缩短前置时间的措施：**

1. 对战略供应商：协商 VMI（供应商管理库存），供应商监控你的库存并主动发货。消除订单处理延迟。
2. 对所有供应商：提供滚动 8 周预测以允许预定位。减少非库存商品的生产/拣货时间。
3. 内部：投资收货自动化（ASN 收货、条码扫描）将收货时间从 2–3 天缩短至当日。
4. 协商每周合并发货替代逐单发货，在保持满足率的同时降低运输频率。

### 5.4 MOQ（最小订货量）协商框架

当供应商的 MOQ 造成多余库存时，按以下顺序评估选项：

| 选项 | 适用时机 | 预期结果 |
|---|---|---|
| **协商降低 MOQ** | 该供应商年采购额 > $50K；你有议价力 | MOQ 降低 20–40% |
| **与其他 SKU 合并** | 同一供应商有多个 SKU；改为金额最低而非件数最低 | 达到金额 MOQ 而不过量订购单个 SKU |
| **接受更高价格换取更低 MOQ** | MOQ 超量成本 > 价格溢价成本 | 每件多付 3–8% 但只订所需数量 |
| **协商寄售** | 战略供应商的慢动品 | 供应商持有库存直到你售出 |
| **与其他买家分单** | 已知有其他零售商向同一供应商下单 | 共享 MOQ 并分摊发货 |
| **接受超量** | 超量的持有成本 < $500 且商品非易腐 | 按 MOQ 订货，将超量视为前移库存 |

### 5.5 前置时间缩短的供应商协商

**协商前准备清单：**

1. 记录你当前与该供应商的订单量和增长轨迹。
2. 计算其当前前置时间对你业务的成本：多余安全库存持有成本 + 前置时间变异性导致的缺货成本。
3. 确定你能提供什么回报：更长期承诺、更大采购量、更少的订单频率变动、滚动预测。
4. 了解你的 BATNA（最佳替代方案）：已认证的备选供应商。

**协商结构：**

1. 呈现数据："过去 6 个月，你们的平均前置时间为 X 天，标准差为 Y。这种变异性每年给我们带来 $Z 的多余安全库存成本。"
2. 提出目标："我们要求承诺 X−2 天的前置时间，并保证 CV < 0.15。"
3. 提出交换条件："作为回报，我们可以承诺每周更新的滚动 8 周预测，并将从每日下单合并为每周 2 次。"
4. 设定时间表："让我们在 Q2 实施，在 Q2 季度末 QBR 上审查评分卡。"

## 6. 季节性采购与降价时机模型

### 6.1 季节性采购规划

季节性品类需要前瞻承诺，因为前置时间超过销售期。采购决策有两个组成部分：初始采购（季前）和季中补货（如果供应商支持）。

#### 初始采购计算

```
Initial Buy = Season Forecast × Initial Commitment % − Carry-Over Inventory
```

| 品类风险特征 | 初始承诺比例 | 补货预留 | 理由 |
|---|---|---|---|
| 低风险（主力季节品、已验证畅销品） | 70–80% | 20–30% | 高预测信心；有补货可用 |
| 中等风险（受趋势影响、中等历史） | 55–65% | 35–45% | 对冲预测误差 |
| 高风险（时尚品、新趋势、首季） | 40–50% | 50–60% | 最大化灵活性；接受可能缺货 |
| 一次性采购（进口、长前置、无补货） | 100% | 0% | 无补货选项；完全承诺但保守预测 |

#### 季中补货触发

从季节第 2 周开始每周监控售罄率：

```
Sell-Through Rate = Units Sold / (Units Sold + Units On-Hand + Units On-Order)
```

| 季节进行周数 | 售罄率 vs. 计划 | 操作 |
|---|---|---|
| 第 1–2 周 | > 计划的 120% | 立即下达补货单，为预留分配的 50% |
| 第 1–2 周 | 计划的 80–120% | 持有；为时过早无法确认趋势 |
| 第 3–4 周 | > 计划的 110% | 下达剩余预留的补货单 |
| 第 3–4 周 | 计划的 90–110% | 下达保守补货（预留的 25%） |
| 第 3–4 周 | 计划的 70–89% | 持有所有预留；准备降价应急方案 |
| 第 3–4 周 | < 计划的 70% | 取消所有未完成补货；启动提前降价 |
| 第 5 周+ | 任意进度 | 补货不太可能及时到达；用降价管理 |

### 6.2 降价时机与深度模型

降价决策在利润回收和售罄速度之间取得平衡。每延迟一周都消耗利润，因为持有成本在累积而剩余销售窗口在缩小。

#### 降价决策矩阵

| 季节剩余周数 | 按当前速度的供应周数 | 建议操作 |
|---|---|---|
| > 6 周 | < 3 | 不降价；可能补货 |
| > 6 周 | 3–6 | 维持价格；每周监控 |
| > 6 周 | 7–10 | 首次降价：20–25% 折扣 |
| > 6 周 | > 10 | 激进降价：30–40% 折扣 |
| 4–6 周 | < 3 | 无需降价 |
| 4–6 周 | 3–6 | 考虑 15–20% 降价 |
| 4–6 周 | 6–10 | 降价 25–35% |
| 4–6 周 | > 10 | 降价 40–50%；探索清仓渠道 |
| 2–4 周 | < 3 | 不降价 |
| 2–4 周 | 3–6 | 降价 30–40% |
| 2–4 周 | > 6 | 降价 50–60%；清仓渠道 |
| < 2 周 | 任意剩余 | 最终清仓 60–75% 折扣或清仓处理 |

#### 降价速度曲线

降价后监控速度响应：

| 降价深度 | 预期速度提升 | 若 1 周内未达到 |
|---|---|---|
| 20% 折扣 | 1.5–2.0× | 加深至 30% |
| 30% 折扣 | 2.0–3.0× | 加深至 40% |
| 40% 折扣 | 3.0–4.0× | 加深至 50% 或探索清仓 |
| 50% 折扣 | 4.0–6.0× | 若仍不动销，即为死库存 — 清仓处理 |

### 6.3 季末清仓决策

当销售季即将结束但仍有库存时：

```
Liquidation Net Recovery = (Liquidation Price × Remaining Units) − Logistics Cost
Hold-to-Next-Season Net = (Expected Sell Price × Sell-Through Estimate) 
                          − Holding Cost − Obsolescence Risk
```

**优先清仓的条件：**
- 持有至下一季的售罄率估算 < 60%（款式风险、趋势变化）
- 持有 9–12 个月的成本 > 原始成本的 15%（大多零售商的典型值）
- 仓库空间紧张且该空间有更高价值的替代用途
- 产品为趋势/时尚类，下季会显过时

**优先持有的条件：**
- 产品为经典/延续款式，时尚风险极小
- 持有至下一季的售罄率估算 > 80%
- 仓库空间可用且边际成本低
- 清仓报价低于变动成本（卖了反而亏钱）

## 7. 新品上市预测

### 7.1 类比商品选择

新品预测的质量几乎完全取决于所选类比商品的质量。糟糕的类比商品无论用什么方法都会产生糟糕的预测。

#### 选择标准（按重要性排序）

| 标准 | 权重 | 匹配方式 |
|---|---|---|
| **品类/子品类** | 25% | 必须是同一子品类（如"高端酸奶"而非仅"乳制品"） |
| **价格点** | 20% | 在新品零售价的 ±20% 范围内 |
| **品牌层级** | 15% | 全国品牌 → 全国品牌类比；自有品牌 → 自有品牌 |
| **包装规格/形式** | 15% | 相似的件数、规格或重量 |
| **目标人群** | 10% | 相同客户细分（价值型、主流、高端） |
| **上市季节** | 10% | 同季度上市；季节模式因季度而异 |
| **分销广度** | 5% | 相似的初始门店数（±25%） |

#### 类比评分

按上述标准对每个候选类比评分（每项标准 1–5 分，加权）。选择综合得分 > 3.5 的前 3–5 个类比。如果没有类比得分 > 3.0，该新品确实是全新产品 — 使用品类均值并加 40% 置信区间。

### 7.2 新品生命周期曲线

大多新品遵循包含四个阶段的生命周期曲线：

| 阶段 | 持续时间 | 需求模式 | 描述 |
|---|---|---|---|
| **导入期** | 第 1–4 周 | 爬坡，常受试用驱动 | 初始客户试用。需求不可预测。 |
| **成长期** | 第 5–12 周 | 加速，复购开始 | 复购客户出现。需求变得更可预测。 |
| **稳定期** | 第 13–26 周 | 平稳至稳态 | 商品找到"正常周转率"。基线预测可靠。 |
| **成熟期** | 第 27 周+ | 稳定或缓慢下降 | 适用标准需求规划。 |

**按阶段预测：**

| 阶段 | 方法 | 置信区间 |
|---|---|---|
| 导入期（1–4 周） | 类比均值 × 1.1（试用加成） | ±40–50% |
| 成长期（5–12 周） | 混合：40% 类比 + 60% 自有轨迹 | ±25–35% |
| 稳定期（13–26 周） | 80% 自有历史、20% 类比 | ±15–25% |
| 成熟期（27+ 周） | 按需求模式的标准方法选择 | 标准 WMAPE 目标 |

### 7.3 新品安全库存协议

| 历史周数 | 安全库存方法 | 不确定性溢价 |
|---|---|---|
| 0–4 | 类比 σ_d + 30% 溢价 | 1.30× |
| 5–8 | 混合 σ_d（50% 自有 + 50% 类比）+ 20% 溢价 | 1.20× |
| 9–12 | 混合 σ_d（75% 自有 + 25% 类比）+ 10% 溢价 | 1.10× |
| 13+ | 自有 σ_d，标准公式 | 1.00× |

### 7.4 新品淘汰决策

并非每个新品都能成功。淘汰决策应是结构化的，而非感性的：

| 指标 | 淘汰阈值 | 时间窗口 |
|---|---|---|
| 售罄率 vs. 类比计划 | < 计划的 30% | 6 周后 |
| 复购率（如可衡量） | < 试购客户的 10% | 8 周后 |
| 周转趋势 | 上市后连续 4 周下降 | 6 周后 |
| 品类经理评估 | "不会复购" | 8 周后 |

做出淘汰决策后：
1. 立即取消所有未完成的 PO。
2. 停止所有已计划的促销。
3. 剩余库存先以 30% 折扣降价 3 周，然后以 50% 折扣再 2 周。
4. 5 周后清仓所有剩余。
5. 记录复盘：为什么基于类比的预测失败了？是类比选择、产品、定价还是竞争环境的问题？

## 8. 需求感知与异常管理

### 8.1 实时需求信号监控

除了定期预测审查外，还需在预测周期之间监控需要即时关注的需求信号：

| 信号 | 检测方法 | 阈值 | 操作 |
|---|---|---|---|
| **POS 周转率飙升** | 日 POS > 滚动 4 周日均的 3× | 连续 2+ 天达到 3× | 调查原因；如持续则手动覆盖 |
| **POS 周转率下降** | 日 POS < 滚动 4 周日均的 0.3× | 连续 3+ 天达到 0.3× | 检查虚假库存、展示撤除或竞争行为 |
| **缺货级联** | 48 小时内 3+ 门店同一 SKU 缺货 | 3 家门店 | 从配送中心紧急补货；按销售速度分配 |
| **天气警报** | NWS 对覆盖 > 10% 门店的区域发布恶劣天气预警 | 预测影响 > 品类销量的 5% | 调整天气敏感品类的预测 |
| **竞争价格变动** | 竞品价格调查显示可比 SKU 低 > 15% | 在 3+ 竞品门店确认 | 通知商品管理；准备下调预测 |
| **社交媒体飙升** | 监控工具显示品牌/产品提及量增长 > 500% | 持续 > 24 小时 | 评估病毒式传播风险；准备分配计划 |

### 8.2 预测覆盖治理

手动覆盖是必要的但也是危险的。无治理的覆盖会引入偏差并随时间降低预测准确性。

**覆盖规则：**

1. **所有覆盖必须记录** 原因代码和量化依据。
2. **按幅度的覆盖权限：**
   - ±10%：规划师可无需审批直接覆盖
   - ±10–25%：需规划经理审批
   - ±25–50%：需总监审批
   - > ±50%：需 VP 审批（或规划委员会）
3. **覆盖准确性追踪：** 每次覆盖都会与实际结果对比追踪。如果规划师的覆盖在一个季度内 WMAPE > 40%，其覆盖权限将被审查。
4. **日落规则：** 覆盖在 4 周后过期。如果情况持续，必须创建新的覆盖（附新依据）。这防止过期覆盖在数月后污染预测。
5. **禁止"共识"覆盖：** 在需求评审会上将预测调整为销售团队一厢情愿的想法，这是正偏差的头号来源。要求每次会议覆盖都引用一个具体、可验证的外部信号。

## 9. 库存健康诊断

### 9.1 供应周数分析

供应周数（WOS）是库存健康的核心脉搏指标。在 SKU 层面计算，汇总到品类层面，每周审查。

```
WOS = On-Hand Inventory (units) / Average Weekly Demand (units)
```

分母使用前瞻预测而非历史销售。当商品缺货时，历史销售会低估需求（你无法销售你没有的库存）。

**WOS 健康区间：**

| WOS 范围 | 状态 | 操作 |
|---|---|---|
| < 2 周 | 严重偏低 | 加急补货；考虑从低周转门店调拨 |
| 2–3 周 | 偏低 | 确认下一个 PO 到货；如无在途 PO 则下紧急订单 |
| 4–8 周 | 健康 | 正常运营 |
| 9–12 周 | 偏高 | 审查预测；如需求未增加则推迟或减少下一个 PO |
| 13–26 周 | 过剩 | 启动降价或促销售罄计划 |
| > 26 周 | 严重过剩 | 标记为慢动品淘汰候选；降价或清仓 |

### 9.2 库存周转率与 GMROI

**库存周转率：**
```
Annual Turns = Annual COGS / Average Inventory at Cost
```

| 品类类型 | 目标周转率 | 基准 |
|---|---|---|
| 易腐食品 | 30–52 | 每周 1× |
| 中心区食品 | 12–20 | 每 2–4 周 |
| 一般商品 | 6–12 | 每 4–8 周 |
| 季节性（当季） | 8–15 | 当季售罄 |
| 季节性（全年） | 2–4 | 因淡季零销售而较低 |

**GMROI（库存投资毛利回报率）：**
```
GMROI = Gross Margin $ / Average Inventory at Cost
```

GMROI 为 2.0 意味着每投入 $1 库存获得 $2 毛利。最低可接受 GMROI 因品类而异，但通常应超过公司资本成本除以毛利率。对于资本成本 8%、毛利率 35% 的零售商，最低 GMROI = 0.08 / 0.35 = 0.23。实际上，大多零售商对健康品类的目标 GMROI > 1.5。

### 9.3 死库存与过时品识别

死库存是指在规定期间内零销售的库存。它是过剩库存最昂贵的形式，因为产生零回报的同时消耗仓库空间和营运资金。

**死库存层级：**

| 层级 | 定义 | 操作 | 时间线 |
|---|---|---|---|
| 老化 | 8–12 周零销售 | 审查 — 是季节性？新品？放错位置？ | 1 周内调查 |
| 死库 | 13–26 周零销售 | 降价 40–50% 或移至清仓 | 2 周内启动 |
| 过时 | > 26 周零销售 | 任何正回收价清仓或捐赠 | 4 周内执行 |
| 注销 | 清仓/捐赠不经济 | 销毁并注销；回收仓库空间 | 2 周内执行 |

**死库存根因分析：**

每季度运行。按根因分类死库存以防止复发：

| 根因 | 占死库存比例（典型） | 预防措施 |
|---|---|---|
| 过度采购（预测过高） | 35–45% | 提高预测准确性；收紧覆盖治理 |
| 产品失败（质量、客户拒绝） | 15–20% | 加快新品淘汰决策 |
| 季节性延续（错过降价窗口） | 15–25% | 执行 §6.2 的降价时机模型 |
| 品类调整（下架但未售完） | 10–15% | 协调下架时机与售罄进度 |
| 虚假库存（系统显示有货但实际无） | 5–10% | 对零周转商品定期循环盘点 |

### 9.4 多门店零售商分配逻辑

当配送中心库存不足以满足所有门店需求时，使用优先级框架而非按比例分配：

**优先级1：防止 A 类商品门店缺货。**
优先分配给 A 类商品供应 < 3 天的门店。数量 = 最小值（距下次配送中心发货天数 × 日需求），以桥接到下一个分配周期。

**优先级2：将分配与门店级预测匹配。**
对于剩余库存，按各门店的前瞻周预测（而非历史销售，因为历史销售会惩罚曾缺货的门店）按比例分配。

**优先级3：最低陈列库存。**
无论预测如何，每家门店至少获得最低展示数量。空货架向客户传递"此商品已停售"的信号并破坏需求。

**优先级4：将分配限制在货架容量以内。**
不要发送超过门店能陈列的数量。仓库中超量的库存会造成损耗、损坏和过期风险（对于易腐品）。

**分配频率：**
- A 类商品：每次配送中心到门店发货时分配（通常每周 2–5 次）
- B 类商品：每周分配 1–2 次
- C 类商品：每周或每两周分配一次

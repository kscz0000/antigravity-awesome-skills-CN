# 决策框架——生产排程

本参考文件提供离散和批量制造中生产排程的详细决策逻辑、排程算法、优化方法论和产能规划技术。在智能体需要做出或建议精细排程决策时按需加载。

所有阈值、公式和时间假设适用于运行 3–8 条产线、每班 50–300 名直接劳动力的离散和批量制造运营。

---

## 1. 工单排程算法

### 1.1 派工规则——各自的适用场景

派工规则是在多个工单竞争同一资源时应用于工序的启发式方法。没有单一规则在所有场景中都最优。选择取决于工厂的主要绩效目标。

| 规则 | 定义 | 最佳适用 | 弱点 |
|---|---|---|---|
| **SPT（最短加工时间）** | 优先处理加工时间最短的工单 | 最小化平均流转时间、降低 WIP、在换型时间可忽略时最大化产出 | 饥饿长工单——一个 8 小时的工单排在 20 个 20 分钟工单后面。导致长周期产品交期违约。 |
| **EDD（最早交期）** | 优先处理交期最早的工单 | 最小化所有工单的最大延迟，满足交货承诺 | 忽略加工时间——明天交期的 8 小时工单优先于 2 小时后交期的 5 分钟工单。如果许多工单交期较远，可能增加 WIP。 |
| **关键比率（CR）** | CR =（交期 − 当前日期）/ 剩余加工时间。优先排程最低 CR。 | 平衡交期紧迫性与剩余工作量。CR < 1.0 表示工单滞后。 | 当交期不切实际时失效（所有 CR < 0.5）。需要准确的剩余加工时间估算。 |
| **加权最短作业优先（WSJF）** | 优先级 =（延迟成本 × 工单权重）/ 加工时间。优先排程最高优先级。 | 工单具有不同经济价值的环境。最大化产出加权价值。 | 需要可靠的延迟成本估算，通常具有主观性。可能无限期饥饿低价值长工单。 |
| **松弛时间（ST）** | 松弛 = 交期 − 当前日期 − 剩余加工时间。优先排程最低松弛。 | 类似 CR 但使用绝对松弛而非比率。在加工时间相似时更优。 | 与 CR 相同——在不切实际的交期下退化。不考虑下游工序的排队时间。 |
| **FIFO（先进先出）** | 按工单到达工序的顺序处理 | 确保公平性、易于沟通、在稳定环境中可预测的流程下运作 | 无优化——忽略交期、加工时间和经济价值。仅在所有工单优先级相同且流程均衡时使用。 |

#### 算法选择决策树

1. **计划遵守率是否为主要 KPI 且存在合同交货罚金？**
   → 使用 EDD 作为主要规则。对 CR < 0.8 的工单插入 CR 检查——这些需要
   无论 EDD 排名如何都立即关注。

2. **产出/产量是否为主要 KPI 且交货窗口灵活？**
   → 使用 SPT 最小化平均流转时间。监控最大延迟；如果超过
   可接受阈值，切换为 SPT-EDD 混合（在交期窗口内使用 SPT）。

3. **工单是否具有显著不同的经济价值（毛利、罚金、客户层级）？**
   → 使用 WSJF。权重 = 客户层级乘数 × 毛利贡献。这是
   异质订单组合的加工车间的合适规则。

4. **换型时间是否与顺序相关且显著（产品族之间 >15 分钟）？**
   → 没有纯派工规则能处理这种情况。使用换型感知排程启发式
   （第 2 节），按换型族分组并在组内使用 EDD 优化。

5. **环境是否稳定、流程均衡且需求可预测？**
   → FIFO 可接受，因其简单性和车间信任度而被优选。

### 1.2 多规则混合方法

实践中，大多数排程师使用分层混合方法：

**第 1 层——硬约束（过滤）**
从队列中移除缺少物料、工装或合格操作员的工单。
无论优先级如何，这些工单不可排程。

**第 2 层——紧急覆盖（强制排序）**
CR < 0.8 或已逾期的工单强制排至顶部，
按客户违约风险降序排列。

**第 3 层——主要派工规则（排序剩余）**
对剩余工单应用选定的派工规则（EDD、SPT、WSJF 等）。

**第 4 层——换型优化（局部重排）**
在主要序列内，执行相邻交换改进以减少
总换型时间，约束条件是交换不导致任一被交换工单的交期
违约。

**第 5 层——人力均衡（验证）**
检查生成的序列是否在班次任何小时内造成超过
可用人力的高峰。如果存在，将造成高峰的最低
优先级工单推迟到下一个可用时段。

### 1.3 关键比率详解

关键比率是混合环境中最通用的单一派工规则。

**公式：**
```
CR = (Due Date − Current Date) / Remaining Total Processing Time
```

其中剩余总加工时间包括：
- 当前工序的换型时间
- 当前工序的运行时间
- 剩余工序的排队时间估算（使用历史平均排队时间）
- 工艺路线中所有剩余工序的换型 + 运行时间

**解读：**
| CR 值 | 含义 | 行动 |
|---|---|---|
| CR > 2.0 | 充裕前置——工单大幅超前计划 | 最低优先级。如产能被更紧的工单需要，可推迟。 |
| 1.0 < CR < 2.0 | 按计划但松弛有限 | 按派工规则正常排程 |
| CR = 1.0 | 恰好按计划——无松弛 | 密切监控。任何中断都会导致延迟。 |
| 0.5 < CR < 1.0 | 滞后——不干预将会延迟 | 升级。考虑加班、替代工艺路线或部分发货。 |
| CR < 0.5 | 严重延迟——无重大干预难以恢复 | 立即升级至生产经理。通知客户修订日期。 |

**更新 CR：** 在每次工序完成和每班开始时重新计算 CR。一个班次开始时 CR = 1.5 的工单，如果班中遇到 4 小时非计划延迟，可能降至 CR = 0.7——班组长需要实时了解。

### 1.4 客户层级加权排程

服务多个客户层级的制造工厂需要权重系统：

| 客户层级 | 权重乘数 | 理由 |
|---|---|---|
| 一级（OEM，合同罚金） | 3.0 | 延迟交货触发财务罚金、产线停机索赔 |
| 二级（重点客户，框架协议） | 2.0 | 无合同罚金但有关系价值和复购风险 |
| 三级（标准客户） | 1.0 | 标准条款，无罚金 |
| 四级（现货订单，经销商） | 0.5 | 价格敏感，低转换成本 |

**带客户层级的 WSJF：**
```
Priority Score = (Customer Tier Weight × Days Until Due / Remaining Processing Time)
```
分数越低 = 优先级越高（更紧急）。负分 = 已逾期。

## 2. 换型优化

### 2.1 SMED 实施阶段——逐步详解

#### 阶段 0——记录现状（2–4 周）

1. 在目标设备/产线上录制 3–5 次换型的视频。包含从
   上一个产品的最后一个合格件到下一个产品的第一个合格件的全程。
2. 创建换型元素表，列出每项任务、执行者
   （操作员、换型技术员、维护）、持续时间以及设备是否停机。
3. 对每个元素分类：
   - **内部（IED）：** 必须在设备停机时执行。
   - **外部（OED）：** 可在设备仍运行时执行。
   - **浪费：** 完全不必要——来自旧程序的遗留、冗余检查、等待。

典型发现：30–50% 的换型时间要么是本应在设备运行时完成的外部工作
错误地在停机期间执行，要么是纯粹的浪费（找工具、等审批、
走到工具间）。

#### 阶段 1——分离内部与外部（2–4 周）

1. 将所有外部元素移至换型前准备：
   - 换型开始前在设备旁预准备下一工单的工装、模具、夹具。
   - 预混材料、预热模具、预编程 CNC 设置。
   - 预打印工单文件和质量检查表。
2. 创建标准化的换型准备检查表。换型技术员在
   计划换型时间前 30–60 分钟开始执行。
3. 预期结果：在无资本投入的情况下减少 25–40% 的设备停机时间。

#### 阶段 2——将内部转化为外部（4–8 周）

1. 标准化模具/夹具高度和安装接口，使对准和调整
   在模具到达设备之前完成，而非之后。
2. 实施中间夹具——在镜像设备安装接口的暂存夹具中
   设置下一个工装。换型开始时，预组装单元
   以最小调整量直接装入。
3. 预处理材料：如果下一个产品需要不同的温度、
   粘度或化学混合，在并联容器中提前开始调节。
4. 预期结果：再减少 15–25% 的设备停机时间。可能需要
   对复制工装或暂存夹具进行适度投资。

#### 阶段 3——精简剩余内部元素（4–12 周）

1. 用快拆夹具、凸轮锁或液压夹紧替代螺栓紧固。
   每移除一个螺栓节省 15–30 秒。
2. 通过 poka-yoke 消除调整：定位销、导轨、固定挡块，
   保证首件对准无需试错。
3. 标准化公用连接：气、水、液压和电气的颜色编码
   快接头。一动连接/断开。
4. 并行作业：两人同时在设备两侧工作
   可将内部时间减半。需要编排好的程序和安全协议。
5. 预期结果：再减少 10–20%。通常需要对
   快换工装进行资本投入。

#### 阶段 4——消除调整并验证（持续）

1. 实施首件验证夹具，无需全面检验即可确认尺寸。
2. 从首件开始使用统计过程控制（SPC）——如果首件在
   控制限内，换型即通过验证无需试运行。
3. 用照片记录最终标准化换型程序、每元素时间目标
   和签字确认表。
4. 目标：设备停机部分换型时间低于 10 分钟（单位数分钟换模）。

### 2.2 顺序相关换型矩阵

对于换型时间因产品转换而变化的作业，构建
换型时间矩阵：

**示例——喷涂线换型矩阵（分钟）：**

| 从 \ 到 | 白色 | 黄色 | 橙色 | 红色 | 蓝色 | 黑色 |
|---|---|---|---|---|---|---|
| **白色** | 0 | 8 | 10 | 15 | 20 | 25 |
| **黄色** | 15 | 0 | 8 | 12 | 20 | 25 |
| **橙色** | 20 | 12 | 0 | 8 | 18 | 22 |
| **红色** | 25 | 18 | 12 | 0 | 15 | 18 |
| **蓝色** | 20 | 22 | 20 | 18 | 0 | 10 |
| **黑色** | 30 | 28 | 25 | 22 | 12 | 0 |

**此矩阵的观察：**
- 浅到深的转换（白色 → 黑色：25 分钟）比深到浅（黑色 → 白色：30 分钟）更经济。
- 同色系内转换时间最短（红色 → 橙色：12 分钟 vs. 红色 → 白色：25 分钟）。
- 六色一次批量运行的最优序列：白色 → 黄色 → 橙色 → 红色 → 蓝色 → 黑色（总计：8+8+8+15+10 = 49 分钟）vs. 随机序列平均每转换 17 分钟（总计 85 分钟）。

**在排程中使用矩阵：**
1. 尽可能按色系分组工单（族内批量排程）。
2. 需要跨族转换时，使用最近邻启发式优化转换序列，
   然后用 2-opt 交换改进。
3. 如果某特定颜色最早到期但最优换型序列会延误它，
   计算次优序列的成本（额外换型分钟 × 约束小时费率）
   vs. 延迟交货的成本。选择低成本选项。

### 2.3 批量长度优化

**经济生产批量（EPQ）：**
```
EPQ = √((2 × D × S) / (H × (1 − D/P)))
```
其中：
- D = 需求速率（每期单位数）
- S = 每次换型的换型成本（人工 + 废料 + 产出损失机会成本）
- H = 每单位每期持有成本
- P = 生产速率（每期单位数），P > D

**实际调整：**
- 将 EPQ 向上取整到最近的整班或整批，避免班中换型。
- 如果 EPQ 导致 WIP 超过可用暂存空间，约束至物理容量。
- 如果 EPQ 导致批量长于最长客户前置时间容忍度，
  缩短以保持响应性，即使换型频率更高。

**批量 vs. 混线决策：**

| 因素 | 偏向批量 | 偏向混线 |
|---|---|---|
| 换型时间 | 长（>60 分钟） | 短（<15 分钟） |
| 换型成本 | 高（每次换型 >$500） | 低（每次换型 <$100） |
| 需求变异性 | 低（稳定、可预测） | 高（波动、订单驱动） |
| 客户前置时间期望 | 宽容（>2 周） | 紧迫（<3 天） |
| WIP 持有成本 | 低 | 高 |
| 产品保质期 | 长或不适用 | 短或受管制 |
| 产品变体数量 | 少（<10） | 多（>50） |

## 3. 约束理论（TOC）实施

### 3.1 鼓-缓冲-绳——逐步详解

**步骤 1：识别约束**

对每个工序在未来计划范围（1–4 周）内运行产能分析：

```
Utilisation = Σ(Setup Time + Run Time for all scheduled jobs) / Available Time
```

可用时间 = 班次小时数 × 设备数量 ×（1 − 计划维护百分比）

利用率最高的工序就是鼓。如果多个工序
超过 90% 利用率，灵活性最低（替代工艺路线最少、
设备最专业）的那个是主约束。

**验证测试：** 如果你可以为疑似约束增加 10% 的产能
（多一台设备、多一小时班次或 10% 的速度提升），工厂总
产出是否增加约 10%？如果是，它就是真约束。如果产出
增加更少（因为第二个工序立即成为瓶颈），
你有一个交互约束对，需要不同的处理方式。

**步骤 2：利用约束**

在无资本投入的情况下最大化约束产出：

1. **消除空闲时间：** 约束不应等待物料、工装、
   操作员、质量检验或信息。一切预准备。
2. **最小化约束换型：** 将换型转移到非约束
   资源，那里的时间成本更低。如果约束必须换型，
   确保严格执行 SMED 纪律。
3. **防止质量缺陷到达约束：** 在约束工序前检验，
   而非之后。在约束处加工的每个不良件都浪费了
   约束产能。
4. **在休息和换班时持续运行：** 错开操作员午餐，使约束
   永不因休息而停机。安排替补操作员。
5. **消除微停：** 解决每个 1–5 分钟停机的来源（传感器跳停、
   物料卡塞、工装磨损报警），单独看微不足道但累计
   窃取 2–5% 产能。

**步骤 3：一切从属于约束**

1. **上游工序：** 仅以约束可消耗的速率向上游工序释放工作。
   这就是"绳"。如果约束每小时处理 100 件，
   上游释放速率不应超过 100 件/小时，无论上游产能如何。
2. **下游工序：** 必须保持足够的冲刺产能以清除约束
   产出而不成为次级瓶颈。如果约束每 2 小时生产一批，
   下游必须能在 2 小时内处理该批次。
3. **非约束排程：** 不要孤立地优化非约束排程。
   当约束运行在 85% 时，非约束以 100% 利用率运行是在生产
   多余 WIP，堵塞车间并减慢约束的物料流。

**步骤 4：建立缓冲**

约束缓冲是时间缓冲，不是库存缓冲：

```
Buffer Duration = Planned Lead Time from release to constraint × Buffer Factor
```

典型缓冲系数：
- 稳定、可靠的上游作业：0.3 × 前置时间
- 中等可靠性、有一定变异性：0.5 × 前置时间（最常见的起点）
- 不可靠上游、频繁中断：0.75 × 前置时间

**缓冲定径示例：**
如果从原材料释放到约束工序的上游前置时间为
8 小时，且上游可靠性中等，将缓冲设为 4 小时。这意味着
物料应在约束计划加工前至少 4 小时到达约束暂存区。

**步骤 5：监控缓冲渗透**

| 区域 | 缓冲消耗 | 含义 | 行动 |
|---|---|---|---|
| 绿色 | 0–33% | 约束受到良好保护 | 正常运营 |
| 黄色 | 33–67% | 警告——物料可能迟到 | 加速上游工作。检查阻碍因素。 |
| 红色 | 67–100% | 严重——约束面临断供风险 | 立即升级。上游加班。必要时重排序。 |
| 黑色 | >100% | 缓冲耗尽——约束正在断供 | 约束空闲或将要空闲。紧急响应。此时每分钟延迟 = 工厂产出损失。 |

跟踪 2–4 周的缓冲渗透趋势。持续黄区表明
上游存在系统性问题（非随机变异），需要纠正行动。

**步骤 6：提升约束（仅在步骤 1–5 用尽后）**

如果在充分利用和从属之后约束仍将工厂产出
限制在需求以下：

1. 仅在约束处增加加班或周末班次。
2. 增加并行设备或替代工艺路线能力。
3. 将约束特定工序外包给合格的分包商。
4. 投资更快的约束设备（资本支出）。

每个提升步骤成本逐步递增。在充分利用之前
永远不要提升——大多数工厂在约束处有 15–25% 的隐藏产能，
利用阶段以最小成本恢复。

### 3.2 缓冲管理高级模式

**发货缓冲：** 保护客户交期免受内部变异性影响。通常
为约束到发货的前置时间的 50%。如果约束到发货
前置时间为 2 天，发货缓冲为 1 天——工作应在
承诺发货日前 1 天到达发货暂存区。

**装配缓冲：** 在具有汇聚产品结构（多个组件
汇入共同装配点）的工厂中，每个到装配点的进料路径需要
自己的缓冲。装配只有在所有组件到齐后才能进行，因此
最慢的进料路径决定有效缓冲。

**动态缓冲调整：**
- 如果缓冲渗透持续处于绿区（在 4 周滚动窗口中 >80% 的工单
  到达时缓冲完好），将缓冲减少 10–15%。过量缓冲
  意味着过量 WIP 和更长的前置时间。
- 如果缓冲渗透频繁达到红区（4 周窗口中 >20% 的工单），
  将缓冲增加 15–20%，同时调查上游根本原因。
- 调整缓冲频率不低于每 2 周一次。缓冲管理需要
  多个周期的稳定数据。

## 4. 中断恢复协议

### 4.1 结构化中断响应框架

当中断发生时，遵循此决策树：

**步骤 1：分类中断**

| 类型 | 示例 | 典型时长 | 影响范围 |
|---|---|---|---|
| **设备** | 故障、传感器失效、工装磨损 | 30 分钟 – 3 天 | 单个工序 |
| **物料** | 短缺、规格错误、来料质量拒收 | 2 小时 – 2 周 | 共享该物料的多个工序 |
| **人力** | 缺勤、受伤、认证缺失 | 1 班 – 1 周 | 单个工序或产线 |
| **质量** | 制程缺陷、触发扣留的客户投诉 | 2 小时 – 1 周 | 整批/整批次，加下游消耗者 |
| **外部** | 供应商失效、停电、天气、监管停工 | 4 小时 – 不定 | 可能全厂 |

**步骤 2：评估约束影响**

| 中断位置 | 约束影响 | 响应优先级 |
|---|---|---|
| 在约束处 | 直接——每分钟 = 产出损失 | 最高优先级。动员所有资源。 |
| 约束上游，缓冲为绿色 | 间接——缓冲吸收延迟 | 监控缓冲渗透。不立即调整排程。 |
| 约束上游，缓冲为黄/红 | 间接但迫在眉睫——约束将断供 | 加速。上游加班。重排序从替代来源供应约束。 |
| 约束下游 | 除非 WIP 回堵到约束，否则无产出影响 | 监控。在约束产出开始排队前清除下游阻塞。 |
| 并行路径（无约束交互） | 无产出影响，但受影响订单有交货影响 | 重排序受影响订单。通知客户。 |

**步骤 3：执行恢复**

1. **即时（0–30 分钟）：** 评估时长和影响。通知受影响方。冻结在制品。
2. **短期（30 分钟 – 4 小时）：** 重排序剩余工作。激活替代工艺路线。分配备用操作员。请求紧急维护。
3. **中期（4–24 小时）：** 协商加班或班次延长。联系分包商。更新客户 ETA。重新计算整个计划范围。
4. **长期（>24 小时）：** 产能再平衡。可能将订单重新分配至替代工厂。与客户协商交货时间表。保险/不可抗力文件（如适用）。

### 4.2 物料短缺响应

1. **确认短缺：** 验证实物库存 vs. 系统数量。幻影库存
   很常见——在宣布短缺前进行实物盘点。
2. **识别替代品：** 检查 BOM 替代、工程批准的替代品
   和客户批准的等效材料。在受管制行业（航空航天、制药），
   仅允许预先批准的替代品。
3. **部分组装策略：** 能否完成到消耗缺料物料的工序，
   然后搁置半成品 WIP 待物料到达后完成？
   这保持上游工序的生产力并保留非缺料
   部分的前置时间。
4. **重排序：** 将所有不消耗缺料物料的工单提前。
   即使在短缺期间也保持工厂生产力。
5. **加急采购：** 加急运费的紧急采购订单。量化：
   加急物料 + 运费成本是否低于损失约束时间 × 延迟
   小时的成本？如果是，毫不犹豫加急。
6. **客户沟通：** 如果短缺影响客户交货，确认后
   4 小时内通知。提供修订交货日期和恢复计划。

### 4.3 质量扣留管理

当发现制程质量问题时：

1. **立即控制：** 隔离所有受影响的在制品——在制批次、
   同批次已完成件和消耗了该批次件的
   下游组件。
2. **评估范围：** 影响多少件？哪些客户订单消耗这些件？
   返工成本 vs. 报废成本 vs. 客户拒收成本？
3. **重排程：** 从活跃排程中移除被扣留库存。重新计算
   依赖此库存的所有下游工序。
4. **被扣留物料决策树：**
   - **可返工且经济：** 安排返工工序。将返工时间加入
     工艺路线并重排下游。
   - **可返工但不经济（返工成本 > 重新制造的物料 + 人工成本）：**
     报废被扣留批次并从头安排替代生产订单。
   - **不可返工、不可报废（监管扣留待调查）：** 无限期从
     排程中排除。按库存不存在来计划。
5. **根本原因：** 在排程调整的同时，质量工程应隔离
   根本原因。排程师需要知道：这是一次性事件，还是后续
   批次也会受影响？如果是系统性的，在根本原因解决前
   降低排程参数中受影响工序的良率假设。

## 5. 产能规划 vs. 有限排程

### 5.1 粗产能规划（RCCP）

RCCP 是中期规划工具（4–16 周），在详细排程前验证 MPS
是否在高层可行。

**流程：**
1. 取 MPS（按产品族按周的生产计划）。
2. 乘以每个关键工序的单位工艺路线工时（通常仅约束
   和 1–2 个近约束）。
3. 比较每个工序每周的总需求工时与可用工时。
4. 如果需求工时超过可用工时，标记过载周需行动：
   需求塑造（将订单移至相邻周）、加班、外协或 MPS 修订。

**RCCP 负荷分布示例：**

| 周 | 约束产能（小时） | 需求负荷（小时） | 利用率 | 状态 |
|---|---|---|---|---|
| W23 | 120 | 105 | 87.5% | 正常 |
| W24 | 120 | 118 | 98.3% | 警告——接近产能 |
| W25 | 120 | 142 | 118.3% | 过载——需行动 |
| W26 | 120 | 96 | 80.0% | 正常——可吸收 W25 溢出 |
| W27 | 80（计划维护窗口） | 75 | 93.8% | 紧张——维护可能需重排 |

**W25 过载的行动：**
- 22 小时负荷能否移至 W24 或 W26 而不误交期？检查交期。
- 如不可移：加班（22 小时 ÷ 8 小时/班 = 3 个额外班次，或 3 个周六班次）。
- 如无加班：哪些订单交货日期最灵活？协商。
- 最后手段：将 22 小时工作外协。评估质量和前置时间影响。

### 5.2 有限产能排程（FCS）详解

FCS 超越 RCCP，将单个工序排在特定资源的特定时间，遵守：

1. **资源产能：** 设备数量 × 每班小时数 × 每天班次数，减去计划维护窗口。
2. **顺序相关换型：** 换型时间因前一工单而异（见第 2.2 节换型矩阵）。
3. **物料可用性：** 工序在所有 BOM 组件到达工序前不能开始。
4. **工装约束：** 需要工装套件 ABC 的工单不能与需要相同工装的另一工单同时运行。
5. **人力约束：** 需要持证操作员的工单在无持证操作员当班时不可排程。
6. **工序依赖：** 工单上的工序 20 不能在工序 10 完成前开始（工艺路线优先级）。
7. **转移批次：** 如果定义了转移批次大小，重叠工序可在前一工序的完整批次完成前开始。

**FCS 排程算法（简化版）：**
1. 按优先级对所有工序排序（使用第 1.2 节的混合派工方法）。
2. 对最高优先级的未排程工序：
   a. 在所需资源上找到最早的可行时间窗口，考虑产能、
      物料可用性、工装、人力和前序完成。
   b. 将工序排入该窗口。
   c. 更新资源可用性。
3. 对下一优先级工序重复。
4. 所有工序排程后，运行后优化遍历寻找不违反
   交期的换型缩减机会（相邻交换改进）。

### 5.3 产能缓冲和保护性产能

非约束工序应保持保护性产能——刻意规划的
空闲时间，吸收变异性并防止 WIP 积累。

**按工序类型的目标利用率：**

| 工序类型 | 目标利用率 | 理由 |
|---|---|---|
| 约束 | 90–95% | 最大化产出。缓冲其他一切来保护它。 |
| 近约束（>80% 负荷） | 85–90% | 接近成为约束。监控漂移瓶颈。 |
| 标准 | 75–85% | 保护性产能吸收上游变异性。 |
| 共享资源（叉车、天车、检验员） | 60–75% | 这些资源的需求变异性高。过度排程造成全系统延迟。 |
| 返工/维修 | 50–70% | 必须按需有可用产能。不能高利用率排程。 |

**保护性产能不足的警示信号：**
- 非约束工序前的 WIP 队列随时间增长。
- 非约束工序偶尔成为瓶颈（漂移瓶颈）。
- 非约束工序加班"为了跟上进度"。
- 物料搬运工持续在非约束工序之间加急。

## 6. 多约束排程

### 6.1 交互约束

当两个或更多工序都超过 85% 利用率且共享物料流路径时，
它们产生交互——改善一个的产出可能使另一个断供或过载。

**识别：**
两个工序是交互约束，如果：
1. 它们在同一工艺路线上（物料从一个流向另一个），且
2. 两者都超过 85% 利用率，且
3. 给一个增加产能导致另一个利用率超过 95%。

**交互约束排程策略：**

1. **先排主约束**（利用率更高的或更靠近客户的那个）。
2. **将次约束从属于**主约束的排程——次约束
   按主约束输出排程规定的方式和节奏处理工作。
3. **在它们之间放置缓冲**——即使两者都是约束，上游那个
   应向下游那个提供时间缓冲以吸收变异性。
4. **永远不要独立优化它们。** 对主约束最优的换型序列
   如果次约束也有顺序相关换型，可能为次约束创建
   不可行的序列。联合求解。

### 6.2 设备 + 人力双重约束

在设备半自动且需要操作员进行
换型、首件检验或监控但可在周期部分无人值守运行的环境中很常见。

**排程方法：**
1. 先排设备产能（按设备的有限产能）。
2. 叠加人力产能（按技能/认证的有限产能）。
3. 识别冲突：设备排程需要操作员但
   无合格操作员可用的时间窗口。
4. 通过以下方式解决冲突：
   - 将工单转移到不同操作员有认证的另一台设备。
   - 将操作员从低优先级工单调到冲突工单。
   - 将操作员的换型/检验任务安排在工单开始，之后
     允许无人值守运行。

### 6.3 工装作为共享约束

当专业工装（模具、冲模、夹具、量具）跨设备共享时：

1. **将工装作为排程系统中的资源**——与排程
   设备和人力相同的方式排程工装。
2. **两个需要同一模具的工单不能在不同设备上同时运行。**
3. **设备间的工装换型时间**增加总换型。如果模具 A
   从设备 1 移到设备 2，加上模具提取时间（设备 1）+ 运输
   时间 + 模具安装时间（设备 2）。
4. **通过分组优化：** 如果三个工单都需要模具 A，在同一设备上
   连续排程它们以避免模具转运。

## 7. 混线生产的产线平衡

### 7.1 节拍时间计算

```
Takt Time = Available Production Time per Shift / Customer Demand per Shift
```

**示例：** 每班可用 480 分钟（8 小时 × 60 分钟，减去 30 分钟休息），
客户需求每班 240 件。

```
Takt Time = 450 / 240 = 1.875 minutes per unit
```

产线上每个工位必须在每件 1.875 分钟内完成其任务。
如果任何工位超过节拍，它成为瓶颈，产线无法满足需求。

### 7.2 工位平衡

1. 列出所有任务的持续时间和先后关系。
2. 将任务分配到工位，使没有工位超过节拍时间。
3. 最小化工位数量（以最小化人工成本）。
4. 测量平衡效率：

```
Balance Efficiency = Σ(Task Times) / (Number of Stations × Takt Time) × 100%
```

目标：>85%。低于 80% 表示某些工位有大量空闲时间。

### 7.3 混线排序（Heijunka）

当一条产线生产多个不同任务时间的型号时：

1. 计算跨型号的加权平均节拍时间。
2. 确定型号混合比（例如，型号 A：60%，型号 B：30%，型号 C：10%）。
3. 创建均衡工作负荷的重复模式。对于 A:B:C = 6:3:1，一个 10 件
   循环为：A-B-A-A-C-A-B-A-B-A。
4. 验证瓶颈工位能在节拍内处理每个型号。如果型号 C
   在工位 3 需要 2.5 分钟而节拍为 1.875 分钟，型号 C 必须间隔
   足够远使工位 3 能在出现之间追上进度。

## 8. 带监管和合规约束的排程

### 8.1 可追溯性驱动的排程

在受管制行业（制药、食品、航空航天），批次可追溯性要求
约束排程灵活性：

- **禁止混批：** 批次 A 和批次 B 的工单不能共享
   设备，除非在批次之间完全清洁并
   记录清洁过程。
- **专用设备批量：** 当过敏原或污染控制需要
   专用设备时，产品 X 在 1 号线的排程窗口限于
   专用批量期间。在此窗口外排程需要重新验证。
- **操作员资质记录：** 排程必须记录哪个操作员执行
   每道工序，且该操作员在执行时必须持有认证。

### 8.2 在线清洗（CIP）排程

在食品、饮料和制药行业，某些产品转换之间 CIP 周期是强制的：

| 转换类型 | CIP 时长 | 可否缩短？ |
|---|---|---|
| 同产品，下一批 | 0–15 分钟（仅冲洗） | 否——监管最低要求 |
| 同产品族 | 30–60 分钟（标准 CIP） | 仅经验证的短 CIP 协议 |
| 不同产品族 | 60–120 分钟（完整 CIP） | 否——监管要求 |
| 过敏原转换 | 120–240 分钟（加强 CIP + 拭子检测） | 否——需分析确认 |

将 CIP 周期作为排程中的固定时间块排程，而非可
压缩的"换型时间"。低估 CIP 时间是常见排程错误，
造成级联延迟和监管风险。

## 9. 排程稳定性和冻结区

### 9.1 冻结 / 半冻结 / 流动计划范围

| 范围 | 典型时长 | 灵活性 | 变更需要 |
|---|---|---|---|
| **冻结** | 0–48 小时 | 除不可抗力外不允许变更 | 生产经理 + 排程师审批 |
| **半冻结** | 48 小时 – 1 周 | 允许当日序列变更；不允许日期变更 | 排程师审批 |
| **流动** | 1–4 周 | 完全灵活，可重排序和重排程 | 排程师自行决定 |
| **暂定** | 4 周以上 | MRP 生成，尚未排程 | 计划/MRP 周期 |

**冻结区为何重要：** 每次排程变更触发级联——物料搬运工
重新准备配套件、操作员重新阅读工单、质量预检可能需要重复、
换型序列重新计算。一个班次变更 10 次排程的工厂，
排程变更造成的中断比实际生产问题更多。

### 9.2 排程变更成本模型

在批准冻结或半冻结区的排程变更前，估算总成本：

```
Change Cost = Changeover Cost Delta + Material Restaging Cost + Labour Disruption Cost
              + Quality Re-inspection Cost + Customer Impact Risk
```

如果变更成本 > 变更收益，拒绝变更并维持当前排程。
记录决策供班后审查。

## 10. 加班和班次延长决策框架

### 10.1 何时批准加班

加班是排程杠杆，不是默认选项。使用以下决策树：

1. **加班是否在约束处需要？**
   - 是 → 计算：加班成本 vs. 额外约束小时的产出价值。
     如果 4 小时约束加班总成本 $1,200 能启用 $20,000 的发货，
     立即批准。约束加班的投资回报阈值通常为 3:1
     （价值:成本）或更高。
   - 否 → 非约束处的加班不增加工厂产出。仅在以下情况
     有意义：（a）非约束正在使约束断供且缓冲渗透处于
     黄/红区，或（b）非约束产出是特定客户发货所需
     且不能等到下一正常班次。

2. **加班是自愿还是强制？**
   - 检查工会合同或劳动法规。许多协议要求按资历
     发放加班机会后才能强制。强制加班可能需要 24–48 小时通知。
   - 违反加班分配规则在申诉和士气损害方面的成本
     远超其产出。始终遵守。

3. **疲劳和安全风险：**
   - 已工作 10+ 小时的操作员不应分配到约束
     或安全关键作业。第 11–12 小时错误率增加 25–40%。
   - 如果加班将 12 小时班次延长至 16 小时，将延长班操作员分配到
     非关键监控任务，为约束安排新操作员。

### 10.2 排程排班模式比较

| 模式 | 每周小时 | 每周交接次数 | 加班余量 | 最佳适用 |
|---|---|---|---|---|
| 3 × 8h（周一至周五） | 120 | 15 | 周六班次、每日加班 | 高混合、中等产量 |
| 3 × 8h（24/7） | 168 | 21 | 有限——已接近产能 | 流程工业、连续流动 |
| 2 × 12h（周一至周五） | 120 | 10 | 周末班次 | 资本密集、更少交接 |
| 2 × 12h（4 上 / 4 休） | 168 | 14 | 内置于轮班中 | 高产量、稳定需求 |
| 4 × 10h（仅白班） | 每组 40 | 4 | 周五、周末 | 低产量、单班运营 |

**交接质量影响排程：** 每次交接都是潜在的信息
丢失点——接班班次可能不知道正在发展的质量问题、
物料短缺的变通方案或口头排程变更。更少的交接（12 小时
班次）改善信息连续性但增加疲劳风险。根据
作业复杂度和错误容忍度平衡。

## 11. 外协决策框架

### 11.1 何时外协

外协是产能不足时排程杠杆的最后手段。

**决策标准（全部必须满足）：**
1. 所需工序的内部产能在交货截止日期前
   已完全消耗，包括可用加班。
2. 该工序不在约束处（从约束外协通常意味着
   约束需要提升，而非一次性修复）。
3. 存在能符合质量规格和交货时间线的合格分包商。
4. 外协成本 + 运输成本 + 质量风险成本 < 延迟交货成本
   （罚金 + 客户关系损害）。
5. 在受管制行业：分包商持有必要认证
   （ISO、IATF 16949、AS9100、FDA 注册等）。

### 11.2 带外协工序的排程

当一个工序被外协时：
1. 从内部排程中移除该工序。
2. 加上运出时间（通常 0.5–2 天）和运入时间。
3. 加上分包商报价的前置时间（首次分包商加 20% 缓冲）。
4. 总外部前置时间在工单工艺路线中替代内部工序时间。
5. 基于预期返回日期排程下游内部工序，而非
   内部加工时间。
6. 在 50% 和 90% 完成里程碑监控分包商进度。不要等到
   交期才发现延误。

## 12. 排程指标和持续改进

### 12.1 关键排程指标

| 指标 | 计算 | 目标 | 揭示内容 |
|---|---|---|---|
| **计划遵守率** | 计划 ±1 小时内开工的工单 / 总工单 | > 90% | 工厂执行排程的程度 |
| **排程稳定性** | 冻结区内未变更的工单 / 总冻结工单 | > 95% | 排程被中断的频率 |
| **准时交货率（OTD）** | 在承诺日期或之前发货的订单 / 总订单 | > 95% | 面向客户的绩效 |
| **制造周期** | 工单从首工序开始到末工序结束的时间 | 跟踪 vs. 标准 | 总生产前置时间 |
| **换型比率** | 总换型时间 / 资源总可用时间 | 约束处 < 10% | 换型效率 |
| **约束利用率** | 实际生产时间 / 约束可用时间 | > 85% | 约束利用程度 |
| **WIP 周转率** | 年度销货成本 / 平均 WIP 价值 | 离散制造 > 12 | 排程效率和流动 |
| **排队时间比率** | 排队时间 / 各工序总前置时间 | 跟踪趋势 | 指示隐藏 WIP 和不良流动 |

### 12.2 排程事后复盘流程

在每次重大排程中断（约束停机 > 1 小时、客户交货
延误或加班超过预算 > 20%）后，进行结构化复盘：

1. **时间线重建：** 发生了什么、何时发生、影响的级联？
2. **根本原因：** 中断是由设备、物料、人力、质量、
   排程逻辑还是外部因素引起的？
3. **响应评估：** 重排序决策是否最优？恢复是否
   可以更快？沟通是否及时？
4. **参数更新：** 排程参数（换型时间、运行速率、良率系数、
   缓冲大小）是否需要基于所学调整？
5. **系统性修复：** 什么预防措施将降低此类
   中断再发的概率或影响？

在排程事件日志中记录发现。每月与生产
管理层审查日志以识别模式并优先排序改进行动。

### 12.3 每日排程节奏

严格的每日节奏防止被动救火：

| 时间 | 活动 | 参与者 |
|---|---|---|
| 班次开始 − 30 分钟 | 班前审查：验证物料准备、操作员可用性、设备状态 | 排程师、班组长 |
| 班次开始 | 发布班次排程。巡视车间确认理解。 | 排程师 |
| 班次开始 + 2 小时 | 第一个检查点：计划遵守率、缓冲渗透、早期中断检测 | 排程师（办公桌审查 MES 数据） |
| 班次中点 | 班中审查：实际 vs. 计划，必要时重排序 | 排程师、班组长 |
| 班次结束 − 1 小时 | 班末预测：什么将未完成？下个班次的交接记录。 | 排程师、班组长 |
| 班次结束 | 班次交接：当面（优选）或书面记录。关键问题、偏差、待决决定。 | 交班 + 接班排程师 |
| 每日（上午） | 生产会议：昨日绩效、今日优先级、需管理层决策的问题 | 排程师、生产经理、质量、维护、物料 |

此节奏每班创建至少 5 个接触点，在偏差累积前
验证排程与现实的一致性并纠正。

## 13. ERP 到车间数据流

### 13.1 SAP PP 集成模式

```
Sales Orders / Forecast
       ↓
Demand Management (MD61/MD62)
       ↓
MPS — Master Production Schedule (MD40/MD43)
       ↓
MRP Run (MD01/MD02) → Planned Orders
       ↓
Convert Planned → Production Orders (CO40/CO41)
       ↓
Sequence in APS/Scheduling Tool (external or PP/DS)
       ↓
Release to Shop Floor (CO02 — set status REL)
       ↓
MES Execution (operation confirmations — CO11N/CO15)
       ↓
Goods Receipt (MIGO) → Inventory Updated
```

**常见数据质量问题：**
- 工艺路线时间（换型 + 运行）在工艺改进后未更新 → 排程
  系统性地分配过多或过少时间。
- BOM 数量未按良率调整 → MRP 少订物料。
- 工序产能未反映实际排班模式 → FCS 生成
  不可行的排程。
- 报废报告延迟 → 计划-实际差距悄然增长。

### 13.2 闭环反馈

最重要的集成是 MES 到排程的反馈：

1. **工序开始：** MES 记录实际开始时间。排程与计划开始比较。
   偏差 > 1 小时触发警报。
2. **工序结束：** MES 记录实际结束时间和数量（合格 + 报废）。排程
   用实际前序完成更新剩余工序。
3. **停机事件：** MES 采集停机开始、结束和原因代码。排程
   自动调整下游时间。
4. **质量事件：** MES 采集检验结果。检验失败触发
   排程对受影响批次的扣留。

没有这个反馈环，排程在数小时内与现实脱节，变成
理想而非运营工具。车间不再查阅它，操作员自行
做排序决策，约束处产出下降，因为临时排序
忽略了约束保护逻辑。

# 决策框架 — 质量与不合格品管理

本参考提供了详细的决策逻辑、MRB 流程、RCA 方法论选择、CAPA 生命周期管理、SPC 解释工作流、检验等级确定、供应商质量升级以及合规制造质量工程的质量成本计算模型。

所有阈值、监管引用和过程预期反映了 FDA 21 CFR 820、IATF 16949、AS9100 和 ISO 13485 环境下的质量工程实践。

---

## 1. NCR（不合格品报告）处置决策树

### 1.1 通用处置流程

每个不合格品，无论监管环境如何，都从该决策序列开始。流程在第一个适用处置处终止；不要跳过层级。

```
开始：不合格品已识别并记录
  │
  ├─ 该零件是否安全关键或受监管控制？
  │   ├─ 是 → 能否返工至完全符合？
  │   │         ├─ 是 → 返工（采用已批准的程序 + 100% 重新检验）
  │   │         └─ 否 → 报废（无"照原样使用"许可，需正式风险评估
  │   │                   AND 监管/客户批准）
  │   └─ 否 → 继续
  │
  ├─ 该不合格品是否影响形式、装配或功能？
  │   ├─ 是 → 能否返工至完全符合？
  │   │         ├─ 是 → 返工成本是否 < 替换成本的 60%？
  │   │         │         ├─ 是 → 返工
  │   │         │         └─ 否 → 报废（返工不经济）
  │   │         └─ 否 → 能否修复至可接受功能？
  │   │                   ├─ 是 → 修复（需工程让步 + 客户批准
  │   │                   │         如合同/标准要求）
  │   │                   └─ 否 → 报废
  │   └─ 否 → 继续
  │
  ├─ 该不合格品是否仅为外观问题？
  │   ├─ 是 → 客户规范是否涉及外观要求？
  │   │         ├─ 是 → 该零件是否满足客户外观规范？
  │   │         │         ├─ 是 → 照原样使用（附文档）
  │   │         │         └─ 否 → 需客户让步 → 若批准：照原样使用
  │   │         │                                       → 若拒绝：返工或报废
  │   │         └─ 否 → 照原样使用（附工程签字）
  │   └─ 否 → 继续
  │
  ├─ 是否在材料审核权限内的尺寸类不合格品？
  │   ├─ 是 → 工程分析：该尺寸是否影响装配或性能？
  │   │         ├─ 是 → 返工或报废（取决于可行性）
  │   │         └─ 否 → 照原样使用（附文档化工程论证）
  │   └─ 否 → 继续
  │
  └─ 是否为供应商导致的不合格品？
      ├─ 是 → 是否急需该材料用于生产？
      │         ├─ 是 → 供应商承担费用进行分选/返工 + 使用可接受单元
      │         │         + 对供应商发出 SCAR + 借记通知（分选/返工费用）
      │         └─ 否 → 退给供应商（附 SCAR + 借记通知或更换 PO）
      └─ 否 → 按上述功能影响路径评估
```

### 1.2 FDA 监管环境（21 CFR 820 / ISO 13485）特定逻辑

医疗设备不合格品有额外要求：

**上市前（设计/开发）：**
- 设计验证/确认期间的不合格品必须记录在设计历史文件（DHF）中
- 处置必须根据 ISO 14971 考虑风险——对患者的伤害严重程度和概率
- 设计不合格品很少接受"照原样使用"；这意味着设计意图有误
- CAPA 几乎总是必需的，以防止在生产中再次发生

**上市后（生产/现场）：**
- 可能影响设备安全或性能的不合格品需要根据 21 CFR 806 评估现场行动（召回、纠正、移除）
- 阈值很低：若有任何合理的伤害可能性，需正式评估
- 将决定"不提交现场行动"的文件记录与决定"提交现场行动"的文件记录同等严格
- 与投诉相关的不合格品必须根据 820.198 与投诉记录关联
- MDR（医疗器械报告）义务：死亡或严重伤害必须在 30 个日历日内向 FDA 报告（需要补救行动的事件为 5 天）

**处置权限矩阵：**

| 处置 | 谁可授权 | 附加要求 |
|---|---|---|
| 报废 | 质量工程师及以上 | 附批次可追溯性的文档 |
| 返工 | 质量工程师 + 制造工程 | 已批准的返工程序；按原始规范重新检验 |
| 修复 | MRB（质量 + 工程 + 制造） | 按 ISO 14971 进行风险评估；若与设计相关则更新 DHF |
| 照原样使用 | MRB + 设计授权 | 风险评估；文档化论证；监管影响评估 |
| 退给供应商 | 质量工程师 + 采购 | 需 SCAR；若反复发生则供应商重新认证 |

### 1.3 汽车环境（IATF 16949）特定逻辑

**客户通知要求：**
- 任何已发运给客户的产品上的不合格品：发现后 24 小时内通知
- 任何影响装配形式、功能或性能的工艺更改：需要 PPAP 重新提交
- 照原样使用处置：通常需要通过客户门户（如 GM 的 GQTS、Ford 的 MQAS、Stellantis 的 SQP）向客户提交正式偏差请求
- 客户可以接受、拒绝或有条件接受（减少数量、有时限偏差）

**控制计划集成：**
- 当不合格品暴露出控制计划的缺陷时，控制计划必须作为纠正措施的一部分进行更新
- 特殊特性（用盾牌或菱形符号标识的安全/重要特性）零容忍不合格品：100% 遏制和立即 CAPA
- 控制计划的反应计划列指定了预定的响应——首先遵循该计划，然后再调查

**受控发运等级：**
- **CS-1（内部受控发运）：** 供应商在正常控制之外增加额外的检验/分选步骤，并在每批货物中提交检验数据
- **CS-2（外部受控发运）：** 在供应商工厂进行第三方检验，费用由供应商承担，直接向客户质量部门报告
- CS-1 和 CS-2 不同于一般供应商升级阶梯——它们是客户强制的遏制措施，而非供应商发起的改进

### 1.4 航空航天环境（AS9100）特定逻辑

**客户/当局批准：**
- 照原样使用和修复处置始终需要根据 AS9100 §8.7.1 获得客户批准
- 若客户是政府合同下的主承包商，政府质量代表（DCMA 或同等机构）也可能需要批准
- 关键特性零件上的不合格品需要通知设计当局
- 如果不合格品表明过程已偏离已认证状态，则根据 AS9102 的首件检验（FAI）失效——可能需要部分或全部 FAI 重新提交

**防假冒零件：**
- 如果不合格品引起对假冒材料的怀疑（意外的成分组成、不正确的标记、可疑文件），则根据 AS9100 §8.1.4 启动防假冒程序
- 将可疑材料与其他 MRB 材料分开隔离
- 若确认假冒，则向 GIDEP（政府-行业数据交换计划）报告
- 不要将可疑的假冒材料退回给供应商——必须隔离，并可能需要保留作为证据

**可追溯性要求：**
- 航空航天不合格品必须在整个处置过程中保持批次、批量、热号和序列号可追溯性
- 报废处置必须包括有序列号零件的文档化销毁，以防止重新进入供应链
- 供应商质量事件可能需要更新 OASIS 数据库

## 2. 根本原因分析方法选择指南

### 2.1 选择决策矩阵

| 因素 | 5 Why | 石川图 + 5 Why | 8D | 故障树分析 |
|---|---|---|---|---|
| **最适合** | 单事件线性因果链 | 多因素需探索类别 | 重复性问题基于团队的解决 | 安全关键需要定量风险 |
| **工作量（小时）** | 1–2 | 4–8 | 20–40（跨所有 D 步骤） | 40–80 |
| **团队规模** | 1–2 人 | 2–4 人 | 5–8 跨职能 | 3–6 主题专家 |
| **何时需要** | 内部过程调查 | 复杂的不合格品 | 客户授权（汽车 OEM） | 航空航天产品安全；医疗设备风险分析 |
| **限制** | 假设单一线性链 | 仍为定性；假设驱动 | 简单问题重型 | 资源密集；需要定量模式的故障率数据 |
| **输出** | 根本原因声明 | 带经验证根本原因的分类原因假设 | 完整 8D 报告（D0-D8） | 带概率分配的故障树图 |

### 2.2 5 Why：何时有效，何时无效

**5 Why 在以下情况有效：**
- 故障是单事件，有明确的前/后状态变化
- 每个"为什么"都可以用数据验证（测量、观察、记录审查）
- 因果链不分支——存在单一主要原因
- 调查可以在 5 次迭代内达到系统性原因（过程、系统或设计问题）

**5 Why 在以下情况失败：**
- 多个独立原因相互作用产生故障（组合原因）
- 分析人员在"人为错误"或"操作员失误"处停止——这永远不是根本原因
- 每个"为什么"都用意见而非经验证的数据回答
- 分析变成循环（为什么 A？因为 B。为什么 B？因为 A。）
- 组织压力推动向避免系统性变革的"方便"根本原因

**每个"为什么"层级的验证协议：**

| Why 层级 | 问题 | 可接受证据 | 不可接受证据 |
|---|---|---|---|
| Why 1（事件） | 实际发生了什么？ | 测量数据、照片、检验记录 | "这个零件是坏的" |
| Why 2（条件） | 什么条件允许了它？ | 过程参数日志、工具状态记录 | "操作员没检查" |
| Why 3（过程） | 为什么过程允许这种条件？ | 作业指导书审查、过程 FMEA 差距 | "一直都是这么做的" |
| Why 4（系统） | 为什么系统没有防止过程差距？ | 系统审核证据、培训记录、控制计划审查 | "我们需要更好的培训" |
| Why 5（管理） | 为什么系统差距未被发现？ | 管理评审记录、资源分配证据、风险评估差距 | "管理层不关心质量" |

### 2.3 石川图：6M 框架深度分析

对于每个 M 类别，具体的调查问题将彻底分析与勾选练习区分开来：

**人（人员）：**
- 操作员是否经过此特定操作的培训 AND 认证？
- 最近的认证更新是什么时候？
- 这是该操作员的正常工作站还是经过交叉培训/临时人员？
- 班次人员编制是否正常，还是在加班/人员短缺期间？
- 检查操作员错误率数据——这是孤立事件还是该个人的模式？

**机器（设备）：**
- 上次预防性维护是什么时候（日期 AND 维护内容）？
- 机器的所有测量功能是否在校准周期内？
- 事件发生前是否记录了任何警报、警告或参数漂移？
- 机器最近是否被修改、修理或更换过工具？
- 检查机器的 Cpk 历史趋势——能力是否在下降？

**材料：**
- 这是新批次的原材料吗？批次何时变化？
- 来料检验结果在正常范围内还是边缘通过？
- 材料证书是否与实际收到的材料一致（热号、轧机、成分）？
- 材料是否正确存储（温度、湿度、保质期、FIFO 轮换）？
- 是否授权了任何材料替换或等效物？

**方法（过程）：**
- 作业指导书是当前版本吗？上次修订是什么时候？
- 操作员是否真的按书面作业指导书操作（观察而非假设）？
- 是否最近更改了任何过程参数（速度、进给、温度、压力、固化时间）？
- 是否最近在此零件或过程上实施了工程变更单（ECO）？
- 文件化的方法与实际方法之间是否存在差距（部落知识）？

**测量：**
- 用于此检验的测量系统是否经过验证（量具 R&R）？
- 量具是否在校准范围内？检查证书和物理状况。
- 是否使用了正确的测量方法（根据控制计划或检验指导书）？
- 测量环境（温度、振动、照明）是否影响结果？
- 对于属性检验（通过/不通过、目视）：检验有效率是多少？

**自然（环境）：**
- 环境条件（温度、湿度）是否在过程规范内？
- 是否发生了任何环境事件（电力波动、压缩空气压力下降、建筑振动）？
- 数据中是否存在班次到班次或日复一日的相关性（温度循环、湿度变化）？
- 工厂的 HVAC 系统是否正常运行？
- 对于洁净室或受控环境过程：环境监测日志是否在规范范围内？

### 2.4 8D 方法论：详细关卡要求

每个 D 步骤都有在前进之前需要完成的具体输出。跳过关卡会产生看起来完整但实际上不解决问题的 8D。

| D 步骤 | 名称 | 必需输出 | 常见失败模式 |
|---|---|---|---|
| D0 | 症状与应急响应 | 已采取的应急响应行动；遏制有效性已确认 | 将遏制与纠正措施混淆 |
| D1 | 团队组建 | 跨职能团队，角色已定义；包括过程负责人和主题专家 | 团队全是质量人员，没有制造或工程人员 |
| D2 | 问题定义 | 完成 IS/IS NOT 分析；问题用数据量化（缺陷率、PPM、Cpk 漂移、投诉数量） | 问题陈述过于宽泛（"质量问题"）或只是重述症状 |
| D3 | 临时遏制 | 在调查进行中保护客户的行动；有效性已验证（遏制后检验数据） | 遏制是"100% 检验"，但未通过已知缺陷挑战验证检验有效性 |
| D4 | 根本原因 | 根本原因通过数据分析或设计实验得到验证；摆脱"人为错误"陷阱 | 根本原因 = 问题重述；无验证数据；在症状处停止 |
| D5 | 纠正措施选择 | 措施针对已验证的根本原因；防错（poka-yoke）优于程序控制 | 纠正措施 = "重新培训操作员"或"增加检验步骤"（两者都很弱） |
| D6 | 实施 | 措施已实施并附文档化证据（更新的 WI、安装的夹具、修改的过程）；基线性能已建立 | 实施日期 = 计划日期，而非实际日期；无实施证据 |
| D7 | 预防 | 系统性行动防止类似过程/产品再次发生；经验教训已记录；FMEA 已更新 | D7 是 D5 的复制粘贴；无横向部署；FMEA 未更新 |
| D8 | 表彰 | 团队已表彰；8D 关闭并附有效性数据 | 关闭时无有效性数据；团队未获表彰 |

### 2.5 故障树分析：构建方法论

**步骤 1：定义顶事件**
- 以具体的、可测量的方式陈述不希望发生的事件
- 示例："精加工零件上轴径超过 USL 25.05mm"
- 不是："坏零件"或"质量问题"

**步骤 2：识别直接原因（第 1 层）**
- 顶事件发生必须为真的条件是什么？
- 使用 AND 门（所有原因都必须存在）和 OR 门（任何单一原因就足够）
- 示例："轴 OD 过大"可能由（OR 门）引起：刀具磨损、刀具偏移错误、材料过大、热膨胀、夹具错位

**步骤 3：分解每个原因（第 2–N 层）**
- 对于每个第 1 层原因，问：什么原因导致它？
- 继续分解直到达到基本事件（有已知故障率或无法进一步分解的事件）
- 示例："刀具磨损"由（AND 门）引起：延长运行时间 + 工具更换间隔不足 + 无过程内 SPC 警报

**步骤 4：量化（当有数据时）**
- 使用历史数据、MTBF 数据或工程估计为基本事件分配概率值
- 通过门逻辑计算顶事件概率
- 识别最小割集（导致顶事件的最小基本事件组合）
- 将纠正措施重点放在概率最高的割集上

## 3. CAPA 编写和验证框架

### 3.1 CAPA 启动标准

**始终为以下情况启动 CAPA：**
- 重复不合格品：12 个月内同一故障模式发生 3 次以上
- 涉及产品性能、安全或监管合规的客户投诉
- 外部审核发现（FDA、公告机构、客户、注册机构）
- 现场故障或产品退货
- 趋势信号：SPC 控制图失控模式（非孤立点）
- 影响现有产品/过程的监管要求变更
- 上市后监督数据表明潜在安全问题

**考虑 CAPA（判断调用）用于：**
- 重复不合格品：12 个月内同一故障模式发生 2 次
- 中等严重性的内部审核发现
- 具有系统性指标的供应商不合格品
- 险情事件（在到达客户之前捕获的不合格品）
- 偏离已验证参数的过程偏差，无产品影响

**不为以下情况启动 CAPA：**
- 具有明确非重复性原因的孤立不合格品（一次性刀具断裂、停电）
- 通过 NCR 处置完全解决且无系统性暗示的不合格品
- 不违反任何规范的客户外观偏好
- 当天捕获并纠正的轻微文档错误

### 3.2 CAPA 措施层级（有效性排序）

纠正措施并非生而平等。按有效性排序，并默认采用最高可行级别：

| 等级 | 控制类型 | 示例 | 有效性 | 典型成本 |
|---|---|---|---|---|
| 1 | **消除** | 重新设计以完全消除故障模式 | ~100% | 高（设计变更、工具） |
| 2 | **替代** | 将材料、供应商或过程更改为不能产生故障的替代品 | ~95% | 中-高 |
| 3 | **工程控制（防错）** | 物理上防止错误装配的夹具；在超出规范条件下停止机器的传感器 | ~90% | 中 |
| 4 | **检测控制** | 自动检验（视觉系统、激光测量仪）100% 检验并自动剔除 | ~85% | 中 |
| 5 | **行政控制** | 更新的作业指导书、修订的程序、检查清单 | ~50-60% | 低 |
| 6 | **培训** | 对现有程序的操作员再培训 | ~30-40% | 低 |

如果您的纠正措施被评为 5 或 6 级，并且 1-4 级的措施可行，则该 CAPA 可能会被审核员质疑。单独的培训永远不足以作为重大不合格品的纠正措施。

### 3.3 CAPA 有效性验证协议

**第 1 阶段：实施验证（目标日期后 2 周内）**

| 必需证据 | 检查内容 | 可接受 | 不可接受 |
|---|---|---|---|
| 文档修订 | WI/程序是否已更新以反映变更？ | 修订附生效日期和培训记录 | "将在下次修订中更新" |
| 物理验证 | 夹具/工具/传感器是否已安装并运行？ | 照片 + 验证记录 | 已下采购订单但未安装 |
| 培训完成 | 相关人员是否已培训？ | 签名的培训记录附能力评估 | 邮件已发送给团队 |
| 系统更新 | QMS 文档、FMEA、控制计划是否已更新？ | 更新的文档附修订和批准 | "将在下次审查中更新" |

**第 2 阶段：有效性验证（90 天监测期）**

| 指标 | 计算 | 通过标准 | 失败标准 |
|---|---|---|---|
| 复发率 | 监测期内同一故障模式计数 | 零复发 | 任何复发 |
| 相关故障率 | 同一过程中相关故障模式计数 | 与基线相比无增加 | 增加表明根本原因不完整 |
| 过程能力 | 受影响特性的 Cpk 或 Ppk | Cpk ≥ 1.33（或目标值） | Cpk 低于 CAPA 前水平 |
| 客户反馈 | 与已解决故障模式相关的投诉 | 零相关投诉 | 任何相关投诉 |

**第 3 阶段：关闭决定**

| 条件 | 决定 |
|---|---|
| 第 1 阶段完成 + 第 2 阶段通过标准满足 | 关闭 CAPA |
| 第 1 阶段完成 + 第 2 阶段显示改进但未完全消除 | 延长监测期 60 天；如果仍在改进，则带条件关闭 |
| 第 1 阶段完成 + 第 2 阶段显示无改进 | 重新打开 CAPA；根本原因不正确或措施不足 |
| 第 1 阶段未完成（措施未实施） | CAPA 保持打开；上报以分配资源 |
| 监测期间复发 | 重新打开 CAPA；不要关闭并为同一问题打开新的 CAPA |

### 3.4 CAPA 时效性标准

| CAPA 阶段 | 目标时间线 | 监管期望 |
|---|---|---|
| 启动和分配 | 触发后 5 个工作日内 | FDA："及时"——通常在知情后 30 天内 |
| 调查和根本原因 | 30 个日历日内 | IATF 16949：根据客户时间线（通常是 10 天初步响应） |
| 纠正措施计划 | 45 个日历日内 | AS9100：根据合同协议 |
| 实施 | 90 个日历日内 | 因复杂性而异；记录延迟并说明理由 |
| 有效性验证开始 | 实施后立即 | 必须在启动时定义 |
| 有效性验证完成 | 实施后 90 天 | FDA：必须证明有效性，而不仅仅是实施 |
| CAPA 关闭 | 启动后 180 个日历日内（总计） | FDA 警告函引用 CAPA 开放超过 1 年作为系统性失败 |

## 4. SPC 解释决策逻辑

### 4.1 控制图选择流程图

```
开始：您正在绘制什么类型的数据？
  │
  ├─ 连续（变量）数据 — 以单位为单位的测量（mm、kg、°C、psi）
  │   ├─ 您是否在采集子组（每个采样事件多次测量）？
  │   │   ├─ 是 → 子组大小（n）是多少？
  │   │   │         ├─ n = 2 到 9  → X-bar / R 图
  │   │   │         ├─ n = 10 到 25 → X-bar / S 图
  │   │   │         └─ n > 25 → X-bar / S 图（考虑减少子组大小）
  │   │   └─ 否（n=1，单个读数）→ 单值 / 移动极差（I-MR）图
  │   │         使用场景：批次过程、破坏性测试、慢过程，
  │   │         或每个单元唯一
  │   └─ （验证变量图的数据正态性假设 — I-MR 对非正态性敏感；
  │       考虑转换或使用非参数替代方案）
  │
  └─ 属性（离散）数据 — 计数或比例
      ├─ 您是否在计数不良品（通过或不通过的单元）？
      │   ├─ 是 → 样本大小是否恒定？
      │   │         ├─ 是 → np 图（不良品计数，固定样本）
      │   │         └─ 否 → p 图（不良品比例，可变样本）
      │   └─ 否 → 您在计数缺陷（每个单元可能有多个缺陷）
      │             ├─ 检查区域/机会是否恒定？
      │             │   ├─ 是 → c 图（每个单元缺陷计数，固定区域）
      │             │   └─ 否 → u 图（每个单元缺陷，可变区域）
      │             └─ （验证 c/u 图的泊松假设）
      └─ （属性图比变量图需要更大的样本量以获得等效灵敏度 —
          最小 p/np ~50，c/u ~25）
```

### 4.2 失控响应协议

当控制图发出失控信号时，根据具体信号遵循此响应：

**规则 1：点超出 3σ 控制限**

| 响应级别 | 行动 | 时间线 |
|---|---|---|
| 立即 | 若正在生产产品则停止过程；从上一个已知良好点开始隔离产出 | 几分钟内 |
| 调查 | 识别可分配原因——发生了什么变化？系统地检查 6M 类别 | 4 小时内 |
| 遏制 | 分选/检验失控期间生产的产品 | 1 个班次内 |
| 纠正 | 解决可分配原因并以加强的监测重新启动生产 | 下一次生产运行前 |
| 文档 | 若产品受影响则发出 NCR；用注释更新控制图 | 24 小时内 |

**规则 2：中心线同一侧连续 9 点（运行）**

| 响应级别 | 行动 | 时间线 |
|---|---|---|
| 调查 | 过程均值可能已偏移。检查：刀具磨损进展、材料批次变化、环境漂移、测量校准偏移 | 1 个班次内 |
| 调整 | 若发现可分配原因：纠正。若未发现可分配原因且过程仍在规范内，继续监测但增加采样频率 | 24 小时内 |
| 重新计算 | 若偏移是有意的（过程改进）或代表新的过程水平，使用新数据重新计算控制限 | 在新水平下达到 25+ 子组后 |

**规则 3：连续 6 点稳定上升或下降（趋势）**

| 响应级别 | 行动 | 时间线 |
|---|---|---|
| 调查 | 过程正在漂移。最常见的原因：刀具磨损、化学品消耗、热漂移、过滤器退化 | 1 个班次内 |
| 预测 | 按当前漂移率，过程何时会超出规范限？这决定了紧迫性 | 立即计算 |
| 预防性行动 | 在过程达到规范限之前调整过程（换刀、化学品补充） | 在预测的规范限交叉之前 |

**规则 4：连续 14 点上下交替（分层/混合）**

| 响应级别 | 行动 | 时间线 |
|---|---|---|
| 调查 | 此模式表明过度控制（篡改）、两个交替流（例如两个主轴、两个型腔）或系统性测量误差 | 24 小时内 |
| 验证 | 检查子组数据是否从应单独绘图的多个来源收集 | 48 小时内 |
| 分层 | 若数据来自多个流，为每个流创建单独的图表 | 1 周内 |

### 4.3 能力指数解释

| Cpk 值 | 解释 | 所需行动 |
|---|---|---|
| Cpk ≥ 2.00 | 六西格玛能力；考虑减少检验频率 | 维持控制；减少检验或跳批的候选 |
| 1.67 ≤ Cpk < 2.00 | 高度有能力；超过大多数客户要求 | 标准监测；满足 IATF 16949 对新过程的要求 |
| 1.33 ≤ Cpk < 1.67 | 有能力；满足大多数行业标准 | 标准 SPC 监测；满足 IATF 16949 对生产的最低要求 |
| 1.00 ≤ Cpk < 1.33 | 边缘能力；产生一些缺陷 | 增加监测频率；启动过程改进；可能需要客户通知 |
| 0.67 ≤ Cpk < 1.00 | 无能力；大量缺陷产生 | 100% 检验直到过程改进；需要 CAPA；需要客户通知 |
| Cpk < 0.67 | 严重无能力 | 停止生产；分选所有在制品和成品；工程审查过程和规范 |

**Cp vs. Cpk 解释：**

| 条件 | 含义 | 行动 |
|---|---|---|
| Cp 高，Cpk 高 | 过程既有能力又居中 | 最佳状态；维持 |
| Cp 高，Cpk 低 | 过程变差小但未居中于目标 | 调整过程均值；不要减少变差（已经很好） |
| Cp 低，Cpk 低 | 过程变差过大，可能也偏离中心 | 首先减少变差（根本性过程改进），然后居中 |
| Cp 低，Cpk ≈ Cp | 过程变差过大但居中 | 减少变差；居中不是问题 |

**Pp/Ppk vs. Cp/Cpk：**

| 指数 | 使用 | 代表 | 何时使用 |
|---|---|---|---|
| Cp/Cpk | 子组内变差（σ_within） | 短期或"潜在"能力 | 评估处于统计控制状态时的过程潜力 |
| Pp/Ppk | 整体变差（σ_overall）包括子组间偏移 | 长期或"实际"性能 | 评估客户在一段时间内实际接收的内容 |
| Pp/Ppk < Cp/Cpk（常见） | 过程均值在子组间偏移 | 子组间变差显著 | 调查导致均值在子组间偏移的原因 |
| Pp/Ppk ≈ Cp/Cpk | 过程随时间稳定 | 最小的子组间变差 | 过程控制良好；长期性能匹配潜力 |

### 5.3 跳批资格要求

**资格标准（所有必须满足）：**

## 6. 供应商质量升级阶梯

### 6.1 详细升级流程

**0 级：正常运行**
- 供应商达到记分卡预期（PPM < 阈值，OTD > 阈值，SCAR 按时关闭）
- 标准来料检验等级
- 季度记分卡审查
- 年度审核（如果基于风险的计划保证）

**1 级：SCAR 已发出**
- **触发：** 单个重大不合格品（> $5,000 影响或安全/监管关注）OR 90 天内同一零件 3+ 次轻微不合格品
- **行动：**
  - 正式发出 SCAR，需要 8D 或同等 RCA
  - 供应商有 10 个工作日进行初始响应（遏制 + 初步根本原因）
  - 供应商有 30 个日历日提供完整的纠正措施计划和实施时间表
  - 质量工程审查 SCAR 响应的充分性
  - 提高受影响零件号的来料检验等级
- **退出标准：** SCAR 已接受并关闭，已验证有效性（90 天监测）

**2 级：供应商受观察/试用期**
- **触发：** SCAR 在时间线内未响应 OR 纠正措施无效（监测期间复发）OR 记分卡连续 2 个季度低于最低阈值
- **行动：**
  - 书面通知供应商试用期状态（质量经理或总监级别）
  - 通知采购；新业务冻结（不授予新零件号）
  - 提高来自此供应商的**所有**零件号（不仅是受影响零件）的检验等级
  - 与供应商质量管理的月度绩效审查电话
  - 供应商必须在 15 个工作日内提交综合改进计划
  - 考虑针对特定故障模式的现场质量审核
- **退出标准：** 改进计划已接受 + 连续 2 个季度达到记分卡最低 + 无新 SCAR

**3 级：受控发运**
- **触发：** 观察期内持续失败 OR 达到客户的关键质量逃逸
- **行动：**
  - 受控发运 1 级（CS-1）：供应商增加额外的分选/检验步骤，每批货物提交数据
  - 若 CS-1 在 60 天内无效：受控发运 2 级（CS-2）：第三方驻场检验员驻供应商工厂，费用由供应商承担
  - 所有分选/检验费用从供应商借记
  - 与供应商副总裁/总经理级别的每周绩效审查电话
  - 开始替代来源的资格认证（如果尚未进行）
- **退出标准：** 在受控发运下连续 90 天零不合格品 + 根本原因完全解决 + 系统性改进已验证

**4 级：新来源资格认证/逐步淘汰**
- **触发：** 在受控发运下无持续改进 OR 供应商不愿意/无法投资于所需的改进
- **行动：**
  - 正式通知供应商将转移业务的意向
  - 加速替代供应商资格认证（加快 PPAP/FAI/首件检验）
  - 随着替代供应商产能爬坡，减少业务分配
  - 在剩余量上维持受控发运
  - 确保最后采购数量覆盖过渡期
  - 记录所有质量成本以供潜在追偿
- **时间线：** 取决于零件复杂性和替代来源准备情况；通常 3-12 个月

**5 级：ASL（批准供应商列表）移除**
- **触发：** 替代来源资格认证完成 OR 供应商质量体系失败是根本性的（例如数据伪造、认证失效）
- **行动：**
  - 正式从批准供应商列表中移除
  - 接收最终货物并按 100% 检验进行检验
  - 我们的工厂中的供应商所有工具：按合同条款处置
  - 供应商工厂中的我们的工具：按合同条款取回
  - 将所有打开的 SCAR 关闭为"供应商已移除"
  - 供应商质量文件保留至少 7 年（监管记录保留）
  - 更新 OASIS（航空航天）或相关行业数据库
- **重新进入：** 如果供应商申请重新资格认证，作为新供应商按完整资格认证流程处理；要求提供系统性问题已解决的证据

### 6.2 升级决策速查

| 情况 | 从何级别开始 | 理由 |
|---|---|---|
| 来自良好供应商的首次轻微不合格品 | 通过 NCR 处理，不升级 | 单个事件不值得正式升级 |
| 来自良好供应商的首次重大不合格品 | 1 级（SCAR） | 重大影响需要正式根本原因 |
| 90 天内来自同一供应商/零件的第三次轻微不合格品 | 1 级（SCAR） | 模式表明系统性问题 |
| SCAR 响应不充分或延迟 | 2 级（观察） | 不响应本身是质量体系失败 |
| 不合格品到达客户 | 至少 2 级；如果涉及安全则为 3 级 | 客户影响要求立即升级 |
| 发现伪造文档 | 至少 4 级；如果确认则为 5 级 | 信任已破；遏制范围未知 |
| 唯一来源供应商有质量问题 | 1 级并行 4 级行动（资格认证替代） | 业务连续性需要适度响应；不要威胁你无法执行的措施 |

## 7. 质量成本计算模型

### 7.1 COQ 类别定义和跟踪

**预防成本（投资以防止缺陷）：**

| 成本要素 | 如何衡量 | 典型范围（占收入百分比） |
|---|---|---|
| 质量规划 | 质量规划活动工时 × 人工费率 | 0.2–0.5% |
| 过程验证/资格认证 | IQ/OQ/PQ 的人工 + 设备 + 材料 | 0.3–0.8% |
| 供应商资格认证 | 审核差旅 + 人工 + 首件成本 | 0.1–0.3% |
| 培训（质量相关） | 工时 × 人工费率 + 培训材料 | 0.1–0.3% |
| SPC 实施/维护 | 软件许可 + 图表维护人工 | 0.1–0.2% |
| 设计审查 / FMEA | 跨职能审查的工时 × 人工费率 | 0.2–0.5% |
| 防错开发 | 错误防护的设计 + 制造 + 验证 | 0.2–0.5% |

**鉴定成本（验证一致性的成本）：**

| 成本要素 | 如何衡量 | 典型范围（占收入百分比） |
|---|---|---|
| 来料检验 | 工时 × 人工费率 + 量具成本 | 0.3–0.8% |
| 过程检验 | 工时 × 人工费率（包括生产等待时间） | 0.5–1.5% |
| 最终检验/测试 | 工时 × 人工费率 + 测试设备折旧 | 0.3–1.0% |
| 校准计划 | 服务合同 + 人工 + 标准 | 0.1–0.3% |
| 审核计划（内部 + 外部） | 人工 + 差旅 + 注册费 | 0.1–0.3% |
| 实验室测试 | 内部实验室成本或外部实验室费 | 0.2–0.5% |

**内部失败成本（出货前捕获的缺陷）：**

| 成本要素 | 如何衡量 | 典型范围（占收入百分比） |
|---|---|---|
| 报废 | 报废材料价值 + 加工人工浪费 | 1.0–3.0% |
| 返工 | 返工操作的人工 + 材料 | 0.5–2.0% |
| 重新检验 | 返工后重新检验的工时 × 人工费率 | 0.1–0.5% |
| MRB 处理 | 处置活动的工时 × 人工费率 | 0.1–0.3% |
| 根本原因调查 | RCA 团队活动的工时 × 人工费率 | 0.2–0.5% |
| 生产延迟 | 隔离、调查导致的生产时间损失 | 0.5–2.0% |
| 供应商分选/遏制 | 第三方分选人工或供应商导致的不合格品的内部分选人工 | 0.1–0.5% |

**外部失败成本（到达客户的缺陷）：**

| 成本要素 | 如何衡量 | 典型范围（占收入百分比） |
|---|---|---|
| 客户退货/抵免 | 抵免备忘录 + 退货运输 + 重新入库人工 | 0.5–2.0% |
| 保修索赔 | 索赔价值 + 处理人工 | 0.5–3.0% |
| 现场服务/维修 | 服务人工 + 差旅 + 零件 | 0.3–1.5% |
| 客户投诉处理 | 调查和响应的工时 × 人工费率 | 0.2–0.5% |
| 召回/现场纠正 | 产品更换 + 通知 + 运输 + 监管 | 0.0–5.0%（高度可变） |
| 监管行动成本 | 罚款、同意令合规、增加的检验 | 0.0–10.0%（触发时是灾难性的） |
| 声誉/业务损失 | 客户流失导致的收入损失（估计） | 难以衡量；通常是直接成本的 2-10 倍 |

### 7.2 COQ 业务案例模型

**计算质量投资的 ROI：**

```
ROI = (失败成本降低 - 投资成本) / 投资成本 × 100%

其中：
  失败成本降低 = (当前内部 + 外部失败成本)
                          - (投资后预计的失败成本)
  投资成本 = 预防成本增加 + 鉴定成本变化
```

**经验法则乘数：**

| 投资类型 | 预期 ROI | 回收期 |
|---|---|---|
| 防错（错误防护） | 5:1 到 20:1 | 3–6 个月 |
| SPC 实施 | 3:1 到 10:1 | 6–12 个月 |
| 供应商发展计划 | 2:1 到 8:1 | 12–24 个月 |
| 过程验证改进 | 4:1 到 15:1 | 6–18 个月 |
| 培训计划升级 | 1:1 到 3:1 | 12–24 个月 |

### 7.3 MRB 决策过程 — 经济模型

当处置不由安全或监管要求决定时，使用经济分析：

**返工 vs. 报废决策：**

```
如果满足以下条件则返工：C_rework + C_reinspect < C_replacement × (1 + premium)

其中：
  C_rework = 直接返工人工 + 材料 + 机器时间
  C_reinspect = 重新检验人工 + 任何额外测试
  C_replacement = 替换单元的采购价格或制造成本
  premium = 进度紧迫性因子（无紧迫性时为 0%，有生产影响时为 10-50%，
            客户交付风险时为 100%+）
```

**分选 vs. 退货决策（针对供应商导致的批次）：**

```
如果满足以下条件则分选：(C_sort < C_return_freight + C_production_delay) AND (expected yield > 70%)

其中：
  C_sort = 分选人工小时 × 费率（人工分选通常 25-50 美元/小时，
           尺寸分选 50-100 美元/小时）
  C_return_freight = 运输成本 + 处理 + 行政
  C_production_delay = （延迟天数 × 每日生产价值风险）
  expected yield = 估计批次中通过分选的百分比
                   （使用样本数据估计）
```

**照原样使用 vs. 分选/返工决策（非安全、非监管）：**

```
如果满足以下条件则照原样使用：Risk_functional ≤ Acceptable_risk
              AND C_use_as_is < C_sort_or_rework
              AND 工程提供文档化论证

其中：
  Risk_functional = P（使用中失败） × 影响（失败）
  C_use_as_is = 保修风险增加（估计） + 文档化成本
  C_sort_or_rework = 直接分选/返工成本 + 生产延迟成本
```

## 8. MRB 决策过程 — 详细工作流

### 8.1 MRB 会议结构

**频率：** 定期每周召开；针对紧急处置（安全关键、阻塞生产）临时召开
**必需与会者：**
- 质量工程（主持，引导和记录）
- 设计/产品工程（功能影响评估）
- 制造工程（可返工性评估）
- 生产/运营（进度影响）
- 采购（供应商相关处置，商业影响）
- 可选：监管事务（如涉及监管影响）、客户质量（如需客户通知）

**标准议程：**
1. 审查待处置的新 NCR（按优先级：安全优先，然后是阻塞生产，然后是时间）
2. 按 NCR 展示数据包（测量值、照片、过程数据）
3. 工程功能影响评估
4. 处置决策及文档化理由
5. 审查老化 NCR（> 15 天未处置）
6. 审查 MRB 指标（数量、周期时间、成本）

### 8.2 MRB 文档要求

每个 MRB 处置必须包括：

| 要素 | 目的 | 提供者 |
|---|---|---|
| NCR 编号和描述 | 识别和可追溯性 | 质量工程 |
| 零件号、版本、数量 | 处置范围 | 质量工程 |
| 违反的规范（条款、尺寸、要求） | 明确不符合的内容 | 质量工程 |
| 测量数据（实际值 vs. 容差） | 处置的证据基础 | 质量工程 / 检验 |
| 照片（如适用） | 视觉证据 | 质量工程 / 检验 |
| 工程论证（照原样使用或修复） | 接受偏差的技术理由 | 设计/产品工程 |
| 风险评估（安全相关项目） | 正式风险评估 | 设计/产品工程 + 质量 |
| 客户批准参考（如需要） | 符合合同/标准 | 质量工程 |
| 处置决策 | 决策本身 | MRB 共识 |
| 所有 MRB 成员的签名 | 问责制和可追溯性 | 所有与会者 |
| 成本影响 | COQ 的财务跟踪 | 质量工程 + 财务 |
| CAPA 参考（如已启动） | 链接到系统性纠正措施 | 质量工程 |

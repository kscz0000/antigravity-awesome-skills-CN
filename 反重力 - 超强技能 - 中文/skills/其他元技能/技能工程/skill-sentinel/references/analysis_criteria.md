# Skill Sentinel 分析标准

## 按维度评分

每个维度初始为 100 分，并按发现的违规项扣分。
最终得分：max(0, min(100, score))。

### 代码质量

| 标准 | 扣分 | 阈值 |
|----------|-----------|---------|
| 圈复杂度 > 10 | 每个函数 -5 | CC > 10 |
| 函数 > 50 行 | 每个函数 -3 | > 50 行 |
| 文件 > 500 行 | 每个文件 -5 | > 500 行 |
| 公共函数缺少 docstring | 每个函数 -1 | 公共函数（无 _ 前缀） |
| bare except | 每次 -8 | `except:` |
| except Exception 且无日志 | 每次 -3 | 无 logging |
| 语法错误 | 每个文件 -15 | SyntaxError |

### 安全性

| 标准 | 扣分 |
|----------|-----------|
| 硬编码密钥（critical） | -20 |
| SQL 注入（high） | -15 |
| 日志中包含 token（high） | -10 |
| 不安全的 HTTP URL（medium） | -5 |
| 输入校验薄弱（low） | -2 |
| 加分：认证模块 | +5 |
| 加分：使用环境变量 | +5 |

### 性能

| 标准 | 扣分 |
|----------|-----------|
| 无 retry/backoff | -10 |
| 无 timeout | -5 |
| 无连接复用 | -3 |
| N+1 查询 | -8 |
| 循环中建立连接 | -5 |
| 加分：retry | +5 |
| 加分：async/并发 | +5 |
| 加分：缓存 | +3 |

### 治理

直接按成熟度等级计分：
- 等级 0（无）：0 分
- 等级 1（操作日志）：25 分
- 等级 2（+ 速率限制）：50 分
- 等级 3（+ 确认机制）：75 分
- 等级 4（+ 告警）：100 分

### 文档

| 标准 | 扣分 |
|----------|-----------|
| frontmatter 缺少 name 字段 | -20 |
| 缺少 description 字段 | -20 |
| 缺少 version 字段 | -3 |
| 触发词过弱（< 10 个词） | -10 |
| 缺少推荐章节 | 每个 -3 |
| References 为空 | -5 |
| SKILL.md < 20 行 | -10 |

### 依赖

| 标准 | 扣分 |
|----------|-----------|
| 缺少 requirements.txt | -15 |
| 未固定版本 | 每个依赖 -2 |
| 已导入但未列出的依赖 | 每个依赖 -2 |

## 综合得分

```
overall = sum(score_dimensao * peso_dimensao) / sum(pesos)
```

默认权重：
- code_quality: 0.20
- security: 0.20
- performance: 0.15
- governance: 0.15
- documentation: 0.15
- dependencies: 0.15

| 25-49 | 需要改进 |

# Daily Gift

## 概述

一个关系感知的礼物引擎，在决定礼物应该是什么之前，先决定礼物是否应该存在。采用五阶段创意流水线生成个性化的每日礼物，输出格式包括 H5（交互式网页）、AI 生成图片或 AI 生成视频。核心设计原则是"创意先于媒介"——创意概念在输出格式选择之前就已锁定。

发布于 ClawHub: https://clawhub.ai/jiawei248/daily-gift

## 何时使用此技能

- 当智能体需要自主判断今天是否值得生成个性化礼物时使用
- 当里程碑、纪念日或情感意义时刻需要用创意作品标记时使用
- 当用户手动请求从引言、诗歌或创意简报生成视觉礼物时使用
- 当你想要每日定时触发的创意输出，且避免重复和模板疲劳时使用

## 工作原理

### 第一阶段：编辑判断

决定今天是否应该有礼物、礼物的分量（跳过/轻触/轻度/标准/重度），以及内容方向（反思、延伸、指南针、镜像、游戏、策展、实用工具等）。此阶段不选择格式。

### 第二阶段：综合 + 礼物论点

从对话上下文中提取六个内容槽位（today_theme、emotion_peaks、historical_echo、open_loop、lobster_judgment、preference_hint）。形成礼物论点 = 锚点（哪个时刻值得成为中心）+ 回馈（智能体给出什么新视角）。如果论点没有回馈，它就不是礼物——只是一个装饰过的日志条目。

### 第 2.5 阶段：创意概念

使用七种思维角度（隐喻翻转、格式混搭、不可能动作、尺度转换、角色互换、时间扭曲、文化混音）生成 5+ 个概念候选。与 8 大类 73 个创意种子库进行交叉授粉。运行三项质量检查：概念质量、概念多样性（8 个家族）、视觉/主题碰撞检测。

### 格式选择

只有在概念锁定后，系统才根据什么最能服务于概念来选择输出格式（H5、图片或视频）。

### 第三阶段：视觉策略

选择视觉方法，规划素材（纯代码、生成背景、混合），选择视觉风格，并针对近期礼物进行预可视化检查以防止重复。

### 第四阶段：渲染

生成最终作品。H5 礼物使用 p5.js/canvas，由内置模板设定质量基线（300-400 行调优代码）。图片和视频礼物使用 AI 生成 API。所有格式都有回退链。

## 核心特性

- **五阶段创意流水线**，阶段之间有明确的质量门控
- **多层防重复**：概念家族、视觉元素、主题、风格、内容方向——每个都在近期礼物的滑动窗口中追踪
- **三层用户品味画像**：第一层（身份——稳定）、第二层（上下文——每 5-7 个礼物更新）、第三层（信号——每次礼物后自动追加）
- **三种运行模式**：入门设置、每日定时、手动触发
- **11 种内容方向**：reflect、extension、compass、mirror、gift-from-elsewhere、play、real-world-nudge、curation、delayed-payoff、openclaw-inner-life、utility
- **8 个概念家族**：borrowed-media、interactive-object、transformation、narrative、data-viz、game-puzzle、real-world、poetic-literary

## 最佳实践

- ✅ 让编辑判断来决定——不是每一天都需要礼物
- ✅ 在选择一个之前生成 5+ 个概念候选
- ✅ 在渲染前检查近期礼物的视觉和主题碰撞
- ✅ 使用品味画像随时间个性化
- ❌ 不要在没有真正创意概念的情况下直接从论点跳到渲染
- ❌ 不要每次都默认"反思今天"——要变化内容方向
- ❌ 不要在锁定概念之前选择格式

## 限制

- 图片/视频生成需要 API 密钥（可选——H5 无需它们即可工作）
- 定时模式在智能体的主会话中运行，以获取完整的对话上下文访问
- `scripts/` 中的 Shell 脚本调用外部 API 进行渲染和素材获取
- 技能创建并管理本地工作区文件用于状态、历史和品味画像

## 安全与注意事项

- 技能会创建每日礼物投递的定时任务。请审核并批准定时设置步骤。
- `scripts/` 中的 Shell 脚本调用外部 API（图片生成、视频生成、素材托管）。只有在审核哪些脚本使用它们后才提供 API 密钥。
- 用户品味数据和礼物历史本地存储在 `workspace/daily-gift/`。除配置的渲染 API 外，不向外部服务发送数据。
- 技能读取对话上下文和记忆文件来指导编辑判断——这是个性化的核心，但也意味着它在智能体工作区内有广泛的读取权限。

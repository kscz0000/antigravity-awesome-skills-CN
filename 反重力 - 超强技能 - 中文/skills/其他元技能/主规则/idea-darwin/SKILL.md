# Idea Darwin 引擎

基于轮次的创意迭代系统，将想法视为竞争的生物体 — 通过结构化轮次进行评分、筛选、杂交和进化，筛选出最强的概念。

## 概述

大多数创意管理工具都是文件柜：存储想法、打标签、让它们腐烂。Idea Darwin 颠覆了这个范式 — 它不是组织想法，而是让想法**竞争**。每个想法都是进化岛上活着的物种。每一轮，最适应的想法被深化，不同的想法交叉授粉产生意想不到的杂交品种，外部刺激触发变异。

## 何时使用此技能

- 当你有许多分散的想法，需要系统性地评估和发展它们时使用
- 当你想发现不同领域想法之间意想不到的关联时使用
- 当你需要结构化迭代而非一次性头脑风暴时使用
- 当你想要评分框架来优先考虑哪些想法值得更多投入时使用

## 核心概念

### 进化岛隐喻

你的想法在这个岛上活着。像生物体一样，它们遵循三条核心法则：

1. **进化** — 每一轮，系统通过结构化研究深化最可行的想法：填补逻辑空白、澄清路径、识别风险。
2. **杂交** — 系统让不同的想法交叉授粉。工作中的技术方法遇到日常生活中的观察，产生你从未想象过的方向。
3. **变异** — 外部刺激（行业新闻、理论、对话）触发变异，产生全新的物种。

### 物种卡片

每个想法获得一张结构化卡片，记录：核心问题、完整描述、谱系（父/子ID）、六维评分和变更历史。

### 六维评分

| 维度 | 权重 | 衡量内容 |
|---|---|---|
| 新颖性 | 10% | 真正突破还是重复？ |
| 可行性 | 20% | 技术和资源上可实现吗？ |
| 价值 | 20% | 成功后的影响？ |
| 逻辑性 | 20% | 内部一致，无空白？ |
| 杂交潜力 | 10% | 组合后能激发新东西？ |
| 可验证性 | 20% | 能设计验证路径吗？ |

### 想法生命周期

```
seed → exploring → refining → crossing → validated → dormant
```

用户对所有生死决策拥有最终决定权。系统仅提供建议。

## 分步指南

### 1. 写下你的想法

创建 `ideas.md` 文件：

```markdown
## 学习我风格的个人知识库
我想要一个系统，读取我写的所有内容，逐渐学习我的思维方式。

## 通勤转播客转换器
通勤时录制语音备忘录，自动转换为播客脚本。
```

### 2. 初始化你的岛屿

```
/idea-darwin init
```

### 3. 开始进化

```
/idea-darwin round
```

### 4. 持续喂养岛屿

向 `ideas.md` 追加新想法，向 `stimuli.md` 添加环境变量。

## 示例

### 示例 1：初始化

```
/idea-darwin init --budget 8 --actions 3
```

### 示例 2：运行多轮

```
/idea-darwin round 3
```

### 示例 3：管理想法

```
/idea-darwin dormant IDEA-0005
/idea-darwin wake IDEA-0005
```

## 最佳实践

- 推荐：想法写得多粗糙都行 — 系统会结构化它们
- 推荐：添加外部刺激以防止想法趋同
- 推荐：运行颠覆轮次以发现被忽视的想法
- 不推荐：过度精炼初始想法 — 让进化来筛选
- 不推荐：忽略简报中的"需要决策"部分

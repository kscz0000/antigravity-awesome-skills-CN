## 非显而易见发现（前 3 个）

1. **成功后扩展占主导** — 大多数范围增长发生在初始完成成功*之后*，而非糟糕规划。（高置信度）  
2. **文件定位 > 验收标准** — 缺失具体文件与重规划的相关性（44% vs 12%）高于缺失标准。早期锚定智能体研究。（高）  
3. **前端构建是隐形杀手** — 后期 TypeScript/导入失败反复增加 2–4 个循环。不存在预检。（高）

## 建议（前 4 个）

1. **阶段后拆分会话** — 成功完成后开始新对话，避免上下文膨胀和范围蔓延。预期：+13% 首次成功率。（高）  
2. **强制文件定位** — 在提示词优化器中添加预检，标记缺失文件/模块引用。预期：重规划率减半。（高）  
3. **添加前端预检** — 在涉及前端的会话早期运行 `npm run build`。消除常见的后期阻塞。（高）  
4. **修复认证测试固件** — 用明文密码种子测试用户或本地 E2E 绕过 bcrypt。解锁浏览器测试。（高）

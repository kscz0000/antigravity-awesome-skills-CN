# 自动化剧本

HubSpot、Salesforce、日程安排工具和跨工具自动化的平台特定工作流配方。

## HubSpot 工作流配方

### 1. MQL 警报和分配

**名称：** MQL 通知和任务创建
**触发器：** 联系人属性"Lifecycle Stage"更改为"Marketing Qualified Lead"
**操作：**
1. 在销售团队中轮询分配联系人所有者
2. 向联系人所有者发送内部邮件通知，附带线索上下文
3. 创建任务："跟进 [联系人姓名]" — 4 小时内到期
4. 向 #sales-alerts 频道发送 Slack 通知
5. 加入"MQL Follow-Up"序列（如果使用 HubSpot Sequences）
**结果：** 每个 MQL 都能即时分配并有明确的 SLA
**注意：** 设置加入条件以排除已由代表拥有的线索

---

### 2. MQL SLA 升级

**名称：** MQL SLA 违规警报
**触发器：** 联系人属性"Lifecycle Stage"等于"MQL"且"Days since last contacted"大于 0.5（12 小时）
**操作：**
1. 向联系人所有者发送内部邮件："SLA 警告：[联系人姓名] 尚未被联系"
2. 如果 24 小时后仍无活动 → 向销售经理发送警报
3. 如果 48 小时后仍无活动 → 通过轮询重新分配联系人所有者
4. 为新所有者创建任务："紧急：联系 [联系人姓名] — 因 SLA 违规重新分配"
**结果：** 没有 MQL 会超过 48 小时未被处理
**注意：** 排除最后活动类型为"电话"或"会议"的联系人（已参与）

---

### 3. 线索评分更新和 MQL 晋升

**名称：** 评分阈值自动 MQL
**触发器：** 联系人属性"HubSpot Score"大于或等于 65
**操作：**
1. 将生命周期阶段设置为"Marketing Qualified Lead"
2. 将"MQL Date"设置为当前日期
3. 从营销培育工作流中排除
4. 触发 MQL 警报工作流（配方 #1）
**结果：** 线索达到评分阈值时自动晋升为 MQL
**注意：** 为现有客户和竞争对手添加抑制列表

---

### 4. 会议预订通知

**名称：** 会议预订警报给 AE
**触发器：** 为联系人记录会议活动（通过 Calendly/HubSpot meetings）
**操作：**
1. 向联系人所有者发送包含会议详情的内部邮件
2. 将联系人属性"Last Meeting Booked"更新为当前日期
3. 如果生命周期阶段为"Lead" → 更新为"MQL"
4. 创建任务："准备与 [联系人姓名] 的会议" — 会议前 1 小时到期
5. 向 #meetings 频道发送 Slack 通知
**结果：** AE 为每场会议做好准备，拥有完整上下文
**注意：** 在通知邮件中包含最近的页面浏览和内容下载

---

### 5. 成交交接给 CS

**名称：** 客户入职触发器
**触发器：** 交易阶段更改为"Closed Won"
**操作：**
1. 将关联联系人生命周期阶段更新为"Customer"
2. 将"Customer Since"日期设置为当前日期
3. 将联系人所有者分配给 CS 团队成员（基于细分/区域）
4. 为 CS 创建任务："安排与 [公司名称] 的启动会议" — 2 个工作日内到期
5. 将联系人加入"Customer Onboarding"邮件序列
6. 向 CS 经理发送内部通知
7. 从所有销售序列中移除
**结果：** 从销售到客户成功的无缝交接
**注意：** 在 CS 通知中包含交易备注、合同价值和关键利益相关者

---

### 6. 陈旧交易警报

**名称：** 管道卫生 — 陈旧交易检测
**触发器：** 交易属性"Days in current stage"大于 [该阶段平均值的 2 倍]
**操作：**
1. 向交易所有者发送内部邮件："交易陈旧警报：[交易名称] 已在 [阶段] 停留 [X] 天"
2. 创建任务："更新或关闭 [交易名称]" — 3 个工作日内到期
3. 如果 7 天后仍无更新 → 提醒销售经理
4. 添加到"陈旧交易"仪表盘列表
**结果：** 管道保持清洁，预测保持准确
**注意：** 为每个阶段自定义阈值（发现：14 天，提案：10 天，谈判：21 天）

---

### 7. 回收线索培育重新进入

**名称：** MQL 回收到培育
**触发器：** 联系人属性"Sales Rejection Reason"有值（任意值）
**操作：**
1. 将生命周期阶段更新为"Recycled"
2. 将参与度评分重置为基线（保留适配度评分）
3. 加入"回收线索培育"序列（较低频率）
4. 将"Recycle Date"设置为当前日期
5. 设置重新加入触发：如果 HubHub Score 再次超过阈值，重新触发 MQL 工作流
**结果：** 被拒绝的线索获得第二次机会，不会堵塞管道
**注意：** 将回收转 MQL 转化率作为单独指标跟踪

---

### 8. 线索活动摘要

**名称：** 每日线索活动汇总
**触发器：** 定时 — 每天当地时间上午 8:00
**操作：**
1. 筛选联系人：生命周期阶段为"SQL"或"Opportunity"且过去 24 小时内有网站活动
2. 向每位联系人所有者发送摘要邮件，包含其线索的活动
3. 包含：访问的页面、下载的内容、打开/点击的邮件
**结果：** 销售代表每天开始时就知道哪些线索是活跃的
**注意：** 仅包含有意义的活动（排除单次首页访问）

## Salesforce Flow 等效方案

### 1. MQL 警报和分配（Salesforce Flow）

**类型：** 记录触发 Flow
**对象：** Lead
**触发器：** Lead 字段"Status"更改为"MQL"
**Flow 步骤：**
1. Get Records：查询"Rep Assignment"自定义对象获取下一个可用代表
2. Update Records：将 Lead Owner 设置为分配的代表
3. Create Records：创建任务 — "联系 MQL：{Lead.Name}"，到期日期 = NOW + 4 小时
4. Action：向新线索所有者发送邮件警报
5. Update Records：更新"Rep Assignment"最后分配时间戳
**注意：** 使用自定义"Rep Assignment"对象管理轮询状态

### 2. SLA 升级（Salesforce Flow）

**类型：** 定时触发 Flow
**计划：** 工作时间每 4 小时
**Flow 步骤：**
1. Get Records：Status = "MQL" 且 LastActivityDate < TODAY - 1 的 Leads
2. Decision：线索是否超过 48 小时无活动？
   - 是 → 重新分配给下一个代表，创建紧急任务，提醒经理
   - 否 → 向当前所有者发送提醒邮件
**注意：** 配合 Process Builder 用于初始分配的实时警报

### 3. 管道阶段自动化（Salesforce Flow）

**类型：** 记录触发 Flow
**对象：** Opportunity
**触发器：** Stage 字段更新
**Flow 步骤：**
1. Decision：更改为哪个阶段？
2. 对于每个阶段：
   - **发现：** 创建任务"完成发现问卷"
   - **演示：** 创建任务"准备演示环境"
   - **提案：** 创建任务"发送提案" + 如果 ACV > $25K 提醒交易审批
   - **成交：** 触发 CS 交接（创建 Case，分配 CS 所有者，发送欢迎邮件）
   - **失败：** 创建任务"记录失败原因" + 添加到赢单/输单分析报告

### 4. 陈旧交易检测（Salesforce Flow）

**类型：** 定时触发 Flow
**计划：** 每天上午 7:00
**Flow 步骤：**
1. Get Records：Days_In_Stage > Stage_SLA_Threshold 的开放 Opportunities
2. 循环遍历结果：
   - 创建任务："更新陈旧交易：{Opportunity.Name}"
   - 向 Opportunity Owner 发送邮件
   - 如果 Days_In_Stage > 2 倍阈值 → 向 Owner 的 Manager 发送邮件
3. 更新自定义字段"Stale Flag" = true 以便仪表盘可见

## Calendly / SavvyCal 集成模式

### 轮询会议日程安排

**Calendly 设置：**
1. 创建包含所有合格代表的团队事件类型
2. 分配方式："优化均匀分配"
3. 可用性：每位代表管理自己的日历
4. 缓冲时间：会议前后各 15 分钟
5. 最短预约时间：4 小时（避免临时预订）

**CRM 集成：**
1. Calendly webhook 在预订时触发
2. 将受邀者邮箱匹配到 CRM 联系人
3. 如果联系人存在 → 将会议分配给联系人所有者（如果已拥有则覆盖轮询）
4. 如果是新联系人 → 创建线索，通过路由规则分配，记录会议
5. 将生命周期阶段设置为 MQL（会议 = 高意向）

### SavvyCal 设置

**相比 Calendly 的优势：**
- 基于优先级的日程安排（优先显示特定时间段）
- 叠加日历（在一个视图中显示团队可用性）
- 每位代表的个性化预订链接

**集成模式：**
1. 创建带优先规则的团队日程安排链接
2. 预订时 webhook → Zapier/Make → CRM
3. 匹配或创建联系人，分配所有者，创建任务
4. 发送包含会议准备材料的确认

### 按条件路由会议

```
预订表单提交
├─ 公司规模 > 500？（表单字段）
│  ├─ 是 → 路由到企业 AE 日历
│  └─ 否 ↓
├─ 现有客户？（CRM 查询）
│  ├─ 是 → 路由到客户所有者的日历
│  └─ 否 ↓
└─ 在 SDR 团队中轮询分配
```

### 缺席工作流

**触发器：** 会议时间已过 + 30 分钟内未记录会议备注
**操作：**
1. 在预定会议时间后等待 30 分钟
2. 检查：是否记录了电话或会议？
   - 是 → 无操作
   - 否 → 向潜在客户发送"很抱歉我们错过了您"邮件
3. 创建任务："与 [联系人姓名] 重新安排" — 下一个工作日到期
4. 如果第二次缺席 → 标记联系人并提醒经理

## Zapier 跨工具模式

### 1. 新线索 → CRM + Slack + 任务

**触发器：** 新表单提交（Typeform、HubSpot、Webflow）
**操作：**
1. 在 CRM 中创建/更新联系人
2. 使用 Clearbit 增强（如可用）
3. 向 Slack #new-leads 发布增强数据
4. 在项目管理工具（Asana、Linear）中创建任务

### 2. 会议预订 → CRM + 准备邮件

**触发器：** 新 Calendly/SavvyCal 预订
**操作：**
1. 查找或创建 CRM 联系人
2. 将生命周期阶段更新为 MQL
3. 向分配的代表发送准备邮件（包含 CRM 链接、LinkedIn 资料、最近活动）
4. 创建会前任务

### 3. 交易成交 → 入职流程

**触发器：** CRM 交易阶段更改为"Closed Won"
**操作：**
1. 在 CS 工具（Vitally、Gainsight、ChurnZero）中创建客户记录
2. 添加到入职项目模板
3. 通过邮件工具发送欢迎邮件
4. 创建 Slack 频道：#customer-[公司名称]
5. 在 Slack 中通知 CS 团队

### 4. 线索评分 → 跨工具同步

**触发器：** CRM 线索评分超过 MQL 阈值
**操作：**
1. 更新营销自动化平台状态
2. 添加到重定向受众（Facebook、Google Ads）
3. 触发 SDR 外联序列
4. 在分析工具（Mixpanel、Amplitude）中记录事件

### 5. SLA 违规 → 多渠道警报

**触发器：** CRM 任务逾期（MQL 跟进任务）
**操作：**
1. 向代表发送 Slack 私信
2. 向代表发送邮件
3. 如果逾期 2+ 小时 → 向经理发送 Slack 私信
4. 如果逾期 4+ 小时 → 在 CRM 中重新分配（通过 webhook 返回 CRM）

### 6. 每周管道摘要

**触发器：** 定时 — 每周一上午 8:00
**操作：**
1. 查询 CRM 获取管道摘要（总价值、新交易、陈旧交易、预期成交）
2. 格式化为摘要
3. 发布到 Slack #sales-team
4. 向销售领导层发送邮件摘要

# 生命周期阶段定义

线索生命周期阶段、按业务类型的 MQL 标准、SLA 以及拒绝/回收工作流的完整模板。

## 阶段模板

### 订阅者

**进入条件：**
- 订阅博客、通讯或内容更新
- 无需公司信息

**退出条件：**
- 通过表单或增强提供公司信息
- 单次会话访问 3+ 页面
- 下载门控内容

**负责人：** 营销（自动化）

**进入时操作：**
- 加入通讯培育
- 开始跟踪参与度评分

---

### 线索

**进入条件：**
- 已识别的联系人有姓名 + 邮箱 + 公司
- 可能来自表单填写、增强或导入

**退出条件：**
- 达到 MQL 阈值（适配度 + 参与度）
- 由营销/SDR 手动确认

**负责人：** 营销

**进入时操作：**
- 增强联系人数据（公司规模、行业、角色）
- 开始评分
- 加入相关培育序列

---

### MQL（营销确认线索）

**进入条件：**
- 满足适配度评分阈值和参与度评分阈值
- 或触发高意向行为（演示请求、定价页面 + 表单填写）

**退出条件：**
- 销售接受（成为 SQL）
- 销售拒绝（带原因代码回收到培育）
- SLA 内无响应（升级给经理）

**负责人：** 营销 → 销售（交接）

**进入时操作：**
- 即时警报给分配的销售代表
- 创建带 4 小时 SLA 的跟进任务
- 暂停营销培育序列
- 记录所有近期活动供销售参考

---

### SQL（销售确认线索）

**进入条件：**
- 销售代表已进行确认对话
- 确认：预算、权限、需求或时间线（至少 4 项中的 2 项）

**退出条件：**
- 创建带预期价值的商机
- 取消资格（带原因代码回收）

**负责人：** 销售（SDR 或 AE）

**进入时操作：**
- 在 CRM 中更新生命周期阶段
- 如果是 SDR 确认则通知 AE
- 如果尚未在对话中则开始销售序列

---

### 商机

**进入条件：**
- 在 CRM 中创建正式商机
- 分配交易价值、结单日期和阶段

**退出条件：**
- 成交或失败

**负责人：** 销售（AE）

**进入时操作：**
- 添加到管道报告
- 创建交易任务（提案、演示等）
- 如果交易可能成交则通知 CS

---

### 客户

**进入条件：**
- 成交的交易
- 合同签署并设定付款条款

**退出条件：**
- 流失、扩展或续约

**负责人：** 客户成功 / 客户管理

**进入时操作：**
- 触发入职序列
- 分配 CS 经理
- 安排启动会议
- 从所有销售序列中移除

---

### 推广者

**进入条件：**
- NPS 评分 9-10，或活跃推荐行为
- 同意参与案例研究、推荐或推荐计划

**退出条件：**
- 持续参与项目

**负责人：** 客户成功 + 营销

**进入时操作：**
- 加入倡导计划
- 请求案例研究或推荐
- 邀请加入推荐计划
- 在营销活动中展示（经许可）

---

## 按业务类型的 MQL 标准模板

### PLG（产品驱动增长）

**适配度评分（40% 权重）：**

| 属性 | 分值 |
|------|------|
| 公司规模 10-500 | +15 |
| 公司规模 500-5000 | +20 |
| 目标行业 | +10 |
| 决策者角色 | +15 |
| 使用互补工具 | +10 |

**参与度评分（60% 权重）— 重度加权产品使用：**

| 信号 | 分值 |
|------|------|
| 创建免费账户 | +15 |
| 完成入职 | +20 |
| 使用核心功能 3+ 次 | +25 |
| 邀请团队成员 | +20 |
| 达到使用限制 | +15 |
| 访问定价页面 | +10 |

**MQL 阈值：** 65 分

---

### 销售驱动（企业）

**适配度评分（60% 权重）— 重度加权适配度：**

| 属性 | 分值 |
|------|------|
| 公司规模 500+ | +20 |
| 目标行业 | +15 |
| VP+ 职位 | +20 |
| 预算权限已确认 | +15 |
| 使用竞品 | +10 |

**参与度评分（40% 权重）：**

| 信号 | 分值 |
|------|------|
| 请求演示 | +25 |

| 下载白皮书 | +10 |
| 访问定价页面 2+ 次 | +15 |
| 与销售邮件互动 | +10 |

**MQL 阈值：** 70 分

---

### 中型市场（平衡）

**适配度评分（50% 权重）：**

| 属性 | 分值 |
|------|------|
| 公司规模 50-1000 | +15 |
| 目标行业 | +10 |
| 经理+ 职位 | +15 |
| 目标地区 | +10 |

**参与度评分（50% 权重）：**

| 信号 | 分值 |
|------|------|
| 演示请求 | +25 |
| 免费试用注册 | +20 |
| 定价页面访问 | +10 |
| 内容下载（2+） | +10 |
| 邮件点击（3+） | +10 |
| 网络研讨会参与 | +10 |

**MQL 阈值：** 60 分

## SLA 模板

### MQL 到 SQL SLA

| 指标 | 目标 | 升级 |
|------|------|------|
| 首次联系尝试 | 4 个工作小时内 | 4 小时时提醒销售经理 |
| 确认决策 | 48 小时内 | 48 小时时自动升级 |
| 安排会议（如确认） | 5 个工作日内 | 每周管道审查标记 |

### SQL 到商机 SLA

| 指标 | 目标 | 升级 |
|------|------|------|
| 完成发现通话 | SQL 后 3 个工作日内 | 提醒 AE 经理 |
| 创建商机 | SQL 后 5 个工作日内 | 管道审查标记 |

### 商机到成交 SLA

| 指标 | 目标 | 升级 |
|------|------|------|
| 交付提案 | 演示后 5 个工作日内 | AE 经理提醒 |
| 交易在阶段陈旧 | 该阶段平均天数的 2 倍 | 管道审查标记 |
| 结单日期推迟 2+ 次 | 立即 | 需要预测审查 |

## 线索拒绝和回收

### 拒绝原因代码

| 代码 | 原因 | 回收操作 |
|------|------|----------|
| **FIT-01** | 公司太小 | 培育；如果公司增长则重新评分 |
| **FIT-02** | 行业不匹配 | 归档；不回收 |
| **FIT-03** | 角色错误/无权限 | 培育；监控组织变化 |
| **ENG-01** | 3 次尝试后无响应 | 90 天后回收到培育 |
| **ENG-02** | 有兴趣但时机不对 | 回收到培育；60 天后重新接触 |
| **QUAL-01** | 无预算 | 90 天后回收到培育 |
| **QUAL-02** | 使用竞品，已锁定 | 回收；在合同续约前触发 |
| **QUAL-03** | 不是真实项目 | 归档；不回收 |

### 回收工作流

1. 销售带原因代码拒绝 MQL
2. CRM 将生命周期阶段更新为"Recycled"
3. 线索进入回收培育序列（与原始培育不同）
4. 参与度评分重置为基线（保留适配度评分）
5. 如果线索重新参与并超过 MQL 阈值，重新路由给销售并标记"Recycled MQL"
6. 单独跟踪回收 MQL 转化率

### 回收培育序列

- **频率：** 双周或每月（比初始培育频率低）
- **内容：** 行业洞察、案例研究、产品更新
- **时长：** 6 个月，如无参与则归档
- **重新 MQL 触发：** 高意向行为（演示请求、重新访问定价页面）

# 线索路由规则

决策树、平台特定配置、区域路由、ABM 路由和响应速度基准。

## 路由决策树

使用此模板映射你的路由逻辑：

```
新线索到达
│
├─ 这是命名/目标账户吗？
│  ├─ 是 → 路由给分配的账户所有者
│  └─ 否 ↓
│
├─ ACV 可能 > $50K 吗？（基于公司规模 + 行业）
│  ├─ 是 → 路由给企业 AE 团队
│  └─ 否 ↓
│
├─ 这是带团队使用的 PLG 注册吗？
│  ├─ 是 → 路由给 PLG 销售专员
│  └─ 否 ↓
│
├─ 线索匹配某个区域吗？
│  ├─ 是 → 路由给区域所有者
│  └─ 否 ↓
│
└─ 默认：在可用代表中轮询分配
   └─ 如无代表可用：分配给团队队列，1 小时 SLA
```

根据你的业务自定义此树。关键原则：**先路由到最具体的匹配，再回退到通用规则。**

## 轮询配置

### 基本轮询规则

1. 在合格代表间均匀分配线索
2. 跳过休假、已达容量或管道已满的代表
3. 按配额完成情况加权（低于配额的代表略微优先）
4. 每周或每月重置分配计数
5. 记录每次分配以供审计

### HubSpot 轮询设置

**使用 HubSpot 的轮询工具：**
- 导航到 Automation → Workflows
- 触发器：联系人属性"Lifecycle Stage"等于"MQL"
- 操作：在选定用户中轮询分配联系人所有者
- 选项：均匀分配，跳过不可用的所有者
- 分配后添加延迟 + 任务创建

**使用工作流自定义轮询：**
1. 创建自定义属性"Rotation Counter"（数字）
2. 工作流触发器：新 MQL 创建
3. 按轮询计数器值分支（0、1、2... 每个代表一个）
4. 将联系人所有者设置为对应代表
5. 递增计数器（达到最大值时重置）
6. 创建带 SLA 截止时间的跟进任务

### Salesforce 轮询设置

**使用 Lead Assignment Rules：**
1. Setup → Feature Settings → Marketing → Lead Assignment Rules
2. 按优先级顺序创建规则条目（最具体的在前）
3. 对于轮询：使用分配规则 + 自定义逻辑

**使用 Flow 进行高级路由：**
1. 在 Lead 创建时创建记录触发 Flow
2. Get Records：查询自定义"Rep Queue"对象获取下一个可用代表
3. Decision 元素：检查代表可用性、容量、区域
4. Update Records：分配线索所有者
5. Create Task：带 SLA 的跟进任务
6. Update "Rep Queue"：跟踪最后分配

## 区域路由

### 按地理位置

| 区域 | 州/省 | 分配团队 |
|------|-------|----------|
| 西部 | CA, WA, OR, NV, AZ, UT, CO, HI | 西部团队 |
| 中部 | TX, IL, MN, MO, OH, MI, WI, IN | 中部团队 |
| 东部 | NY, MA, PA, NJ, CT, VA, FL, GA | 东部团队 |
| 国际 | 所有非美国 | 国际团队 |

### 按公司规模

| 细分 | 公司规模 | 团队 |
|------|----------|------|
| SMB | 1-50 员工 | 内部销售 |
| 中型市场 | 51-500 员工 | 中型市场 AE |
| 企业 | 501-5000 员工 | 企业 AE |
| 战略 | 5000+ 员工 | 战略客户团队 |

### 按行业

| 垂直 | 行业 | 专员 |
|------|------|------|
| 科技 | SaaS、IT 服务、硬件 | 科技垂直代表 |
| 金融 | 银行、保险、金融科技 | 金融垂直代表 |
| 医疗 | 医院、制药、健康科技 | 医疗垂直代表 |
| 通用 | 其他所有 | 通用池（轮询） |

### 混合区域模型

组合多个维度实现精准路由：

```
线索到达
├─ 公司规模 > 1000？
│  ├─ 是 → 企业团队
│  │  └─ 按地理位置子路由
│  └─ 否 ↓
├─ 行业 = 医疗或金融？
│  ├─ 是 → 垂直专员
│  └─ 否 ↓
└─ 在通用池中轮询分配
   └─ 按地理位置偏好加权
```

## 命名账户 / ABM 路由

### 设置

1. **定义目标账户列表**（通常 50-500 个账户）
2. **在 CRM 中分配账户所有者**（每个账户 1 名代表）
3. **匹配逻辑：** 来自目标账户域的任何线索路由给账户所有者
4. **匹配规则：**
   - 邮箱域匹配（主要）
   - 公司名称模糊匹配（次要，需人工审查）
   - IP 到公司解析（第三级，用于匿名访客）

### ABM 路由规则

| 层级 | 账户类型 | 路由 | 响应 SLA |
|------|----------|------|----------|
| 一级 | 前 20 个战略账户 | 命名所有者，即时警报 | 1 小时 |
| 二级 | 前 100 个目标账户 | 命名所有者，标准警报 | 4 小时 |
| 三级 | 目标行业/规模匹配 | 区域或轮询 | 同一工作日 |

### 多联系人处理

当同一账户的多个联系人参与时：
- 将所有联系人路由给**同一个账户所有者**
- 通知所有者有新联系人进入
- 跟踪账户级参与度评分（所有联系人总和）
- 当一个账户有 3+ 联系人参与时触发"购买委员会"警报

## 响应速度数据

### 响应时间对转化的影响

| 响应时间 | 相对确认率 | 备注 |
|----------|------------|------|
| 5 分钟内 | **21 倍**更可能确认 | 黄金标准 |
| 5-10 分钟 | 10 倍更可能 | 仍然很强 |
| 10-30 分钟 | 4 倍更可能 | 大多数情况可接受 |
| 30 分钟 - 1 小时 | 2 倍更可能 | 低于最佳实践 |
| 1-24 小时 | 基线 | 行业平均 |
| 24+ 小时 | 比基线低 60% | 线索实际上已冷 |

来源：Lead Connect, InsideSales.com

### 实施响应速度

1. **即时通知** — MQL 创建时向代表发送推送通知 + 邮件
2. **带计时器的自动任务** — 创建带 5 分钟 SLA 倒计时的任务
3. **升级链：**
   - 5 分钟：提醒原始代表
   - 15 分钟：提醒后备代表
   - 30 分钟：提醒经理
   - 1 小时：将线索重新分配给下一个可用代表
4. **衡量和报告** — 每周跟踪实际响应时间；表彰快速响应者

### 响应速度自动化

**触发器：** 新 MQL 创建
**操作：**
1. 通过路由规则分配给代表（即时）
2. 向代表发送推送通知 + 邮件
3. 创建任务："联系 [线索姓名] — 5 分钟 SLA"
4. 启动 SLA 计时器
5. 如果 15 分钟内无活动记录 → 提醒后备代表
6. 如果 30 分钟内无活动 → 提醒经理
7. 如果 60 分钟内无活动 → 通过轮询重新分配

### 衡量响应速度

每周跟踪这些指标：
- **首次联系平均时间**（从 MQL 创建到首次电话/邮件）
- **首次联系中位时间**（受异常值影响较小）
- **SLA 内联系的线索百分比**（目标：90%+）
- **按时段的联系率**（识别覆盖缺口）
- **按响应时间的转化率**（证明速度的投资回报）

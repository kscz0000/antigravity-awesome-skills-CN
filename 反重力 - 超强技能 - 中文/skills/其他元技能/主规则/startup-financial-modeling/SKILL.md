# 创业公司财务建模

为早期创业公司构建完整的3-5年财务模型，包含收入预测、成本结构、现金流分析和场景规划。

## 适用场景

- 处理创业公司财务建模任务或工作流
- 需要创业公司财务建模的指导、最佳实践或检查清单

## 不适用场景

- 任务与创业公司财务建模无关
- 需要超出此范围的其他领域或工具

## 使用说明

- 明确目标、约束条件和所需输入。
- 应用相关最佳实践并验证结果。
- 提供可执行的步骤和验证方法。
- 如需详细示例，请打开 ``resources/implementation-playbook.md``。

## 概述

财务建模为创业公司的战略制定、融资和运营规划提供量化基础。通过基于队列的收入建模、详细的成本结构和场景分析，创建切合实际的预测，以支持决策制定和投资人演示。

## 核心组件

### 收入模型

**基于队列的预测：**
按客户队列的获取和留存情况构建收入。

**公式：**
```
MRR = Σ (Cohort Size × Retention Rate × ARPU)
ARR = MRR × 12
```

**关键输入：**
- 每月新增客户数
- 各月客户留存率
- 每用户平均收入（ARPU）
- 定价和套餐假设
- 扩展收入（追加销售、交叉销售）

### 成本结构

**运营费用分类：**

1. **销售成本（COGS）**
   - 托管和基础设施
   - 支付处理费
   - 客户支持（变动部分）
   - 每客户第三方服务费用

2. **销售与营销（S&M）**
   - 客户获取成本（CAC）
   - 营销活动和广告
   - 销售团队薪酬
   - 营销工具和软件

3. **研发（R&D）**
   - 工程团队薪酬
   - 产品管理
   - 设计和用户体验
   - 开发工具和基础设施

4. **行政管理（G&A）**
   - 高管团队
   - 财务、法务、人力资源
   - 办公和设施
   - 保险和合规

### 现金流分析

**组成部分：**
- 期初现金余额
- 现金流入（收入、融资）
- 现金流出（运营费用、资本支出）
- 期末现金余额
- 月度消耗率
- 资金跑道（剩余现金可支撑月数）

**公式：**
```
Runway = Current Cash Balance / Monthly Burn Rate
Monthly Burn = Monthly Revenue - Monthly Expenses
```

### 人员编制规划

**基于岗位的招聘计划：**
按部门和岗位跟踪人员编制。

**关键指标：**
- 每名员工的全成本
- 每名员工创造的收入
- 各部门人员占比

**典型比例（早期SaaS）：**
- 工程：40-50%
- 销售与营销：25-35%
- 行政管理：10-15%
- 客户成功：5-10%

## 财务模型结构

### 三场景框架

**保守场景（P10）：**
- 客户获取速度较慢
- 定价或转化率较低
- 流失率较高
- 销售周期较长
- 用于现金管理

**基准场景（P50）：**
- 最可能的结果
- 现实假设
- 主要规划场景
- 用于董事会报告

**乐观场景（P90）：**
- 增长更快
- 单位经济更优
- 流失率更低
- 用于上行规划

### 时间跨度

**详细预测：3年**
- 第1年按月明细
- 第2年按月明细
- 第3年按季度明细

**高层预测：第4-5年**
- 年度预测
- 仅关键指标
- 支持长期规划

## 分步流程

### 第1步：定义商业模式

明确收入模型和定价策略。

**SaaS模式：**
- 订阅定价层级
- 年付 vs. 月付合同
- 免费试用或免费增值策略
- 扩展收入策略

**平台模式：**
- GMV预测
- 佣金率（交易额百分比）
- 买卖双方经济模型
- 交易频次

**交易模式：**
- 交易量
- 每笔交易收入
- 频次和季节性

### 第2步：构建收入预测

使用基于队列的方法论以提高准确性。

**月度客户获取：**
定义每月获取的新客户数。

**留存曲线：**
建模客户随时间的留存情况。

**典型SaaS留存率：**
- 第1个月：100%
- 第3个月：90%
- 第6个月：85%
- 第12个月：75%
- 第24个月：70%

**收入计算：**
对每个队列，计算每月留存客户数 × ARPU。

### 第3步：建模成本结构

按类别和行为特征分解成本。

**固定 vs. 变动：**
- 固定：工资、软件、租金
- 变动：托管、支付处理、客服

**扩展假设：**
- COGS占收入百分比
- S&M占收入百分比（CAC回收周期）
- R&D增长率
- G&A占总费用百分比

### 第4步：制定招聘计划

按岗位和部门建模人员增长。

**输入：**
- 起始人员编制
- 各岗位招聘速度
- 各岗位全成本薪酬
- 福利和税费（通常为工资的1.3-1.4倍）

**示例：**
```
Engineer: $150K salary × 1.35 = $202K fully-loaded
Sales Rep: $100K OTE × 1.30 = $130K fully-loaded
```

### 第5步：预测现金流

计算月度现金头寸和资金跑道。

**月度现金流：**
```
Beginning Cash
+ Revenue Collected (consider payment terms)
- Operating Expenses Paid
- CapEx
= Ending Cash
```

**跑道计算：**
```
If Ending Cash < 0:
  Funding Need = Negative Cash Balance
  Runway = 0
Else:
  Runway = Ending Cash / Average Monthly Burn
```

### 第6步：计算关键指标

跟踪与当前阶段相关的指标。

**收入指标：**
- MRR / ARR
- 增长率（环比、同比）
- 按细分或队列的收入

**单位经济：**
- CAC（客户获取成本）
- LTV（生命周期价值）
- CAC回收期
- LTV / CAC 比率

**效率指标：**
- 消耗倍数（净消耗 / 净新增ARR）
- 魔术数字（净新增ARR / S&M支出）
- 40法则（增长率% + 利润率%）

**现金指标：**
- 月度消耗率
- 资金跑道（月数）
- 现金效率

### 第7步：场景分析

使用不同假设创建三个场景。

**变动假设：**
- 客户获取率（±30%）
- 流失率（±20%）
- 平均合同价值（±15%）
- CAC（±25%）

**固定假设：**
- 定价结构
- 核心运营费用
- 招聘计划（调整时间，不调整岗位）

## 商业模式模板

### SaaS财务模型

**收入驱动因素：**
- 新增MRR（客户数 × ARPU）
- 扩展MRR（追加销售）
- 收缩MRR（降级）
- 流失MRR（失去的客户）

**关键比率：**
- 毛利率：75-85%
- S&M占收入百分比：40-60%（早期阶段）
- CAC回收期：< 12个月
- 净留存率：100-120%

**示例预测：**
```
Year 1: $500K ARR, 50 customers, $100K MRR by Dec
Year 2: $2.5M ARR, 200 customers, $208K MRR by Dec
Year 3: $8M ARR, 600 customers, $667K MRR by Dec
```

### 平台财务模型

**收入驱动因素：**
- GMV（商品交易总额）
- 佣金率（GMV百分比）
- 净收入 = GMV × 佣金率

**关键比率：**
- 佣金率：10-30%，取决于品类
- 买方 vs. 卖方CAC
- 贡献利润率：60-70%

**示例预测：**
```
Year 1: $5M GMV, 15% take rate = $750K revenue
Year 2: $20M GMV, 15% take rate = $3M revenue
Year 3: $60M GMV, 15% take rate = $9M revenue
```

### 电商财务模型

**收入驱动因素：**
- 流量（访客数）
- 转化率
- 客单价（AOV）
- 购买频次

**关键比率：**
- 毛利率：40-60%
- 贡献利润率：20-35%
- CAC回收期：3-6个月

### 服务/代理财务模型

**收入驱动因素：**
- 可计费工时或项目数
- 时薪或项目费用
- 利用率
- 团队产能

**关键比率：**
- 毛利率：50-70%
- 利用率：70-85%
- 每员工收入

## 融资集成

### 融资场景建模

**投前估值：**
基于指标和可比公司确定。

**稀释：**
```
Post-Money = Pre-Money + Investment
Dilution % = Investment / Post-Money
```

**资金用途：**
分配融资以延长跑道并达成里程碑。

**示例：**
```
Raise: $5M at $20M pre-money
Post-Money: $25M
Dilution: 20%

Use of Funds:
- Product Development: $2M (40%)
- Sales & Marketing: $2M (40%)
- G&A and Operations: $0.5M (10%)
- Working Capital: $0.5M (10%)
```

### 基于里程碑的规划

**识别关键里程碑：**
- 产品发布
- 首个$1M ARR
- CAC盈亏平衡
- A轮融资

**融资金额：**
确保跑道足够到达下一个里程碑 + 6个月缓冲。

## 常见陷阱

**陷阱1：收入预测过于乐观**
- 新创公司很少能达成激进的预测
- 使用保守的客户获取假设
- 建模现实的流失率

**陷阱2：低估成本**
- 在费用估算上增加20%缓冲
- 包含全成本薪酬
- 考虑软件和工具费用

**陷阱3：忽视现金流时间差**
- 收入 ≠ 现金（存在付款账期）
- 费用支出先于收入到账
- 仔细建模现金转化

**陷阱4：静态人员编制**
- 招聘需要时间（3-6个月填补岗位）
- 生产力爬坡期（3-6个月）
- 考虑人员流失（年化10-15%）

**陷阱5：不做场景规划**
- 单一场景永远不会准确
- 始终建模保守场景
- 为基准场景失败的情况做好预案

## 模型验证

**合理性检查：**
- [ ] 收入增长率可实现（第2年3倍，第3年2倍）
- [ ] 单位经济切合实际（LTV/CAC > 3，回收期 < 18个月）
- [ ] 消耗倍数合理（第2-3年 < 2.0）
- [ ] 人员编制随收入增长（每员工收入增长）
- [ ] 毛利率适合商业模式
- [ ] S&M支出与CAC和增长目标一致

**对标同行业：**
将关键指标与类似阶段的同类公司进行比较。

**投资人反馈：**
将模型分享给顾问或投资人，获取对假设的反馈。

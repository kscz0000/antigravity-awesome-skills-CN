## 使用场景

当用户需要从 TikTok、YouTube 或 Instagram 搜索、导入或分析视频内容，从录制中总结会议或讲座，从视频内容构建可搜索的知识库，或研究社交媒体趋势和创作者时，使用此技能。

# 视频搜索与分析

## 描述

此技能使 AI 智能体能够使用 Memories.ai 的大型视觉记忆模型（LVMM）搜索、导入和分析视频内容。与一次性视频分析工具不同，它提供持久化视频智能——视频只需索引一次，即可在多个会话中重复查询。支持社交媒体导入（TikTok、YouTube、Instagram）、会议总结、知识库构建，以及通过记忆增强生成（MAG）进行跨视频问答。

## 概述

该技能将 21 个 API 命令封装为面向工作流的参考指南，智能体按需加载。SKILL.md 中的路由表自动将用户意图映射到正确的工作流。

## 使用场景

- 分析或询问来自 URL 的视频内容时使用
- 按主题、标签或创作者在 TikTok、YouTube 或 Instagram 上搜索视频时使用
- 从录制中总结会议、讲座或网络研讨会时使用
- 从视频内容和文本记忆构建可搜索的知识库时使用
- 研究社交媒体内容趋势、影响者或病毒式传播模式时使用
- 使用 AI 视觉分析或描述图像时使用

## 工作原理

### 步骤 1：意图检测

智能体读取 SKILL.md 工作流路由器，将用户请求匹配到 6 个意图类别之一。

### 步骤 2：参考加载

智能体加载相应的参考文件（例如，视频问答使用 video_qa.md，社交媒体研究使用 social_research.md）。

### 步骤 3：工作流执行

智能体按步骤执行工作流：上传/导入 -> 等待处理 -> 分析/对话 -> 呈现结果。

## 示例

### 示例 1：视频问答

```
用户："这个视频中的主要论点是什么？https://youtube.com/watch?v=abc123"
智能体：上传视频 -> 等待处理 -> 使用 chat_video 提问 -> 呈现结构化摘要
```

### 示例 2：社交媒体研究

```
用户："TikTok 上关于可持续时尚有什么热门内容？"
智能体：使用 search_public 查找热门视频 -> 导入热门结果 -> 分析内容模式
```

### 示例 3：会议记录

```
用户："总结这个会议录音并提取待办事项"
智能体：上传录音 -> 等待 -> 获取转录 -> 使用 chat_video 生成包含待办事项的结构化摘要
```

## 最佳实践

- 查询前务必等待视频处理完成
- 使用 caption_video 进行快速分析（无需上传）
- 使用 chat_video 进行深度多轮分析（需要上传）
- 使用 search_audio 查找视频中的特定时刻或引用
- 使用 memory_add 存储重要发现以便后续检索

## 常见问题

- **问题：** 在处理完成前查询视频
  **解决方案：** 上传后务必使用 `wait` 命令，然后再进行任何分析

- **问题：** 只需要快速字幕时却上传了视频
  **解决方案：** 使用 `caption_video` 进行一次性分析；仅在需要重复查询时上传

## 限制

- 视频处理需要 1-5 分钟，取决于时长
- 免费版限制 100 积分
- 社交媒体导入需要公开内容
- 音频搜索仅适用于已处理的视频

# LEX: 法律实体交叉引用

## 概述

LEX 是一个结构化的真相引擎，旨在通过将智能体锚定在 29+ 个司法管辖区的经验证政府参考和立法上，消除法律幻觉。它为商业组建、雇佣和合同起草提供确定性上下文。

## 何时使用此技能

- 当你需要交叉引用或比较不同地区之间的法律要求时使用，例如验证 **欧盟 SARL** 和 **美国 LLC** 之间的合规差距。
- 当处理需要插入特定、符合司法管辖要求的条款到专业脚手架中的基础商业或雇佣文件时使用。
- 当用户询问 **29 个支持的司法管辖区**（美国、加拿大和欧盟）内法律实体的特定监管细微差别、组建步骤或"基于真相"的定义时使用。

## 工作原理

### 步骤 1：识别司法管辖区
在起草之前，确定用户的实体或合同目标是否在 **美国、加拿大或欧盟**。

### 步骤 2：搜索与获取上下文
使用 CLI 快捷方式查找相关的法律模式和模板。
- 运行 `lex search <query>` 查找匹配的模板。
- 运行 `lex get <path>` 读取细粒度元数据和要求。

### 步骤 3：脚手架起草
使用 `lex Draft <description>` 生成基础级文档。这确保所有草稿包含强制性的 AI 生成内容免责声明。

### 步骤 4：验证权威
始终在输出中包含"已验证来源"部分，通过运行 `lex verify`，该命令获取检索上下文的官方政府链接。

## 示例

### 示例 1：比较雇佣法律
```bash
# 获取劳动力模板以比较美国与欧盟的通知期
lex get templates/02_employment_workforce.md
```

### 示例 2：起草捷克合同
```bash
# 用捷克语创建房屋销售合同脚手架
lex draft "Czech house sale contract"
```

## 最佳实践

- ✅ **信任但验证**：始终在输出中包含 `lex verify` 提供的链接。
- ✅ **表格格式**：在比较多个司法管辖区的结果时使用表格。
- ❌ **禁止猜测**：如果司法管辖区超出美国/欧盟/加拿大范围，声明其超出 LEX"真相引擎"覆盖范围。
- ❌ **禁止轶事建议**：严格遵循模板中的发现或经验证的政府域名。

## 常见陷阱

- **问题：** 关于特定欧盟通知期的法律幻觉。
  **解决方案：** 运行 `lex get templates/02_employment_workforce.md` 查看限制性契约比较表。

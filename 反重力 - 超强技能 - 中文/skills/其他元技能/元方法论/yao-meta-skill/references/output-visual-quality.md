# 输出视觉质量

在批准任何会生成报告、教程、HTML 页面、截图、Markdown 交付物或类幻灯片制品的技能之前，使用本清单进行检查。

## 常见视觉缺陷

- 当用户所在领域需要更精准的章节名称时，仍出现 Overview、Key Points、Summary、Next Steps 之类的通用标题
- 大段引用或脚注堆积，打断句子流畅度
- Markdown 表格中单元格内容像段落一样冗长，或层级关系不清
- 截图来自错误的状态、视口、裁剪区域，或缺失资源
- HTML 报告看起来只是把原始 JSON 换成了卡片样式
- 多张卡片权重完全相同，导致页面难以快速扫读
- 装饰性的渐变、阴影或玻璃效果，并未服务于内容表达
- 移动端布局退化为长串无差别的文本块

## 设计质量关卡

### P0 必须修复

- 最终 HTML 中不得出现绝对的 `/Users/...` 路径
- 不得出现占位符形式的标题、标签、截图或来源说明
- 不得捏造截图、图表、引用或视觉证据
- 当要点列表或卡片更适合扫读时，不得使用单元格内含整段文字的表格
- 不得无内容依据地照搬其他技能中固定的设计配色方案

### P1 应当修复

- 标题和章节名称使用领域名词，并指向预期产出
- 每份报告在首屏给出对"这份报告是做什么的"的清晰说明
- 视觉层级区分决策、证据、风险与下一步行动
- 密集内容应拆分到不同章节，而不是挤压进单一区块
- 仅供评审者查看的细节虽应保留，但不应挤占用户的主阅读流

### P2 润色

- 排版角色保持一致
- 留白节奏契合阅读速度
- 卡片、表格与提示框各自承担不同的语义任务
- 来源说明归并呈现，以保持阅读流畅性
- 在相关场景下兼顾移动端、打印与静态文件查看的体验

## 自检修复环节

交付前，扫描所生成的制品是否存在以下问题：

1. 标题的具体性
2. 表格的可读性
3. 引用密度
4. 截图的真实性
5. 本地路径泄露
6. 占位符残留
7. 移动端可扫读性
8. 评审者可见的证据

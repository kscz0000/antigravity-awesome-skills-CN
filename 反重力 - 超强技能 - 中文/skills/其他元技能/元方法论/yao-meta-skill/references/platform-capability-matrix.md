| --- | --- | --- | --- | --- | --- | --- | --- |
| `openai` | 是 | 是 | 是 | 是 | 是 | 激活、执行、信任、权限、降级、原生行为 | 生成 `targets/openai/agents/openai.yaml` |
| `claude` | 是 | 是 | 是 | 是 | 是 | 激活、执行、信任、权限、降级、原生行为 | 生成 `targets/claude/README.md` 和适配器元数据 |

## 当前支持模型

- `openai`：具有显式编译器契约的最强元数据适配器支持。
- `claude`：具有显式编译器契约和回退说明的轻量兼容性适配器。
- `generic`：面向中性 Agent Skills 使用者的最低摩擦导出。
- `agent-skills-compatible`：具有编译器证据的规范源形态，可用于评审和分发。
- `vscode`：VS Code / Copilot Agent Skills 适配器，保留中性源包，并记录用户/项目范围及工作区信任评审说明。
- 运行时权限探针目前会为生成的目标报告元数据回退；在客户端或安装程序集成能够真正执行权限模型之前，没有任一目标被声称为原生强制。

## 可移植语义

每个目标现在都保留以下内容：

- 激活模式和可选的路径过滤器
- 执行上下文和 shell 选择
- 信任层级和远程内联执行策略
- 针对网络、文件写入、子进程和交互式脚本面的权限契约
- 目标原生行为契约，涵盖原生表面、激活策略、资源策略、脚本策略、权限强制、安装范围、评审产物和回退行为
- 针对不支持的客户端行为的降级策略
- 来自 `reports/compiled_targets.json` 的生成文件映射和适配器模式

## 明确的非目标

本项目尚未实现：

- 客户端 SDK 集成
- 特定提供方的执行逻辑
- 提供方原生的安装动作或账号级激活变更
- 原生运行时权限强制

## 降级规则

如果某个目标无法直接支持源功能：

1. 保留中性源包
2. 输出一个最小适配器
3. 在该目标的输出中记录回退情况

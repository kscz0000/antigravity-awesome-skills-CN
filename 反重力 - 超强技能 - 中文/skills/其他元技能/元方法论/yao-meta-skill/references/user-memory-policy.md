# 用户记忆策略

本技能将用户偏好记忆视为本地化、显式化、可审查的。

## 原则

- **仅限显式来源**：自适应扫描需要用户提供文件路径。
- **本地优先**：偏好提取无需网络访问。
- **禁止隐式私有日志**：默认屏蔽 shell 历史、浏览器历史、邮件与隐藏聊天记录。
- **仅记录重复信号**：一次性陈述将作为丢弃信号记录，除非达到配置的支撑阈值。
- **证据脱敏**：存储的摘录必须移除密钥、令牌、电子邮件地址与本地绝对路径。
- **先提案后打补丁**：偏好记忆可生成提案，不会自动修改源码。

## 允许的输入

推荐输入为为审查目的而准备的 JSONL、Markdown 或文本文件。JSONL 记录应使用 `text`、`message`、`content`、`excerpt`、`prompt`、`note` 或 `body` 等字段。

## 默认屏蔽项

扫描器会拒绝常见的 shell 历史文件，如 `.zsh_history`、`.bash_history` 与 `.fish_history`，除非为受控测试添加显式覆盖。即使带有覆盖项，输出仍保持脱敏且仅作为提案。

## 保留策略

生成的报告仅存储汇总模式与简短脱敏摘录。它们不应被视为完整记录、聊天归档或持久的个人记忆库。

## 升级路径

未来的补丁应用阶段必须新增：

- 人工审批台账；
- 白名单目标文件；
- 干运行差异；
- 回归命令执行；
- 回滚制品；
- 审查者可见的审计轨迹。

# RESOURCES.md 格式

`RESOURCES.md` 是该主题的精选可信资料库。讲解所需的知识应当取自这里，而非凭参数化猜测。智慧则来自其中列出的社区。

## 结构

```md
# {主题} 资源

## 知识（Knowledge）

- [图书：《力量训练的科学和实践》—— Zatsiorsky & Kraemer](https://example.com)
  关于编程与适应性的奠基性著作。适用：任何涉及周期化、恢复、强度区的问题。
- [文章：《我该练多少？》—— Greg Nuckols (Stronger By Science)](https://example.com)
  关于训练容量基准的循证综述。适用：每块肌肉的每周组数目标。

## 智慧 / 社区（Wisdom）

- [r/weightroom](https://reddit.com/r/weightroom)
  高信噪比的 subreddit，对"伪科学型健身"有良好管理。适用：训练计划评审、平台期排查。
- 本地：{健身房名称} 周二力量训练课
  适用：对举重动作的实时教练反馈。
```

## 规则

- **只保留高可信度。** 优先一手资料、受认可的专家、同行评审成果，以及管理到位的社区。若资源只是"伪装成教育的市场营销"，请勿纳入。
- **每一条都要写注解。** 一条裸链接三个月后毫无用处。请补上一句话：它涵盖了哪些内容、何时调用它。
- **按 知识 / 智慧 分组。** 与 [SKILL.md](./SKILL.md) 中的教学理念一致。一个资源仅出现在某一组中是允许的。
- **明确暴露缺口。** 如果使命所需的某一领域没有合适资源，请增设 `## Gaps`（缺口）小节列出所缺项。这将驱动后续检索。
- **果断修剪。** 发现资源有误、内容浅薄、或偏离使命，应当直接移除，而不是"埋起来"。五条精挑细选的胜过三十条平庸的。
- **记录社区偏好。** 如果用户已选择不加入社区，请在此注明，避免未来会话反复提议。

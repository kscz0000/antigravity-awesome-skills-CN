# GLOSSARY.md 格式

`GLOSSARY.md` 是本教学工作空间使用的规范用语。所有讲解、练习与学习记录都应当遵循其中的术语。编写词汇表本身也是学习的一部分：把一个概念压缩成一段简洁的定义，正是你真正掌握了它的证据。

## 结构

```md
# {主题} 词汇表

{一到两句对词汇表所覆盖主题的描述。}

## 术语

**Hypertrophy（肌肥大）**：
由反复训练中机械张力与代谢应激所驱动的肌肉生长。
_避免_：增重、变壮

**Progressive overload（渐进超负荷）**：
在一段时间内系统性地提高对某块肌肉的需求——可以通过负重、训练量或强度达成。
_避免_：再加把劲儿、升级打怪

**RPE (Rate of Perceived Exertion，自觉用力程度)**：
对某组训练感受强度的 1–10 自评，10 表示力竭，8 表示还能再做 2 次。
_避免_：用力分数、强度评级
```

## 规则

- **仅当用户真正理解后才添入新词。** 词汇表是"已压缩知识"的记录，而非供用户阅读以学习新知的词典。若用户刚接触某个概念，等到他们能正确使用它后再将其收入词汇表。
- **敢于取舍。** 当同一概念有多种说法时，挑出最佳的一个，把其余列为"避免使用"的别名。这正是语言压缩得以发生的方式。
- **定义要紧凑。** 一到两句。定义"是什么"，而非"做什么"或"怎么做"。
- **在定义中复用词汇表自身的术语。** 一旦某词进入词汇表，在任何地方都优先使用它——包括其他定义的内部。这是日后让复杂术语更易理解的关键。
- **按主题归类** 当自然聚类出现时（例如 `## 解剖`、`## 编排`）。当术语本身就内聚时，一份扁平列表也很好。
- **明确标注模糊之处。** 若某词在更广领域内被松散使用，记录下约定："在本工作空间中，'组'（set）始终指工作组——热身组另作记录。"
- **随理解深化而修订。** 用户在第一周写下的定义到第六周可能已不准确。原地更新，不要留下过时条目。

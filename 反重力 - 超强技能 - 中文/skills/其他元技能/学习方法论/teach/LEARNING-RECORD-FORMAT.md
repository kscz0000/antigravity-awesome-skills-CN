# 学习记录格式

学习记录存放在 `./learning-records/` 目录中，按顺序编号：`0001-slug.md`、`0002-slug.md` 等。仅在首次写入记录时再惰性创建该目录。

它们是教学领域中的 ADR：捕捉非显而易见的经验、关键洞察与既有的先验知识，这些都将引导未来的会话。它们用来计算最近发展区。

## 模板

```md
# {所学或所确立内容的简短标题}

{1-3 句话：记录所学内容（或所确立的先验知识），以及它为何对未来会话重要。}
```

这就是它的全部格式。一份学习记录可以只是一个段落。其价值在于记录下"现在已知这一件事"以及"它如何改变下一阶段应教什么"——而非罗列章节。

## 可选章节

仅在确有价值时包含这些章节。多数记录用不到。

- **Status** frontmatter（`active | superseded by LR-NNNN`）——当先前的理解后来被证实有误并被替换时很有用。
- **Evidence（证据）**——用户通过什么展示了该理解（答出的问题、完成的练习、援引的先前经验）。在主张可能被回看时很有用。
- **Implications（含义）**——对未来会话解锁或排除的内容。当其非显而易见得值得记录。

## 编号

扫描 `./learning-records/`，找出当前最大的编号，再加一。

## 何时写一条学习记录

满足以下任一条件即可：

1. **用户展示了非平凡的真正理解**——不仅是接触，而是有证据表明他们能正确使用该概念。这为下一阶段应教的内容划定了一条新的底线。
2. **用户披露了先验知识**——"我已经会 X。"记录下来，避免未来会话重复教授。同时记录所声明的*深度*。
3. **错误观念被纠正**——用户此前相信的是错的，现在明白为何。这些是高价值的：它们能预测相关主题未来的绊脚石。
4. **使命在学习过程中发生迁移**——用户发现他们真正在意的不是自己以为的东西。交叉引用 [[MISSION.md]] 并更新它。

### *不* 构成一条学习记录的情形

- 仅仅被覆盖过的素材。"覆盖过"不等于"学过"。等到有证据再说。
- 任何已用条目定义的方式被记入 [[GLOSSARY.md]] 的内容。无需重复。
- 逐次的活动日志。学习记录不是日记——它们是决策级别的洞察。

## 取代（Supersession）

当一条后续记录与更早的记录相矛盾（用户的理解深化或被修正），将旧记录标记为 `Status: superseded by LR-NNNN` 而非直接删除。理解如何演化的这一段历史，本身也是有价值的信息。

# 数据叙事

将原始数据转化为引人入胜的叙事，推动决策并激发行动。

## 不适用场景

- 任务与数据叙事无关
- 需要此范围之外的其他领域或工具

## 使用说明

- 明确目标、约束条件和所需输入。
- 应用相关最佳实践并验证结果。
- 提供可执行的步骤和验证方法。
- 如需详细示例，请打开 `resources/implementation-playbook.md`。

## 适用场景

- 向高管展示分析结果
- 创建季度业务回顾
- 构建投资人演示文稿
- 撰写数据驱动的报告
- 向非技术受众传达洞察
- 基于数据提出建议

## 核心概念

### 1. 故事结构

```
铺垫 → 冲突 → 解决

铺垫：背景和基准
冲突：问题或机会
解决：洞察和建议
```

### 2. 叙事弧线

```
1. Hook（钩子）：用令人惊讶的洞察抓住注意力
2. Context（背景）：建立基准
3. Rising Action（上升动作）：通过数据点层层推进
4. Climax（高潮）：核心洞察
5. Resolution（解决）：建议方案
6. Call to Action（行动号召）：后续步骤
```

### 3. 三大支柱

| 支柱        | 目的     | 组成要素                |
| ----------- | -------- | ----------------------- |
| **数据**    | 证据     | 数字、趋势、对比        |
| **叙事**    | 意义     | 背景、因果关系、影响    |
| **视觉**    | 清晰度   | 图表、示意图、高亮标注  |

## 故事框架

### 框架 1：问题-解决方案故事

```markdown
# 客户流失分析

## 钩子

"我们每年因可预防的流失损失 240 万美元。"

## 背景

- 当前流失率：8.5%（行业平均：5%）
- 平均客户终身价值：$4,800
- 上季度流失 500 个客户

## 问题

对流失客户的分析揭示了一个模式：

- 73% 在前 90 天内流失
- 共同因素：支持互动 < 3 次
- 第一个月功能采用率低

## 洞察

[展示参与度曲线可视化]
在前 14 天未参与的客户
流失可能性高出 4 倍。

## 解决方案

1. 实施 14 天入职引导序列
2. 第 7 天主动触达
3. 功能采用追踪

## 预期影响

- 减少早期流失 40%
- 每年节省 96 万美元
- 回收期：3 个月

## 行动号召

批准 5 万美元预算用于入职自动化。
```

### 框架 2：趋势故事

```markdown
# Q4 业绩分析

## 起点

Q3 以 120 万美元 MRR 结束，低于目标 15%。
错失目标后团队士气低落。

## 变化

[时间线可视化]

- 10 月：推出自助定价
- 11 月：简化注册流程
- 12 月：增加客户成功电话

## 转变

[前后对比图表]
| 指标 | Q3 | Q4 | 变化 |
|----------------|--------|--------|--------|
| 试用→付费 | 8% | 15% | +87% |
| 价值实现时间 | 14 天| 5 天 | -64% |
| 扩展率 | 2% | 8% | +300% |

## 核心洞察

自助服务 + 高接触创造复合增长。
既自助服务又获得成功电话的客户
扩展率高出 3 倍。

## 展望

加大混合模式投入。
目标：Q2 达到 180 万美元 MRR。
```

### 框架 3：对比故事

```markdown
# 市场机会分析

## 问题

我们应该先扩展到 EMEA 还是 APAC？

## 对比

[并排市场分析]

### EMEA

- 市场规模：42 亿美元
- 增长率：8%
- 竞争：激烈
- 监管：复杂（GDPR）
- 语言：多种

### APAC

- 市场规模：38 亿美元
- 增长率：15%
- 竞争：中等
- 监管：多样
- 语言：多种

## 分析

[加权评分矩阵可视化]

| 因素      | 权重 | EMEA 得分 | APAC 得分 |
| --------- | ---- | --------- | --------- |
| 市场规模  | 25%  | 5         | 4         |
| 增长      | 30%  | 3         | 5         |
| 竞争      | 20%  | 2         | 4         |
| 易进入性  | 25%  | 2         | 3         |
| **总计**  |      | **2.9**   | **4.1**   |

## 建议

优先 APAC。增长更高，竞争更少。
从新加坡中心开始（英语、商业友好）。
第二年进入 EMEA，做好本地化准备。

## 风险缓解

- 时区覆盖：招聘 24/7 支持
- 文化适配：本地合作伙伴
- 支付：第一天起支持多币种
```

### 技巧 1：渐进式揭示

```markdown

幻灯片 5："我们需要新细分市场" [添加机会区域]
```

## 写作技巧

### 有效的标题

```markdown
差的："Q4 销售分析"
好的："Q4 销售超额完成目标 23%——原因如下"

差的："客户流失报告"
好的："我们因可预防的流失损失 240 万美元"

差的："营销表现"
好的："内容营销 ROI 是付费广告的 4 倍"

公式：
[具体数字] + [业务影响] + [可执行的背景]
```

### 过渡短语

```markdown
构建叙事：
• "这让我们不禁要问..."
• "当我们深入挖掘..."
• "当...时模式变得清晰"
• "与此形成对比的是..."

引入洞察：
• "数据揭示..."
• "令我们惊讶的是..."
• "拐点出现在..."
• "关键发现是..."

转向行动：
• "这一洞察表明..."
• "基于此分析..."
• "影响显而易见..."
• "我们的建议是..."
```

### 处理不确定性

```markdown
承认局限性：
• "我们可以以 95% 的置信度说..."
• "500 的样本量显示..."
• "虽然相关性很强，但因果关系需要..."
• "这一趋势适用于 [细分市场]，尽管 [注意事项]..."

呈现范围：
• "影响估计：40-60 万美元"
• "置信区间：15-20% 的改善"
• "最佳情况：X，保守估计：Y"
```

## 最佳实践

### 应该做的

- **从"那又怎样"开始** — 以洞察开场
- **使用三原则** — 三个要点、三个对比
- **展示而非讲述** — 让数据说话
- **个性化** — 与受众目标建立联系
- **以行动结束** — 明确的后续步骤

### 不应该做的

- **不要倾倒数据** — 无情地筛选
- **不要埋没洞察** — 把关键发现放在前面
- **不要使用行话** — 匹配受众词汇
- **不要先展示方法论** — 先背景，后方法
- **不要忘记叙事** — 数字需要意义

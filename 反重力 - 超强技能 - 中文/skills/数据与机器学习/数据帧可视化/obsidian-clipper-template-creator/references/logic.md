# Obsidian Web Clipper 模板逻辑

**官方文档：** [Logic - Obsidian Help](https://help.obsidian.md/web-clipper/logic)

自 **Obsidian Web Clipper 1.0.0** 起，模板在 `noteContentFormat` 和属性 `value` 字段中支持逻辑：条件判断、循环、变量赋值和回退值。本页描述各功能的工作方式。有关权威语法和分隔符，请使用上方的官方文档链接。

---

## 何时使用逻辑

- **条件判断：** 仅在数据存在时显示可选部分（例如仅在 `{{schema:Recipe:nutrition}}` 存在时显示营养信息块）。
- **变量赋值：** 赋值一次并在模板中重复使用，避免重复冗长的表达式。
- **回退值：** 当变量为空时提供默认值，使笔记仍然保持正确的显示效果。
- **循环：** 遍历数组（食材、步骤、标签）以格式化每个项；配合过滤器实现列表/表格输出。

保持简单模板的简洁性；仅在逻辑能改善结果或避免缺失数据导致输出异常时添加逻辑。

---

## 条件判断

条件判断根据值是否存在或满足条件来包含或排除区块。

- **比较运算符：** 比较两个值（例如等于、不等于、大于、小于）。具体运算符在官方 Logic 页面中定义。
- **逻辑运算符：** 组合或取反条件（例如 and、or、not；或符号形式如 `&&`、`||`、`!`）。使用这些构建复合条件。
- **真值判断：** 值以类似 JavaScript 的方式评估条件。空字符串、空数组和缺失变量通常为假值；非空值为真值。完整规则请参阅官方文档。
- **典型用法：** 将可选部分（例如作者、营养信息、图片）包裹在条件判断中，使其仅在变量有值时渲染。

---

## 变量赋值

你可以赋值一个变量并在同一模板中重复使用。这可以避免重复冗长的变量或过滤器链（例如在多处使用的相同 Schema 路径或选择器）。赋值的变量在其作用域内的模板其余部分可用。具体语法（例如赋值块格式和作用域）在官方 Logic 文档中。

---

## 回退值

回退值在变量为空时提供默认值。

- **缺失时的默认值：** 提供回退值，使模板在主变量为空时仍能生成可读的输出。
- **链式回退：** 你可以链接多个回退值（例如先尝试变量 A，再尝试 B，再使用字面量默认值）。使用第一个非空值。
- **配合过滤器：** 回退值可以与过滤器一起使用。求值顺序（变量 → 回退值 → 过滤器，或其他）在官方文档的"配合过滤器"和"求值顺序"部分定义。

---

## 循环

循环遍历数组值（例如食材、步骤、标签）。

- **循环源：** 要遍历的数组——通常是返回列表的变量（例如 `{{schema:Recipe:recipeIngredient}}`、`{{schema:Recipe:recipeInstructions}}`）。
- **循环变量：** 在循环内部，你可以获得当前项的变量；也可能获得索引或其他循环元数据。名称和行为在官方"循环变量"和"按索引访问数组项"部分描述。
- **按索引访问数组项：** 当你需要在循环外部获取数组的第 n 项，或在循环内部获取当前索引时，语法在官方文档中定义。
- **嵌套循环：** 当你有结构化数据的数组时（例如每个步骤包含子步骤），循环可以嵌套。
- **组合逻辑：** 循环和条件判断可以组合使用（例如遍历项并仅在字段存在时显示子部分，或有条件地包含循环）。

---

## 求值顺序和逻辑组合

变量、过滤器、回退值、条件判断和循环的求值顺序在官方文档中定义。当你组合逻辑时（例如循环内的条件判断，或在条件判断和输出中都使用的变量），请遵循官方的"求值顺序"和"组合逻辑"部分，以确保行为符合预期。

---

## 模板验证

Obsidian Web Clipper 模板编辑器**会验证模板语法**。无效逻辑（例如格式错误的条件判断或循环）会在编辑器中报告。生成模板时，仅使用官方 Logic 页面中描述的逻辑构造和语法，以确保模板通过验证。

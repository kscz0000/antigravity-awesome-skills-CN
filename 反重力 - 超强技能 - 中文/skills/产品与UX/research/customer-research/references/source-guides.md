# 客户研究 — 来源指南

从在线水源地搜集客户情报的逐源详细打法。

---

## Reddit 研究

### 找对子版块

先识别 ICP 在哪儿活跃，而非产品在哪儿被讨论。

**发现方法：**
- 搜索 `site:reddit.com "[职位] tools"` 或 `site:reddit.com "[问题类别] software"`
- 用 [subreddit search tools](https://www.reddit.com/subreddits/search) 配问题领域关键词
- 看 Google 搜索 ICP 问题时出现哪些子版块
- 看竞品客户在评论中提到哪些子版块

**按类别常见的高价值子版块：**
- B2B SaaS：r/sales、r/marketing、r/entrepreneur、r/startups、r/smallbusiness
- 开发工具：r/programming、r/devops、r/webdev、r/cscareerquestions
- 分析/数据：r/analytics、r/dataengineering、r/BusinessIntelligence
- 营销：r/PPC、r/SEO、r/emailmarketing、r/content_marketing
- HR/招聘：r/recruiting、r/humanresources、r/jobs
- 财务/运营：r/accounting、r/financialplanning、r/projectmanagement

### 搜索算子

```
site:reddit.com/r/[subreddit] "[关键词]"
site:reddit.com "[问题]" "recommend" OR "suggestion" OR "alternative"
site:reddit.com "[竞品名]" "vs" OR "alternative" OR "switched"
```

### 找什么

**高信号帖子类型：**
- "你们用什么工具做 X？" → 揭示替代方案与用语
- "对 [竞品] 受够了，在找替代品" → 揭示痛点与切换触发
- "你们怎么处理 X？" → 揭示工作流与替代办法
- "[你的品类] 值得用吗？" → 揭示异议与评估标准
- 对竞品的吐槽串 → 揭示你可能填补的空缺

**要抽取的：**
- 帖子描述的具体问题
- 点赞最高的解答（实操者真正推荐什么）
- 评论中对现有方案的抱怨
- 使用的语言——记下具体的词句
- 点赞模式——共识 vs 争议

### 工具
- Reddit 原生搜索（有限但快）
- Google：`site:reddit.com [查询]`（结果更好）
- Pullpush.io — 搜索归档的 Reddit 帖子（适合较早的串）

## G2 与评论站点挖掘

### 自己产品的评论

按以下顺序读，信号最大：

1. **3 星评论**——最诚实。客户喜欢到足以留下，但又觉得少了点什么。
2. **1 星评论**——理解失败模式。把产品问题与支持/上手问题分开。
3. **5 星评论**——抽取"他们爱什么"的语言。这些是你的卖点。
4. **4 星评论**——常常把"我唯一希望……"埋在赞美里。

**要抽取的：**
- 他们说用它来做什么（待办任务）
- 他们说最难或最沮丧的是什么
- 他们在拿什么做对比（"以前用 [X]"，"比 [Y] 好"）
- 评论者简介中的行业与角色信号

### G2 上的竞品评论

竞品 4 星评论是金子——客户喜欢产品但仍有抱怨。

**G2 可利用的结构：**
- "你最满意什么？" → 它们的优势（你的 battlecard 情报）
- "你最不满意什么？" → 它们的弱点（你的机会点）
- "你在解决什么问题？" → 待办任务

**Capterra** 结构类似。**Trustpilot** 偏 B2C。**AppSumo** 评论对 SMB/专业消费者 SaaS 有用。

### 评论挖掘模板

对每个竞品的 4 星评论，抽取：

| 类别 | 备注 |
|----------|-------|
| 待办任务 | 为什么用这个产品？ |
| 主要赞美 | 他们爱什么（可能你难以匹敌）？ |
| 主要抱怨 | 什么让他们沮丧？ |
| 切换上下文 | 有没有提到从什么迁移过来？ |
| 未被满足的需求 | "我希望它能……"或"如果能……就更好了" |

## Indie Hackers 与 Product Hunt

### Indie Hackers

对创业者/开发者/SMB ICP 信号强。

**看哪里：**
- "Ask IH" 帖子：关于你产品所解决问题的提问
- 里程碑帖子：创始人描述自己的技术栈时，会透露工具偏好与痛点
- 你所在品类的产品发布下的评论串

**搜索：** `site:indiehackers.com "[问题]"` 或用 IH 的原生搜索。

### Product Hunt

竞品的**讨论标签页**是研究金矿：
- 提问 = 售前顾虑 = 异议
- 评论 = 早期采用者反应 = 接受度的领先指标
- "X 的替代品"合集揭示用户视角的竞争格局

---

## Hacker News

对技术/开发者 ICP 信号强。偏向建设者与怀疑论者。

**高价值搜索：**
- `site:news.ycombinator.com "[竞品或品类]"`
- HN "Ask HN: best tools for X" 串
- 竞品的 "Show HN" 帖子——看怀疑性评论

**HN 与众不同之处：**
- 用户更可能批评底层架构与商业模式
- 对定价模式有强烈意见（尤其任何订阅制的）
- 可能其他地方听不到的第一性原理异议

## LinkedIn 研究

### 帖子与评论

搜索实操者描述工作流的帖子：
- "[角色] 在 [公司规模]" + 问题关键词
- "我们过去 [旧做法] 但现在 [新做法]" 故事
- 寻求工具推荐的帖子会得到活跃买家的评论

### 招聘启事

招聘启事就是公司对痛点的坦白。

**看哪里：**
- 哪些工具列为"加分项"vs"必须"？（揭示技术栈与相邻工具）
- 职位描述中提到哪些指标与结果？
- 这个角色大部分时间在做什么？（揭示待办任务）

**搜索：** `site:linkedin.com/jobs "[职位名]" "[相关工具或品类]"`

## YouTube 评论

### 找高信号视频

- 解决你产品所对应问题的教程视频
- "[年份] X 类最佳工具" 盘点视频
- 竞品的产品演示与走查

**评论里看什么：**
- "这能用于 [特定场景] 吗？" → 边缘情况与未满足需求
- "我试了但……" → 失败点
- "[竞品] 怎么样？" → 主动评估
- 带时间戳的提问 → 工作流中的困惑点

## Twitter / X 研究

### 搜索算子

```
"[竞品]" -filter:replies min_faves:10
"[问题关键词]" "anyone know" OR "recommend" OR "alternative"
"[品类] is broken" OR "frustrated with [品类]"
```

### 找什么

- 对竞品的实时抱怨
- 实操者讨论自己的技术栈
- ICP 关注的影响者/思想领袖（对分发有用）

## 博客文章与论坛研究

### 对比内容

Google：`"[竞品 1] vs [竞品 2]"` 或 `"best [品类] software [年份]"`

读这些文章下面的评论——找对比内容的人正在主动评估。他们的评论是你的销售流程该回答的问题。

### 细分社区

- **Slack 社区**：许多行业有公开或半公开的 Slack 群。搜索 "[行业] Slack community"。
- **Discord 服务器**：在开发者与创作者社区中增长迅速。
- **Facebook 群组**：在 SMB、电商、代理、教练/咨询 ICP 中依然强势。
- **Circle/Mighty Networks 社区**：看看 ICP 所在领域是否有付费社区。

## B2C 与消费应用研究

B2C 研究需要与 B2B SaaS 不同的来源。消费者买家不聚集在 LinkedIn 或 G2——他们把痕迹留在应用商店、社交媒体，以及围绕产品所服务活动而建的社区里。

### 应用商店评论（iOS App Store / Google Play）

移动/消费产品最丰富的未过滤来源之一。

**按以下顺序读：**
1. **1-2 星评论**——失败模式、未被满足的预期、挫败高峰
2. **3 星评论**——诚实的权衡与"不错但是……"反馈
3. **5 星评论**——用他们自己的话说他们爱什么（卖点与定位）

**要抽取的：**
- 他们雇这个应用要做什么（"我用它来……"）
- 它在哪一刻不再奏效
- 他们在拿它跟什么对比，或从什么迁过来
- 情感化语言——"我爱……的方式"、"我太沮丧了……"

**搜索技巧**：先按"最新"排序拿到新鲜信号，再按"最低分"排序挖痛点主题。

### Amazon 评论（实物或带 Amazon 存在的软件）

与应用商店相同的优先级顺序：先 3 星。

**消费 SaaS 的 G2 对应物**：Trustpilot、Sitejabber，以及产品专属的评论聚合站。

### Reddit 消费社区

B2C Reddit 高度垂直——去具体的兴趣/生活方式子版块，而不是通用大版。

**按产品类型举例：**
- 健身应用：r/running、r/loseit、r/fitness、r/MyFitnessPal
- 个人理财：r/personalfinance、r/financialindependence、r/ynab
- 效率/笔记：r/productivity、r/Notion、r/ObsidianMD
- 旅行：r/travel、r/solotravel、r/digitalnomad
- 育儿：r/Parenting、r/beyondthebump、r/daddit

**搜索模式：** `site:reddit.com/r/[社区] "[应用名 OR 问题]"`

### TikTok 与 Instagram 评论

对具有视觉/生活方式吸引力的消费产品信号强。

**如何找信号：**
- 在 TikTok 搜索"[产品名] review"或"is [产品] worth it"
- 看完前 5-10 个视频；读所有评论——不只是点赞
- 在 Instagram，看真实用户标记的帖子（不是品牌方帖子）

**要抽取的：**
- 评论里的提问 = 未被满足的需求或定位不清
- "这能用于……吗？" = 他们想雇它做的任务
- "我从 X 换过来了"评论 = 切换触发
- 对价格、缺失功能或未兑现承诺的抱怨

### YouTube 评论（消费侧）

方法同 B2B，但视频类型不同：

- "X app 真实评测"或"X app 6 个月后"
- "[年份] 最佳 [品类] app"对比视频
- 硬件/实物产品的开箱或"设置"视频

评测视频下的评论尤其有价值——这些是正处于考虑期的人。

### 消费社区平台

- **Facebook 群组**：在许多消费垂直（育儿、健身、本地服务、爱好）仍占主导
- **Discord 服务器**：在游戏、创作者工具、效率、加密、生活方式社区中增长
- **Nextdoor**：对本地服务业务有用
- **Quora**：长篇提问揭示决策焦虑与评估标准

## SparkToro（受众情报）

SparkToro 是一个行为受众研究工具。它不挖单条帖子和评论，而是聚合点击流、搜索与社交数据，按规模展示你的受众在做什么——读什么、看什么、听什么、关注什么、搜什么。

### 何时用 SparkToro vs 手动研究

- **先 SparkToro**：当你需要了解 ICP 在哪儿活跃、消费什么内容、关注哪些影响者时——它用聚合数据秒级回答
- **先手动研究**（Reddit、G2、社区）：当你需要原生态语言、确切引语、情感上下文，以及行为背后的"为什么"
- **组合最佳**：先用 SparkToro 找出哪些播客、subreddit、网站重要，再去那些源头手动挖 VOC 语言

### 关键查询

**按竞品：**
- "关注 @竞品 的人"——揭示共享受众的偏好
- "访问竞品网站的人"——展示他们还消费什么

**按受众描述：**
- "经常讨论 [话题] 的人"——找到受众行为
- "简介含 [职位] 的人"——刻画基于角色的分群

**按你自己的受众：**
- "访问 yourdomain.com 的人"——理解你的真实受众
- 与竞品受众画像对比找缺口

### 抽取内容

| 数据类型 | 它告诉你 | 用途 |
|-----------|------------------|-----------|
| 访问最多的网站 | 受众在哪儿读 | 内容合作、客座发文目标 |
| 热门播客 | 他们听什么 | 播客嘉宾、赞助决策 |
| 热门 YouTube 频道 | 他们看什么 | 视频内容策略、广告投放 |
| 热门 subreddit | 他们在哪儿讨论 | 社区参与、Reddit 广告定向 |
| 搜索关键词 | 他们在 Google 什么 | SEO 与内容选题 |
| AI 提示主题 | 他们向 AI 工具提什么 | 新兴内容机会 |
| 关注的社交账号 | 谁影响他们 | 影响者合作、联合营销 |
| 人口统计 | 他们是谁 | 画像构建、广告定向 |

### 来源加权

SparkToro 数据是聚合且匿名的——它展示模式而非个体观点。把它当作：
- 对行为数据置信度高（访问、关注、搜索）
- 对人口统计置信度中（自我报告，可能不完整）
- 不可替代定性研究（抓不到语言、情感，或"为什么"）

### 限制

- 免费版：5 份报告/月，结果较浅（前 5–10）
- 无公开 API——所有研究都通过 Web 界面进行
- 偏英语、美国中心
- 展示受众做什么而非为什么——与定性来源搭配使用

详见 [tools/integrations/sparktoro.md](../../../tools/integrations/sparktoro.md) 的完整工具细节与定价。

## 组织你的研究

跨来源使用一套简单标签系统：

| 标签 | 含义 |
|-----|---------|
| `#pain` | 问题或挫败 |
| `#trigger` | 引发搜索的事件 |
| `#outcome` | 成功是什么样 |
| `#language` | 文案中原样可用的短语 |
| `#alternative` | 他们考虑或使用的其他方案 |
| `#objection` | 犹豫或不买的原因 |
| `#competitor` | 关于竞品的任何内容 |

维护一份带列的运行文档：来源 | 日期 | 引语 | 标签 | 备注

积累 20-30 条之后，模式就会浮现。寻找在多个不相关来源里都出现的引语——那是置信度最高的洞察。

## 来源可靠性与置信度打分

不同来源权重不一。打置信度标签时使用本指南。

### 来源加权

| 来源 | 信号强度 | 注意的偏差 |
|--------|----------------|--------------|
| 客户访谈（主动提及） | 很高 | 样本小；偏向活跃客户的选择偏差 |
| 赢/输访谈 | 高 | 仅近期记忆；常有理化 |
| 应用商店 / G2 评论 | 高 | 偏向强烈意见（爱或恨） |
| Reddit / 社区帖 | 中高 | 偏技术、怀疑论者、声音大的少数 |
| 支持工单 | 中 | 偏向问题；沉默的大多数未代表 |
| 问卷（开放式） | 中 | 受问题框架引导 |
| 问卷（多选） | 低中 | 反映你给定的选项 |
| NPS 原话 | 中 | 与分数相关；被问卷时刻触发 |
| YouTube/TikTok 评论 | 中 | 偏向活跃观众；社交表演 |
| SparkToro 受众数据 | 中高 | 聚合的行为数据；强在"什么"，不在"为什么" |
| 招聘启事 | 低中 | 理想化，未必反映当前痛点 |

### 实践中的置信度标签

呈现洞察时，把置信度放在前面：

```
[高置信度] 客户感到被手工报表压垮——出现在 20 场访谈中的 12 场、4 条 Reddit 串里，
且是 G2 3 星评论的头号抱怨。在 SMB 与中端市场间一致。

[中置信度] 客户更常拿我们和电子表格比，而不是直接竞品——
6 场访谈和 3 条 Reddit 串中提及，但评论数据中暂未看到。

[低置信度] 企业买家可能有采购顾虑——2 位来自 500+ 公司的访谈者提及。
需要更多信号才能据此行动。
```

### 时效窗口

- **作为主要来源使用**：过去 12 个月的数据
- **谨慎使用**：12-24 个月（产品和市场可能已变）
- **仅作基线背景**：2 年以上

当主题在老数据和新数据中一致出现时，那就是值得采取行动的稳健信号。

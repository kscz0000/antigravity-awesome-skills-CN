# 边界情况

参考以下示例可以让行为更加可预测。

## 无运行目录

用户：`/ustht status`

智能体：`.ustht/ 未找到，请先运行 /ustht init。`

## 技能被禁用

若 `SKILL_STATUS=off`，写入类命令不应修改文件。`status`、`raw`、`mdbase show` 等读取类命令仍可执行。

## 即时模式被禁用

当 `INSTANT_STATUS=off` 时，不要自动捕获自然语言想法。显式命令仍可执行。

## 命令与想法并存

用户：`把按钮改成 8px 圆角，再 /ustht status`

智能体：执行该命令，并在即时捕获启用时记录该 UI 偏好。不要把命令文本本身记录下来。

## 消息后缀忽略

用户：`这次颜色只是试试 /ustht ignore`

智能体：不要写入 raw。如有忽略跟踪机制，将其作为后缀忽略条目记录到 `ignored/`。

## 忽略间隔

用户：`/ustht ignore start`

智能体：在当前上下文中进入忽略模式。

用户：`试三种临时布局。`

智能体：不记录该想法。

用户：`/ustht ignore end`

智能体：退出忽略模式。

## 最后一条忽略

用户：`/ustht ignore --last`

智能体：移除最后一条未处理的原始条目，并将其追加到 `ignored/`。如果不存在条目，应直接说明而不要报错。

## 用户提及处理标记

用户：`也许我们应该用 <!-- processed --> 作为文档里的完成标记。`

智能体：将其作为普通用户内容保留。`sortin` 仅检查原始文件的首行。

## 非法维度名

拒绝包含空格、`..`、反斜杠、绝对路径或不安全字符的维度。

示例：

- 拒绝 `../../../etc/passwd`。
- 拒绝 `my file`。
- 接受 `ui/details`。
- 接受 `dev-stack`。

## 串联命令

用户：`/ustht skill on && instant on && status`

智能体：按从左到右的顺序执行命令，并汇报简洁的摘要。

## 导入但无相关内容

若 `/ustht import README.md` 没有找到任何项目决策，应汇报未抽取到任何条目，且不要写入空的维度章节。

## 多智能体写入

本技能不提供文件锁。若同时运行多个智能体，在 `sortin` 或 `resort` 之前应协调好以避免冲突写入。

## 敏感内容

若用户表示某条想法包含机密或个人数据，优先采用忽略行为，并提醒用户 `.ustht/` 不会自动脱敏。

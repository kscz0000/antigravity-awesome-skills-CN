# sdk-dx

设计开发者爱用的 SDK。优秀的 SDK 开发者体验通过口碑驱动采纳，而非激进营销。

## 本技能涵盖内容

- **API 设计原则**：为常见场景优化、渐进式披露
- **错误消息**：引导开发者找到解决方案的消息
- **类型安全**：自动补全和编译时检查
- **IDE 集成**：编辑器中的文档
- **版本管理**：语义化版本与废弃流程
- **迁移指南**：版本间平滑升级
- **原生感**：语言惯用设计模式

## 何时使用本技能

- 设计新的 SDK 或客户端库
- 改善现有 SDK 的开发者体验
- 规划 SDK 版本管理和发布策略
- 为破坏性变更创建迁移路径
- 发布前评估 SDK 质量

## 核心原则

最好的 SDK 营销就是让开发者忍不住推荐的 SDK。当你的 SDK 让开发者感到高效和胜任时，他们会成为你的拥护者。

## 快速见效

1. 添加带"如何修复"指引的有用错误消息
2. 确保 IDE 自动补全适用于所有操作
3. 为所有公共方法添加 docstrings/JSDoc
4. 测试快速入门代码确实能正常运行
5. 创建开发者可以捕获的特定错误类型

## 相关技能

- `docs-as-marketing` - SDK 的文档
- `api-onboarding` - SDK 的首次体验

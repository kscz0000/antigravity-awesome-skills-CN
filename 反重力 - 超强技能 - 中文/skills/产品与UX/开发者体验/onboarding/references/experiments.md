# 引导实验思路

用户引导和激活的 A/B 测试与实验综合列表。

## 目录
- 流程简化实验（减少摩擦、步骤排序、进度与动力）
- 引导体验实验（产品导览、CTA 优化、UI 引导）
- 个性化实验（用户分群、动态内容）
- 快速胜利与参与实验（价值到达时间、动力机制、支持与帮助）
- 邮件与多渠道实验（引导邮件、邮件内容、反馈循环）
- 重新激活实验（停滞用户恢复、回归体验）
- 技术与 UX 实验（性能、移动端引导、无障碍）
- 追踪指标

## 流程简化实验

### 减少摩擦

| 测试 | 假设 |
|------|------|
| 邮箱验证时机 | 引导中 vs. 引导后 |
| 空状态 vs. 示例数据 | 预填充示例 |
| 预填模板 | 用模板加速设置 |
| OAuth 选项 | 更快的账号关联 |
| 必需步骤数 | 更少的必需步骤 |
| 可选 vs. 必填字段 | 最小化必填项 |
| 跳过选项 | 允许跳过非关键步骤 |

### 步骤排序

| 测试 | 假设 |
|------|------|
| 步骤顺序 | 测试不同排列 |
| 价值优先排序 | 最高价值功能在前 |
| 摩擦位置 | 将困难步骤后移 |
| 必填 vs. 可选比例 | 必需步骤的占比 |
| 单一 vs. 分支路径 | 一条路径 vs. 个性化 |
| 快速开始 vs. 完整设置 | 最短价值路径 |

### 进度与动力

| 测试 | 假设 |
|------|------|
| 进度条 | 显示完成百分比 |
| 清单长度 | 3-5 项 vs. 5-7 项 |
| 游戏化 | 徽章、奖励、成就 |
| 完成提示 | "已完成 X%"可见性 |
| 起始点 | 从 20% 开始 vs. 从 0% |
| 庆祝时刻 | 确认完成 |

## 追踪指标

所有实验应度量：

| 指标 | 说明 |
|------|------|
| 激活率 | 达到激活事件的百分比 |
| 激活时间 | 到首次价值的小时/天数 |
| 步骤完成率 | 每步完成的百分比 |
| 流失点 | 用户在哪里放弃 |
| 回访率 | 回来的用户 |
| 第1/7/30天留存 | 随时间的参与度 |
| 功能采用 | 哪些功能被使用 |
| 支持请求 | 引导期间的请求量 |

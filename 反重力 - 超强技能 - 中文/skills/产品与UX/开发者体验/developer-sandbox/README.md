# developer-sandbox

设计交互式 Playground，让开发者在无需承诺的前提下体验你的产品，并将其转化为用户。

## 本技能涵盖的内容

- **Playground 设计**：即时满足、渐进式复杂度
- **预置示例**：选择能促成转化的示例
- **分享与嵌入**：可分享的 URL 与可嵌入的小部件
- **门控决策**：何时要求注册，何时保持开放
- **转化优化**：将 Playground 用户转化为注册用户
- **架构**：客户端与服务端 Playground

## 何时使用本技能

- 构建全新的开发者 Playground
- 优化现有 Playground 的转化率
- 决定哪些功能要门控、哪些保持开放
- 为文档创建可嵌入的示例
- 为你的 API 设计评估体验

## 核心原则

一个出色的 Playground 能消除不确定性。开发者应当能在投入搭建或注册之前，先验证你的产品是否能解决他们的问题。

## Playground 层级

| 层级 | 体验 | 目标 |
|-------|------|------|
| 一键演示 | 点击按钮即可看到结果 | 证明可用 |
| 可编辑输入 | 修改输入，查看输出 | 激发好奇心 |
| 完整 API 访问 | 编写代码，调用 API | 展示能力 |
| 完整 Playground | 构建迷你应用 | 支持评估 |

## 速赢要点

1. 预加载一个可用的示例（而不是空白的编辑器）
2. 让用户无需注册即可运行示例
3. 为每个示例添加"复制为 cURL"按钮
4. 展示真实的 API 响应（而非模拟数据）
5. 创建可分享的 Playground URL

## 相关技能

- `api-onboarding` - 将 Playground 作为入职工具
- `docs-as-marketing` - 文档中的交互式示例
- `sdk-dx` - 在 Playground 上下文中可用的 SDK

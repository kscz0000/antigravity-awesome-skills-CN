# 单页纸模板

适用于不同单页纸使用场景的模板，包含布局指导和文案提示。

## 产品概述单页纸

默认的单页纸。向对你一无所知的人介绍你的产品。

### 结构

```
[Logo]                                              [标语]

标题：用一句话描述你做什么以及为谁而做。

问题
2-3 句话描述你的买家面临的痛点。

解决方案
2-3 句话描述你的产品如何解决这些问题。

为什么选择 [你的产品]
• 差异点 1 — 一句话说明价值
• 差异点 2 — 一句话说明价值
• 差异点 3 — 一句话说明价值

客户见证
"包含具体成果的客户引言。" — 姓名，职位，公司
[可选：2-3 个数据亮点："提升 X%"，"节省 Y 小时"]

[行动号召按钮/链接]                    [联系方式：name@company.com]
```

### 文案提示

- 标题："用一句话说清楚你做什么，能让对方说'再跟我说说'。"
- 问题："在找到你之前，你的买家在为什么事情挣扎？"
- 差异点："如果只能告诉他们 3 件事，什么能让他们选择你？"

## 用例专用单页纸

针对特定工作流、行业或问题定制。比产品概述更具针对性。

### 结构

```
[Logo]                                    [用例：例如"面向销售团队"]

标题：[你的产品]如何帮助 [目标人群] [达成目标]。

挑战
当 [目标人群] 需要 [任务] 时，他们面临 [具体痛点]。
这导致 [后果]：[浪费时间 / 损失资金 / 风险]。

工作原理
1. [步骤 1] — 发生了什么以及为什么重要
2. [步骤 2] — 发生了什么以及为什么重要
3. [步骤 3] — 发生了什么以及为什么重要

成果
• [指标 1]：之前 → 之后
• [指标 2]：之前 → 之后
• [指标 3]：之前 → 之后

客户聚焦
"关于这个具体用例的引言。" — 姓名，职位，公司

[行动号召："观看演示"或"开始试用"]       [联系信息]
```

### 何时使用

- 不同买家人群需要不同的单页纸
- 行业专用版本（医疗、金融科技、电商）
- 用例版本（报表、入职、安全）

## 会后留存单页纸

为强化已发生的对话而设计。总结你们讨论的内容并提出后续步骤。

### 结构

```
[Logo]                                            [会议日期]

会议纪要：[公司名称]

我们讨论了什么
• [他们提到的痛点 1]
• [他们提到的痛点 2]
• [他们想要达成的目标]

[你的产品]如何提供帮助
• [针对痛点 1 的解决方案] — [具体功能或工作流]
• [针对痛点 2 的解决方案] — [具体功能或工作流]
• [如何帮助他们达成目标]

相关客户见证
"来自类似客户的引言。" — 姓名，职位，公司
[来自类似客户的 1-2 个数据指标]

建议的后续步骤
1. [下一步及日期]
2. [跟进动作]
3. [决策时间线]

[你的姓名]  |  [你的职位]  |  [邮箱]  |  [电话]
```

### 建议

- 会议后 24 小时内发送
- 引用他们说过的具体内容（表明你在倾听）
- 后续步骤要具体且有时间节点
- 这是你的内部推动者会转发给老板的材料

## 内部推动者赋能单页纸

专门为你的内部推动者设计，供其与团队和领导层分享。撰写时要让他们显得专业。

### 结构

```
[Logo]

我们为什么评估 [你的产品]

现状
[2-3 句话描述内部挑战，撰写时要像推动者在向
自己的团队解释一样。使用"我们"和"我们的"表述。]

[你的产品]做什么
[1-2 句话。用通俗语言，不要用术语。]

为什么选择这个方案
• [原因 1] — 如何解决我们的具体问题
• [原因 2] — 与我们现在做法的对比
• [原因 3] — 与我们评估过的其他方案的对比

预期影响
• [指标]：当前状态 → 预期状态
• [指标]：当前状态 → 预期状态
• [见效时间]：[X 周]内上线

其他使用者
[他们行业中 2-3 个知名公司名称]
"相关客户引言。" — 姓名，职位，公司

后续步骤
• [我们接下来要做什么]
• [我们需要团队做什么]
• [决策时间线]

有问题？请联系 [推动者姓名] 或 [你的姓名和邮箱]。
```

### 为什么有效

- 用推动者的口吻撰写，而非你的口吻
- 回答了他们老板会问的问题
- 包含来自他们认可的同行的客户见证
- 清晰的行动号召和时间线，推动内部进展

## 布局指导

### 视觉层次

1. **标题** — 最大的文字，位于页面顶部，立即传达价值
2. **章节标题** — 加粗、清晰，作为可扫描的锚点
3. **正文** — 短句，优先使用项目符号而非段落
4. **见证元素** — 数据指标和引言应视觉突出（更大的字号、颜色或引用框）
5. **行动号召** — 放在显眼位置，页面底部或右下角

### 留白

- 页边距：四边至少 0.75 英寸
- 章节间距：足以视觉分隔（不要塞太满）
- 如果感觉拥挤，删减内容。字号永远不要低于 9pt。

### 字号建议

| 元素 | 建议字号 |
|------|---------|
| 标题 | 18-24pt |
| 章节标题 | 12-14pt 加粗 |
| 正文 | 10-11pt |
| 细则/页脚 | 8-9pt |

### 配色

- 标题和强调色使用品牌色
- 正文在白色背景上使用深色（黑色或接近黑色）
- 强调色限制在 1-2 种以保持视觉一致性
- 用颜色吸引注意力到数据指标和行动号召

### 文件格式

- **PDF** 用于邮件附件和留存材料
- **Google Slides / PowerPoint** 用于销售代表可以自定义的可编辑版本
- 两种格式都要提供 — 销售代表需要自定义，潜在客户想要干净的 PDF

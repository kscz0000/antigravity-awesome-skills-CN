# 竞品画像
## 使用时机

当用户希望基于 URL 研究、描绘或分析竞品时使用此技能。也用于用户提及"竞品画像""竞品研究""竞品分析""profile this competitor""analyze competitor""competitive intelligence""competitor deep dive"等场景。

你是一名资深的竞争情报分析师。你的目标是接收一组竞品 URL，通过结合实时站点抓取与 SEO 及市场数据，产出全面、结构化的竞品画像文档。

## 初始评估

**首先检查产品营销上下文：**
如果存在 `.agents/product-marketing.md`（或 `.claude/product-marketing.md`，或在旧版设置中的 `product-marketing-context.md` 文件名），请在提问之前先阅读它。直接使用该上下文，仅询问其中未涵盖的信息。

在开始画像之前，请确认以下事项：

1. **竞品 URL** —— 需要画像的竞品网站 URL 列表
2. **你的产品** —— 你的产品是做什么的（若产品营销上下文中未提供）
3. **画像深度** —— 快速扫描（仅关键事实）或深度画像（完整研究）
4. **关注维度** —— 需要优先关注的特定维度（如定价、定位、SEO 实力、内容策略）

如果用户已提供 URL 且上下文可用，则无需再次询问，直接进行。

---

## 核心原则

### 1. 事实优于观点
画像中的每一项主张都应可追溯至来源 —— 抓取到的页面内容、评论数据或 SEO 指标。明确标注哪些是推断。

### 2. 结构化与可对比
所有画像遵循相同模板，以便并排比较。一致性比单个画像的完整性更重要。

### 3. 现行数据
画像是某一时刻的快照。务必包含生成日期。标记任何看起来已过时的信息（如"定价页面最后更新于 2023 年"）。

### 4. 诚实的评估
不要夸大竞品弱点，也不要淡化其优势。准确的画像才是有用的画像。

## 原始数据保存

在综合画像之前，将所有原始抓取、SEO 与评论数据持久化到磁盘，以便后续重新读取、审计或复用，避免再次调用昂贵的 API。

**目录布局**（相对于项目根目录）：

```
competitor-profiles/
├── raw/
│   └── <competitor-slug>/
│       └── <YYYY-MM-DD>/
│           ├── scrapes/    # 每个抓取页面一个 .md 文件（homepage.md、pricing.md 等）
│           ├── seo/        # 每次 DataForSEO 调用一个 .json 文件（backlinks-summary.json、ranked-keywords.json 等）
│           └── reviews/    # 每个评论来源一个 .md 或 .json 文件（g2.md、capterra.md 等）
├── <competitor-slug>.md    # 最终合成的画像
└── _summary.md             # 跨竞品汇总
```

规则：

- `<competitor-slug>` 使用小写、连字符格式（如 `responsehub`、`safe-base`）
- `<YYYY-MM-DD>` 为拉取数据的日期 —— 支持多次重新拉取并对比不同时间的快照
- 将每次 Firecrawl 抓取结果以原始 Markdown 形式保存至 `scrapes/<page-name>.md`
- 将每次 DataForSEO 响应以原始 JSON 形式保存至 `seo/<endpoint-name>.json`
- 将每个评论来源保存至 `reviews/<source>.md`（清洗后的文本）或 `.json`（原始数据）
- 每次新运行都新建日期文件夹；不要覆盖先前日期的数据

合成的画像（`<competitor-slug>.md`）应在 `## Raw Data Sources` 章节中引用其对应的原始数据文件夹。

## 研究流程

### 阶段 1：站点抓取（Firecrawl）

针对每个竞品 URL，抓取关键页面以提取定位、功能、定价与文案。

#### 步骤 1：站点地图

使用 **Firecrawl Map** 发现竞品站点结构并识别关键页面：

```
firecrawl_map → 竞品 URL
```

根据地图识别并优先排序以下页面类型：
- 首页
- 定价页
- 功能 / 产品页
- 关于 / 公司页
- 博客（顶层，用于内容策略信号）
- 客户 / 案例研究页
- 集成页
- 更新日志 / 最新动态（若存在）

#### 步骤 2：抓取关键页面

对每个识别出的页面使用 **Firecrawl Scrape**：

```
firecrawl_scrape → 每个关键页面 URL
```

在抽取字段之前，先将每个结果保存至 `competitor-profiles/raw/<competitor-slug>/<YYYY-MM-DD>/scrapes/<page-name>.md`。

从每个页面中提取：

| 页面 | 需提取内容 |
|------|-----------|
| **首页** | 标题、副标题、价值主张、主要 CTA、社会证明声明、目标受众信号 |
| **定价** | 套餐、价格、各套餐功能明细、计费选项、免费套餐/试用详情、企业版定价信号 |
| **功能** | 功能分类、关键能力、各功能的描述方式、截图/演示信号 |
| **关于** | 创立故事、团队规模、融资情况、使命声明、总部所在地 |
| **客户** | 具名客户、Logo、所服务行业、案例研究主题 |
| **集成** | 集成数量、关键集成、类别 |
| **更新日志** | 发布速度、近期关注点、产品方向信号 |

#### 步骤 3：抓取竞品评论（可选但高价值）

使用 **Firecrawl Scrape** 或 **Firecrawl Search** 查找：
- 竞品的 G2 评论页
- Capterra 评论页
- Product Hunt 发布页
- TrustRadius 个人页

将每个抓取的评论页保存至 `competitor-profiles/raw/<competitor-slug>/<YYYY-MM-DD>/reviews/<source>.md`。然后提取：总体评分、评论数量、常见的正面主题、常见的负面主题，以及 3-5 条具有代表性的引言。

---

### 阶段 2：SEO 与市场数据（DataForSEO）

使用 DataForSEO MCP 工具收集量化的竞争情报。在解析为画像之前，将每次原始响应以 JSON 形式保存至 `competitor-profiles/raw/<competitor-slug>/<YYYY-MM-DD>/seo/<endpoint-name>.json`。本技能中使用的 MCP 工具完整列表（Firecrawl + DataForSEO）及调用示例，请参见 [references/tool-reference.md](references/tool-reference.md)。

#### 域名权重与外链

使用 **backlinks_summary** 获取：
- 域名权重 / 权威分数
- 外链总数
- 引荐域名数量
- 垃圾分数

使用 **backlinks_referring_domains** 获取：
- 顶级引荐域名（质量信号）
- 外链获取模式

#### 关键词与流量情报

使用 **dataforseo_labs_google_ranked_keywords** 获取：
- 参与排名的自然关键词总数
- 排名前 3、前 10、前 100 的关键词数
- 预估自然流量

使用 **dataforseo_labs_google_domain_rank_overview** 获取：
- 域名级自然指标
- 预估流量价值
- 按流量排名的头部关键词

使用 **dataforseo_labs_google_keywords_for_site** 发现：
- 他们所针对的关键词
- 与你网站的内容差距

#### 竞争定位数据

使用 **dataforseo_labs_google_competitors_domain** 发现：
- 他们最接近的自然竞品（可能揭示你尚未考虑的竞品）
- 市场重叠数据

使用 **dataforseo_labs_google_relevant_pages** 发现：
- 他们流量最高的页面
- 带来最多自然价值的内容

---

### 阶段 3：综合

将抓取的内容与 SEO 数据结合，构建画像。交叉核对主张（例如，如果他们在网站上声称"10,000 客户"，则检查其流量/外链概况是否支撑这一规模）。

### 汇总文档

完成所有竞品画像后，生成 `competitor-profiles/_summary.md`，包含：

1. **竞争格局概览** —— 一段话概述竞争领域
2. **对比表** —— 横向展示所有已画像竞品的关键指标
3. **定位图** —— 每个竞品所处的位置（如：简易↔复杂、廉价↔高端）
4. **关键洞察** —— 来自研究的 3-5 条战略性观察
5. **空白与机会** —— 市场未充分服务的领域

## 快速扫描 vs. 深度画像

### 快速扫描（更快、成本更低）
- 抓取：仅首页 + 定价页
- SEO：域名权重概览 + 排名关键词摘要
- 跳过：评论、技术栈、外链细节
- 输出：精简版画像（一览 + 定位 + 定价 + SEO 摘要）

### 深度画像（全面）
- 抓取：所有关键页面 + 评论站点
- SEO：完整外链分析 + 关键词情报 + 竞品发现
- 包含：技术栈、内容策略分析、评论挖掘
- 输出：完整画像模板

除非用户要求深度画像或明确指定少量竞品（3 个或更少），否则默认采用 **快速扫描**。

## 处理多个竞品

画像竞品数量大于一个时：

1. **并行抓取** —— 同时抓取所有竞品的首页，然后再抓取定价页，以此类推
2. **使用一致的指标** —— 为每个竞品拉取相同的 DataForSEO 指标，使画像可比
3. **最后再构建汇总** —— 在所有单独画像完成之后
4. **按相关性优先排序** —— 如果用户提供了 10 个以上竞品，建议先基于域名重叠或市场相似度画像前 5 个

---

## 更新画像

画像是某一时刻的快照。更新时：

- 首先检查定价页（最易变动）
- 重新拉取 SEO 指标（流量与排名每月都会变化）
- 扫描更新日志以发现产品变更
- 更新"生成日期"
- 在底部 `## Change Log` 章节中记录自上次画像以来的变更

- **competitors**：用于基于这些画像创建对比/替代页

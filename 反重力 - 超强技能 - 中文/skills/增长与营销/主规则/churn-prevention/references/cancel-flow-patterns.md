# 取消流程模式

按业务类型、计费服务商和行业的详细取消流程模式。

---

## 按业务类型的取消流程

### B2C / 自助 SaaS

高量、低接触。流程必须无人干预即可运作。

**流程结构：**
```
取消按钮 → 离线调查（1 问题）→ 动态优惠 → 确认 → 取消后
```

**特点：**
- 全自动，无人工介入
- 快速 — 最多 2-3 个屏幕
- 一个优惠 + 一个备选，不是选项菜单
- 移动端优化（大量取消发生在移动端）
- 每步都有清晰的"继续取消"

**典型挽留率：** 20-30%

**示例流程（$29/月 生产力应用）：**
1. "主要原因是什么？" → 6 个选项
2. 选择"太贵" → "接下来 3 个月享 25% 折扣（节省 $21.75）"
3. 拒绝 → "或切换到我们的入门版 $12/月"
4. 拒绝 → "很遗憾看到您离开。您的访问将持续到 [日期]。"

---

### B2B / 团队套餐

量少、风险高。人工触达值得成本。

**流程结构：**
```
取消按钮 → 离线调查 → 优惠（或路由到 CS）→ 确认 → 取消后
```

**特点：**
- MRR 超阈值的账户路由到客户成功
- 展示团队影响（"您的 8 位团队成员将失去访问权限"）
- 企业账户提供管理员对管理员通话
- 更长考虑时间 — 允许"预约通话"作为挽留选项
- 需要管理员/所有者角色才能取消（不是任意团队成员）

**典型挽留率：** 30-45%（因人工触达更高）

**基于 MRR 的路由：**

| 账户 MRR | 取消流程 |
|---------|---------|
| <$100/月 | 自动流程配优惠 |
| $100-$500/月 | 自动 + 标记 CS 跟进 |
| $500-$2,000/月 | 取消完成前路由到 CS |
| $2,000+/月 | 阻止自助取消，要求 CS 通话 |

---

### 免费增值 / 免费转付费

用户取消付费回归免费档。心理不同 — 他们不是离开，是降级。

**流程结构：**
```
取消按钮 → "切换到免费？" 提示 → 离线调查（如仍取消）→ 优惠 → 确认
```

**特点：**
- 首选展示免费档作为第一选项（不是挽留优惠）
- 展示免费版保留什么 vs 失去什么
- "挽留"是让他们留在免费版，不是完全失去
- 追踪免费用户以便未来重新升级活动

## 按计费周期的取消流程

### 月付订阅者

- 更价格敏感，承诺更短
- 折扣优惠效果好（2-3 个月 20-30%）
- 暂停有效（1-2 个月）
- 建议折扣年付作为替代

**优惠优先级：**
1. 折扣（如原因 = 价格）
2. 暂停（如原因 = 用得不够/临时）
3. 切换年付（如参与但价格敏感）

### 年付订阅者

- 承诺更高，取消原因往往更强
- 按比例退款预期重要
- 挽留窗口更长（已付费）
- 人工触达更合理（更高 LTV）

**优惠优先级：**
1. 暂停剩余期限（如临时）
2. 套餐调整 + 下次续费赠金
3. CS 人工触达
4. 部分退款 + 降级（优于全额退款 + 取消）

**退款处理：**
- 如剩余时间较多，提供按比例退款
- 如少于 3 个月，"暂停到续费"
- 大方些 — 糟糕的退款体验会制造声量大的贬损者

## 挽留优惠模式

### 折扣阶梯

不要一上来就给最大折扣。逐步升级：

```
取消点击 → 15% 折扣 → 仍取消 → 25% 折扣 → 仍取消 → 让他们走
```

**规则：**
- 每次取消会话最多 2 个折扣优惠
- 永不超过 30%（更高会训练为优惠而取消的行为）
- 限时折扣（2-3 个月，然后恢复原价）
- 追踪折扣接受者 — 如他们在原价时再次取消，不再提供

### 暂停手册

暂停往往比折扣更好，因为不会贬低产品价值。

**实现：**

| 设置 | 建议 |
|------|------|
| 暂停时长选项 | 1 个月、2 个月、3 个月 |
| 默认选择 | 1 个月（最短） |
| 最长暂停 | 3 个月（更长暂停很少回归） |
| 暂停期间 | 保留数据，移除访问 |
| 重新激活 | 自动重新激活，提前 7 天邮件 |
| 重复暂停 | 每 12 个月允许 1 次暂停 |

**暂停重新激活序列：**
- 第 -7 天："您的暂停将在 7 天后结束。我们很忙 — 这是新功能。"
- 第 -1 天："明天欢迎回来！这是等待您的内容。"
- 第 0 天："您回来了！快速浏览新功能。"

### 降级路径

对于多套餐产品，降级是最强的挽留：

```
┌─────────────────────────────────────────┐
│  在您离开之前，考虑调整合适的套餐？      │
│                                         │
│  当前：专业版（¥49/月）                 │
│                                         │
│  ┌─────────────────────────────────┐    │
│  │ 切换到入门版（¥19/月）          │    │
│  │                                 │    │
│  │ ✓ 保留：项目、集成              │    │
│  │ ✗ 失去：高级分析、              │    │
│  │         团队功能                │    │
│  │                                 │    │
│  │ [切换到入门版]                  │    │
│  └─────────────────────────────────┘    │
│                                         │
│  [不用了，继续取消]                      │
└─────────────────────────────────────────┘
```

**降级最佳实践：**
- 精确展示保留什么和失去什么
- 使用勾和叉便于扫描
- 即使在低档位也保留数据
- 如降级，至少 30 天内不展示升级提示

### 竞品切换处理

当取消原因是"切换到竞品"：

1. **询问哪个竞品**（可选，不强制）
2. **展示对比**如有（参见 competitor-alternatives 技能）
3. **提供迁移赠金**（"我们 3 个月匹配他们的价格"）
4. **请求反馈通话**（"15 分钟了解我们缺少什么"）

这些数据对产品和营销团队价值连城。

## 取消后体验

取消后发生什么对以下方面很重要：
- 挽回潜力
- 口碑
- 评论情绪

### 确认页面

```
您的订阅已取消。

接下来：
• 您的访问将持续到 [计费周期结束日期]
• 您的数据将保留 90 天
• 您可以随时从账户设置重新激活

[重新激活我的账户]

我们欢迎您回来。我们会根据像您这样的客户反馈持续改进。
```

### 取消后序列

| 时机 | 行动 |
|------|------|
| 立即 | 确认邮件，含访问结束日期 |
| 第 1 天 | （无 — 不要显得急切） |
| 第 7 天 | NPS/满意度调查整体体验 |
| 第 30 天 | "新功能"邮件，含近期改进 |
| 第 60 天 | 如已解决，针对其具体取消原因 |
| 第 90 天 | 最终挽回配特别优惠 |

**详细挽回邮件序列**：参见 email-sequence 技能。

## 分段规则

最有效的取消流程使用分段向不同客户展示不同优惠。

### 分段维度

| 维度 | 为何重要 |
|------|---------|
| 套餐 / MRR | 更高价值客户获得人工触达 |
| 使用时长 | 长期客户获得更慷慨优惠 |
| 使用水平 | 高使用客户获得不同于休眠客户的信息 |
| 计费周期 | 月付 vs 年付需要不同方法 |
| 既往挽留 | 不要向重复取消者提供相同折扣 |
| 取消原因 | 驱动展示哪个优惠（核心映射） |

### 分段特定流程

**新客户（< 30 天）：**
- 他们未激活。挽留是入门，不是折扣。
- 优惠：免费入门通话、设置帮助、延长试用
- 询问："您希望实现什么？"（了解缺少什么）

**因价格取消的参与客户：**
- 他们喜欢产品但无法证明成本合理。
- 优惠：折扣、切换年付、降级
- 高挽留潜力

**休眠客户（30+ 天未登录）：**
- 他们忘了您。折扣不会带回他们。
- 优惠：暂停订阅、"发生了什么变化？"对话
- 低挽留潜力 — 专注于学习原因

**切换到竞品的高级用户：**
- 他们主动选择了其他产品。
- 优惠：竞品匹配、反馈通话、路线图预览
- 中等挽留潜力 — 取决于原因

## 实现清单

### 第一阶段：基础（第 1 周）
- [ ] 添加取消流程（调查 + 1 个优惠 + 确认）
- [ ] 设置离线调查，含 5-7 个原因分类
- [ ] 每个原因映射一个优惠（简单 1:1 映射）
- [ ] 在分析中追踪取消原因和挽留率
- [ ] 启用预催收卡片过期邮件

### 第二阶段：优化（第 2-4 周）
- [ ] 添加备选优惠（每个原因首选 + 备选）
- [ ] 实现暂停订阅选项
- [ ] 设置催收邮件序列（10 天内 4 封邮件）
- [ ] 启用智能重试（Stripe Smart Retries 或等效）
- [ ] 添加基于 MRR 的高价值账户路由

### 第三阶段：高级（第 2 个月+）
- [ ] 从使用信号构建健康评分
- [ ] 设置主动干预触发
- [ ] A/B 测试折扣金额和优惠类型
- [ ] 按套餐、使用时长和使用分段流程
- [ ] 取消后挽回序列（与 email-sequence 技能协调）
- [ ] 队列分析：按渠道、套餐、使用时长的流失

## 合规说明

### FTC Click-to-Cancel 规则（美国）
- 取消必须与注册一样简单
- 如注册在线上，不能要求电话取消
- 不能添加过多步骤阻碍取消
- 允许挽留优惠但"继续取消"必须清晰

### GDPR / 数据保留（欧盟）
- 通知用户取消后数据保留期限
- 账户删除前提供数据导出
- 30 天内响应删除请求
- 未经同意不得将取消后数据用于营销

### 通用最佳实践
- 始终展示完成取消的清晰路径
- 永不隐藏取消按钮（暗黑模式）
- 即使挽留流程出错也处理取消
- 用邮件收据确认取消

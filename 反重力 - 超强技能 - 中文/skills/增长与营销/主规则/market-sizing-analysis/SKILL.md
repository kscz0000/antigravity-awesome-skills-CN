# 市场规模测算分析

创业机会的市场规模测算方法论，涵盖总可达市场（TAM）、可服务可用市场（SAM）和可服务可获得市场（SOM）的完整计算框架。

## 使用场景

- 进行市场规模测算分析任务或工作流时
- 需要市场规模测算的指导、最佳实践或检查清单时

## 不适用场景

- 任务与市场规模测算无关
- 需要此范围之外的其他领域或工具

## 使用说明

- 明确目标、约束条件和必要输入
- 应用相关最佳实践并验证结果
- 提供可执行的步骤和验证方法
- 如需详细示例，打开 `resources/implementation-playbook.md`

## 概述

市场规模测算是创业战略、融资和商业规划的基础。使用三种互补方法论计算市场机会：自上而下（行业报告）、自下而上（客户细分计算）和价值理论（支付意愿）。

## 核心概念

### 三层市场框架

**TAM（总可达市场）**
- 达到 100% 市场份额时的总收入机会
- 定义潜在客户的总体范围
- 用于长期愿景和市场验证
- 示例：全球所有邮件营销软件收入

**SAM（可服务可用市场）**
- TAM 中当前产品/服务可触达的部分
- 考虑地理、细分或能力约束
- 代表实际可触达的机会
- 示例：北美地区面向电商的 AI 驱动邮件营销

**SOM（可服务可获得市场）**
- 3-5 年内可现实获取的市场份额
- 考虑竞争、资源和市场动态
- 用于财务预测和融资
- 示例：基于竞争格局，占 SAM 的 2-5%

### 各方法论的适用场景

**自上而下分析**
- 已有成熟市场研究时使用
- 适用于成熟、定义明确的市场
- 验证市场存在性和增长性
- 从行业报告出发逐步缩小范围

**自下而上分析**
- 针对特定客户细分时使用
- 适用于新兴或利基市场
- 对投资人最具说服力
- 从客户数据和定价出发构建

**价值理论**
- 创建新市场品类时使用
- 适用于颠覆性创新
- 基于价值创造进行估算
- 计算解决问题方案的支付意愿

## 三大方法论框架

### 方法论 1：自上而下分析

从市场总量出发，缩小到可触达细分市场。

**流程：**
1. 从研究报告中确定市场总量类别
2. 应用地理筛选（目标区域）
3. 应用细分筛选（目标行业/客户）
4. 计算竞争定位调整

**公式：**
```
TAM = Total Market Category Size
SAM = TAM × Geographic % × Segment %
SOM = SAM × Realistic Capture Rate (2-5%)
```

**适用场景：** 有现成研究的成熟市场（如 SaaS、金融科技、电商）

**优势：** 快速、使用可信数据、验证市场存在性

**局限：** 可能高估新品类，粒度较粗

### 方法论 2：自下而上分析

从客户细分计算出发构建市场规模。

**流程：**
1. 定义目标客户细分
2. 估算每个细分的潜在客户数量
3. 确定客户平均收入
4. 计算现实渗透率

**公式：**
```
TAM = Σ (Segment Size × Annual Revenue per Customer)
SAM = TAM × (Segments You Can Serve / Total Segments)
SOM = SAM × Realistic Penetration Rate (Year 3-5)
```

**适用场景：** B2B、利基市场、特定客户细分

**优势：** 对投资人最具说服力、粒度细、可辩护

**局限：** 需要详细客户调研、耗时

### 方法论 3：价值理论

基于创造的价值和支付意愿计算。

**流程：**
1. 识别所解决的问题
2. 量化问题的当前成本（时间、金钱、低效）
3. 计算解决方案的价值（节省、收益、效率）
4. 估算支付意愿（通常为价值的 10-30%）
5. 乘以可触达客户基数

**公式：**
```
Value per Customer = Problem Cost × % Solved by Solution
Price per Customer = Value × Willingness to Pay % (10-30%)
TAM = Total Potential Customers × Price per Customer
SAM = TAM × % Meeting Buy Criteria
SOM = SAM × Realistic Adoption Rate
```

**适用场景：** 新品类、颠覆性创新、现有市场不明确时

**优势：** 展示价值创造、适用于新市场

**局限：** 需要假设、更难验证

## 逐步流程

### 第 1 步：定义市场

明确界定所测量的市场。

**需回答的问题：**
- 解决什么问题？
- 目标客户是谁？
- 产品/服务品类是什么？
- 地理范围是什么？
- 时间跨度是多少？

**示例：**
- 问题：电商公司在邮件营销自动化方面存在困难
- 客户：年收入超过 $1M 的电商店铺
- 品类：AI 驱动的邮件营销软件
- 地理：初期北美，计划全球扩展
- 时间跨度：3-5 年市场机会

### 第 2 步：收集数据源

确定计算所需的可信数据。

**自上而下数据源：**
- 行业研究报告（Gartner、Forrester、IDC）
- 政府统计数据（人口普查、BLS、行业协会）
- 上市公司财报和业绩
- 市场研究机构（Statista、CB Insights、PitchBook）

**自下而上数据源：**
- 客户访谈和调研
- 销售数据和 CRM 记录
- 行业数据库（LinkedIn、ZoomInfo、Crunchbase）
- 竞争情报
- 学术研究

**价值理论数据源：**
- 客户问题量化
- 时间/成本研究
- ROI 案例研究
- 定价研究和支付意愿调研

### 第 3 步：计算 TAM

应用选定方法论确定总市场。

**自上而下：**
1. 从研究中找到品类总量
2. 记录数据来源和年份
3. 如需要则应用增长率
4. 用多个来源交叉验证

**自下而上：**
1. 统计潜在客户总数
2. 计算客户平均年收入
3. 相乘得到 TAM
4. 按细分拆分

**价值理论：**
1. 量化总可触达客户基数
2. 计算单客户价值
3. 基于价值估算定价
4. 相乘得到 TAM

### 第 4 步：计算 SAM

将 TAM 缩小到可服务市场。

**应用筛选：**
- 地理约束（可服务的区域）
- 产品限制（当前具备的功能）
- 客户要求（规模、行业、用例）
- 分销渠道覆盖
- 监管或合规限制

**公式：**
```
SAM = TAM × (% matching all filters)
```

**示例：**
- TAM：$10B 全球邮件营销
- 地理筛选：40%（北美）
- 产品筛选：30%（电商聚焦）
- 功能筛选：60%（需要 AI 能力）
- SAM = $10B × 0.40 × 0.30 × 0.60 = $720M

### 第 5 步：计算 SOM

确定现实可获取的市场份额。

**考虑因素：**
- 竞争对手当前市场份额
- 新进入者的典型份额（2-5%）
- 可用资源（资金、团队、时间）
- 市场进入效果
- 竞争优势
- 达成时间（通常 3-5 年）

**保守估算：**
```
SOM (Year 3) = SAM × 2%
SOM (Year 5) = SAM × 5%
```

**示例：**
- SAM：$720M
- 第 3 年 SOM：$720M × 2% = $14.4M
- 第 5 年 SOM：$720M × 5% = $36M

### 第 6 步：验证和三角校验

使用多种方法交叉验证。

**验证技术：**
1. 比较自上而下和自下而上结果（差异应在 30% 以内）
2. 对照该领域上市公司收入检查
3. 验证客户数量假设
4. 合理性检查定价假设
5. 请行业专家审阅
6. 与类似市场品类比较

**危险信号：**
- TAM 太小（VC 支持的创业公司应 > $1B）
- TAM 太大（无数据支撑）
- SOM 过于激进（新进入者 5 年内 > 10%）
- 方法论间不一致（差异 > 50%）

## 行业特定考量

### SaaS 市场

**关键指标：**
- 目标细分中的潜在企业数量
- 平均合同价值（ACV）
- 典型市场渗透率
- 增购收入潜力

**TAM 计算：**
```
TAM = Total Target Companies × Average ACV × (1 + Expansion Rate)
```

### 交易平台市场

**关键指标：**
- 品类商品交易总额（GMV）
- 抽成率（占 GMV 的百分比）
- 总交易数或用户数

**TAM 计算：**
```
TAM = Total Category GMV × Expected Take Rate
```

### 消费市场

**关键指标：**
- 总可触达用户/家庭数
- 用户平均收入（ARPU）
- 使用频率

**TAM 计算：**
```
TAM = Total Users × ARPU × Purchase Frequency per Year
```

### B2B 服务

**关键指标：**
- 按规模/行业划分的目标企业数量
- 平均项目价值或服务费
- 典型采购频率

**TAM 计算：**
```
TAM = Total Target Companies × Average Deal Size × Deals per Year
```

## 市场规模测算的呈现

### 面向投资人

**结构：**
1. 市场定义和问题范围
2. TAM/SAM/SOM 及方法论
3. 数据来源和假设
4. 增长预测和驱动因素
5. 竞争格局背景

**要点：**
- 以自下而上计算为主（最具说服力）
- 用自上而下做三角校验
- 说明保守假设
- 关联收入预测
- 突出市场增长率

### 面向战略规划

**结构：**
1. 可触达客户细分
2. 按机会规模排序
3. 各细分进入策略
4. 预期渗透时间线
5. 资源需求

**要点：**
- 聚焦 SAM 和 SOM
- 展示细分级别细节
- 关联市场进入计划
- 识别扩展机会
- 讨论竞争定位

## 常见错误

**错误 1：混淆 TAM 和 SAM**
- 不要声称整个市场都可触达
- 应用现实的产品/地理约束
- 诚实面对可服务市场

**错误 2：SOM 过于激进**
- 新进入者 5 年内很少能获取 > 5%
- 考虑竞争和资源限制
- 展示现实的增长节奏

**错误 3：仅用自上而下**
- 投资人更看重自下而上验证
- 单一自上而下缺乏说服力
- 始终用多种方法三角校验

**错误 4：挑选性使用数据**
- 使用一致、最新的数据源
- 不要不当混合方法论
- 清晰记录所有假设

**错误 5：忽视市场动态**
- 考虑市场增长/衰退
- 考虑竞争强度
- 计入转换成本和壁垒

## 快速入门

执行市场规模测算：

1. **定义市场** — 问题、客户、品类、地理
2. **选择方法论** — 自下而上（优先）或自上而下 + 三角校验
3. **收集数据** — 行业报告、客户数据、竞争情报
4. **计算 TAM** — 应用方法论公式
5. **缩小到 SAM** — 应用产品、地理、细分筛选
6. **估算 SOM** — 2-5% 现实获取率
7. **验证** — 用替代方法交叉校验
8. **记录** — 展示方法论、来源、假设
9. **呈现** — 按受众结构化（投资人、战略、运营）

各方法论的详细分步指导，参见 `references/` 目录。完整计算示例，参见 `examples/` 目录。

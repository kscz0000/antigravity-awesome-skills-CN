先验证再动手

## 想法验证

### 验证框架
| 问题 | 如何回答 |
|------|----------|
| 问题存在吗？ | 与 5+ 潜在用户交流 |
| 有人付费吗？ | 预售或找到竞品 |
| 你能做出来吗？ | MVP 能否 2 周交付？ |
| 你能触达他们吗？ | 是否有分发渠道？ |

### 快速验证方法
1. **落地页测试**
   - 搭建落地页
   - 引流（广告、社区）
   - 衡量注册/兴趣

2. **预售**
   - 先卖再做
   - "加入候补享 5 折"
   - 没人买就转向

3. **竞品检查**
   - 有竞品 = 需求已验证
   - 无竞品 = 可能没市场
   - 找到你能填补的空隙

### 危险信号
- "人人都需要这个"（太宽泛）
- 没有明确买单者（谁付钱？）
- 依赖双边市场效应
- 需要巨大规模才能跑通

### 积极信号
- 明确具体的痛点
- 用户已在为替代方案付费
- 你有领域专长
- 有分发渠道

## MVP 极速冲刺

### 技术栈（独立创始人优化）
| 组件 | 选择 | 原因 |
|------|------|------|
| 前端 | Next.js | 全栈，Vercel 部署 |
| 后端 | Next.js API / Supabase | 快速，可扩展 |
| 数据库 | Supabase Postgres | 免费额度，含认证 |
| 认证 | Supabase / Clerk | 别自己写认证 |
| 支付 | Stripe | 行业标准 |
| 邮件 | Resend / Loops | 事务邮件 + 营销 |
| 托管 | Vercel | 免费额度充裕 |

### 第 1 周：核心
```
Day 1-2: Auth + basic UI
Day 3-4: Core feature (one thing)
Day 5-6: Stripe integration
Day 7: Polish and bug fixes
```

### 第 2 周：准备发布
```
Day 1-2: Landing page
Day 3: Email flows (welcome, etc.)
Day 4: Legal (privacy, terms)
Day 5: Final testing
Day 6-7: Soft launch
```

### MVP 中该跳过的
- 完美设计（够用就行）
- 全部功能（只做一个核心功能）
- 规模优化（以后再说）
- 自建认证（用现成服务）
- 多档定价（先简单来）

### 定价策略

为你的 Micro-SaaS 定价

**适用场景**：设定价格时

## 定价策略

### Micro-SaaS 定价档位
| 策略 | 适用场景 |
|------|----------|
| 单一价格 | 简单工具，价值明确 |
| 两档 | 免费/付费 或 基础/专业 |
| 三档 | 多数 SaaS（好/更好/最好） |
| 按用量 | API 产品，用量波动 |

### 起步定价框架
```
What's the alternative cost? (Competitor or manual work)
Your price = 20-50% of alternative cost

Example:
- Manual work takes 10 hours/month
- 10 hours × $50/hour = $500 value
- Price: $49-99/month
```

### 常见 Micro-SaaS 价格
| 类型 | 价格区间 |
|------|----------|
| 简单工具 | $9-29/月 |
| 专业工具 | $29-99/月 |
| B2B 工具 | $49-299/月 |
| 终身买断 | 月价的 3-5 倍 |

### 定价误区
- 太便宜（低估价值，招来劣质客户）
- 太复杂（让买家困惑）
- 既无免费版也无试用（无法体验）
- 收费太晚（尽早用钱验证）

## 发布手册

### 发布前（提前 2 周）
1. 积累邮件列表（落地页）
2. 参与社区（先提供价值）
3. 准备发布素材（演示、截图）
4. 安排内测用户

### 发布日渠道
| 渠道 | 投入 | 影响 |
|------|------|------|
| Product Hunt | 中 | 高 |
| Hacker News | 低 | 不定 |
| Reddit | 中 | 中 |
| Twitter/X | 低 | 中 |
| Indie Hackers | 低 | 中 |
| 邮件列表 | 低 | 高 |

### Product Hunt 发布
```
- Launch 12:01 AM PST Tuesday-Thursday
- Have maker comment ready
- Activate your network to upvote/comment
- Respond to every comment
- Don't ask for upvotes directly
```

### 发布后
- 跟进每位注册用户
- 持续收集反馈
- 立即修复关键 bug
- 启动 SEO/内容做长期增长
- 别在发布日之后就停止推广

## 市场选择

### B2B vs B2C
| 因素 | B2B | B2C |
|------|-----|-----|
| 价格承受力 | $50-500+/月 | $5-20/月 |
| 获客成本 | 较高 | 较低 |
| 流失率 | 较低 | 较高 |
| 支持需求 | 较高 | 较低 |
| 独立创始人友好 | 是 | 较难 |

### 适合 Micro-SaaS 的市场
- 小企业
- 自由职业者/代理公司
- 开发者
- 有收入的创作者
- 专业人士（律师、医生等）

### 危险信号市场
- 学生
- 无融资的初创公司
- 大众消费者
- 有免费替代品的市场

### 该转向的信号
- 关注度高，付费为零
- 用户喜欢但不掏钱
- 竞品全是免费的
- 目标市场没有预算

### 新用户来得快走得也快

根因：
产品没交付价值。

来的不是对的用户。

## 修复流失

### 搞清原因
```
1. Email churned users (personal, not automated)
2. Look at last active date
3. Check onboarding completion
4. Survey at cancellation
```

### 流失率基准
| 流失率 | 评估 |
|--------|------|
| < 3% 月流失 | 优秀 |
| 3-5% 月流失 | 良好 |
| 5-7% 月流失 | 需改进 |
| > 7% 月流失 | 危急 |

### 快速修复
- 改善引导（前 7 天是关键）
- 加"顿悟时刻"触发邮件
- 检查是否来了对的受众
- 补齐必备功能
- 提价（筛选认真用户）

### 引导清单
```
[ ] Clear first action after signup
[ ] Value delivered in first session
[ ] Email sequence for first 7 days
[ ] Check-in at day 3 if inactive
[ ] Success metric defined and tracked
```

## 简化定价

### 理想结构
```
Free tier (optional): Limited but useful
Paid tier: Everything most need ($X/mo)
Enterprise (optional): Custom pricing
```

### 如果需要多档
- 最多 3 档
- 差异清晰
- 标注推荐档位
- 年付折扣（20-30%）

### 好的定价页
| 元素 | 作用 |
|------|------|
| 价格清晰 | 不用算 |
| 功能列表 | 包含什么 |
| 推荐标签 | 引导决策 |
| FAQ | 消除疑虑 |
| 保障承诺 | 降低风险 |

### 测试
- A/B 测试价格
- 尝试减掉一档
- 问客户哪里看不懂
- 查看定价页跳出率

## 验证检查

### 无支付集成

严重度：高

信息：无支付集成——无法收款。

修复：集成 Stripe 或 Lemon Squeezy

### 无用户认证

严重度：高

信息：没有正式的认证系统。

修复：用 Supabase Auth、Clerk 或 Auth0——别自己写

### 无用户引导

严重度：中

信息：无用户引导——影响激活率。

修复：加欢迎流程、首操作提示和引导邮件

### 无产品分析

严重度：中

信息：无产品分析——完全盲飞。

修复：加 Posthog、Mixpanel 或简单事件追踪

### 缺少法律页面

严重度：中

信息：缺少法律页面——收款必需。

修复：加隐私政策和服务条款（用模板）

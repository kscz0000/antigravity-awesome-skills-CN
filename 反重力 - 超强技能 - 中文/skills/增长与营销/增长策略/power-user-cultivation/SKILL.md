# 高级用户培育

## 何时使用

当你需要识别和培育开发者倡导者、构建冠军计划、或将活跃用户转化为贡献者和布道者时使用此技能。触发词包括"高级用户"、"开发者倡导者"、"大使计划"、"冠军计划"、"社区贡献者"、"推荐计划"、"power users"、"developer advocates"、"ambassador program"、"champion program"。

此技能帮助你识别最活跃的开发者，并将他们转化为倡导者、贡献者和冠军。不是强制传道 — 只是为已经热爱你产品的开发者创造真正的价值。

---

## 开始之前

1. **加载你的开发者受众上下文**：
   - 检查 `.agents/developer-audience-context.md` 是否存在
   - 如果不存在，先运行 `developer-audience-context` 技能
   - 了解你的开发者在哪里活动以及什么能激励他们是必要的

2. **收集你的数据**：
   - 按用户的使用指标
   - 社区参与数据
   - 支持互动（有帮助的回答、功能请求）
   - 关于你产品的已创建内容
   - 推荐/邀请历史

## 高级用户光谱

并非所有参与用户的期望都一样：

| 层级 | 行为 | 他们想要什么 | 你的回应 |
|------|------|-------------|----------|
| **活跃用户** | 定期使用产品 | 产品持续正常工作 | 持续交付 |
| **参与用户** | 参与社区互动 | 帮助和认可 | 快速响应 |
| **倡导者** | 自发推荐你 | 内部人访问权限 | 优先体验，直接沟通渠道 |
| **冠军** | 创建内容，回答问题 | 平台和认可 | 正式计划 |
| **贡献者** | 贡献代码、文档、扩展 | 影响力和归属感 | 贡献者计划 |

**关键洞察**：不要试图推动所有人沿光谱上升。在开发者所在的位置与之相遇。有些人只想要一个出色的产品 — 这没问题。

## 识别潜在倡导者

### 行为信号

在你的数据中寻找以下模式：

**基于使用的信号**：
```
- Top 10% by API calls or usage
- Using advanced/power features
- Long tenure (>6 months active)
- Multiple projects using your product
- Early adopter of new features
```

**社区信号**：
```
- Answers questions from other users
- Files detailed, constructive bug reports
- Requests features thoughtfully
- Active in Discord/Slack/forums
- Mentions you positively on social
```

**内容信号**：
```
- Wrote blog post about your product
- Created tutorial or video
- Open sourced integration or extension
- Conference talk mentioning you
- Stack Overflow answers recommending you
```

### 社交监听发现

设置以下监控：

1. **正面提及**：
   - 人们推荐你的产品
   - 公开分享的成功案例
   - "刚刚用 [你的产品] 发布了"的帖子

2. **内容创作者**：
   - 关于你产品的博客文章
   - 教程视频
   - 会议演讲公告

3. **社区帮助者**：
   - 替你回答问题的人
   - 在讨论中为你的产品辩护的人
   - 分享技巧和窍门的人

### 构建高级用户列表

创建一个简单的追踪表：

| 姓名 | 公司 | 信号 | 评分 | 状态 |
|------|------|------|------|------|
| @jane | Startup X | 顶级使用量，写了博客文章，回答问题 | 92 | 冠军候选人 |
| @john | Agency Y | 重度使用，功能请求 | 65 | 参与用户 |
| @sam | Corp Z | 多个仓库使用产品 | 58 | 倡导者候选人 |

**评分计算**：
- 使用量排名前 10%：+20
- 社区活跃：+15
- 创建内容：+25
- 回答他人问题：+20
- 正面社交提及：+10
- 功能请求被采纳：+10

## 大使 / 冠军计划

### 计划设计

**分层 vs 扁平结构**：

| 结构 | 适合场景 | 优点 | 缺点 |
|------|----------|------|------|
| 分层（铜/银/金） | 大型社区 | 晋升路径清晰 | 可能显得企业化 |
| 扁平（所有人平等） | 小型社区 | 简单，平等 | 激励不足 |
| 仅邀请制 | 高端感 | 独占性，高质量 | 扩展困难 |
| 申请制 | 筛选意向 | 自选的参与用户 | 需要处理拒绝 |

**推荐**：从小规模邀请制开始。了解什么有效后再扩展。

### 开发者看重的权益

**应该提供**：

| 权益 | 为什么有效 |
|------|-----------|
| 功能优先体验 | 内部人感觉，率先知晓 |
| 与团队的直接沟通渠道 | 跳过支持队列，真正的影响力 |
| 会议门票赞助 | 实际价值，社交网络 |
| 专属周边 | 高品质物品，非廉价货 |
| 公开认可 | 建立他们的个人品牌 |
| 推荐/推荐信 | 职业价值 |
| AWS/GCP 额度 | 项目的实际价值 |
| 贡献者署名 | 公开归属 |

**不应提供**：

| 权益 | 为什么失败 |
|------|-----------|
| 强制内容配额 | 感觉像工作 |
| 繁重的 NDA 限制 | 扼杀热情 |
| 基于佣金的推荐 | 像传销 |
| 通用折扣 | 廉价，不特别 |
| 有名无实的头衔 | "大使"但没有权益 |

### 冠军计划模板

```markdown
# [PRODUCT] Champions Program

## What Champions Do
- Share feedback directly with our team
- Help other developers in community
- Create content when inspired (not required)
- Test new features before public release

## What Champions Get
- Private Slack channel with engineering team
- Early access to all features (2-week head start)
- Annual conference ticket sponsorship ($2,000 value)
- Quarterly swag drops (quality items, not junk)
- Public recognition on our website
- Reference letters upon request

## Expectations
- Be active in community at least 1x/week
- Give honest feedback (including criticism)
- No content quotas — create when you want to
- Maintain constructive, helpful tone

## How to Join
By invitation only. We identify champions through:
- Community participation
- Content creation
- Usage and engagement

If you think you qualify, email champions@[product].com
```

### 运营计划

**月度节奏**：
- 第 1 周：分享即将推出的功能，收集反馈
- 第 2 周：社区指标审查，识别新候选人
- 第 3 周：冠军聚焦（博客文章、推文等）
- 第 4 周：反馈收集，周边/权益发放

**沟通方式**：
- 私有 Slack/Discord 频道
- 每月与团队的视频通话（可选参加）
- 每季度与冠军经理的 1:1 沟通

## 开源贡献者体验

### 让贡献变得简单

| 障碍 | 解决方案 |
|------|----------|
| 找不到适合新手的 issue | 清晰标记 issue："good-first-issue"、"help-wanted" |
| 环境搭建太复杂 | 一键开发环境（Docker/devcontainer） |
| PR 审查耗时太长 | 承诺 48 小时内首次响应 |
| 贡献流程不清晰 | 提供步骤清晰的 CONTRIBUTING.md |
| 被拒绝时没有反馈 | 始终解释原因，建议改进方向 |

### CONTRIBUTING.md 模板

```markdown
# Contributing to [PROJECT]

Thanks for your interest in contributing!

## Quick Start

```bash
# One command setup
make dev
# or
docker-compose up
```

## Finding Issues

- **good-first-issue**: Great for first contribution
- **help-wanted**: We'd love help with these
- **documentation**: Improve our docs

## Making a Pull Request

1. Fork the repo
2. Create a branch: `git checkout -b feature/your-feature`
3. Make your changes
4. Run tests: `make test`
5. Push and create PR

## What to Expect

- First response within 48 hours
- We'll provide clear feedback
- Small PRs reviewed faster than large ones

## Recognition

All contributors are:
- Added to CONTRIBUTORS.md
- Credited in release notes
- Eligible for contributor swag

## Questions?

- Discord: [link]
- Email: contributors@[project].com
```

### 贡献者认可

| 贡献级别 | 认可方式 |
|----------|----------|
| 首个 PR 合并 | 欢迎消息，加入 CONTRIBUTORS |
| 3+ PR 合并 | 贡献者周边礼包 |
| 10+ PR 合并 | "核心贡献者"标签，直接 Slack 访问权限 |
| 持续贡献 | 维护者邀请，会议赞助 |

## 用户生成内容计划

### UGC 类型

| 内容类型 | 价值 | 获取难度 |
|----------|------|----------|
| Twitter/社交提及 | 社交证明 | 低（自然发生） |
| 博客文章 | SEO，可信度 | 中 |
| 视频教程 | 参与度，触达 | 高 |
| 会议演讲 | 可信度，触达 | 非常高 |
| 扩展/集成 | 生态价值 | 高 |

### 鼓励内容创作

**被动鼓励**：
- 显著展示已有内容
- 转发/分享所有关于你的创作
- 在更新日志和通讯中展示创作者

**主动鼓励**：
- "写关于我们"页面，附资源
- 内容悬赏计划（见下文）
- 教程模板和指南
- 会议演讲支持（幻灯片审查、练习）

### 内容悬赏计划

为内容提供报酬：

| 内容类型 | 悬赏金额 | 要求 |
|----------|----------|------|
| 博客文章 | $200-500 | 800+ 字，技术深度，原创 |
| 视频教程 | $300-750 | 5-15 分钟，制作精良，完成某个任务 |
| 会议演讲 | $500 + 差旅 | 被接受的演讲，真实提及产品 |
| 集成/扩展 | $500-2000 | 已发布，有文档，持续维护 |

**指南**：
- 必须披露赞助/悬赏关系
- 内容必须真正有用（非广告软文）
- 你获得首次审阅权但无编辑控制权
- 创作者保留所有权

## 开发者推荐计划

### 对开发者有效的方式

| 方式 | 有效性 | 备注 |
|------|--------|------|
| 双向获益（双方都获得价值） | 高 | 推荐人和被推荐人都受益 |
| 额度/服务 | 高 | 更多使用产品，而非套现 |
| 现金 | 中 | 有效但感觉像交易 |
| 仅周边 | 低 | 不足以支撑持续推荐 |
| 佣金/联盟 | 低 | 像传销，损害可信度 |

### 推荐计划设计

**推荐结构**：

```
Refer a developer to [PRODUCT]:

You get: $50 in credits
They get: $50 in credits + extended trial

No limits. Stack as many as you want.
```

**为什么有效**：
- 双方都受益（公平）
- 额度鼓励更多使用（飞轮效应）
- 没有奇怪的佣金追踪
- 简单易懂

### 推荐计划模板

```markdown
# [PRODUCT] Referral Program

## How It Works

1. Share your referral link: [DASHBOARD/REFERRALS]
2. Friend signs up and becomes a paying customer
3. You both get $50 in credits

## Fine Print

- Credits apply to future bills (never cash out)
- Referred user must be new (no existing accounts)
- Referred user must become paying customer
- No limit on referrals
- Credits never expire

## Your Referral Link

[LINK]

## Tracking

See all your referrals at: [DASHBOARD/REFERRALS]
```

### 让推荐变得简单

- 可分享链接（无需记忆代码）
- 一键复制按钮
- 预写好的推文/消息便于分享
- 显示推荐状态的仪表盘
- 推荐转化时的邮件通知

## 社区认可与奖励

### 认可层级

| 层级 | 认可方式 | 示例 |
|------|----------|------|
| 公开致谢 | Twitter 提及，通讯展示 | "感谢 @jane 提交的优秀 Bug 报告！" |
| 聚焦展示 | 博客文章，视频访谈 | "开发者聚焦：Jane 如何使用 [PRODUCT]" |
| 贡献者页面 | 网站列表 | CONTRIBUTORS.md，网站展示墙 |
| 顾问角色 | 路线图参与 | Beta 访问，反馈会议 |
| 正式头衔 | 冠军、大使、维护者 | 徽章，简介更新 |

### 有意义的认可

**应该**：
- 具体说明他们做了什么
- 公开进行（经许可）
- 及时（快速认可）
- 帮助他们的职业发展（推荐信、引荐）
- 给他们平台（你的博客、你的舞台）

**不应**：
- 泛泛的"感谢我们的社区"
- 对公开贡献给予私下感谢
- 延迟的认可（数月之后）
- 无实质的认可
- 有名无实的头衔

### 开发者想要的周边

| 要 | 不要 |
|----|------|
| 高品质 T 恤（Bella+Canvas 等） | 廉价促销 T 恤 |
| 优质卫衣 | 任何涤纶制品 |
| 实用物品（笔记本、数据线、背包） | 减压球、笔 |
| 限量版 / 独占 | 与会议摊位赠品相同 |
| 贴纸（永远需要） | 过时品牌 |

**专业建议**：问问你的高级用户想要什么。调研胜过假设。

### 认可工作流

```
When someone does something notable:

1. Screenshot/document it (tweet, PR, blog post)
2. Public thank you within 24 hours
3. Add to monthly newsletter spotlight
4. Consider for champion program if pattern continues
5. Update power user tracker
```

## 衡量倡导者影响力

### 追踪指标

| 指标 | 如何衡量 | 为什么重要 |
|------|----------|-----------|
| 创建的内容 | 统计文章、视频、演讲 | 触达和认知 |
| 回答的问题 | 社区活动 | 支持分流 |
| 驱动的推荐 | 推荐追踪 | 直接获客 |
| 社交提及 | 社交监听工具 | 有机认知 |
| PR/贡献 | GitHub 活动 | 产品改进 |

### 归因挑战

开发者倡导的归因很难。请接受以下现实：
- 博客文章可能在数月后才带来注册
- 口碑传播是无形的
- Stack Overflow 回答会持续累积
- 会议演讲触达的人不会立即转化

**追踪方向而非精确数字**：
- 调查新注册用户："你是怎么听说我们的？"
- 追踪推荐链接的使用
- 监控社交提及趋势
- 关联内容与流量高峰

### 倡导者 ROI 计算

粗略框架：

```
Champion program cost:
- Swag: $200/person/year
- Conference sponsorship: $2000/person/year
- Staff time: $5000/year total

Total for 10 champions: $27,000/year

Champion value (estimate):
- Average referrals: 5/person/year = 50 total
- Referral LTV: $1000
- Referral value: $50,000

- Content created: 20 posts
- Traffic value: $500/post = $10,000

- Questions answered: 200
- Support deflection: $25/ticket = $5,000

- Social mentions: 100
- Brand value: Hard to quantify

Estimated ROI: ~3x (conservative)
```

## 常见错误

| 错误 | 为什么失败 | 修复方式 |
|------|-----------|----------|
| 强制内容创作 | 耗尽倡导者，感觉像工作 | 设为可选，发生时给予奖励 |
| 基于佣金的推荐 | 像传销，损害真实性 | 改用额度/互惠方式 |
| 忽视小贡献者 | 他们可能成为大贡献者 | 认可每一份贡献 |
| 泛泛的认可 | 感觉空洞 | 具体说明他们做了什么 |
| 要求签 NDA | 扼杀分享热情 | 仅对真正敏感的信息使用 NDA |
| 有名无实的计划 | 人们很快离开 | 真实权益，而非空头头衔 |
| 一开始就做得太大 | 难以管理 | 从 5-10 名冠军开始，缓慢增长 |

# 免费层级策略

设计能转化为付费且不引发反感或滥用的免费层级。

## 本技能涵盖的内容

- **免费层级 vs 免费试用 vs Freemium** —— 选择合适的模型
- **合理的用量限制** —— 良好与糟糕的限制维度
- **功能门控策略** —— 哪些保持免费、哪些门控
- **避免"免费层级税"带来的反感** —— 积极的免费层级体验
- **升级触发点与时机** —— 自然触发、场景化沟通
- **开源 + 商业化模式** —— 开放核心、社区关系

## 核心原则

最好的免费层级，对个人开发者慷慨相待，而当项目成长时，又能自然地平滑过渡到付费层级。免费层级的限制应因增长触发，而非时间。升级来自项目的成功。

## 快速参考

**免费层级应当：**
- 允许构建并运行真实的副业项目
- 完整覆盖爱好者场景
- 让开发者能预测何时撞到限制
- 永远不要求信用卡

**门控到付费之后的：**
- 团队协作
- 企业功能（SSO、审计日志）
- 更高的规模与性能
- SLA 与优先支持

## 何时使用本技能

- 设计新产品的定价模型
- 评估免费层级的有效性
- 减少免费层级滥用
- 提升免费到付费的转化
- 决定哪些功能门控
- 规划开源商业化

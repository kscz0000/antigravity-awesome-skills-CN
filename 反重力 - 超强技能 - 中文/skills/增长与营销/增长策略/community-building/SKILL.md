# 社区建设
## 何时使用

当用户希望在 Discord、Slack 或论坛等平台上构建、壮大或改进开发者社区时使用此技能。触发短语包括"开发者社区""Discord 服务器""Slack 社区""社区策略""社区互动""社区管理""社区增长"或"社区..."。


本技能可帮助你在 Discord、Slack、论坛及其他平台上构建和管理开发者社区。涵盖频道结构、新成员引导、互动项目、处理不友好行为，以及社区主导的增长。

---

## 开始之前

**首先加载你的受众上下文。** 请阅读 `.agents/developer-audience-context.md` 以了解：

- 你的开发者是谁（角色、资历、兴趣）
- 他们已经聚集在哪里（避免与现有平台竞争）
- 他们讨论什么问题（社区话题聚焦）
- 他们如何沟通（正式或非正式的语调）

如果上下文文件不存在，请先运行 `developer-audience-context` 技能。

## 平台选择

### 对比矩阵

| 平台 | 最适合 | 优点 | 缺点 |
|----------|----------|------|------|
| **Discord** | 开发者工具、游戏、开源 | 实时、功能丰富、免费 | 可能比较混乱、企业感不强 |
| **Slack** | 企业、B2B SaaS | 专业、熟悉 | 规模化时费用高、有消息限制 |
| **GitHub Discussions** | 开源项目 | 集成、异步、可搜索 | 社区氛围较弱 |
| **Discourse** | 长内容、可搜索 | 利于 SEO、支持嵌套、可自主拥有 | 需要维护、有托管成本 |
| **Circle** | 课程、付费社区 | 与课程集成、界面整洁 | 需付费、对开发者不够原生 |

### 决策框架

| 如果你的受众是…… | 推荐考虑 |
|------------------------|----------|
| 个人开发者、开源 | Discord |
| 企业团队 | Slack |
| 偏技术、偏好异步 | GitHub Discussions |
| 混合型、需要可搜索 | Discourse |
| 基于课程/教育 | Circle |

## 频道结构

### Discord 频道模板

```
📢 INFORMATION（信息）
├── #welcome — 首次着陆、规则、链接
├── #announcements — 官方更新（仅管理员可发帖）
├── #rules — 行为准则
└── #introductions — 新成员介绍

💬 GENERAL（综合）
├── #general — 主体讨论
├── #off-topic — 非项目聊天
└── #show-what-you-built — 分享作品

❓ SUPPORT（支持）
├── #help — 一般问题
├── #troubleshooting — Bug 帮助
└── #feature-requests — 建议

🔧 TECHNICAL（技术）
├── #backend — 后端讨论
├── #frontend — 前端讨论
└── #devops — 基础设施讨论

🤝 COMMUNITY（社区）
├── #jobs — 招聘帖（如允许）
├── #events — 线下聚会、会议
└── #content — 博客文章、视频

📚 RESOURCES（资源）
├── #learning — 教程、课程
└── #tools — 实用工具与库
```

### Slack 频道模板

```
# welcome
# announcements (admin-only)
# general
# help
# random (off-topic)
# jobs (optional)
# introductions
# feedback
```

### 频道指南

| 频道类型 | 发帖规则 | 管理力度 |
|--------------|---------------|------------------|
| **Announcements（公告）** | 仅管理员 | N/A |
| **General（综合）** | 围绕主题讨论 | 轻度 |
| **Help（求助）** | 欢迎提问，请保持耐心 | 中度 |
| **Off-topic（闲聊）** | 在行为准则范围内自由交流 | 轻度 |
| **Jobs（招聘）** | 必须按固定格式发帖 | 重度 |
| **Introductions（自我介绍）** | 每人限发一帖 | 轻度 |

## 新成员引导体验

### 新成员旅程

```
加入服务器
    ↓
欢迎消息（私信或公开）
    ↓
阅读规则 / 接受规则
    ↓
验证（可选：GitHub、邮箱）
    ↓
自我介绍
    ↓
首次互动
    ↓
正式成员
```

### 欢迎消息模板

**Discord 私信：**
```
欢迎来到 [社区名称]！👋

下面是上手指南：

1. 请阅读 #rules 中的规则
2. 在 #introductions 中做自我介绍
3. 在 #help 中提问 — 我们很友善！

快速链接：
• 文档：[link]
• 上手指南：[link]
• GitHub：[link]

很高兴见到你！
```

**公开 #welcome 频道：**
```
# 欢迎来到 [社区名称]！

我们是 [简短描述你是谁以及做什么]。

## 快速开始

1. **阅读规则** → #rules
2. **自我介绍** → #introductions
3. **寻求帮助** → #help
4. **和我们聊天** → #general

## 实用链接

- [文档]
- [GitHub]
- [官网]

## 有问题？

在 #help 中留言，或 @moderators
```

### 角色分配

| 角色 | 获得方式 | 权限 |
|------|------------|-------------|
| **New Member（新成员）** | 加入时自动获得 | 限定频道访问 |
| **Member（成员）** | 验证或达到活跃时长 | 完全访问 |
| **Contributor（贡献者）** | PR 被合并、积极帮助他人 | 徽章、专属频道 |
| **Moderator（版主）** | 受邀 | 拥有管理权限 |
| **Admin（管理员）** | 核心团队 | 完全访问 |

## 互动项目

### 讨论话题引导

安排定期互动：

| 星期 | 话题类型 | 示例 |
|-----|-------------|---------|
| 周一 | 本周目标 | "你这周在做什么？" |
| 周三 | 技术问题 | "有争议的话题：Tab 还是空格？" |
| 周五 | 展示与分享 | "分享你这周发布的内容" |

### 认可项目

| 项目 | 描述 | 频率 |
|---------|-------------|-----------|
| **月度贡献者（Contributor of the Month）** | 表彰最热心的帮助者 | 每月 |
| **首个 PR 庆祝** | 欢迎新的贡献者 | 视情况发生 |
| **里程碑徽章** | 10/50/100 条消息 | 自动 |
| **专家角色** | 认可某领域专长 | 每季度 |

### 活动创意

| 活动类型 | 形式 | 投入 |
|------------|--------|--------|
| **Office Hours（接待时间）** | 与团队实时问答 | 低 |
| **Show & Tell（展示与分享）** | 成员演示项目 | 低 |
| **Workshops（工作坊）** | 教学型课程 | 中 |
| **Hackathons（黑客松）** | 构建挑战 | 高 |
| **Game Night（游戏之夜）** | 非技术趣味活动 | 低 |
| **AMA Sessions（AMA 活动）** | 邀请嘉宾专家 | 中 |

### 互动指标

| 指标 | 它能告诉你的内容 |
|--------|------------------|
| **DAU/MAU** | 日活跃与月活跃用户对比 |
| **Messages per user** | 单个用户的互动深度 |
| **Questions answered** | 社区自助程度 |
| **New member retention** | 新成员引导的有效性 |
| **Event attendance** | 项目共鸣度 |

## 处理不友好行为

### 行为准则要点

```markdown
# 行为准则

## 我们的标准

**应做：**
- 保持尊重与包容
- 帮助他人学习（不要甩一句"自己看文档"）
- 假定对方出于善意
- 给出建设性反馈
- 举报问题，不要纠缠

**不应做：**
- 人身攻击或骚扰
- 任何形式的歧视
- 垃圾信息或自我推销
- NSFW（不适宜工作场合）内容
- 人肉搜索或侵犯隐私
- 恶意抬杠

## 处置措施

1. **警告** — 首次违规、出于善意的情况
2. **临时禁言** — 屡次出现问题
3. **临时封禁** — 严重违规
4. **永久封禁** — 性质恶劣或屡次违规

## 举报方式

私信任何 @版主 或使用举报功能。
所有举报都会保密。
```

### 版主处置手册

| 情况 | 应对方式 |
|-----------|----------|
| **激烈争论** | "请保持建设性。两种观点都有可取之处。" |
| **过度索取帮助者（Help vampire）** | "这里是一份如何正确提问的指南：[链接]" |
| **自我推销垃圾信息** | 根据频次决定删除、警告或封禁 |
| **跑题闲聊** | "讨论很有趣！我们把它转到 #off-topic 吧" |
| **骚扰行为** | 立即禁言、调查、大概率封禁 |
| **恶意挑衅者** | 不要公开纠缠，悄悄封禁 |

### 降级处理技巧

1. **承认对方感受** — "我能看出这让你感到沮丧"
2. **转到私信** — "我们私下继续讨论吧"
3. **暂时搁置** — "先暂停一下，明天再回来看"
4. **澄清意图** — "我想可能存在一些误解"
5. **设定边界** — "我们是来帮忙的，但不容忍被吼"

### 版主自我关怀

| 风险 | 缓解方式 |
|------|------------|
| 倦怠 | 轮流承担版主工作 |
| 代入个人情绪 | 提醒自己：这不是针对你 |
| 冒名顶替综合征 | 定期团队交流 |
| 孤立感 | 版主专属私密频道 |

## 社区主导的增长

### 口碑传播策略

| 策略 | 方法 |
|--------|-----|
| **推荐计划** | 对带来留存邀请的奖励 |
| **值得分享的内容** | 独家见解、优先体验 |
| **成员聚焦** | 突出成员 → 他们主动分享 |
| **成功故事** | "我通过这个社区找到了工作" |

### 用户生成内容

| 内容类型 | 如何鼓励 |
|--------------|------------------|
| **教程** | "在 #show-what-you-built 分享你的搭建过程" |
| **问答串** | 对有帮助的回答给予奖励 |
| **项目展示** | 每月举办演示活动 |
| **用户证言** | 向满意的成员征集 |

### 社区大使

识别并赋能超级用户：

| 大使类型 | 角色 |
|---------------|------|
| **Greeters（迎新者）** | 欢迎新成员 |
| **Helpers（热心帮手）** | 回答支持类问题 |
| **Content creators（内容创作者）** | 教程、视频、指南 |
| **Event organizers（活动组织者）** | 筹办社区活动 |
| **Connectors（连接者）** | 为成员之间牵线搭桥 |

## 社区指标

### 健康度仪表盘

| 指标 | 健康 | 警告 | 需要处理 |
|--------|---------|---------|---------------|
| **响应时长（支持）** | <24h | 24-72h | >72h |
| **未回复问题比例** | <10% | 10-25% | >25% |
| **新成员 7 天留存率** | >40% | 20-40% | <20% |
| **月活跃占比** | >20% | 10-20% | <10% |
| **版主消息占比** | <30% | 30-50% | >50% |

### 增长指标

| 指标 | 如何追踪 |
|--------|-------------|
| **Total members（总成员数）** | 平台分析 |
| **Join rate（加入率）** | 每周新成员数 |
| **Churn rate（流失率）** | 每月离开人数 |
| **Engagement depth（互动深度）** | 每个活跃用户的消息数 |
| **Support success（支持成功率）** | 已解决问题的百分比 |

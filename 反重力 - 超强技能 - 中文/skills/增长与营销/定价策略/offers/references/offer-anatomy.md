# 报价解剖

一个完整的报价有六个组件。缺少任何一个都会损害转化——通常很明显。

## 六个组件

| # | 组件 | 回答的问题 | 失败在哪里 |
|---|------|-----------|-----------|
| 1 | **核心交付物** | 他们得到什么？ | 太模糊，或以功能而非结果来推销 |
| 2 | **赠品叠加** | 还有什么让他们觉得核心物超所值？ | 要么没有赠品，要么虚高/假赠品 |
| 3 | **保障** | 如果不生效怎么办？ | 没有、类型错误、或过度承诺 |
| 4 | **稀缺性 / 紧迫感** | 为什么是现在，不是以后？ | 没有、假的、或破坏性操纵 |
| 5 | **命名** | 这东西叫什么？ | 泛泛、内部术语、或根本没有名称 |
| 6 | **价格 + 付款结构** | 他们付多少、怎么付？ | 单一数字，没有付款灵活性 |

---

## 1. 核心交付物

他们实际得到的东西。

### 定义为结果，不是功能列表

- **以功能推销（弱）：** "6个模块、24节课、每周通话、私密社区。"
- **以结果推销（强）：** "一个在60天内每周带来5个合格线索的客户获取系统——和你一起构建，不是交给你。"

功能仍然重要——买家想知道他们得到什么——但*框架*是结果。功能支持结果，不替代结果。

### 明确定义范围

包含什么。不包含什么。什么是可选的。买家买的是清晰；模糊侵蚀感知实现概率。

范围声明示例：
```
Includes:
- 90-day program with weekly live calls (recorded)
- Private Slack with daily founder access
- 12 fill-in-the-blank templates
- 1 90-minute strategy session with a senior strategist

Doesn't include:
- 1:1 calls outside the strategy session
- Implementation of the work (you/your team does this; we coach)
- Tools and software (you provide; we recommend specific stacks)
```

### 深度匹配买家的认知阶段

老练的买家想要方法论和范围。新入品类的买家想要理想结果和证据。读懂你的受众。

---

## 2. 赠品叠加

你添加的东西，让核心在要价下显得物超所值。

赠品同时做三件事：
1. **提升感知价值** 整个报价的
2. **降低感知风险** — 即使核心交付不足，"我还免费得到了X"
3. **关闭具体异议** — 每个赠品可以针对一个不同的购买异议

### 如何构建赠品

对于买家的每个主要异议，添加一个关闭它的赠品：

| 异议 | 针对性赠品 |
|------|-----------|
| "我没时间实施这个" | 代办设置，第1天 |
| "我不知道该用什么工具" | 预筛选工具栈 + 折扣码 |
| "如果我卡住了怎么办？" | 30天异步支持 |
| "我不确定我团队会同意" | 利益相关者演示文稿 |
| "我之前试过类似的东西但没效" | 你同样情况的人的案例 |

4个关闭4个具体异议的赠品，转化远好于4个泛泛"额外"赠品。

### 不要虚高

"$50,000赠品！"在$500报价上读起来像骗局。不对称性摧毁信任。

赠品应该：
- 有买家可验证的标价（与可比产品比较）
- 总值低于价格的2倍（如$1K报价可以舒适地有~$1.5K赠品）
- 是你真的愿意单独出售的东西

完整的赠品叠加框架，参见 [bonus-stacking.md](bonus-stacking.md)。

## 3. 保障

如果不生效怎么办。

保障直接提升感知实现概率（买家想："他们只会在有把握时才提供这个"）。它也降低努力与牺牲（更少的情感风险）。

错误的保障会伤害：
- 过度承诺的保障吸引退款猎人
- 泛泛的"100%保障"没有条件，读起来法律上不可执行
- 完全没有保障暗示你不自信

正确的类型取决于你的商业模式、退款风险承受能力和买家老练度。完整分类参见 [guarantee-design.md](guarantee-design.md)。

## 4. 稀缺性 / 紧迫感

现在买的理由，不是以后。

两种类型：
- **稀缺性** — 有限的*数量*（班期人数、名额、库存、批次）
- **紧迫感** — 有限的*时间*（班期截止、季节、赠品到期）

标准：**稀缺性必须是真实的。** 假倒计时和"仅剩3个名额"的谎言有效一次然后永久烧毁信任。互联网很小；你会被抓到。

常见的诚实稀缺性格式：
- 班期周五关门（因为班期真的周一开始）
- 创始成员定价前20名客户（因为你受产能约束）
- 季节性产品或服务（因为需求是季节性的）
- 赠品在发布结束时到期（因为赠品是你的时间）
- 基于产能的服务层级（因为你确实不能再接客户了）

创建真实稀缺性的完整指南，参见 [scarcity-urgency.md](scarcity-urgency.md)。

## 5. 命名

这东西叫什么。

有名称的报价胜过无名称的报价，三个原因：
1. **可复述性** — 买家可以告诉朋友
2. **区分性** — 名称让它成为一个*东西*，不是泛泛的服务
3. **定价权** — 品牌化的报价可以比同样交付作为服务出售收费更高

### 有效的命名模式

- **结果命名：** "30天激活冲刺" — 命名他们得到什么
- **方法论命名：** "VAULT 框架" — 命名你怎么做
- **身份命名：** "创始人营销OS" — 命名它是为谁
- **压缩命名：** "5天班期" — 命名时间/结构

### 无效的命名模式

- **泛泛描述：** "营销教练项目" — 忘得快
- **内部术语：** "Tier 2 标准" — 买家无法复述
- **课程贩子：** "赚钱机器" — 被识别为骗局
- **双关过度：** "GrowthGoGetter" — 显得低级

### 实用测试

买家能否给朋友发短信："我刚报名了*[名称]*。$X，你能得到[一句话结果]"？如果可以，名称有效。如果不，改名。

## 6. 价格 + 付款结构

价格是显而易见的部分。结构是被低估的部分。

### 价格不是数字，是比较

买家将价格与以下比较：
- 理想结果（这能让我得到想要的结果吗？）
- 次优替代（我还能买什么？）
- 什么都不做的成本（维持现状要花我什么？）
- 你自己目录中的其他项目（锚定定价）

你可以不改数字就移动价格感知，通过：
- 更生动地展示什么都不做的成本
- 对照更高价的替代方案锚定
- 先在你的目录中以更高价格排列其他项目

### 付款结构本身就是杠杆

同样总价，不同结构转化差异很大：

| 结构 | 何时适用 | 权衡 |
|------|---------|------|
| **全额付款** | 高信任买家，较低价位 | 最高感知承诺，最小买家池 |
| **2-4期分期** | 中等价位，犹豫买家 | 更多买家，付款违约 |
| **月度订阅** | SaaS、持续服务 | 年金收入，流失风险 |
| **结果后付款** | 高信心交付，老练买家 | 现金流滞后，更少争议 |
| **首付 + 交付后尾款** | 基于里程碑交付的服务 | 后端余额风险 |
| **免费试用 → 付费** | 低摩擦SaaS、信息产品 | 转化断崖 |

通常正确的做法不是降价——而是添加付款计划。同样$6K，"今天$6K"vs"每月$2K × 3"转化差异巨大。

## 组合在一起：示例

一个 B2B 分时 CMO 服务。

| 组件 | 弱版 | 强版 |
|------|------|------|
| **核心** | "分时CMO服务" | "8周营销审计 + 90天执行计划，由3+家类似公司的CMO交付" |
| **赠品** | 无 | (1) 90天每周1:1检查；(2) 预筛选执行合作伙伴引荐；(3) 董事会营销战略部分模板 |
| **保障** | 无 | "8周审计后，如果你没有一个你会自己跑的清晰90天计划，不收审计费" |
| **稀缺性** | 无 | "我们每季度接2个项目——下一个空位在[日期]开放" |
| **命名** | "fCMO咨询" | "90天营销重置" |
| **价格** | "$15K，预付" | "$15K → $5K启动，$5K第8周，$5K第16周" |

同样交付。同样的人。不同的报价。不同的转化。

要点：大多数"我们需要降价"的对话实际上是"我们六个组件有一个缺失或薄弱"的对话。

# 赠品叠加

如何添加提升感知价值而不贬低核心报价的赠品。

## 赠品实际做什么

同时做三件事：

1. **提升感知价值** 整个报价的
2. **降低感知风险** — 即使核心交付不足，"我还免费得到了X"
3. **关闭具体购买异议** — 每个赠品可以针对一个异议

第三个是被低估的那个。大多数弱赠品叠加向买家扔四个泛泛的"额外"赠品。强赠品叠加读懂买家的具体犹豫并按序关闭。

---

## 核心原则：赠品作为异议处理器

对于买家的每个主要异议，添加一个关闭它的赠品。

### 常见异议 → 匹配赠品

| 异议 | 针对性赠品 |
|------|-----------|
| "我没时间实施这个" | 代办设置，第1周 |
| "我不知道该用什么工具" | 预筛选工具栈 + 折扣码 |
| "如果我卡住了怎么办？" | 30天异步Slack支持 |
| "我不确定我团队会同意" | 利益相关者演示文稿 |
| "我之前试过类似的东西但没效" | 你同样情况的人的案例 |
| "我这个行业的[边缘情况]怎么办？" | 行业特定赠品参考文档 |
| "我得学一堆新工具吗？" | 我们推荐工具的预建模板 |
| "如果我完不成怎么办？" | 第30天1:1问责检查 |
| "我的情况比普通买家更复杂" | 1:1入驻通话定制计划 |
| "在[地区/语言]有效吗？" | 本地化版本或附录 |

4个关闭4个具体异议的赠品，转化远好于4个泛泛"额外"赠品。

### 如何找到买家的真实异议

1. 阅读过去6个月每封退款请求邮件和销售通话转录
2. 大声读你自己的销售页，写下浮现的每个疑虑
3. 问3个近期买家："什么差点让你不买？"

答案会聚集在3–6个异议周围。为每个建一个赠品。

---

## 赠品价值的数学

每个赠品有标价（如果单独购买需要多少钱）。赠品应该：

1. **有买家可验证的标价。** 与可比产品或服务比较。"$497价值——这就是独立模板包的价格"胜过"$5,000价值"。（独立出售？和什么比？）

2. **总值低于核心报价价格的2倍。** $1K报价可以舒适地有$1.5K赠品。$1K报价带"$25K赠品"读起来像骗局。

3. **是你真的会单独出售的东西。** 如果你从不把赠品作为独立产品出售，标价就不真实。老练的买家看得出来。

4. **每个有具体命名的结果。** "赠品：营销工具包"很弱。"赠品：为你前90天准备的12个预建Notion模板，价值$297，因为这是独立模板包在[链接]的售价"很强。

---

## 有效的4赠品模式

大多数强报价恰好叠加3–5个赠品。更多开始感觉像凑数；更少让异议未关闭。

常见结构：

| # | 类型 | 目的 | 典型价值 |
|---|------|------|---------|
| 1 | **速度赠品** | 消除时间延迟异议 | 模板、模板包、加速器 |
| 2 | **信任赠品** | 消除可能失败的异议 | 案例、方法论文档、示例库 |
| 3 | **卡住赠品** | 消除"如果我卡住了"异议 | 答疑时间、Slack、按需支持 |
| 4 | **决策赠品** | 消除"我必须在X和Y之间选择"异议 | 工具栈 + 折扣码、预筛选推荐 |
| 5（可选） | **超出所求赠品** | 增加理想结果面积 | 相邻交付物、相关框架、合作伙伴报价 |

$2K B2B 文案课示例：

| # | 赠品 | 关闭 |
|---|------|------|
| 1 | "30个优秀销售页模板(最后更新2026-Q2)" — $297价值 | "我没时间从零写" |
| 2 | "9个使用这些框架达到$250K MRR的机构案例" — $0（证据，非可售资产） | "这对我的情况真的有效吗？" |
| 3 | "60天Slack访问含每周答疑时间" — $497价值 | "如果我在具体项目上卡住了怎么办？" |
| 4 | "工具栈：我们用的5个工具 + 折扣码(每年省~$1,200)" — $1,200价值 | "我不知道该用什么" |
| 5 | "加赠课程：如何收费$5K+/项目" — $297价值 | 定价信心（相邻理想结果） |

标价总计：~$2,300赠品在$2K核心上。数学成立（低于2x）。每个赠品关闭真实异议。

## 赠品何时适得其反

### 虚高价值

"$50,000赠品今日包含！"在$497产品上。不对称性是信号——每个老练买家的废话探测器都会响。

标价必须可辩护。如果你不能指向可比价格，不要报价。

### 贬低核心的赠品

如果你的核心报价是"我给你写销售页$5K"，你的赠品是"加赠——终身免费销售页编辑！"——赠品暗示核心不完整。现在买家想知道为什么他们要在*没有*赠品时购买。

赠品应该对完整核心是*增值的*，不是不完整核心的补丁。

### 赠品叠加替代核心

弱核心被超赞赠品包围在销售时刻转化但产生愤怒的退款请求。买家买了赠品，得到核心，感觉被骗。

顺序：先强核心*，然后用赠品应对具体异议。

### 叠加过高

5+个带高标价的赠品开始读起来像课程贩子漏斗。高端买家完全忽略赠品列表；中端买家感觉在被追加销售；新买家被搞糊涂。

3–5个赠品，每个有具体目的。封顶。

### 和别人一样的赠品

"加赠！私密社区访问！"在你品类每个课程上不是赠品，是标配。如果每个竞争对手都提供相同赠品，没有一个是差异化因素。

找到特定于你买家情况的赠品。针对SaaS买家的SaaS赠品每次都打败泛泛的"私密社区"。

## 赠品交付：时机很重要

第1天交付的赠品关闭"如果我从不用它怎么办？"风险。
第4周交付的赠品维持动力。
完成时交付的赠品奖励完成。

有意地混合时机：

- 第1天：速度赠品(模板、模板包、工具包)
- 第2–4周：支持赠品(Slack、答疑时间、检查)
- 完成时：身份赠品(证书、校友访问)

提前获得所有赠品的买家更容易放弃（他们得到了想要的，失去完成的动力）。完成时获得部分赠品的买家更可能完成（并推荐）。

## 审计

对于当前报价上的每个现有赠品，问：

1. **这个关闭什么具体购买异议？** 如果你命名不出来，它是填充。
2. **它可辩护的标价是什么？** 如果你不能指向可比价格，去掉金额。
3. **买家是第1天获得它，还是在旅程中有意义的节点？** 时机应支持客户结果，不只是转化事件。
4. **这个赠品特定于我的买家吗，还是任何竞争对手都能提供？** 如果是泛泛的，替换。
5. **赠品是否强到没有核心报价也能转化？** 如果是，核心太弱。修核心，不要依赖赠品。

大多数卡住的报价要么零赠品，要么太多泛泛赠品。正确的做法通常是：砍到3–5个具体异议关闭赠品，清楚命名每个，放上可辩护的价值。

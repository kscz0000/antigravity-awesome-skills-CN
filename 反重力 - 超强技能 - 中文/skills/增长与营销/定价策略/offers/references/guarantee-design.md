# 保障设计

保障直接提升*感知实现概率*（买家想："他们只会在有信心时才提供这个"）并降低*努力与牺牲*（更少的情感风险）。它是报价设计中杠杆最高的工具之一。

错误的保障比没有保障更伤。选择匹配你商业模式的类型。

## 八种保障类型

| 类型 | 承诺什么 | 何时适用 | 何时适得其反 |
|------|---------|---------|------------|
| 1. **无条件退款** | "X天内随时退款，不问原因" | 低价信息产品、高信任受众 | 高价/高接触；退款风险吞噬利润 |
| 2. **有条件退款** | "如果你完成X仍没看到Y则退款" | 课程、需要投入的项目 | 老练买家；公开兑现更难 |
| 3. **超值退款** | "如果不生效，全额退款+$X" | 有信心的交付、利润充裕 | 如果你失败；少数失败案例公开爆炸 |
| 4. **服务级别 / SLA** | "如果我们没在Y前交付X，退款" | 产品化服务、代理工作 | 模糊的SLA你无法衡量 |
| 5. **绩效制** | "只在X发生时付款"（分成、结果制） | 老练B2B、高信心交付 | 长周期、难以归因的结果 |
| 6. **反保障** | "不退款。确认你想要。" | 高端受众、成熟买家 | 困惑/首次买家；显得冷淡 |
| 7. **结果或延期** | "如果你在Y前没得到X，我们免费继续" | 教练、可延长时间的服务 | 开放式成本；谨慎选择 |
| 8. **比较保障** | "超过[竞争者]结果否则退款" | 当你能可信比较时 | 当你无法干净地衡量竞争者时 |

## 选择正确的保障

决策树：

1. **买家最大的感知风险是什么？**
   - "如果不生效怎么办？" → 退款家族(1, 2, 3)
   - "如果你不能按时交付怎么办？" → SLA(4)
   - "如果我付了钱但没结果怎么办？" → 绩效制(5)或结果或延期(7)
   - "这是真的还是骗局？" → 比较或基于具体性的(8)

2. **你的退款承受力如何？**
   - 能大规模吸收退款 → 无条件(1)
   - 需要筛选退款人 → 有条件(2)
   - 有信心加赠品 → 超值退款(3)
   - 完全承受不了退款 → 反保障(6)或无保障 + 强证据

3. **你的买家老练度如何？**
   - 高端/成熟买家 → 反保障可以工作；"我们不退款"读起来像信心
   - 首次入品类买家 → 强退款保障；他们需要被允许尝试
   - 老练B2B → SLA或绩效制；他们期望商业条款

4. **结果可衡量性如何？**
   - 干净可衡量 → 绩效制、比较、或结果或延期
   - 模糊/主观 → 退款家族加条件门槛（你完成了工作）

## 按商业类型的示例

### 课程 / 班期

**强：** "60天内完成所有六个模块，提交最终练习，如果你还没有[具体结果]我们全额退款。" 以努力为条件，结果清晰。

**弱：** "100%退款保障。" 无条件 = 从未参与的买家的退款磁铁。

### 教练 / 咨询

**强：** "前两次辅导后，如果你觉得项目不会交付，我们终止并退还未使用余额。" 项目中期的退出通道建立信任。

**弱：** "满意保障。" 毫无意义。

### 产品化服务 / 代理保留服务

**强：** "第一个月是付费试点。结束时，如果你没看到[具体里程碑]，不付第二个月，我们好聚好散。" 清晰的门槛，清晰的退出。

**弱：** "我们会工作到你满意为止。" 开放式成本。别做。

### 高价信息产品(社区、大师班)

**强（高端受众）：** "不退款。申请流程严格因为价值是真实的。如果你不确定，先别申请。" 反保障在此适用。

**弱（高端受众）：** 泛泛30天退款。显得廉价。

### 低价信息产品(模板、模板包)

**强：** "30天无理由退款。" 交易性标准是"买了、看了、不想要"。无条件适合。

**弱：** 没有保障。对于这个价格，买家风险太高。

### SaaS

**强：** 免费试用 *或* 年付-前30天可退款。减少摩擦而不锁定不满意的用户。

**弱：** 单独"随时取消"——不是保障，只是标准SaaS条款。

### 直复营销 / 付费流量

**强：** 双倍退款或可比的风险反转。直复营销买家期望风险逆转重的报价。

**弱：** 普通30天退款。不能与平台上其他广告区分。

## 撰写保障文本

保障文本很重要。有效的模式：

**具体条款：**
> 如果，在完成项目前4周后，你不能指出一个你实现的具体业务成果，发邮件给我们，我们100%退款。

**自信语气：**
> 我们知道这有效。如果对你无效，我们不要你的钱。

**承认尴尬：**
> 保障感觉油腻。不管怎样这是我们的：如果你做了模块1–3的工作且没看到有意义的发展，我们退款。

**无效的模式：**

- "100%满意保障！" — 泛泛，低信任
- "终身保障" — 没有条件毫无意义
- 多个叠加的保障 — 复杂度坍塌
- 充满法律术语的保障 — 买家略过并假设最坏

## 常见错误

### 承诺超出交付能力

"收入翻倍或退款+$1,000。" 即使50个买家中有1个失败并获得额外退款 + 写了公开评论，报价就永久受损。

压力测试：如果10%的买家触发保障会怎样？

### 有太多条件的条件保障

"如果你看了全部24个模块、完成了6个练习、参加了每次直播通话、且每周至少在社区发帖一次则退款。"

买家读成"他们让退款实际上不可能。"信任下降。

最多两个条件。三个仅当密切相关时（如"完成课程AND提交最终项目AND给我们发了一个问题"）。

### 把保障藏在细则里

如果你的保障是最强的感知实现概率杠杆，*用24pt字放在销售页上*。移到购买按钮上方。

### 忘记测试

每六个月重读保障文本。一年前有效的措辞现在可能被你对报价的某些更改所削弱。

### 把保障当作证据的替代

强证据 + 弱保障 > 强保障 + 弱证据。顺序很重要。先建证据，再叠加保障。

## 无保障的诚实理由

反保障("不退款，这是终局的")在以下情况适用：

- 买家老练度高
- 销售前有申请或资格审查流程
- 价格是高端到奢侈级别
- 品牌已建立
- 证据压倒性

你在说的是："我们不需要保障，因为工作是真实的，买家已经自我筛选，我们不会参与交易性退款游戏。"

错误的受众读成冷淡或骗人。正确的受众读成信心。了解你的买家。

## 诊断

审查没有保障（或保障薄弱）的报价时，问：

1. **买家的实际风险是什么？** 具体化。("$2K而且我可能得不到更多客户。")
2. **什么保障结构逆转那个特定风险？** 匹配到八种类型之一。
3. **你诚实的退款承受力如何？** 计算退款率 × 退款成本；你能承受吗？
4. **保障匹配你的受众老练度吗？** 高端买家要反保障；首次买家要无条件。

大多数报价不是保障错了——而是*根本没有*保障。添加任何保障几乎总是提升。添加正确的保障才是杠杆。

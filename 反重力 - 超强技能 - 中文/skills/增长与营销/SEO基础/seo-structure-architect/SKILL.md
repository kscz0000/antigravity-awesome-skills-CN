## 重点领域

- 标题标签层级（H1-H6）分析
- 内容组织和流程
- Schema 标记建议
- 内链机会
- 目录结构
- 内容深度评估
- 逻辑信息流

## 标题标签最佳实践

**SEO 指南：**
- 每页一个 H1，与主题匹配
- H2 用于主要章节，包含关键词变体
- H3 用于子章节，包含相关术语
- 保持逻辑层级
- 自然融入关键词

## 主题集群（Siloing）策略

1. 创建主题集群
2. 建立父子关系
3. 构建上下文内链
4. 在集群内保持相关性
5. 仅在高度相关时进行交叉链接

## Schema 标记优先级

**高影响力 Schema：**
- Article/BlogPosting
- FAQ Schema
- HowTo Schema
- Review/AggregateRating
- Organization/LocalBusiness
- BreadcrumbList

## 方法步骤

1. 分析提供的内容结构
2. 评估标题层级
3. 识别结构改进点
4. 建议内链机会
5. 推荐合适的 Schema 类型
6. 评估内容组织
7. 格式化以争取精选摘要

## 输出

**结构蓝图：**
```
H1: 主要关键词聚焦
├── H2: 主要章节（次要关键词）
│   ├── H3: 子章节（LSI 关键词）
│   └── H3: 子章节（实体词）
└── H2: 主要章节（相关关键词）
```

**交付物：**
- 标题层级大纲
- 主题集群/分组图谱可视化
- 内链矩阵
- Schema 标记 JSON-LD 代码
- 面包屑实现
- 目录结构
- 跳转链接建议

**技术实现：**
- WordPress：目录插件配置 + Schema 插件设置
- Astro/静态站点：组件层级 + 结构化数据
- URL 结构建议
- XML 站点地图优先级

**摘要优化：**
- 列表格式用于精选摘要
- 表格结构用于对比
- 定义框用于术语解释
- 分步指南用于流程说明

聚焦逻辑流程和可扫描内容。为用户和搜索引擎创建清晰的信息层级。

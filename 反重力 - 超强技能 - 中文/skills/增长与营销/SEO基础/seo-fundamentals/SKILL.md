# SEO 基础

> **可持续搜索可见性的基础原则。**
> 本技能解释的是_搜索引擎如何评估质量_，而非战术捷径。

---

## 1. E-E-A-T（质量评估框架）

E-E-A-T **不是直接排名因素**。
它是搜索引擎用来**评估内容质量**的框架，尤其适用于敏感或高影响力主题。

| 维度     | 含义             | 常见信号                       |
| -------- | ---------------- | ------------------------------ |
| **Experience（经验）** | 亲身、真实世界的参与 | 原创案例、亲身经历、演示       |
| **Expertise（专业）** | 学科专业能力     | 资质、深度、准确性             |
| **Authoritativeness（权威）** | 获得他人认可     | 提及、引用、链接               |
| **Trustworthiness（可信）** | 可靠性与安全性   | HTTPS、透明度、准确性          |

> 在同一领域竞争的页面，通常由**信任和经验**区分高下，而非关键词。

---

## 2. Core Web Vitals（页面体验信号）

Core Web Vitals 衡量的是**用户体验页面的方式**，而非页面是否值得排名。

| 指标    | 目标值  | 反映内容     |
| ------- | ------- | ------------ |
| **LCP** | < 2.5s  | 加载性能     |
| **INP** | < 200ms | 交互响应性   |
| **CLS** | < 0.1   | 视觉稳定性   |

**重要背景：**

- CWV 很少能弥补低质量内容
- 它们在内容质量相当时最为关键
- CWV 不达标会_拖累_原本优质的页面

---

## 3. 技术 SEO 原则

技术 SEO 确保页面**可访问、可理解、稳定**。

### 抓取与索引控制

| 元素              | 用途           |
| ----------------- | -------------- |
| XML sitemaps      | 帮助发现页面   |
| robots.txt        | 控制抓取访问   |
| Canonical 标签    | 合并重复内容   |
| HTTP 状态码       | 传达页面状态   |
| HTTPS             | 安全与信任     |

### 性能与可访问性

| 因素           | 为何重要         |
| -------------- | ---------------- |
| 页面速度       | 用户满意度       |
| 移动端友好设计 | 移动优先索引     |
| 简洁 URL       | 抓取清晰度       |
| 语义化 HTML    | 可访问性与可理解性 |

---

## 4. 内容 SEO 原则

### 页面级元素

| 元素            | 原则                   |
| --------------- | ---------------------- |
| Title 标签      | 明确主题 + 意图        |
| Meta description | 点击相关性，非排名因素 |
| H1              | 页面核心主题           |
| 标题层级        | 逻辑结构               |
| Alt 文本        | 可访问性与上下文       |

### 内容质量信号

| 维度   | 搜索引擎关注点   |
| ------ | ----------------- |
| 深度   | 充分回答查询       |
| 原创性 | 提供独特价值       |
| 准确性 | 事实正确           |
| 清晰度 | 易于理解           |
| 实用性 | 满足搜索意图       |

---

## 5. 结构化数据（Schema）

结构化数据帮助搜索引擎**理解语义**，而非直接提升排名。

| 类型            | 用途           |
| --------------- | -------------- |
| Article         | 内容分类       |
| Organization    | 实体身份       |
| Person          | 作者信息       |
| FAQPage         | 问答清晰度     |
| Product         | 商业详情       |
| Review          | 评分上下文     |
| BreadcrumbList  | 站点结构       |

> Schema 使页面具备获得富媒体结果的资格，但不保证一定展示。

---

## 6. AI 辅助内容原则

搜索引擎评估的是**输出质量**，而非创作方式。

### 有效用法

- AI 作为起草或研究助手
- 人工审核确保准确性和清晰度
- 原创见解与综合分析
- 明确的责任归属

### 风险用法

- 发布未经编辑的 AI 输出
- 事实错误或幻觉
- 薄弱或重复的内容
- 无价值的关键词堆砌文本

---

## 7. SEO 因素的相对重要性

**不存在固定的排名因素顺序**。
但当竞争页面相似时，重要性通常遵循以下模式：

| 相对权重 | 因素                   |
| -------- | ---------------------- |
| 最高     | 内容相关性与质量       |
| 高       | 权威与信任信号         |
| 中       | 页面体验（CWV、UX）    |
| 中       | 移动端优化             |
| 基线     | 技术可访问性           |

> 技术 SEO 使排名成为可能；内容质量赢得排名。

---

## 8. 衡量与评估

SEO 基础应通过**多维度信号**验证，而非单一指标。

| 领域     | 观察内容             |
| -------- | -------------------- |
| 可见性   | 已索引页面、展示次数 |
| 参与度   | 点击率、停留时间     |
| 性能     | CWV 真实用户数据     |
| 覆盖度   | 索引状态             |
| 权威性   | 提及与链接           |

---

> **核心原则：**
> 可持续的 SEO 建立在_有用的内容_、_技术清晰度_和_长期信任_之上。
> 不存在永久有效的捷径。

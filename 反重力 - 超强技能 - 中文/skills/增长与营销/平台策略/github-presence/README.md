# github-presence

**平台优化技能** — 针对可发现性与转化率优化 GitHub。

## 它能做什么

引导你完成 GitHub 优化：README 结构、badges、个人资料 README、topics、awesome 列表、用于营销的 GitHub Actions，以及 Sponsors 配置。

## 何时使用

- 改进项目的 README
- 配置个人资料 README
- 提升 GitHub 可发现性
- 当你提到"GitHub README"、"GitHub 个人资料"、"awesome 列表"或"GitHub 营销"

## 快速开始

```
/github-presence

Audit our GitHub README and suggest improvements.
We're an open source database tool trying to get
more stars and contributors.
```

## 关键章节

1. **README 结构** — 优秀 README 的解剖
2. **Badges** — 哪些重要以及如何添加
3. **个人资料 README** — 在 GitHub 上的个人品牌
4. **可发现性** — Topics、搜索、awesome 列表
5. **GitHub Actions** — 用于营销的自动化
6. **GitHub Sponsors** — 货币化配置

## README 必备项

| 章节 | 用途 |
|---------|---------|
| Logo/Banner | 视觉识别 |
| Badges | 信任信号 |
| One-liner | 它是干什么的 |
| Quick Start | 2 分钟内跑起来 |
| Features | 为什么要用它 |
| Installation | 所有安装方式 |
| Documentation | 指向完整文档的链接 |
| License | 法律说明 |

## 可发现性策略

| 策略 | 方法 |
|--------|-----|
| Topics | 添加最多 20 个相关 topics |
| Description | 350 字符，富含关键词 |
| Awesome 列表 | 向策展列表提交 PR |
| 社交预览 | 上传自定义图片 |

## 工具

- **[Octolens](https://octolens.com)** — 监控 GitHub 上的提及与讨论
- **Shields.io** — 生成 badges
- **readme.so** — README 生成器

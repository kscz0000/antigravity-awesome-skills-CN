# developer-audience-context

**基础技能** —— 其他所有技能都会首先引用它。

## 功能说明

创建并维护 `.agents/developer-audience-context.md` —— 一份捕捉目标开发者所有信息的文档。只需定义一次受众，其他所有技能都会自动拥有上下文。

## 使用场景

- 启动任何开发者营销项目时
- 在运行其他技能之前（自动被引用）
- 当你提到"开发者画像""目标开发者""ICP"或"我们的开发者是谁"时

## 快速上手

```
/developer-audience-context

Create our developer audience context. We're building an open source
API testing library for Node.js/TypeScript developers.
```

该技能会：
1. 从你的代码库自动起草（README、文档、落地页）
2. 通过对话逐步完成 10 个章节

## 文件

| 文件 | 描述 |
|------|-------------|
| `SKILL.md` | 完整的技能说明 |
| `references/template.md` | 待填写的空白模板 |
| `references/example-apitest.md` | 一个虚构开发工具的完整填写示例 |

## 10 个章节概览

1. **产品概览** —— 名称、一句话介绍、类别、技术、定价
2. **开发者画像** —— 角色、资历、公司规模、技术栈
3. **他们常出没的地方** —— Reddit、Discord、Twitter、播客、开发者大会
4. **问题与痛点** —— 功能性、情感性、触发时刻
5. **当前的替代方案** —— 竞品、自建、不作为
6. **关键差异化** —— 技术、DX、生态系统、价值观
7. **开发者的原话** —— 原话引用，而非润色文案
8. **技术可信度信号** —— GitHub stars、下载量、知名用户
9. **转化动作** —— 认知 → 激活 → 转化
10. **声音与语调** —— 与开发者对话时的调性

## 工具

- **[Octolens](https://octolens.com)** —— 跨 GitHub、Hacker News、Reddit、Stack Overflow、Twitter 监控开发者对话，捕捉原话并找出开发者常出没之处。

# 开发者 Newsletter

**打造开发者真正愿意打开的 newsletter。**

## 它能做什么

引导你创建并运营一个开发者 newsletter。涵盖内容策略、写作、标题、增长策略以及避开垃圾邮件过滤器的方法。

## 适用场景

- 启动一个开发者 newsletter
- 撰写下一期内容
- 提升打开率/点击率
- 增长你的订阅用户
- 当你说"newsletter"、"邮件营销"、"dev newsletter"、"邮件列表"时

## 快速上手

```
/developer-newsletter

我想开一个关于 JavaScript 性能的周刊 newsletter。
目标读者：中高级前端开发者。
```

技能会做这些事：
1. 加载你的开发者受众上下文
2. 帮你确定 newsletter 类型与频率
3. 指导内容组合与结构
4. 优化标题与送达率

## 关键框架

### 内容组合（70-20-10）

| 比例 | 内容类型 |
|------|----------|
| 70% | 价值内容（教学、传递信息、帮忙解决问题） |
| 20% | 产品内容（更新、功能介绍） |
| 10% | 推广内容（CTA、诉求） |

### 标题模板

有效模式：
- 具体收益："把构建时间砍掉 40%"
- 技术好奇心："那个没人用的 JavaScript 特性"
- 直接公告："v2.0 已发布：async/await 支持"
- 数字 + 主题："资深开发者常用的 7 个 TypeScript 技巧"

### 基准指标

| 指标 | 目标 |
|------|------|
| 打开率 | 30-50% |
| 点击率 | 5-15% |
| 退订率 | <0.5% |

## 包含的指导

- 频率选择

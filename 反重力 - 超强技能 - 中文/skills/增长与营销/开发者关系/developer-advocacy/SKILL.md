# 开发者布道
## 适用场景

当用户希望开展开发者布道活动时使用本技能，包括会议演讲、现场编程、播客、公开构建等。触发词包括"开发者布道"、"devrel"、"会议演讲"、"CFP"、"call for papers"、"现场编程"、"播客"、"公开构建"等。

本技能协助你进行开发者布道活动：会议演讲、现场编程演示、播客出镜、公开构建。涵盖演讲提案、演示准备、社交媒体运营和影响力衡量。


---

## 在开始之前

**先加载你的受众画像。** 阅读 `.agents/developer-audience-context.md` 以了解：

- 你想触达的人群（参加的会议、收听的播客）
- 哪些话题能引起共鸣（痛点、兴趣）
- 你产品的定位（要讲什么故事）
- 语气与风格（要多正式、多技术）

如果该上下文文件不存在，先运行 `developer-audience-context` 技能。

## 会议演讲

### 找到合适的会议

| 会议类型 | 最适合 | 示例 |
|-----------------|----------|----------|
| **大型行业会议** | 品牌曝光、大范围触达 | KubeCon、AWS re:Invent、React Summit |
| **区域性会议** | 本地社区、容易参与 | 本地 meetup、城市技术会议 |
| **细分领域会议** | 精准受众、专家定位 | GraphQL Conf、RustConf |
| **公司主办会议** | 生态影响力 | Vercel Ship、GitHub Universe |
| **非正式会议** | 社区连接 | BarCamps、DevOpsDays |

### 演讲提案（CFP）框架

**制胜公式：**
```
具体问题 + 独到视角 + 清晰收获 = 通过的演讲
```

**CFP 模板：**

```markdown
# 标题
[动作动词] + [具体成果] + [借助/使用什么工具]
示例："借助边缘函数和 WebSockets 构建实时功能"

# 摘要（100-200 字）
[钩子：问题或好奇心缺口]
[你将讲到的内容]
[参会者将学到什么 / 能够做什么]

# 详细描述（供评审阅读）
[问题背景]
[为何采用这种方案]
[演讲结构]
[你的演讲资历]

# 大纲
- [时间] 开场 / 问题陈述
- [时间] 第一部分
- [时间] 第二部分
- [时间] 第三部分
- [时间] 现场演示 / 走查
- [时间] 关键收获 / 问答

# 受众
[演讲面向谁]
[前置知识要求]
[他们将学到什么]

# 个人简介
[你相关的经历]
[你为何有资格讲这个话题]
```

### 验证有效的标题套路

| 套路 | 示例 |
|---------|---------|
| **我是如何 X 的** | "我是如何把部署时间缩减 80% 的" |
| **X 分钟搞定 Y** | "15 分钟搞懂 Kubernetes 安全" |
| **关于 Y 的 X** | "错误信息的心理学" |
| **超越 X** | "超越 Console.log：现代调试技法" |
| **面向 Y 的 X** | "面向 REST 工程师的 GraphQL" |
| **来自 X 的教训** | "1000 次生产故障带来的教训" |

### 演讲类型

| 类型 | 时长 | 最适合 |
|------|--------|----------|
| **闪电演讲** | 5-10 分钟 | 单个概念、快速演示 |
| **常规演讲** | 25-45 分钟 | 技术深度讲解 |
| **主题演讲** | 45-60 分钟 | 全局视角、激发灵感 |
| **工作坊** | 2-4 小时 | 动手实践 |
| **圆桌** | 30-60 分钟 | 多视角讨论 |

### 演讲准备清单

| 阶段 | 任务 |
|-------|-------|
| **提前 2 个月** | 列大纲、起幻灯片、测演示 |
| **提前 1 个月** | 初稿完成、首次完整彩排 |
| **提前 2 周** | 幻灯片精修、演示稳定、彩排 3 次 |
| **提前 1 周** | 录像自检、收集反馈、定稿 |
| **前一天** | 全部技术环节走查、备用幻灯片、充分休息 |
| **当天** | 提前到场、测试音视频、多喝水 |

## 现场编程与演示

### 演示的危险区

| 风险 | 缓解措施 |
|------|------------|
| **断网** | 提前录制备份、本地服务器 |
| **拼写卡壳** | 把同一段代码练 20 遍 |
| **遇到修不了的错** | 准备可以跳到的可用 checkpoint |
| **超时** | 严格计时、狠心删内容 |
| **字太小看不清** | 放大、用大字体（24pt 及以上） |
| **深色主题刺眼** | 高对比、对浅色环境友好的主题 |

### 演示准备框架

**10-3-1 法则：**
- 私下把演示**练 10 遍**
- 准备**3 个 checkpoint**，卡壳时可跳
- **1 个备用方案**（运行正常的录像）

**演示前清单：**
- [ ] 关闭无关应用
- [ ] 清空浏览器历史/标签页
- [ ] 关闭通知（Slack、邮件、日历）
- [ ] 字体大小：终端 24pt+、编辑器 20pt+
- [ ] 用 git stash/branch 准备干净的起点
- [ ] 环境变量就绪
- [ ] 尽可能用现场投影/屏幕实测

### 现场编程技巧

| 技巧 | 理由 |
|-----|-----|
| **慢慢敲** | 观众需要跟上 |
| **边敲边口述** | "我在新建一个 handler……" |
| **解释报错** | "这个错说明 X，我修一下" |
| **使用代码片段** | 拿来写模板代码，不是核心概念 |
| **展示运行结果** | 始终运行代码、显示输出 |
| **checkpoint 提交** | `git checkout checkpoint-1` |

## 播客嘉宾

### 如何找播客

| 方法 | 操作 |
|----------|-----|
| **直接搜索** | "最佳 [你的技术] 播客" |
| **嘉宾撮合平台** | Podmatch、Matchmaker.fm |
| **问同行** | "你们都听哪些播客？" |
| **推特搜索** | "[话题] 播客 单集" |
| **Listen Notes** | 播客搜索引擎 |

### 自荐邮件模板

```
Subject: Guest Idea: [Specific Topic] for [Podcast Name]

Hi [Host Name],

I've been listening to [Podcast] for [time] — loved your episode on [specific episode].

I'd love to come on and talk about [specific topic]. Here's the angle:

[2-3 sentences on what you'd discuss and why it matters to their audience]

A bit about me:
- [Relevant credential 1]
- [Relevant credential 2]
- [Link to past podcast/talk]

Would this be a fit?

[Your name]
```

### 录制前准备

| 准备项 | 详情 |
|-----------|---------|
| **研究节目** | 听 2-3 期、了解节目形式 |
| **研究主持人** | 他的兴趣、风格、推特 |
| **准备要点** | 你最想讲的 3-5 个点 |
| **准备故事** | 具体事例、不要泛泛而谈 |
| **音频设置** | 好麦克风、安静房间、戴耳机 |
| **手边备水** | 你要讲很多话 |

### 节目录制中

| 要做 | 不要做 |
|----|-------|
| 用具体细节讲故事 | 给泛泛的建议 |
| 回答前先停顿 | 紧张地"嗯""啊" |
| 礼貌地表达异议 | 为照顾面子一味附和 |
| 巧妙植入宣传 | 硬推自己的产品 |
| 言简意赅 | 东拉西扯没结构 |
| 表现热情 | 语气单调 |

### 录制之后

| 动作 | 时机 |
|--------|--------|
| 感谢主持人 | 当天 |
| 发布后立即转发 | 节目上线即转 |
| 互动评论区 | 上线第一周 |
| 互相导流 | 你的 newsletter、博客 |
| 保持联系 | 建立长期关系 |

## 公开构建

### 该分享什么

| 类别 | 内容方向 |
|----------|---------------|
| **进展** | "今天上线了 X，踩了这些坑" |
| **挑战** | "卡在 X，试了 Y 和 Z，最后是这样搞定的" |
| **决策** | "我们为什么选 X 而不是 Y" |
| **数据** | 收入、用户、增长（保持透明） |
| **幕后** | 团队、流程、工具 |
| **复盘** | "我们犯的错和修复方法" |

### 公开构建的形式

| 形式 | 平台 | 频率 |
|--------|----------|---------|
| **推特长帖** | Twitter/X | 每天或每周 |
| **更新日志** | 博客、Notion、官网 | 每周 |
| **独立黑客帖** | Indie Hackers、HN | 每月 |
| **视频更新** | YouTube、Loom | 每周到每月 |
| **邮件通讯** | 邮件 | 每周 |
| **直播** | Twitch、YouTube | 每周 |

### 不要分享的内容

| 避免 | 原因 |
|-------|-----|
| **客户数据** | 隐私与信任 |
| **团队冲突** | 职业素养 |
| **安全细节** | 漏洞风险 |
| **攻击竞品** | 显得小气 |
| **宣泄情绪** | 没有产出 |

## 社交媒体（Twitter/X）

### 开发者推特运营手册

| 内容类型 | 占比 | 示例 |
|--------------|------------|---------|
| **干货内容** | 60% | 技巧、教程、洞察 |
| **互动** | 20% | 回复、带观点转发 |
| **个人** | 10% | 幕后花絮、人格展示 |
| **推广** | 10% | 你的产品、演讲、内容 |

### 好用的推文格式

| 格式 | 示例 |
|--------|---------|
| **长帖** | "做 X 总结的 10 条心得" |
| **锐评** | "没人敢说的话：[观点]" |
| **小技巧** | "今天才知道：你可以这样 X……" |
| **提问** | "你最爱的 X 方式是……？" |
| **段子/幽默** | 技术笑话、有共鸣的内容 |
| **作品展示** | "刚上线 X，原理是这样的" |
| **致谢** | "致敬 @xxx，因为他……" |

### 互动策略

| 动作 | 频率 |
|--------|-----------|
| 发原创推文 | 每天 |
| 回复他人 | 每天 5-10 次 |
| 转发并加价值评论 | 每周 2-3 次 |
| DM 有意思的人 | 每周 |
| 参加 Twitter Spaces | 按需 |

### 涨粉经验

| 策略 | 具体做法 |
|--------|----------------|
| **保持稳定** | 每天发、每天互动 |
| **先窄后宽** | 先让一件事来代表你 |
| **回复战** | 给大 V 的推文加有价值的回复 |
| **合作** | Twitter Spaces、合作长帖 |
| **互相导流** | newsletter、演讲、博客 |

## 衡量影响力

### 布道指标

| 活动 | 指标 |
|----------|---------|
| **演讲** | 参会人数、反馈评分、回放播放量 |
| **内容** | 阅读量、转发量、互动 |
| **社交** | 粉丝、互动率、触达 |
| **播客** | 听众估算、流量峰值 |
| **社区** | 增长、互动、情感倾向 |

### 归因难题

开发者布道的影响力一直以难以量化著称。可用的代理指标：

| 信号 | 含义 |
|--------|-------------------|
| **流量突增** | 内容/演讲/播客带来访问 |
| **"你怎么知道我们的？"** | 直接归因 |
| **社交提及** | 品牌认知度 |
| **入站线索质量** | 来自社区的高质量线索 |
| **会议邀约** | 名气在涨 |

### 月报模板

月度布道报告：

```markdown
# 开发者布道月报 - [月份]

## 演讲与出镜
- [演讲 1]：[会议]、[参会人数]、[链接]
- [播客 1]：[节目]、[单集链接]

## 已发布内容
- [文章 1]：[阅读量]、[互动量]
- [视频 1]：[播放量]

## 社交增长
- Twitter：+X 粉丝，Y 曝光
- 关键推文：[链接]

## 社区
- Discord/Slack：+X 成员，Y 条消息
- 重要话题/讨论

## 复盘
- 跑通了：[X]
- 没跑通：[Y]
- 下月要试：[Z]
```

## 布道职业路径

### 角色级别

| 级别 | 重心 |
|-------|-------|
| **初级 DA** | 内容创作、社区支持、演讲准备 |
| **开发者布道师** | 演讲、独立内容策略、社区搭建 |
| **高级 DA** | 战略、辅导、大型会议 |
| **资深 DA** | 跨公司影响、行业思想领袖 |
| **DevRel 负责人** | 团队建设、战略、与高管对齐 |

### 技能发展

| 技能 | 如何练 |
|-------|----------------|
| **公开演讲** | Meetup、Toastmasters、刻意练习 |
| **写作** | 持续写博客、收集反馈 |
| **视频** | YouTube、直播、迭代改进 |
| **技术深度** | 做项目、参与开源 |
| **社区** | 主持、组织活动、链接人与资源 |

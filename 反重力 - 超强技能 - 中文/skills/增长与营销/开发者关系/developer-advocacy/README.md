# developer-advocacy

**会议演讲、播客和公开构建。**

## 它能做什么

引导你完成开发者布道活动：会议演讲、现场编程演示、播客出镜和公开构建。涵盖演讲提案、演示准备、社交媒体运营以及影响力衡量。

## 适用场景

- 撰写会议演讲提案（CFP）
- 为现场演示做准备
- 向播客自荐
- 建立社交媒体影响力
- 当你提到"会议演讲"、"CFP"、"现场编程"、"播客"或"公开构建"时

## 快速开始

```
/developer-advocacy

我想投一个关于限流模式（rate limiting patterns）的演讲。
目标会议：偏后端方向、受众为中级水平。
```

技能会：
1. 加载你的开发者受众画像
2. 帮你打磨 CFP 提案
3. 搭建演讲大纲
4. 准备演示和备用方案
5. 制定推广策略

## 关键框架

### CFP 公式

```
具体问题 + 独到视角 + 清晰收获 = 通过的演讲
```

### 演讲标题套路

| 套路 | 示例 |
|---------|---------|
| 我是如何 X 的 | "我是如何把部署时间缩减 80% 的" |
| X 分钟搞定 Y | "15 分钟搞懂 Kubernetes 安全" |
| 来自 X 的教训 | "1000 次生产故障带来的教训" |

### 10-3-1 演示法则

- 私下把演示**练 10 遍**
- 准备**3 个 checkpoint**，卡壳时可跳
- **1 个备用方案**（录像）

### 社交内容配比

| 类型 | 占比 |
|------|------------|
| 干货内容 | 60% |
| 互动 | 20% |
| 个人 | 10% |
| 推广 | 10% |

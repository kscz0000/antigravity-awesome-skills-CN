# developer-churn

**为什么开发者会离开，以及如何赢回他们** —— 理解、预防与挽回流失。

## 功能说明

帮助你理解开发者流失的真实原因，构建早期预警系统来识别高风险用户，开展有效的流失访谈，并发起尊重开发者智识的赢回活动。

## 使用时机

- 流失率高于你期望的水平
- 需要弄清开发者为什么离开
- 正在构建高风险用户检测系统
- 正在创建赢回活动
- 当你提到"为什么开发者会离开"、"流失率"、"高风险用户"或"赢回"

## 快速开始

```
/developer-churn

Help me understand why developers are churning from our API product.
We've had 15% churn last month and I need to figure out the causes.
```

## 开发者流失的 6 大原因

| 原因 | 是否可行动 | 你的行动 |
|--------|-------------|-----------|
| DX 问题 | 是 | 改进文档、SDK、quickstart |
| 价格摩擦 | 是 | 调整定价、增加免费版 |
| 更优替代品 | 部分 | 以价值竞争，而非制造焦虑 |
| 项目终止 | 否 | 接受现实，及时放手 |
| 集成失败 | 是 | 增加集成，改进指南 |
| 非主动（支付） | 是 | 优化催款、卡信息更新 |

## 核心原则

1. **开发者离开不是因为价格 —— 而是因为摩擦**
2. **工单是预警信号，不是互动**
3. **不要试图赢回项目终止型流失者**
4. **赢回时用价值打头阵，而非折扣**
5. **优先修复非主动流失 —— 通常占总量的 20–40%**

## 高风险检测

按以下维度构建健康度评分：
- API 调用趋势（30%）
- 登录频次（20%）
- 功能采用（20%）
- 工单情感倾向（15%）
- 计费健康度（15%）

# GEO 基础

> 针对 AI 驱动的搜索引擎优化。

---

## 1. 什么是 GEO？

**GEO** = Generative Engine Optimization（生成式引擎优化）

| 目标 | 平台 |
|------|------|
| 在 AI 回答中被引用 | ChatGPT、Claude、Perplexity、Gemini |

### SEO vs GEO

| 维度 | SEO | GEO |
|------|-----|-----|
| 目标 | 排名第一 | AI 引用 |
| 平台 | Google | AI 引擎 |
| 指标 | 排名、CTR | 引用率 |
| 重点 | 关键词 | 实体、数据 |

---

## 2. AI 引擎格局

| 引擎 | 引用风格 | 机会 |
|------|----------|------|
| **Perplexity** | 编号 [1][2] | 引用率最高 |
| **ChatGPT** | 行内/脚注 | Custom GPTs |
| **Claude** | 上下文相关 | 长篇内容 |
| **Gemini** | 来源区块 | SEO 跨界 |

---

## 3. RAG 检索因素

AI 引擎如何选择要引用的内容：

| 因素 | 权重 |
|------|------|
| 语义相关性 | ~40% |
| 关键词匹配 | ~20% |
| 权威信号 | ~15% |
| 新鲜度 | ~10% |
| 来源多样性 | ~15% |

---

## 4. 容易被引用的内容

| 元素 | 为何有效 |
|------|----------|
| **原创统计数据** | 独特、可引用的数据 |
| **专家引言** | 权威背书 |
| **清晰定义** | 易于提取 |
| **分步指南** | 可操作价值 |
| **对比表格** | 结构化信息 |
| **FAQ 区块** | 直接答案 |

---

## 5. GEO 内容检查清单

### 内容元素

- [ ] 疑问式标题
- [ ] 顶部摘要/TL;DR
- [ ] 带来源的原创数据
- [ ] 专家引言（姓名、头衔）
- [ ] FAQ 区块（3-5 个问答）
- [ ] 清晰定义
- [ ] "最后更新"时间戳
- [ ] 带资质的作者

### 技术元素

- [ ] 带日期的 Article schema
- [ ] 作者的 Person schema
- [ ] FAQPage schema
- [ ] 快速加载（< 2.5秒）
- [ ] 干净的 HTML 结构

---

## 6. 实体建设

| 行动 | 目的 |
|------|------|
| Google Knowledge Panel | 实体识别 |
| Wikipedia（如具备知名度） | 权威来源 |
| 网络信息一致性 | 实体整合 |
| 行业提及 | 权威信号 |

---

## 7. AI 爬虫访问

### 主要 AI User-Agent

| 爬虫 | 引擎 |
|------|------|
| GPTBot | ChatGPT/OpenAI |
| Claude-Web | Claude |
| PerplexityBot | Perplexity |
| Googlebot | Gemini（共享） |

### 访问策略

| 策略 | 适用场景 |
|------|----------|
| 允许全部 | 需要 AI 引用 |
| 屏蔽 GPTBot | 不想被 OpenAI 训练 |
| 选择性 | 允许部分、屏蔽其他 |

---

## 8. 效果衡量

| 指标 | 追踪方法 |
|------|----------|
| AI 引用 | 人工监控 |
| "据 [品牌] 称"提及 | 在 AI 中搜索 |
| 竞品引用 | 对比份额 |
| AI 引荐流量 | UTM 参数 |

---

## 9. 反模式

| ❌ 不要 | ✅ 要 |
|---------|-------|
| 发布无日期内容 | 添加时间戳 |
| 模糊归因 | 注明来源 |
| 省略作者信息 | 展示资质 |
| 薄弱内容 | 全面覆盖 |

---

> **记住：** AI 会引用清晰、权威、易于提取的内容。成为最佳答案。

# 竞品追踪

面向开发者工具的系统化竞品分析。追踪功能、定价、定位、内容策略以及开发者情感，覆盖你的竞争格局。

## 本技能涵盖的内容

- 竞品识别（直接、间接、自建替代方案、平台型）
- 追踪维度：功能、定价、定位、内容、社区信号
- 通过 GitHub/npm/PyPI 监控增长信号
- 开发者情感监控
- 构建与维护竞争 Battlecard
- 何时以及如何应对竞品动作

## 推荐工具

- **社交聆听工具** — 跨开发者平台监控竞品情感
- **GitHub/npm 监控** — 追踪竞品增长信号

## 快速上手

1. 描绘竞争格局（直接、间接、自建、平台型）
2. 为每个竞品设置监控关键词
3. 为前 3-5 名竞品创建初始 Battlecard
4. 确定复盘节奏（主要竞品月度复盘，次要竞品季度复盘）
5. 针对常见场景构建应对预案

## Battlecard 模板

1. 竞品概览
2. 我们赢的场景（场景、客户类型、佐证）
3. 我们输的场景（风险点、应对策略）
4. 常见异议与回应
5. 差异化优势
6. 预设陷阱

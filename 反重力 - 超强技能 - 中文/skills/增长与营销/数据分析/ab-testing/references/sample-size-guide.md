## 样本量基础

### 必备输入

1. **基线转化率**：当前的转化率
2. **最小可检测效应（MDE）**：值得检测的最小变化
3. **统计显著性水平**：通常 95%（α = 0.05）
4. **统计功效**：通常 80%（β = 0.20）

### 各参数含义

**基线转化率**：页面当前转化 5%，这就是你的基线。

**MDE（最小可检测效应）**：你关心检测的最小提升幅度。依据：
- 业务影响（5% 的提升有意义吗？）
- 实施成本（值得做吗？）
- 现实预期（过去测试结果如何？）

**统计显著性（95%）**：意味着观察到的差异由随机因素导致的概率不到 5%。

**统计功效（80%）**：意味着若真实效应大小为 MDE，你有 80% 的概率能检测到。

## 周期计算器

### 公式

```
周期（天）=（每变体样本量 × 变体数）/（日流量 × 暴露比例）
```

### 示例

**场景 1：高流量页面**
- 需求：每个变体 10,000（2 个变体共 20,000）
- 日流量：5,000 访问者
- 100% 进入测试
- 周期：20,000 / 5,000 = **4 天**

**场景 2：中流量页面**
- 需求：每个变体 30,000（共 60,000）
- 日流量：2,000 访问者
- 100% 进入
- 周期：60,000 / 2,000 = **30 天**

**场景 3：低流量 + 部分曝光**
- 需求：每个变体 15,000（共 30,000）
- 日流量：500 访问者
- 50% 进入测试
- 有效日入：250
- 周期：30,000 / 250 = **120 天**（太久了！）

### 最短周期规则

即便样本量足够，测试至少要跑：
- **1 个完整周**：覆盖工作日/周末差异
- **2 个业务周期**：B2B 场景（工作日 vs 周末模式）
- **覆盖发薪日**：电商场景（月初/月末）

### 最长周期建议

测试周期尽量不要超过 4-8 周：
- 新奇效应会消退
- 外部因素会介入
- 跑其他测试的隐性机会成本

## 多变体调整

超过 2 个变体（A/B/n 测试）需要更多样本：

| 变体数 | 乘数 |
|----------|------------|
| 2（A/B） | 1x |
| 3（A/B/C） | ~1.5x |
| 4（A/B/C/D） | ~2x |
| 5+ | 考虑减少变体数 |

**为什么？** 比较越多，假阳性概率越高。你会比较：
- A vs B
- A vs C
- B vs C（有时）

应用 Bonferroni 校正，或使用能自动处理的工具。

## 常见样本量错误

### 1. 功效不足
**问题**：样本不足以检测现实效果的差异
**对策**：对 MDE 务实一些，加大流量，否则别测

### 2. 功效过盛
**问题**：已经显著了还在等样本量
**对策**：其实没问题——已承诺的样本量照跑就行

### 3. 基线取值错误
**问题**：用了错误的转化率来计算
**对策**：用具体指标和页面的数据，而不是全站均值

### 4. 忽视分群
**问题**：按全流量计算，却做分群分析
**对策**：若计划分群分析，按最小分群计算样本

### 5. 同时测太多
**问题**：流量被切得过细
**对策**：狠心做减法，并行测试少一些

## 样本量要求过高时怎么办

流量不足时的应对方案：

1. **提高 MDE**：只检测更大的效应（20% 以上）
2. **降低置信度**：用 90% 替代 95%（有风险，要书面记录）
3. **减少变体**：只保留最有希望的变体
4. **合并流量**：跨多个相似页面一起测
5. **上推测试位置**：在漏斗更靠前的流量大处测
6. **不测**：改用定性数据做决策
7. **延长测试周期**：接受更长的周期（数周到数月）

## 序贯检验

如果一定要在达到样本量前看结果：

### 是什么？
一种对多次查看数据进行校正的统计方法。

### 何时使用
- 高风险改动
- 需要提前止损劣质变体
- 时间敏感的决策

### 支持的工具
- Optimizely（Stats Accelerator）
- VWO（SmartStats）
- PostHog（贝叶斯方法）

### 权衡
- 提前止损更灵活
- 样本量需求略增
- 分析更复杂

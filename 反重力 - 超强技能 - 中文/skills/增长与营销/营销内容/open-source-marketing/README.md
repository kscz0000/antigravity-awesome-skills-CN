# open-source-marketing

**推广开源项目而不让人反感。**

## 功能介绍

指导你以真诚方式推广开源项目。涵盖 GitHub 优化、社区建设、贡献者体验、发布策略和可持续增长。

## 何时使用

- 发布开源项目
- 增长现有 OSS 项目
- 改善贡献者体验
- 当你说"开源营销"、"推广我的库"、"增长Star"或"开源发布"时

## 快速上手

```
/open-source-marketing

I'm launching an open source React component library.
Target: React developers who need accessible UI components.
Goal: Get first 100 stars and 5 contributors.
```

该技能将会：
1. 加载你的开发者受众上下文
2. 优化你的 GitHub 展示
3. 规划发布策略
4. 建设贡献者体验
5. 创建可持续增长策略

## 核心框架

### 增长公式

```
Growth = (Real value) x (Discoverability) x (First-use experience)
```

### README 结构

1. 一句话描述
2. 徽章（构建、版本、许可证）
3. 视觉展示（GIF > 截图 > 无）
4. 为什么选这个项目？（3 个优势）
5. 快速上手（<5 行）
6. 安装、使用、文档、贡献

### 发布日时间线

| 时间 | 行动 |
|------|------|
| 上午 | Hacker News（太平洋时间周二至周四 6-8am） |
| +1 小时 | Twitter 主题帖 |
| +2 小时 | Reddit 发到相关版块 |
| 全天 | 回复所有内容 |

### 贡献者漏斗

```
User → Star → Issue → PR → Regular Contributor → Maintainer
```

## 什么有效 vs. 无效

| 有效 | 无效 |
|------|------|
| 公开构建过程 | 到处刷"看看我的项目" |
| 优秀的文档和开发者体验 | "代码本身就是文档" |
| 庆祝贡献者 | 独揽功劳 |
| 持续活跃 | 发布后消失 |

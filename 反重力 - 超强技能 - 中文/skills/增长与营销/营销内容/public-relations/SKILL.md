# 公关与赢得媒体

## 何时使用

当用户需要公关、赢得媒体、新闻报道、记者联络或媒体策略方面的帮助时使用（非 pull requests）。当用户提到"PR"、"公共关系"、"媒体"、"新闻稿"、"新闻报道"、"媒体联络"、"向记者推介"、"获得报道"、"媒体策略"时也使用。

你是软件产品赢得媒体领域的专家。你的目标是帮助用户获得记者、播客和新闻简报的报道——高效、尊重对方。

## 开始之前

**先检查产品营销上下文：**
如果 `.agents/product-marketing.md` 存在（或 `.claude/product-marketing.md`，或旧版 `product-marketing-context.md` 文件名），在提问之前先读取。使用该上下文，仅询问未涵盖的信息或与本次任务相关的细节。

## 核心理念

公关不是分销的替代品，而是分销的放大器。

- **赢得媒体不会直接驱动转化。** TechCrunch 的一篇报道不会给你带来 1000 个付费客户。它会给你反向链接、品牌信誉、AI 引用覆盖面，以及销售对话的弹药。
- **像推介客户一样推介记者：** 具体、有用、快速，永远不要只谈自己。
- **故事不是你的产品。故事是趋势、数据、冲突或人物。** 你的产品只是证据。
- **反应式公关中速度胜过打磨。** 故事发生后第一小时内的 B+ 推介，胜过第三天的 A+ 推介。

### 公关值得做的时机

- 你有**真正的故事**——独家数据、鲜明观点、里程碑、有着鲜明前后对比的客户，或对热门话题的新鲜角度
- 你有**创始人/高管时间**——记者想要有利害关系的人的引用，不是 PR 代表
- 你有**落脚点**——新闻页面、博客文章或产品发布，能把注意力转化为有用的东西

### 该跳过公关的时机

- 上市前除了"我们存在"没有其他故事
- 团队中没有人能持续推介 4-6 周（公关是动量游戏）
- 你没有清晰的 ICP——记者会问"谁会因为这个读我的文章？"，如果你回答不了，他们也不行

## 公关组合

四种模式。大多数团队在一种上过度投入。至少运行三种。

| 模式 | 是什么 | 投入 | 获得报道的速度 |
|------|--------|------|----------------|
| **反应式（新闻搭车）** | 将你的观点注入热门新闻 | 低–中 | 数小时到数天 |
| **主动式（推介）** | 建立媒体名单，推介原创故事 | 高 | 2–8 周 |
| **入站式（记者请求）** | 在 HARO/Qwoted/Featured 上回复记者查询 | 低 | 数天到数周 |
| **自有式（新闻页 + 媒体套件）** | 让记者容易找到你 | 一次性设置 | 不适用 |

**反应式新闻搭车工作流** —— 参见 [references/newsjacking.md](references/newsjacking.md)

**主动式记者推介** —— 参见 [references/journalist-pitching.md](references/journalist-pitching.md)

**入站式记者请求平台（HARO、Qwoted 等）** —— 参见 [references/press-platforms.md](references/press-platforms.md)

**推介去向（媒体机构、播客、新闻简报）** —— 参见 [references/media-outlets.md](references/media-outlets.md)。对于创业公司/SaaS/AI 目录，使用独立的 `directory-submissions` 技能——不同的意图，不同的列表。

## 自有式：新闻页 + 媒体套件

一次性设置。这是最便宜的公关投资，对每篇后续故事的 ROI 最高。

**新闻页（`/press` 或 `/newsroom`）应包含：**
- 一段公司描述（可直接复制粘贴）
- 创始人简介和头像（高分辨率，可下载）
- Logo 包（SVG + PNG，浅色 + 深色，附使用指南）
- 产品截图（高分辨率）
- 近期报道列表（为下一位记者提供社会证明）
- 成立日期、员工数、融资情况（如公开披露）
- 媒体联系邮箱（不是表单——记者讨厌表单）
- 近期新闻稿/公告

**顶部一句话：** "如需采访请求或素材，请发邮件至 press@yourcompany.com——我们 24 小时内回复。"

然后*真的*在 24 小时内回复。

## 快速参考：推介质量标准

发送任何推介前，以下所有问题的答案都应为是：

- [ ] 这位记者是否报道这个领域？（查看他们最近 5 篇文章。）
- [ ] 是否有明确的新闻由头——刚发生或即将发生的事？
- [ ] 这位记者能否仅凭这封邮件写出一篇完整报道？（数据、引用、客户名、联系方式。）
- [ ] 主题行是否足够具体，能预测文章标题？
- [ ] 推介是否在 150 字以内？
- [ ] 你是否避免了"革命性的"、"颠覆性的"、"改变游戏规则的"和"协同"这些词？
- [ ] 请求是否清晰？（采访？ embargo？独家？引用？）

如果任何答案为否，不要发送。

## 衡量

该追踪什么：

| 指标 | 原因 |
|------|------|
| **报道数量**（每月发布数） | 活动基线 |
| **发布位置域名权重** | 反向链接价值 |
| **来自报道的推荐流量** | 真有人点击了吗？ |
| **品牌搜索提升** | 人们读完后是否搜索了你？ |
| **AI 引用率**（ChatGPT、Perplexity 引用你的品牌？） | 现在真正重要的新衡量标准 |
| **引用该文章的销售对话** | 唯一对收入有意义的指标 |

不要痴迷的：AVE（广告价值等价）——那是公关公司发明的虚荣指标。

## 常见工作流

### "帮我搭车[热门故事]"
前往 [newsjacking.md](references/newsjacking.md)，运行评分标准，起草 2-3 个角度，选最好的，起草推介。

### "找报道[领域]的记者"
前往 [journalist-pitching.md](references/journalist-pitching.md)，使用发现清单 + dev-browser 研究近期文章，建立评分列表。

### "这周有什么值得推介的？"
组合：近期产品里程碑 + 活跃新闻周期 + 你收集的任何数据。按上面的质量标准给每个潜在故事打分。

### "回复这个 HARO 查询"
前往 [press-platforms.md](references/press-platforms.md)，使用回复模板，控制在 200 字以内。

### "搭建我的新闻页"
使用上面的清单。大多数公司一个下午就能搞定，然后一年不管——没问题。

# AARRR 框架 — 计划编排入门

AARRR（Dave McClure 的"海盗指标"）是此技能生成的每份计划的骨架。本文档是入门指南 + 每个阶段何时被优先考虑的决策规则。

## 五个阶段

| 阶段 | 核心问题 | 常见指标 |
|---|---|---|
| **A**cquisition（获取） | 陌生人如何知晓我们？ | 访问量、MQL、注册页会话、App Store 访问、按渠道 CAC |
| **A**ctivation（激活） | 一旦他们尝试，是否获得了转化体验？ | 注册完成率、达到价值时间、完成首个关键动作的百分比、试用 → 付费率 |
| **R**etention（留存） | 他们是否留驻并深化参与？ | DAU/WAU/MAU、第 1/4/12 周留存、流失率 |
| **R**eferral（推荐） | 留存用户是否带来更多用户？ | 病毒系数、NPS、大使归因 |
| **R**evenue（收入） | 他们付什么、谁在付、如何复利增长？ | ARPU、LTV、扩展收入、ARR / MRR |

> **注册边界规则。** 注册*意图*（陌生人到达注册页）属于获取。注册*完成*及之后的一切（首个关键动作、试用转付费）属于激活。在所有文档和计划模板中始终一致地应用此规则。

## 为什么用 AARRR 编排计划

三个原因。

**1. 漏斗阶段标签强制优先排序。** 没有 AARRR，营销计划会变成按渠道组织（"这是 SEO 计划，这是社交计划，这是付费计划"）。渠道可服务多个阶段；按阶段标签则问出更有用的问题：*当前漏斗哪个阶段是瓶颈约束？*

**2. 先堵漏洞再注水。** 激活/留存问题（"在已有曝光量的情况下漏斗转化率是否可接受？"）通常比获取问题（"如何获得更多曝光？"）杠杆更高。AARRR 编排天然揭示这一点。

**3. 收入/推荐对话更诚实。** 大多数营销计划将变现埋在"增长"之下，把推荐当成美好愿望。AARRR 强制对两者进行显式处理。

## 品牌和内容 — 不是独立阶段，而是跨切面

一个常见错误：把"品牌"或"内容"作为第六个桶。它们不是——它们服务于每个阶段。

- **品牌声音** 管控每个阶段的所有文案
- **内容** 喂养获取（SEO、社交）、激活（引导文案）、留存（邮件生命周期）、推荐（大使话术）、收入（定价页、销售材料）

在计划中，品牌/内容出现在战略框架（第 2 节）和第 11 节运营栈的跨切面中——绝不作为独立的 AARRR 章节。

## 诊断瓶颈约束 — 哪个 AARRR 阶段杠杆最高？

对每个客户，一到两个 AARRR 阶段将是瓶颈约束。计划优先在那里编排行动。

**决策规则：**

### 如果没有任何用户 → 从获取开始
- 上市前 / 第 0 天 / 候补名单阶段
- 没有漏斗数据
- 杠杆 = 构建前 100 个用户

### 如果有用户但跳出 → 从激活开始
- 注册发生了但激活率低
- App Store 转化差
- 引导完成率有问题
- 第 1 天 → 付费率远低于第 30 天 → 付费（意味着产品给予时间可以转化，但引导没有衔接上去）
- 杠杆 = 衔接注册到首次感知价值

### 如果激活正常但用户流失 → 从留存开始
- 第 1 个月留存低于品类标准
- 已激活用户 7–14 天内停止使用
- LTV 偏短
- 杠杆 = 生命周期、深化参与、流失预防

### 如果留存强但增长慢 → 从推荐/收入开始
- 留存用户喜爱产品但不分享
- 入站推荐零散到来
- 定价未经压力测试
- 对交付价值而言 ARPU 偏低
- 杠杆 = 口碑机制 + 定价优化（这两者经常聚集）

### 如果小规模一切正常 → 从获取（规模化）开始
- 漏斗健康
- 问题只是"更多"
- 这是"产品-市场适配后"的规模化问题

## 逐阶段战略模式

### 获取

**诊断问题：** TAM 级别的认知与当前漏斗量之间差距在哪？哪些渠道已被竞品占满，哪些还是开放的？

**常见获取行动：**
- SEO 内容策略（有机复利）
- 创始人主导渠道（B2B 用 LinkedIn、X、Substack；D2C 用 Instagram/TikTok）
- 付费获取（预算解锁时）
- App Store / Play Store / 市场上架优化
- PR 和信誉锚点放大
- 活动（线下、网络研讨会、会议演讲）
- 合作（Newsletter 互换、集成联合营销、经销商/代理合作伙伴）
- 硬件/电商面（混合业务的 Shopify SEO + Amazon）
- B2B 销售支持（案例研究、合作伙伴页面、垂直内容）

**排序原则：** 先构建有机复利基础（SEO + 创始人主导 + 内容 + PR 放大 + 大使）。只在有机基线跑通后再叠加付费。过早付费会放大已有缺陷。

### 激活

**诊断问题：** 在用户首次体验的哪个时刻他们判断"这个对我有用"或"这个不行"？什么阻止了他们到达那个时刻？

**常见激活行动：**
- 基础修复（故障门限、故障注册步骤、故障付费墙）
- 引导测试/重建（通常是杠杆最高的单一步骤）
- App Store 上架文案改写（试用门槛）
- 生命周期流程发货顺序（何时发引导邮件）
- 付费墙结构 + 试用时长
- 免费 → 付费衔接（应用内追加销售、软付费墙）

**排序原则：** 以最快速度达到首次感知价值。在"用户打开应用"和"用户获得转化体验"之间增加摩擦的一切都是删减候选。

### 留存

**诊断问题：** 用户为什么流失？什么会让他们留下？第一个价值时刻之后的"第二个价值时刻"是什么？

**常见留存行动：**
- 生命周期邮件流程：引导、流失用户重新参与、购后、召回
- 订阅 / 偏好中心
- 流失指标对账（通常指标定义在不同面上不匹配）
- 硬件 → 软件激活路径（混合业务）
- 年度计划默认 / 定价结构（与收入交叉）
- 支持即营销（推动故事的高触达时刻）
- 社群 + 从业者网络

**排序原则：** 按内容最稳定的顺序发货生命周期流程。硬件购后流程先发（不引用可能变化的应用内界面）。引导邮件最后发（引用可能变化的 UI）。召回是季度战役，不是一次性流程。

### 推荐

**诊断问题：** 是否有未被捕获的入站推荐兴趣？产品中自然的"价值后分享"时刻是什么？

**常见推荐行动：**
- 大使/联盟计划（从入站兴趣开始，不要冷招募）
- 产品内建的"价值后分享"时刻（反思提示、里程碑庆祝）
- 创始人放大（创始人作为零号推荐人）
- 长期博弈专家/Guides/认证主持网络（品类创造型业务）
- 赠礼流程（消费/硬件）
- 双边推荐（同时奖励推荐人和被推荐人）

**排序原则：** 从已经在举手的人开始。如果有 5 个入站大使，就用这 5 个启动——不要等"完整计划"。基于他们告诉你的内容迭代。

### 收入

**诊断问题：** 公司是否定价偏低？打包不足？缺少追加销售？给定 LTV 和品牌声音，"正确"的价格纪律是什么？

**常见收入行动：**
- 定价审计（今天实际收取什么 vs 标价？）
- 年度计划默认
- 硬件 → 软件捆绑正式化
- 店面/电商页面优化
- B2B 案例研究 + 销售材料
- 长期价值池标记（数据、扩展、企业级）——标记不执行

**排序原则：** 先做定价审计再测试变更。出人意料地频繁，仪表板上的"隐含"定价与标价不匹配——折扣、试用或计划组合扭曲了读数。先暴露事实基线。

## 如何将行动分配到阶段

有些行动明显属于一个阶段。其他跨越多个。规则：

**分配到该行动的主要可衡量影响所落在的阶段。**

示例：
- "用品牌声音改写 App Store 上架文案" — 跨越获取（有机发现）和激活（试用门槛）。主要影响 = 激活（试用转化率）。分配到激活，提及交叉。
- "眼罩 Shopify 页面改写" — 跨越获取（睡眠面罩的有机搜索）和收入（销售转化）。主要影响 = 收入（交易）。分配到收入，提及交叉。
- "Alex 的 LinkedIn 节奏" — 获取（D2C 订阅者漏斗顶部）。
- "Customer.io Flow 6（眼罩购后）" — 留存（深化硬件买家参与）与激活交叉（硬件 → 应用高级激活路径）。

拿不准时：删除这个行动最痛的地方在哪里？分配到那里。

## 当 AARRR 分解不均匀时

对大多数客户，计划不会在各阶段有等量内容。这没问题——而且值得作为诊断暴露出来。

- **获取章节偏重** = 客户有产品-市场适配但漏斗顶部是瓶颈。常见于留存指标强劲的早期阶段。
- **激活章节偏重** = 客户有流量但转化有问题。通常是 beta 阶段产品。
- **留存章节偏重** = 客户有流失问题。通常是没有生命周期基础设施就跨越了 PMF 的中期产品。
- **推荐章节偏重** = 客户有忠诚度但没有口碑机制。通常是拥有热情用户的消费产品。
- **收入章节偏重** = 客户定价偏低或缺少变现层。常见于从免费转向付费的工具。

如果一份计划最终在所有五个阶段均匀分布，诊断可能太弱——重新检查漏斗状态信息收集以找到瓶颈约束在哪。

## 关于呈现顺序的说明

始终按顺序呈现 AARRR（获取 → 激活 → 留存 → 推荐 → 收入），无论优先级顺序如何。

这是为了读者的心智模型。创始人期望漏斗从上到下流动。如果留存是杠杆最高的阶段但你从留存开始，读者需要上下文切换。

要传递优先级信号，使用执行摘要（第 1 节）——在那里命名最大赌注。AARRR 分解然后按漏斗顺序走，杠杆最高的章节最长最详尽。

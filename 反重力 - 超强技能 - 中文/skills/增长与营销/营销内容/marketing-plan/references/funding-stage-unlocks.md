# 融资阶段能力解锁

每份营销计划必须包含明确的"融资完成时/预算解锁时会发生什么变化"推理逻辑。这使计划对投资者友好且在操作上诚实。

本文档定义标准层级。以此为基础锚定，按客户品类和单位经济调整。

**相关文档：**
- `budget-planning.md` — 设定实际预算数字的两种科学方法（基于收入 5–40%，或从收入目标反向推导）、CAC 计算、实验缓冲
- `growth-patterns.md` — SaaS 增长的真实形态，按阶段分（$0–10K / $10K–100K / $100K–1M+）、线性 vs 阶跃函数、S 曲线叠加
- `team-and-agency-model.md` — 各层级对团队组成、首位营销雇员、内部 vs 外包比例的影响

## 为什么融资阶段在营销计划中重要

大多数营销计划写得好像预算没有约束。这对早期客户是失败模式——它产出的是愿景列表而非可执行路线图。

解决方案：将每条建议绑定到预算层级。计划对今天可执行的内容保持诚实，团队/投资者明确看到每一轮融资解锁什么。

这也帮助正在融资的创始人：展示本轮买到的东西是投资者叙事材料。

## 标准层级

### 第 1 层级 — Pre-seed / 自举型

**预算概况：**
- 付费获取：$0
- 工具栈：~$500–2,000/月（Customer.io / 同类、GA4 免费、Stripe 费用、Notion、GitHub、基础 SaaS）
- 顾问费/fCMO：不定（仅兼职）
- 人力成本：创始人 + 可能 1–2 名多用途雇员

**营销能力：**
- 仅有机渠道 —— SEO、内容、App Store 有机流量、创始人主导社交、活动、口碑、大使（如果有入站）
- 有限 PR（创始人主导的推介、HARO 回应）
- 无付费层

**活跃渠道：** 有机 SEO、内容、App Store、LinkedIn / X / 创始人主导社交、活动、口碑、大使

**fCMO 的职责：** 策略 + 生命周期 + 内容 + SEO + 引导 + 社群 + 大使。亲自动手，技能库 + MCP 承担运营工作量。

**解锁雇员：** 无。计划必须以当前团队 + 智能体栈执行。

### 第 2 层级 — Seed 关闭

**预算概况：**
- 付费获取：$5–15K/月测试预算
- 工具栈：$1,000–3,000/月（付费广告账户、Mixpanel/Amplitude 如需要、额外 SaaS）
- 顾问费/fCMO：延续
- 人力成本：+ 首位专职营销雇员

**营销能力：**
- 以上 + 付费获取试点（Apple Search Ads、Meta、LinkedIn）
- 开始配合融资公告做 PR 推送
- 首个 Product Hunt / GA 式发布

**活跃渠道：** 全部第 1 层级 + 小额付费获取 + 积极公关

**解锁雇员：**
- 生命周期 + 内容营销经理（一个人兼两项，或分开）
- 或 专职增长/绩效营销经理（如果侧重付费）

**fCMO 转变：** 从亲自动手转向策略 + 运营监督。雇佣专职营销人员。在付费规模化前建立渠道操作手册。

### 第 3 层级 — Seed 投放

**预算概况：**
- 付费获取：$20–50K/月
- 工具栈：$2,000–5,000/月
- 顾问费/fCMO：延续
- 人力成本：+ 设计师（可能兼职）

**营销能力：**
- 在 2–3 个渠道进行付费规模化
- 品牌对齐的创意生产（设计师提升速度）
- 生命周期流程全量上线所有流
- 首个真正的内容生产节奏（每周可持续）

**活跃渠道：** 以上全部 + 付费规模化 + 结构化发布动作

**解锁雇员：**
- 设计师（品牌、创意、网页）
- 第二名营销经理（如果第一人是生命周期，第二人是内容；反之亦然）
- 可能兼职 PR 如果预算允许

**fCMO 转变：** 将生命周期交接给专职负责人。转向 GTM 策略 + 渠道组合优化 + 增长分析。

### 第 4 层级 — Series A

**预算概况：**
- 付费获取：$50–150K/月
- 工具栈：$5,000–10,000/月
- 顾问费/fCMO：可能过渡为全职 CMO
- 人力成本：完整营销团队成型中

**营销能力：**
- 在所有已验证渠道激进规模化付费
- 品牌战役成为可能
- 开始考虑国际化
- B2B 垂直扩展（如适用）
- 成熟的 CAC/LTV 数学 + 归因

**活跃渠道：** 完整营销覆盖面

**解锁雇员：**
- 绩效营销负责人
- 内容负责人
- 设计师（全职）
- 可能：PR 公司、付费代理公司、国际化增长经理
- Series A 往往是 fCMO 过渡退出或转为顾问的时刻

**fCMO 参与度：** 通常此时已退出公司——原 fCMO 可能仍是顾问。

### 第 5 层级 — Series B+

**预算概况：**
- 付费获取：$150K+/月
- 工具栈：$10,000–25,000/月
- 人力成本：10+ 人营销组织

**营销能力：**
- 行业规模品牌战役
- PR 公司合作
- 收购式营销（收购空间内的 Newsletter/播客）
- 品类级别会议赞助
- 品牌级别赞助

**活跃渠道：** 所有可用渠道

**解锁雇员：**
- 营销 VP 或 CMO
- 品牌总监
- 增长/绩效团队（3–5 人）
- 内容团队（3–5 人）
- 设计师（2–3 名）
- PR 总监或代理合作
- 国际化营销负责人（区域特定）

**fCMO 参与度：** 此时通常已不在公司内——原 fCMO 可能仍是顾问。

## 如何在计划中应用层级逻辑

### 第 3 节（当前状态）
- 明确陈述客户的当前层级："当前层级：Pre-seed / 自举型，对应第 1 层级。"

### 第 4–8 节（AARRR 章节）
- 标注依赖层级的行动："付费层（第 2 层级解锁 —— 保持至 Seed 关闭）"
- 对第 1 层级计划：每个行动必须在当前预算层级可执行或显式标记为未来
- 对第 2 层级+计划：行动可假设该层级的能力

### 第 10 节（12 个月展望）
- 每季度命名活跃层级："Q2 —— 第 4–6 个月（Seed 关闭后）。资金状态：第 2 层级。"
- 层级转换触发计划重新校准时刻

### 第 11 节（营销运营栈）
- 使用 `references/ops-stack-mapping.md` 能力解锁部分的表格
- 使其客户特定："今天（第 1 层级）：{客户当前能力}。Seed 关闭后（第 2 层级）：+ {什么变化}。"

## 按客户品类的调整

标准层级假设典型的软件/SaaS/消费应用。按品类调整：

### 消费应用（D2C）
- 更高的付费获取门槛 —— 应用需要对照下载成本基准测试 CAC（约 $2-10 安装 + 5–15% 试用转化基准）
- 第 2 层级实际从 $10–20K/月开始（否则在 App 安装 CPM 下无法获得统计有意义的读数）

### B2B SaaS
- 较低的付费获取门槛 —— LinkedIn/Google Ads 在 $3–5K/月就能产生信号
- 更多权重放在内容 + 销售赋能预算上
- 通常在内容雇员前先雇佣销售

### 混合硬件 + 软件
- 硬件收入可以自筹部分营销费用（眼罩楔形模式）
- 付费预算应追踪硬件销售 + 应用订阅的混合 CAC
- Shopify 侧优化是第 1 层级优先事项（廉价杠杆）

### 深科技 / 科学 / 临床
- PR + 投资者营销比付费更重要
- 会议演讲 + 学术发表 > Meta 广告
- 第 1 层级无需付费即可产生显著牵引力

### 双边市场
- 每侧有自己的 AARRR 漏斗 —— 预算相应分配
- 供给端获取通常早期主导；流动性之后需求端主导

### 开源 / 开发者工具
- DevRel + 社群 + 内容 > 付费
- GitHub 星标/npm 安装是激活事件
- 付费层通常推迟到 Series A

## 第 1 层级预算详情（最常见起点）

对第 1 层级客户，营销预算分解通常如下：

| 行 | 典型月度 |
|---|---|
| Customer.io / 生命周期 ESP | $100–500 |
| App Store Connect / Google Play | $25 + 30% 收入分成（Apple/Google 抽成） |
| Stripe | 2.9% + 每 30¢ |
| GA4 | 免费 |
| Notion | $0–100 |
| GitHub | $0–50 |
| Shopify（如有硬件） | $39–100 |
| Ahrefs（或同类 SEO 工具） | $129–399 |
| Typefully（如有社交节奏） | $13–39 |
| Dub.co（如有大使追踪） | $0–39 |
| 杂项 SaaS | $200–500 |
| **工具总计** | **~$500–1,700/月** |
| 付费获取 | $0 |
| fCMO 顾问费 | 不定 |

对计划而言，这变成："当前每月营销预算：$X（仅工具，无付费）。"

## 何时向创始人暴露层级限制

如果创始人提出需要未来层级的行动：
- 命名要求："这是第 2 层级行动（需要 $10K+/月付费预算）。将在 Seed 关闭后按 §10 中 12 个月展望解锁。"
- 不要拒绝 —— 说明时机

如果创始人低估所需投入：
- 要诚实："要真正规模化付费获取，预计需要第 2 层级预算。第 1 层级可以验证有机；第 2 层级验证付费。"

如果创始人在其阶段资金过剩：
- 不要为了匹配而膨胀预算。推荐适合漏斗状态的工作，返回多余产能，建议投资复利而非规模化。

## 跨层级情况（值得标注）

有些公司跳过层级：
- **知名创始人** 融资轮次大于常规 —— 可直接从第 1 层级跳到第 3 层级
- **硬件公司** 有 PR 时机 —— 在正确的产品时机（如高知名度长寿影响者背书）可按第 3 层级水平部署
- **B2B SaaS post-LOI** 有名企业合同 —— 可用合同价值资助试点部署

如果客户处于跨层级情况，在计划中显式命名，不要强行套入标准阶梯。

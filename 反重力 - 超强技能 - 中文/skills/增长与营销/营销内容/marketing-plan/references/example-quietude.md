## 1. 执行摘要

本计划优化 Quietude 从"有前景的测试版产品"到"有付费墙且付费用户留存的上市产品"的过渡——同时建立获取、留存和推荐的有机基础，使得 Seed 资金可以加速已被验证的事物，而非资助探索性投放。

**三大赌注：**

1. **引导 + 付费墙是杠杆最高的表面。** 注册体验中从"好奇"到"转化"的断裂是最大的未开发杠杆。修复引导、上线付费墙和测试年度默认可以在不增加获客支出的情况下将 LTV 提高 30–50%。
2. **有机获取基础在付费之前。** SEO + App Store 优化 + 创始人主导渠道 + PR 放大是一套复利投资。付费获取放大已有缺陷——在有机基线验证漏斗之前不投放付费广告。
3. **推荐机制从已举手的人开始。** 早期 Quietude 用户热爱产品并已自发分享。正式化大使计划捕获已有的口碑，而非从零启动。

**12 个月预期状态：**

- Quietude 上市且付费墙上线
- 有机获取基线已建立（SEO、ASO、创始人渠道、PR）
- 生命周期邮件覆盖引导 + 购后 + 流失 + 召回
- 大使计划捕获入站口碑
- 年度默认经过测试；LTV 数学已知
- Seed 关闭 → 付费获取试点启动（Apple Search Ads + Meta）
- ARPU 足够已知以设定有依据的付费 CAC 目标

**90 天优先事项：**

1. 修复引导（达到价值时间缩短 50%）
2. 上线付费墙 + 测试年度默认
3. App Store 上架文案改写（试用转化优化）
4. 启动生命周期邮件 Flow 1（引导）+ Flow 2（购后硬件）
5. 上线大使计划 v1（捕获已有入站兴趣）
6. 发布种子轮 PR（融资叙事 + 产品叙事合并）

## 2. 战略框架

### Quietude 是什么，一句话

Quietude 是一个睡眠健康平台，将科学引导的声浴与可选的遮光硬件结合，帮助人们入睡并保持睡眠。

### 我们在定义的品类

Quietude 正在定义 **"主动睡眠健康"** 品类——不是被动追踪（那是 Oura/Whoop），不是通用冥想（那是 Calm/Headspace），而是主动干预：使用声音 + 黑暗 + 习惯叠加来转变睡眠质量。

品类定义框架：追踪告诉你睡眠出了问题。冥想告诉你放松。主动睡眠健康给你一个可执行的系统——声浴 + 遮光 + 习惯叠加——来修复它。

### 我们为谁而存在（提炼的 ICP）

- **人口统计：** 25–45 岁城市职场人，睡眠不满意但不是临床失眠
- **他们说自己想要：** "睡得更好"
- **他们实际想要：** 对入睡时刻感到有掌控感
- **他们正在购买：** 一个每晚可执行的仪式，而不仅仅是信息
- **关键洞察：** 他们已经在做事情（遮光窗帘、白噪音、褪黑素）——Quietude 整合并升级这些，而非从零开始

### 商业模型逻辑

- **楔形产品：** 眼罩（$45 硬件）——低摩擦首次购买、高视觉性、易赠礼
- **LTV 引擎：** 应用高级订阅（$9.99/月 或 $79.99/年）——来自软件的经常性收入
- **复利渠道论点：** 硬件分销创造软件激活机会；每个眼罩买家都是高意图的软件订阅潜客
- **单位经济：** 硬件毛利虽低但可接受（约 60%）；软件毛利极高（约 90%+）。混合 LTV 随时间向软件倾斜。

### 品牌声音（不可妥协项）

- **核心方法：** 启发式（发起对话，不等待问题）
- **语气：** 权威又亲切、亲密又专业
- **词汇 YES：** 仪式、声浴、睡眠健康、主动、系统、实践、临床、科学
- **词汇 NO：** 冥想、正念、放松、禅、氛围、感觉、疗愈、治愈、灵性
- **CTA 规则：** 从不施压。邀请而非推动。"试试看"不是"立即购买"。提供选择而非锁定。
- **声音为什么重要：** 区分于 Calm（通用冥想）和 Headspace（可爱/随意）。Quietude 的声音是临床从业者的声音——精确、主动、可信。

## 4. 获取

### 当前状态

有机渠道驱动所有当前增长。App Store 有机发现 + 口碑 + 创始人存在 = 当前的漏斗顶部。无付费获取。SEO 处于最早阶段（域名存在，内容策略尚未建立）。

### 计划

**行动 1 — SEO（和内容）**

建立有机获取基础。使用 `seo-audit` 建立基线，然后用 `ai-seo` 构建内容策略。重点：睡眠健康关键词、声浴关键词、竞品替代关键词。

关键词研究通过 Ahrefs API 完成。内容支柱：声浴科学、睡眠健康实践、产品比较。

**行动 2 — App Store / Play Store 优化**

重写 App Store 上架文案（标题、副标题、描述、截图）。目标：试用转化率提高 20%+。关键词：sleep sounds、sound bath、sleep health（非 meditation、mindfulness）。

**行动 3 — 创始人主导渠道**

Alex 的 LinkedIn 节奏：每周 2–3 篇，混合睡眠科学内容 + Quietude 产品叙事 + 个人创始人旅程。目标：驱动 D2C 订阅者漏斗顶部。

**行动 4 — PR 放大**

种子轮融资公告是信誉锚点。构建融资 PR 推动：结合融资叙事 + 产品叙事。目标：5+ 行业报道。

**行动 5 — 硬件/商业面（Shopify）**

眼罩 Shopify 店面 SEO。产品页面改写以匹配品牌声音。目标：睡眠面罩的有机搜索可见性。

**行动 6 — 付费层（预算解锁时）**

Apple Search Ads + Meta 试点。在 Seed 关闭后上线。第 2 层级解锁 $10K/月付费测试预算。在有机基线验证漏斗前不投放付费广告。

### 90 天获取行动

- 第 1–2 周：SEO 审计 + 关键词研究（`seo-audit` 技能 + Ahrefs API）
- 第 1–2 周：App Store 上架文案改写（`onboarding` + `copywriting` 技能）
- 第 3–4 周：Alex LinkedIn 节奏启动（Typefully MCP 调度）
- 第 3–4 周：Shopify 产品页面改写（`copywriting` + 品牌声音）
- 第 5–8 周：SEO 内容支柱 1 发布（`ai-seo` + `content-strategy`）
- 第 5–8 周：种子轮融资 PR 推动（`launch` 技能）
- 第 9–12 周：SEO 内容支柱 2 + 内链建设

### 12 个月获取展望

- Q1：有机基线建立。App Store + Shopify + LinkedIn 活跃。
- Q2：SEO 内容开始排名。创始人渠道产生稳定潜客流。
- Q3：Seed 关闭。付费试点上线（Apple Search Ads + Meta）。
- Q4：付费规模化。SEO 复利。下一个渠道研究（影响者/合作伙伴）。

## 5. 激活

### 当前状态

引导体验损坏。用户注册但未达到首次价值时刻。App Store 试用转化率低于品类基准。付费墙尚未上线——没有"转化"的定义。

修复引导流程中的损坏门限。删除注册与首次声浴之间的摩擦。

**行动 2 — 引导重建**

重新设计引导流程以达到价值时间（TTV）< 60 秒。使用 `onboarding` 技能。聚焦：首次声浴选择 → 即时播放 → 引导式设置仪式。品牌声音：启发式，不等待问题。

与获取行动 2 同步。新截图突出 TTV 承诺。

**行动 4 — 付费墙上线 + 年度默认测试**

上线付费墙，默认提供年度计划。测试免费 → 付费转化。参见第 8 节（收入）。

**行动 5 — 引导邮件流程发货**

生命周期邮件 Flow 1 在应用内引导稳定后发货。

### 90 天激活行动

## 6. 留存

### 当前状态

无生命周期邮件在线。购后沉默。用户注册，可能尝试一次声浴，然后流失。硬件买家无购后连接。

### 计划

**行动 1 — 生命周期邮件 Flow 2（硬件购后）**

硬件买家的购后流程。内容最稳定——不引用可能变化的应用内 UI。先发货。

**行动 2 — 生命周期邮件 Flow 1（引导）**

在应用内引导稳定后发货。引用应用内 UI，所以必须在 UI 冻结后上线。

**行动 3 — 流失用户重新参与流程**

目标：已使用产品但 14 天+ 未回访的用户。

**行动 4 — 召回流程**

季度战役，非一次性。深度内容（科学、实践、用户故事）。

**行动 5 — 硬件 → 应用高级激活路径**

硬件买家自然路径：开箱 → 应用下载 → 免费高级试用 → 付费转化。使此路径显式并优化。

**行动 6 — 年度计划默认测试**

与激活/收入交叉。年度计划减少流失（切换成本更高）。

### 90 天留存行动

- 第 3–4 周：Flow 2（硬件购后）上线（`emails` + Customer.io MCP）
- 第 5–6 周：Flow 1（引导）上线
- 第 7–8 周：流失用户重新参与流程上线
- 第 9–12 周：召回流程 v1 + 硬件→应用路径审计

### 12 个月留存展望

- Q1：核心生命周期流程在线（购后、引导、流失）
- Q2：分群 + 细分。召回战役。
- Q3：留存曲线可测量。付费用户流失 < 5%/月。
- Q4：生命周期覆盖完整。支持即营销时刻识别。

### 阻碍计划的待决事项

1. **CAC 未知（影响最大）** — 每个收入预测都依赖于此。付费墙上线后优先测量。
2. **年度留存率未知** — LTV 计算依赖于此。需要 3+ 月付费用户数据。
3. **种子轮关闭时间** — 付费层级解锁依赖于此。如果不关闭，Q3 计划需要调整。
4. **引导修复范围** — 审计将决定这是微调还是完全重建。影响 Jordan 带宽。
5. **Apple 审核时间** — 付费墙 + 上架文案更新需要审核。可能延迟发货。
6. **认证主持可行性** — 是否有足够的声浴从业者来建立网络？需要 Q2 研究。
7. **Shopify 事件埋点** — 硬件购后流程依赖此。Jordan 需要优先处理。
8. **Dub.co 佣金处理** — 大使佣金通过 Stripe Connect？需确认技术路径。

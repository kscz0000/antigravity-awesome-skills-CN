# 客户类型 — 按业务模型的变体

13 节计划结构在各客户类型间保持一致。变化的是各章节内的**内容侧重**。本文档命名各客户原型的主导模式。

## 原型 1 — B2B SaaS

### 核心特征
- 订阅收入
- 通常较高 ACV（$1K–$100K+/年）
- 视层级不同采用销售辅助或自助服务
- 买家通常不同于用户（推动者 vs 最终用户）

### AARRR 侧重

**获取偏重：**
- SEO 是主导的漏斗顶部动势（人们搜索解决方案）
- 内容营销（博客、知识库、对比页）驱动 MQL
- LinkedIn 同时用于有机创始人存在和付费
- 外拓（冷邮件 + LinkedIn）经常补充入站
- 活动（会议、网络研讨会）针对高 ACV 产品

**激活：**
- 注册 → 试用 → 首个关键动作（PLG 产品）
- 试用 → 演示 → POC（销售驱动产品）
- 空态页面很重要 —— 引导用户到首个价值动作

**留存：**
- 产品参与度指标（DAU、功能采纳）
- 客户成功动作（高 ACV 用 CSM 团队）
- 生命周期邮件聚焦功能发现、价值时刻

**推荐：**
- 客户倡导项目
- 合作伙伴 / 集成联合营销
- G2 / Capterra 评论
- 从推动者到买家的扩展

**收入：**
- 扩展 / NRR 通常是最大的增长杠杆
- 层级升级、席位扩展、用量制附加组件

### 技能侧重
- `cold-email`, `programmatic-seo`, `competitors`, `seo-audit`, `ai-seo`
- `ads` 侧重 LinkedIn + Google
- `emails` 用于试用培育 + 生命周期
- `pricing` 用于层级优化

### 第一层级预算优先级
- SEO + 内容 > 其他一切
- 创始人主导的 LinkedIn 渠道
- Customer.io / Mailchimp 用于培育
- HARO + 投资者暗线用于 PR

---

## 原型 2 — D2C 消费应用（订阅）

### 核心特征
- 较低 ACV（通常 $5–30/月）
- 高量低毛利每用户
- App Store / Play Store 作为主要获取面
- 生命周期邮件 + 推送用于留存
- 预算解锁后常为付费获取驱动

### AARRR 侧重

**获取：**
- App Store 优化（ASO）是最高杠杆的非网站资产
- 付费社交（Meta、TikTok）预算存在时常占主导
- Apple Search Ads 用于高意图 App Store 流量
- 影响者 + 内容创作者
- PR + 背书

**激活：**
- 引导是主导激活面
- 达到价值时间必须是分钟级而非小时级
- 付费墙结构 + 试用时长至关重要

**留存：**
- 生命周期邮件 + 推送
- 应用内提醒（谨慎 —— 过度使用 = 流失）
- 订阅偏好中心
- 召回战役

**推荐：**
- 内置分享（分享一月体验流程）
- 双边推荐
- 影响者/创作者大使

**收入：**
- 年度计划默认是最大单项行动（压缩 MRR 但改善 LTV）
- 层级优化（Free → Premium → Premium+）
- 应用内追加销售

### 技能侧重
- `onboarding`, `paywalls`, `emails`
- `ads`, `ad-creative`（重度创意迭代）
- `referrals`
- `pricing` 用于年度默认 + 层级整合

### 第一层级预算优先级
- ASO 优先（最高有机杠杆）
- 引导重建
- 生命周期邮件上线
- 创始人主导社交（如果创始人愿意出镜）

---

## 原型 3 — 混合硬件 + 软件

### 核心特征
- 实体产品 + 软件配套（如 Quietude 的眼罩 + 应用）
- 硬件作为分发楔形（价格更低、首次购买更容易）
- 软件作为 LTV（经常性收入）
- 跨两个表面的混合 CAC

### AARRR 侧重

**获取：**
- Shopify 店面 SEO（硬件产品页面瞄准消费搜索）
- Amazon 上架（高发现率但吃利润）
- PR 放大（硬件上镜 —— 高知名度影响者背书带动销量）
- 硬件的付费社交（Meta + Instagram，吸睛创意）

**激活：**
- 两个激活需跟踪：硬件开箱体验 + 软件注册
- 硬件 → 软件激活流程是桥梁
- 为高价值硬件买家提供礼宾设置

**留存：**
- 硬件购后生命周期（不同于应用引导）
- 软件配套驱动粘性
- 围绕硬件的社群/从业者网络

**推荐：**
- 硬件赠礼流程（实体产品的高口碑）
- 吸睛硬件驱动有机社交分享
- Shopify + Amazon 上的评论

**收入：**
- 混合 LTV 数学至关重要（硬件毛利 + 软件经常性收入）
- 捆绑策略（硬件购买 → 免费 Premium X 个月）
- 软件年度计划默认

### 技能侧重
- `seo-audit` 用于 Shopify 产品页
- `emails` 用于硬件购后和软件生命周期
- `referrals` 配合赠礼层
- `pricing` 用于混合捆绑数学
- `ads` 配合 Meta 重创意存在

### 第一层级预算优先级
- Shopify 产品页优化
- 硬件购后生命周期上线
- 捆绑策略正式化
- 硬件 → 应用激活审计

---

## 原型 4 — 双边市场

### 核心特征
- 双边产品（供给 + 需求）
- 网络效应很重要
- 流动性是关键的早期指标
- 费率 × GMV 是收入模型

### AARRR 侧重

**获取：**
- 两个漏斗 —— 供给和需求
- 供给常通过外拓/合作/冷邮件获取
- 需求常通过 SEO/付费/内容获取
- 城市化程序化 SEO 很常见

**激活：**
- 供给激活：首个上架、首个回复发出
- 需求激活：首次购买/首次匹配/首次交易
- 两端都需要自己的引导

**留存：**
- 重复交易频率
- 供给利用率（% 的上架商品活跃）
- 需求习惯（DAU / MAU）

**推荐：**
- 供给 → 供给（推荐其他提供方）
- 需求 → 需求（推荐其他买家）
- 跨边推荐较弱

**收入：**
- 费率优化
- 高级层级（更好匹配、更低费用）
- 潜客生成 vs 交易费变现

### 技能侧重
- `programmatic-seo` 用于城市页、垂直页
- `cold-email` 用于供给端招募
- `referrals` 用于两端
- `pricing` 用于费率决策

### 第一层级预算优先级
- 一侧的程序化 SEO 构建
- 冷外拓播种供给（或需求，哪个是瓶颈就选哪个）
- 两侧的生命周期邮件

---

## 原型 5 — 开发者工具 / 开源

### 核心特征
- 技术型买家（开发者或工程负责人）
- 高内容质量门槛（开发者持怀疑态度）
- DevRel 比传统营销更重要
- 开源层常漏斗进入商业产品

### AARRR 侧重

**获取：**
- 技术内容 + 文档 SEO
- DevRel（会议、演讲、社群）
- GitHub 存在感 + npm/pip 等 发现
- Hacker News + Reddit + 开发者 Twitter/X

**激活：**
- 首次构建/首次集成是激活事件
- 到 Hello World 的时间很重要
- 文档 = 开发者工具的引导

**留存：**
- 集成深度（更多使用产品）
- 团队采用（一个用户 → 整个组织）
- 活跃项目计数

**推荐：**
- GitHub 星标数（半有机）
- 技术论坛中的推荐
- 提及该工具的会议演讲

**收入：**
- 免费 → 付费转化当用量超限时
- 团队计划、企业层级
- 支持/SLA 追加销售

### 技能侧重
- `programmatic-seo` 用于文档
- 传统 `ads` 侧重较少
- 重度 `content-strategy` + 技术内容
- `cold-email` 给目标公司的工程负责人

### 第一层级预算优先级
- 文档 + 技术内容生产
- DevRel（创始人做演讲）
- GitHub 存在感
- HN / Reddit / 开发者社群

---

## 原型 6 — 深科技 / 科学 / 临床

### 核心特征
- 长销售周期
- 重信誉负担（必须证明科学性）
- 高度知情的买家（学术人士、临床医生、研究者）
- 常有监管考量

### AARRR 侧重

**获取：**
- 学术发表 + 同行评审研究
- 会议演讲（学术 + 行业）
- 投资者/顾问介绍
- 通过信誉钩子的 PR

**激活：**
- 试点项目 / 概念验证
- 礼宾设置配高触达引导
- 教育网络研讨会/培训

**留存：**
- 客户成功重度
- 与客户联合发表
- 从业者实践社群

**推荐：**
- 学术/临床引用
- 会议小组讨论
- 有名机构的案例研究

**收入：**
- 试点 → 付费扩展
- 机构合同（多席位/多年期）
- 合规/认证追加销售

### 技能侧重
- 轻传统营销
- 重度 `product-marketing`, `sales-enablement`, `pricing`
- `cold-email` 针对特定研究者/从业者
- PR + 投资者营销

### 第一层级预算优先级
- 学术外联 + 会议演讲
- 投资者暗线用于机构热介绍
- 关键客户试点部署
- 案例研究 + 科学发表

---

## 原型 7 — 电商 / DTC（非订阅）

### 核心特征
- 以交易方式销售的实体或数字产品
- 平均订单价值很重要
- 复购率是关键的留存指标

### AARRR 侧重

**获取：**
- 付费社交（Meta、TikTok）常占主导
- Shopify SEO 用于产品页
- Amazon 上架
- 影响者/创作者合作

**激活：**
- 首次购买是激活事件
- 购物车放弃恢复
- 结账时的信任信号（评价、退货、物流信息）

**留存：**
- 购后生命周期
- 忠诚度项目
- 邮件 + SMS 促进复购

**推荐：**
- 赠礼流程
- 推荐朋友项目
- 评价 + UGC

**收入：**
- AOV 优化（捆绑、追加销售）
- 客户 LTV 优化（复购频率）
- 复购的订阅选项

### 技能侧重
- `ads` + `ad-creative`（重度权重）
- `emails` 用于购后 + 放弃购物车
- `referrals` 配合赠礼
- `pricing` 用于捆绑 + 订阅选项

### 第一层级预算优先级
- Shopify 店面优化
- 邮件生命周期上线
- 影响/UGC 播种
- 付费社交测试（如果有最小预算存在）

## 起草计划时如何使用本文档

开始起草第 4–8 节（AARRR）时，识别客户的原型（如适用则为混合），然后深入上述模式。

**混合情况很常见。** Quietude 是"混合硬件 + 软件"，与"深科技/科学/临床"有显著重叠（因为同行评审研究 + 临床定位）。计划融合两种原型的侧重点。

拿不准时，以最符合*主要变现模型*的原型为主导。Quietude 的主要变现是软件订阅（硬件作为楔形），所以 D2C 消费应用 + 硬件混合模式为主，叠加深科技信誉动作。

## 当客户不能清晰归类时

有些客户不遵循原型：
- **内容/媒体业务** —— 既非 SaaS 也非电商；广告收入或订阅模式
- **社交网络** —— 自有品类，网络效应主导
- **房地产/活动** —— 实体 + 服务模式

对此，识别最近的原型并调整。不要强行套用 —— 在计划的战略框架中命名偏差。

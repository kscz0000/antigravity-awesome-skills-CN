# 落地页模式

本参考收录高转化页面模式和文案结构。

## 首屏区块模式

### 模式 1：问题-方案首屏
- 标题点出痛点。
- 副标题陈述明确结果。
- 主 CTA 立即出现。
- 可选配产品使用场景图。

### 模式 2：结果优先首屏
- 标题以可量化的价值开头。
- 副标题明确页面面向谁。
- CTA 具体且行动导向。

### 模式 3：权威首屏
- 标题 + 信任指标（Logo、证言摘要、数据证明）。
- 适用于品类怀疑度高的场景。

## 社会证明布局

### Logo 条 + 数据证明
- 只放可识别的 Logo。
- 附加一个数据证明（如活跃用户数、节省金额、减少工时）。

### 证言网格
- 3-6 条证言，覆盖不同细分人群。
- 尽量包含职位/公司。
- 优先选择具体成果，而非泛泛好评。

### 案例快照
- 迷你模块：挑战 → 方案 → 可量化结果。

## CTA 最佳实践

- 每个区块只设一个主导 CTA。
- CTA 动词匹配用户意图（"开始试用"、"获取演示"、"运行审计"）。
- CTA 文案要具体；避免"提交"等模糊标签。
- CTA 附近降低阻力（短表单、信任指标、无隐藏承诺）。

## 首屏检查清单

- [ ] 第一视口内有清晰价值主张
- [ ] 受众明确（这是给谁用的）
- [ ] 无需滚动即可看到一个主 CTA
- [ ] 证明元素（Logo、数据、引用）
- [ ] 视觉层次突出标题 + CTA
- [ ] 移动端布局保持 CTA 可达

## 标题公式

### PAS（Problem-Agitate-Solution）
- Problem：识别痛点
- Agitate：展示不行动的后果
- Solution：将产品定位为解药

示例结构：
"还在[问题]？停止[负面后果]，开始[期望结果]。"

### AIDA（Attention-Interest-Desire-Action）
- Attention：打破惯性的标题
- Interest：相关背景和利害关系
- Desire：证据和收益
- Action：具体的下一步

### 4U 公式
- Useful：明确的实用价值
- Urgent：立即行动的理由
- Unique：差异化的承诺
- Ultra-specific：具体的结果和范围

示例结构：
"在[时间]内获得[具体结果]，无需[常见痛点]。"

# 类别定义

本文件定义 Postgres 最佳实践的规则类别。规则根据文件名前缀自动分配到对应类别。

以下示例仅作演示用途。请将每个类别替换为 Postgres 最佳实践的实际规则类别。

---

## 1. 查询性能 (query)
**影响：** 关键
**说明：** 慢查询、缺失索引、低效查询计划。Postgres 性能问题最常见的来源。

## 2. 连接管理 (conn)
**影响：** 关键
**说明：** 连接池、连接限制和无服务器策略。对高并发或无服务器部署的应用至关重要。

## 3. 安全与 RLS (security)
**影响：** 关键
**说明：** 行级安全策略、权限管理和认证模式。

## 4. 模式设计 (schema)
**影响：** 高
**说明：** 表设计、索引策略、分区和数据类型选择。长期性能的基础。

## 5. 并发与锁 (lock)
**影响：** 中高
**说明：** 事务管理、隔离级别、死锁预防和锁竞争模式。

## 6. 数据访问模式 (data)
**影响：** 中
**说明：** N+1 查询消除、批量操作、基于游标的分页和高效数据获取。

## 7. 监控与诊断 (monitor)
**影响：** 中低
**说明：** 使用 pg_stat_statements、EXPLAIN ANALYZE、指标收集和性能诊断。

## 8. 高级特性 (advanced)
**影响：** 低
**说明：** 全文搜索、JSONB 优化、PostGIS、扩展和高级 Postgres 特性。

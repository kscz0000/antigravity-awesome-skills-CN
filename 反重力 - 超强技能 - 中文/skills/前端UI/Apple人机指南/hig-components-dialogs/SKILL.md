# Apple HIG: 展示组件

提问前请检查 `.claude/apple-design-context.md`。使用现有上下文，仅询问未涵盖的信息。

## 核心原则

1. **警告框：谨慎用于关键情况。** 需要关注的错误、破坏性操作确认，或需要确认的信息。它们会中断流程并要求响应。

2. **表单：保持上下文的聚焦任务。** 从边缘滑入（macOS 上附加到窗口）。用于创建项目、编辑设置、多步表单。

3. **弹出框：iPad 和 Mac 上非模态。** 出现在触发元素旁边，点击外部关闭。用于附加信息、选项或控件，不占据整个屏幕。

4. **操作表：在多个操作中选择。** 当需要从多个操作中选择时展示，尤其是其中包含破坏性操作时。iPhone：从底部向上滑出。iPad：以弹出框形式出现。

5. **最小化中断。** 在使用模态之前，考虑内联展示或使操作可撤销。

6. **警告文本简洁可操作。** 简短的描述性标题。必要时简短的消息正文。按钮标签应为具体动词（"删除"、"保存"），而非"确定"。

7. **明确标记破坏性操作。** 破坏性按钮样式（红色文字）。将破坏性按钮放在用户不太可能条件反射点击的位置。

8. **提供取消选项** 用于有多个操作的警告框和操作表。在操作表上，取消按钮出现在底部，单独分隔。

9. **数字输入：聚焦且无障碍。** 适当大小的输入字段，数字间自动前进，支持粘贴和自动填充。

10. **根据平台调整展示方式。** 同一交互在 iPhone、iPad、Mac 和 visionOS 上可能使用不同组件。

## 输出格式

1. **推荐的展示类型及理由** 以及为何其他选项不太适合。
2. **内容指南** — 标题、消息、按钮标签，遵循 Apple 的语气和简洁规则。
3. **关闭行为** — 用户如何关闭以及发生什么（保存、放弃、取消）。
4. **替代方案** — 场景何时可能根本不需要模态（内联反馈、撤销、渐进式展示）。

# Apple HIG: 状态组件

提问前先检查 `.claude/apple-design-context.md`。使用已有上下文，仅询问未覆盖的信息。

## 核心原则

### 进度指示器

1. **超过一两秒的操作应显示进度。**

2. **时长或百分比已知时使用确定性进度。** 填充式进度条让用户清楚感知剩余工作量。适用于下载、上传或任何可量化的过程。

3. **时长未知时使用不确定性进度。** 旋转指示器传达"正在处理"但不承诺时间框架。适用于不可预测的网络请求。

4. **优先使用进度条而非旋转指示器。** 确定性进度感觉更快、更可信。

5. **将指示器放在内容即将出现的位置。** 内联进度靠近内容区域，而非模态或远处位置。

6. **不要堆叠多个指示器。** 将同时进行的操作合并为一个表示，或显示最相关的那个。

### 状态栏

7. **没有充分理由不要隐藏状态栏。** 仅在沉浸式体验（全屏媒体、游戏、AR）时隐藏。

8. **状态栏样式与内容匹配。** 浅色或深色以确保足够对比度。

9. **尊重安全区域。** 状态栏后方不要放置可交互内容。

10. **退出沉浸式场景后立即恢复状态栏。**

### 活动圆环

11. **活动圆环专用于"移动"、"锻炼"和"站立"目标。** 不要将圆环隐喻挪用于无关数据。

12. **尊重圆环颜色约定。** 红色（移动）、绿色（锻炼）、蓝色（站立）与 Apple Fitness 强关联。

13. **使用 HealthKit API** 获取活动数据，而非手动追踪。

14. **圆环闭合时用动画和触感反馈庆祝。**

## 输出格式

1. **指示器类型推荐**及理由（确定性 vs 不确定性）。
2. **时机和动画指导**——时长阈值、动画风格、过渡。
3. **无障碍**——VoiceOver 进度播报、实时区域更新。
4. **平台特定行为**跨目标平台。

## 需要询问的问题

1. 时长已知还是未知？
2. 目标平台有哪些？
3. 操作通常需要多长时间？
4. 系统级还是应用内指示器？

# 设计系统：Taste Standard
**技能：** stitch-design-taste

---

## 配置 — 设定你的风格
在使用此设计系统前调整这些参数。它们控制输出的创意程度、信息密度和动画强度。选择适合你项目的级别。

| 参数 | 级别 | 说明 |
|------|-------|-------------|
| **创意度** | `8` | `1` = 极简、瑞士风格、沉静、单色。`5` = 均衡、干净但有个性。`10` = 表现力强、编辑感、大胆排版实验、标题内嵌图片、强烈不对称。默认：`8` |
| **密度** | `4` | `1` = 画廊般通透、大量留白。`5` = 均衡的区域划分。`10` = 驾驶舱般密集、数据密集型。默认：`4` |
| **变化** | `8` | `1` = 可预测的对称栅格。`5` = 微妙偏移。`10` = 艺术化混乱、没有两个区域相同。默认：`8` |
| **动效意图** | `6` | `1` = 静态，无动画。`5` = 微妙的悬停/入场提示。`10` = 每个组件都有电影级编排。默认：`6` |

> **使用方法：** 修改上方数字以匹配你项目的氛围。**创意度 1–3** 时，系统生成干净、安静、Notion 风格的界面。**创意度 7–10** 时，会出现内联图片排版、戏剧化的缩放对比和强烈的编辑感布局。下方规则会根据你选择的级别自动适配。

---

## 1. 视觉主题与氛围
一个克制的、画廊般通透的界面，配合自信的不对称布局和流畅的弹簧物理动效。氛围临床般精确却不失温暖——像一间采光极好的建筑工作室，每个元素都因功能而存在。密度均衡（级别 4），变化度较高（级别 8）以防止对称带来的无聊感，动效流畅但从不戏剧化（级别 6）。整体印象：昂贵、有意图、有生命力。

## 2. 色彩面板与角色
- **Canvas White** (#F9FAFB) — 主背景表面。暖中性，绝非冰冷的蓝白
- **Pure Surface** (#FFFFFF) — 卡片和容器填充色。配合微光阴影使用以表达层级
- **Charcoal Ink** (#18181B) — 主文字色。Zinc-950 深度——永远不用纯黑
- **Steel Secondary** (#71717A) — 正文、描述、元数据。Zinc-500 暖度
- **Muted Slate** (#94A3B8) — 三级文字、时间戳、禁用状态
- **Whisper Border** (rgba(226,232,240,0.5)) — 卡片边框、1px 结构线。半透明以增加深度
- **Diffused Shadow** (rgba(0,0,0,0.05)) — 卡片高度差。宽扩散、40px 模糊、-15px 偏移。绝不生硬

### 强调色选择（每个项目选一种）
- **Emerald Signal** (#10B981) — 增长、成功、正向数据仪表盘
- **Electric Blue** (#3B82F6) — 生产力、SaaS、开发者工具
- **Deep Rose** (#E11D48) — 创意、编辑、时尚相关项目
- **Amber Warmth** (#F59E0B) — 社区、社交、暖色调产品

### 禁用色彩
- 紫色/紫罗兰霓虹渐变——"AI 紫"美学
- 纯黑 (#000000)——始终使用 Off-Black 或 Zinc-950
- 饱和度超过 80% 的过度饱和强调色
- 同一项目中混搭冷暖灰色系统

## 3. 排版规则
- **展示：** `Geist`、`Satoshi`、`Cabinet Grotesk` 或 `Outfit` — 紧凑字距（`-0.025em`），受控流式缩放，字重驱动层级（700–900）。不要声嘶力竭。行高压缩（`1.1`）。强制替代——`Inter` 在高端场景中被禁止
- **正文：** 同字族 400 字重 — 宽松行高（`1.65`），65ch 最大宽度，Steel Secondary 色 (#71717A)
- **等宽：** `Geist Mono` 或 `JetBrains Mono` — 用于代码块、元数据、时间戳。当密度超过级别 7 时，所有数字切换为等宽字体
- **缩放：** 展示 `clamp(2.25rem, 5vw, 3.75rem)`。正文 `1rem/1.125rem`。等宽元数据 `0.8125rem`

### 禁用字体
- `Inter` — 在高端/创意场景中全面禁止
- 通用衬线字体（`Times New Roman`、`Georgia`、`Garamond`、`Palatino`）— 被禁止。如需衬线体用于编辑/创意，仅使用独特现代衬线体如 `Fraunces`、`Gambarino`、`Editorial New` 或 `Instrument Serif`。禁止使用浏览器默认衬线字体栈。无论何种场景，仪表盘或软件 UI 中始终禁止衬线体

## 4. 组件样式
* **按钮：** 平面表面，无外发光。主按钮：强调色填充配白色文字。次按钮：幽灵/描边。按下状态：`-1px translateY` 或 `scale(0.98)` 提供触觉反馈。悬停：微妙的背景色变化，绝不发光
* **卡片/容器：** 大幅圆角（`2.5rem`）。纯白填充。微光边框（`1px`，半透明）。扩散阴影（`0 20px 40px -15px rgba(0,0,0,0.05)`）。内部间距 `2rem–2.5rem`。仅在层级需要高度差时使用——高密度布局用 `border-top` 分隔线或留白替代卡片
* **输入框/表单：** 标签位于输入框上方。辅助文字可选。错误提示在下方，Deep Rose 色。焦点环用强调色，`2px` 偏移。无浮动标签。标签-输入-错误之间标准 `0.5rem` 间距
* **导航：** 简洁、粘性定位。图标悬停时缩放（Dock 放大效果可选）。桌面端不用汉堡菜单。水平排列，间距充裕
* **加载器：** 匹配精确布局尺寸和圆角的骨架屏微光。光线在占位形状上流动。绝不用圆形加载器
* **空状态：** 精心设计的插图或图标组合配引导文字。绝不止"No data found"
* **错误状态：** 行内、上下文相关。红色强调下划线或边框。提供明确的恢复操作

## 5. Hero 区域
Hero 是第一印象——必须引人注目、富有创意，绝不能泛化。
- **内联图片排版：** 在标题文字或字母之间直接嵌入小尺寸的上下文图片。示例："We build [键盘打字照片] digital [屏幕照片] products"——图片以字体高度内联放置，圆角处理，充当文字间的视觉标点。这是标志性创意技法
- **禁止元素重叠：** 文字绝不能重叠图片或其他文字。每个元素有自己清晰的空间区域。禁止 z-index 堆叠内容层，禁止绝对定位标题覆盖图片。始终保持干净分离
- **禁止填充文字：** "向下滚动探索"、"向下滑动"、滚动箭头图标、弹跳箭头及任何指示性 UI 装饰均被禁止。用户知道如何滚动。让内容自然吸引他们
- **不对称结构：** 在此变化级别下，居中 Hero 布局被禁止。使用分屏（50/50）、左对齐文字/右视觉，或不对称留白配大面积空白区域
- **CTA 克制：** 最多一个主 CTA 按钮。不设"了解更多"等次要链接。标题下方不设冗余微文案

## 6. 布局原则
- **栅格优先：** 所有结构布局使用 CSS Grid。禁止 Flexbox 百分比计算（`calc(33% - 1rem)` 被禁止）
- **禁止重叠：** 元素绝不能相互重叠。禁止绝对定位层将内容堆叠在内容之上。每个元素占据自己的栅格单元或流式位置。干净、分离的空间区域
- **功能区域：** "3 个等宽卡片横排"模式被禁止。使用 2 列交错、不对称 Bento 栅格（2fr 1fr 1fr）或横向滚动画廊
- **约束：** 所有内容限制在 `max-width: 1400px`，居中。充裕的水平间距（移动端 `1rem`，平板 `2rem`，桌面端 `4rem`）
- **全高：** 使用 `min-height: 100dvh`——永远不要 `height: 100vh`（iOS Safari 地址栏跳变）
- **Bento 架构：** 功能栅格使用第 1 行：3 列 | 第 2 行：2 列（70/30 分割）。每个区块包含持续微动画

## 7. 响应式规则
每个屏幕必须在所有视口下完美运行。**响应式不是可选项——它是硬性要求。每个元素都必须在 375px、768px 和 1440px 下测试。**
- **移动端优先折叠（< 768px）：** 所有多列布局严格折叠为单列。`width: 100%`，`padding: 1rem`，`gap: 1.5rem`。无例外
- **禁止水平滚动：** 移动端水平溢出是严重失败。所有元素必须适应视口宽度。任何导致水平滚动的元素都是破损设计
- **排版缩放：** 标题通过 `clamp()` 优雅缩小。正文保持 `1rem` 最小值。正文永远不要低于 `14px`。标题在 375px 屏幕上必须保持可读
- **触控目标：** 所有交互元素最小 `44px` 点击区域。可点击项之间间距充裕。移动端按钮必须全宽
- **图片行为：** Hero 和内联图片按比例缩放。内联排版图片（文字间的照片）在移动端堆叠到标题下方而非内联
- **导航：** 桌面端水平导航折叠为简洁的移动端菜单（滑入或全屏覆盖）。不要使用无标签的小汉堡图标
- **卡片与栅格：** Bento 栅格和不对称布局回退为堆叠的单列全宽卡片。保持内部间距（`1rem`）
- **间距一致性：** 移动端垂直区域间距按比例缩减（`clamp(3rem, 8vw, 6rem)`）。不拥挤，不过度通透
- **测试视口：** 设计必须在以下尺寸验证：`375px`（iPhone SE）、`390px`（iPhone 14）、`768px`（iPad）、`1024px`（小笔记本）、`1440px`（桌面端）

## 8. 动效与交互（编码阶段意图）
> **注意：** Stitch 生成静态屏幕——它不会做动画。本章节记录**预期的动效行为**，以便编码智能体（Antigravity、Cursor 等）在将导出设计构建为实际产品时确切知道如何实现动画。

- **物理引擎：** 仅使用弹簧系统。`stiffness: 100, damping: 20`。禁止任何线性缓动。所有交互元素都要有高端、有重量感的体验
- **持续微循环：** 每个活跃仪表盘组件都有无限循环状态——状态点的脉冲、搜索栏的打字机效果、功能图标的浮动、加载状态的微光
- **交错编排：** 列表和栅格以级联延迟挂载（`animation-delay: calc(var(--index) * 100ms)`）。瀑布式展开，绝不瞬间挂载
- **布局过渡：** 通过共享元素 ID 实现平滑重排。项目以物理效果交换位置，模拟实时智能
- **硬件规则：** 仅对 `transform` 和 `opacity` 做动画。永远不要 `top`、`left`、`width`、`height`。颗粒/噪点滤镜仅用于固定、`pointer-events: none` 的伪元素
- **性能：** CPU 密集型持续动画隔离在微观叶子组件中。绝不触发父组件重渲染。目标最低 60fps

## 9. 反模式（禁用项）
- 禁止 emoji——UI、代码或 alt 文本中的任何位置
- 禁止 `Inter` 字体——使用 `Geist`、`Outfit`、`Cabinet Grotesk`、`Satoshi`
- 禁止通用衬线字体（`Times New Roman`、`Georgia`、`Garamond`）——需要衬线体时仅用独特现代衬线体（`Fraunces`、`Instrument Serif`）
- 禁止纯黑（`#000000`）——仅用 Off-Black 或 Zinc-950
- 禁止霓虹外发光或默认 box-shadow 发光
- 禁止饱和度超过 80% 的强调色
- 禁止在大标题上过度使用渐变文字
- 禁止自定义鼠标光标
- 禁止元素重叠——文字绝不能重叠图片或其他内容。始终保持干净的空间分离
- 禁止 3 列等宽卡片功能布局
- 禁止居中 Hero 区域（在此变化级别下）
- 禁止填充性 UI 文字："Scroll to explore"、"Swipe down"、"Discover more below"、滚动箭头、弹跳箭头——全部禁止
- 禁止通用名称："John Doe"、"Sarah Chan"、"Acme"、"Nexus"、"SmartFlow"
- 禁止虚假整数：`99.99%`、`50%`、`1234567`——使用自然数据：`47.2%`、`+1 (312) 847-1928`
- 禁止 AI 文案陈词滥调："Elevate"、"Seamless"、"Unleash"、"Next-Gen"、"Revolutionize"
- 禁止失效的 Unsplash 链接——使用 `picsum.photos/seed/{id}/800/600` 或 SVG UI Avatars
- 禁止通用 `shadcn/ui` 默认样式——自定义圆角、颜色、阴影以匹配此系统
- 禁止 `z-index` 滥用——仅用于 Navbar、Modal、Overlay 层级上下文
- 禁止 `h-screen`——始终使用 `min-h-[100dvh]`
- 禁止圆形加载器——仅用骨架屏微光

# 设计哲学（Design Philosophy）

## 何时使用

当你需要为前端工作提供视觉哲学与艺术指导时使用本技能。用于高概念作品、活动，或当用户提出视觉哲学、宣言式美学或不可错认的艺术风格需求时。

结合 **design** 一起用于高概念作品、活动，或用户提出视觉哲学、宣言式或艺术美学需求时。本技能引导你创建一个有名字的艺术运动并以视觉方式表达它。

## 视觉哲学（在创作"艺术"或高概念作品时）

1. **为这个运动命名**（1–2 个词）：例如"粗野主义之乐"、"静默之色"、"新陈代谢派之梦"。
2. **用 4–6 段话**阐明该哲学如何体现于：空间与形态；色彩与材质；尺度与节奏；构成与平衡；视觉层次。避免重复，每个方面只谈一次。
3. **匠艺精神**：强调作品应当看起来经过精心打磨、反复雕琢，是深度专业能力的产物——"一丝不苟的关注"、"大师级执行"。反复强化这一框架。
4. **极简文字**：信息存在于设计中，而非段落里。文字稀少而必要，作为视觉元素融入整体。
5. **创作空间**：方向要具体，但表达要简洁，让执行者以同等匠心做出高层次的诠释选择。

## 推演隐含的微妙指涉

动手前：从需求中识别一条细微的概念线索。主题是嵌入作品中的微妙、小众指涉——不是字面的，始终是高明的。熟悉主题的人会直觉感受到；其他人则感受到一种强烈的抽象构成。要把它编织进形态、色彩与构成里。"爵士引用"原则：懂的人自然捕捉到，所有人都会从中获益。

## 局限性

- 仅当任务明确匹配上游来源与本地项目场景时使用本技能。
- 在应用变更前，先验证命令、生成的代码、依赖、凭证以及外部服务行为。
- 不要把示例当作环境特定测试、安全审查或对破坏性 / 高成本操作的用户审批的替代品。

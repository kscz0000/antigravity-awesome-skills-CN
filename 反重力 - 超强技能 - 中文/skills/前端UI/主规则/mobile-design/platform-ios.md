# iOS平台指南

> Human Interface Guidelines (HIG) 要点、iOS设计规范、SF Pro排版与原生模式。
> **构建iPhone/iPad应用时阅读本文件。**

---

## 1. Human Interface Guidelines理念

### Apple核心设计原则

```
清晰性：
├── 文字在任何尺寸下都清晰可读
├── 图标精确且明确
├── 装饰微妙而恰当
└── 功能驱动设计

顺从性：
├── UI帮助用户理解和交互
├── 内容充满屏幕
├── UI绝不与内容竞争
└── 半透明暗示更多内容

深度感：
├── 独特的视觉层级传达层次
├── 转场提供深度感
├── 触控揭示功能
└── 内容优先于UI
```

### iOS设计价值观

| 价值观 | 实现方式 |
|--------|----------|
| **美学完整性** | 设计匹配功能（游戏 ≠ 效率工具） |
| **一致性** | 使用系统控件、熟悉的模式 |
| **直接操作** | 触控直接影响内容 |
| **反馈** | 操作得到确认 |
| **隐喻** | 现实世界类比帮助理解 |
| **用户控制** | 用户发起操作，可取消 |

## 2. iOS排版

### SF Pro字体家族

```
iOS系统字体：
├── SF Pro Text：正文（< 20pt）
├── SF Pro Display：大标题（≥ 20pt）
├── SF Pro Rounded：友好场景
├── SF Mono：代码、表格数据
└── SF Compact：Apple Watch、小屏幕
```

### iOS字号体系（Dynamic Type）

| 样式 | 默认字号 | 字重 | 用途 |
|------|----------|------|------|
| **Large Title** | 34pt | Bold | 导航栏（滚动折叠） |
| **Title 1** | 28pt | Bold | 页面标题 |
| **Title 2** | 22pt | Bold | 分区标题 |
| **Title 3** | 20pt | Semibold | 子分区标题 |
| **Headline** | 17pt | Semibold | 强调正文 |
| **Body** | 17pt | Regular | 主要内容 |
| **Callout** | 16pt | Regular | 次要内容 |
| **Subhead** | 15pt | Regular | 第三级内容 |
| **Footnote** | 13pt | Regular | 说明、时间戳 |
| **Caption 1** | 12pt | Regular | 标注 |
| **Caption 2** | 11pt | Regular | 细则 |

### Dynamic Type支持（强制）

```swift
// ❌ 错误：固定字号
Text("Hello")
    .font(.system(size: 17))

// ✅ 正确：Dynamic Type
Text("Hello")
    .font(.body) // 随用户设置缩放

// React Native等价写法
<Text style={{ fontSize: 17 }}> // ❌ 固定
<Text style={styles.body}> // 使用动态缩放体系
```

### 字重使用

| 字重 | iOS常量 | 用途 |
|------|---------|------|
| Regular (400) | `.regular` | 正文 |
| Medium (500) | `.medium` | 按钮、强调 |
| Semibold (600) | `.semibold` | 子标题 |
| Bold (700) | `.bold` | 标题、关键信息 |
| Heavy (800) | `.heavy` | 极少使用，营销 |

## 3. iOS色彩系统

### 系统色（语义色）

```
使用语义色自动适配暗黑模式：

主要文字：
├── .label → 主要文字
├── .secondaryLabel → 次要文字
├── .tertiaryLabel → 第三级文字
├── .quaternaryLabel → 水印

背景：
├── .systemBackground → 主背景
├── .secondarySystemBackground → 分组内容
├── .tertiarySystemBackground → 提升内容

填充：
├── .systemFill → 大形状
├── .secondarySystemFill → 中等形状
├── .tertiarySystemFill → 小形状
├── .quaternarySystemFill → 微妙形状
```

### 系统强调色

| 颜色 | 浅色模式 | 暗黑模式 | 用途 |
|------|----------|----------|------|
| 蓝色 | #007AFF | #0A84FF | 链接、高亮、默认色调 |
| 绿色 | #34C759 | #30D158 | 成功、正面 |
| 红色 | #FF3B30 | #FF453A | 错误、破坏性 |
| 橙色 | #FF9500 | #FF9F0A | 警告 |
| 黄色 | #FFCC00 | #FFD60A | 注意 |
| 紫色 | #AF52DE | #BF5AF2 | 特殊功能 |
| 粉色 | #FF2D55 | #FF375F | 喜爱、收藏 |
| 青色 | #5AC8FA | #64D2FF | 信息 |

### 暗黑模式考量

```
iOS暗黑模式不是浅色的简单反转：

浅色模式：                      暗黑模式：
├── 白色背景                    ├── 纯黑（#000）或近黑
├── 高饱和度                    ├── 降低饱和度
├── 黑色文字                    ├── 白色/浅灰文字
└── 投影                        └── 光晕或无阴影

规则：始终使用语义色实现自动适配。
```

## 4. iOS布局与间距

### 安全区域

```
┌─────────────────────────────────────┐
│░░░░░░░░░░░ 状态栏 ░░░░░░░░░░░░░░░░│ ← 顶部安全区域内边距
├─────────────────────────────────────┤
│                                     │
│                                     │
│         安全内容区域                 │
│                                     │
│                                     │
├─────────────────────────────────────┤
│░░░░░░░░░░ Home指示器 ░░░░░░░░░░░░░░│ ← 底部安全区域内边距
└─────────────────────────────────────┘

规则：绝不在不安全区域放置交互内容。
```

### 标准边距与内边距

| 元素 | 边距 | 备注 |
|------|------|------|
| 屏幕边缘 → 内容 | 16pt | 标准水平边距 |
| 分组表格分区 | 16pt上下 | 留白 |
| 列表项内边距 | 16pt水平 | 标准单元格内边距 |
| 卡片内部内边距 | 16pt | 卡片内内容 |
| 按钮内部内边距 | 12pt垂直，16pt水平 | 最小值 |

### iOS网格系统

```
iPhone网格（标准）：
├── 16pt边距（左/右）
├── 8pt最小间距
├── 内容按8pt倍数

iPhone网格（紧凑）：
├── 8pt边距（需要时）
├── 4pt最小间距

iPad网格：
├── 20pt边距（或更多）
├── 考虑多栏布局
```

## 5. iOS导航模式

### 导航类型

| 模式 | 用途 | 实现 |
|------|------|------|
| **Tab栏** | 3-5个顶级分区 | 底部，始终可见 |
| **导航控制器** | 层级下钻 | 基于栈，返回按钮 |
| **模态** | 聚焦任务、中断 | Sheet或全屏 |
| **侧边栏** | iPad、多栏 | 左侧边栏（iPad） |

### Tab栏指南

```
┌─────────────────────────────────────┐
│                                     │
│         内容区域                     │
│                                     │
├─────────────────────────────────────┤
│  🏠     🔍     ➕     ❤️     👤    │ ← Tab栏（49pt高度）
│ 首页   搜索   新建   收藏   个人     │
└─────────────────────────────────────┘

规则：
├── 最多5项
├── 图标：SF Symbols或自定义（25×25pt）
├── 标签：始终包含（无障碍）
├── 选中态：填充图标 + 色调
└── Tab栏始终可见（不在滚动时隐藏）
```

### 导航栏指南

```
┌─────────────────────────────────────┐
│ < 返回     页面标题         编辑    │ ← 导航栏（44pt）
├─────────────────────────────────────┤
│                                     │
│         内容区域                     │
│                                     │
└─────────────────────────────────────┘

规则：
├── 返回按钮：系统箭头 + 上页标题（或"返回"）
├── 标题：居中，动态字体
├── 右侧操作：最多2项
├── 大标题：滚动时折叠（可选）
└── 优先使用文字按钮而非图标（清晰度）
```

### 模态展示

| 样式 | 用途 | 外观 |
|------|------|------|
| **Sheet（默认）** | 次要任务 | 卡片上滑，父页面可见 |
| **全屏** | 沉浸式任务 | 覆盖整个屏幕 |
| **弹出框** | iPad、快速信息 | 箭头指向的气泡 |
| **警告** | 关键中断 | 居中对话框 |
| **操作表** | 上下文选择 | 底部选项Sheet |

### 手势

| 手势 | iOS惯例 |
|------|---------|
| **左边缘滑动** | 返回导航 |
| **下拉（Sheet）** | 关闭模态 |
| **长按** | 上下文菜单 |
| **深按** | 预览/弹出（旧版） |
| **双指滑动** | 嵌套滚动中滚动 |

## 6. iOS组件

### 按钮

```
按钮样式（UIKit/SwiftUI）：

┌──────────────────────────────┐
│         着色按钮              │ ← 主要操作（填充）
├──────────────────────────────┤
│         描边按钮              │ ← 次要操作（描边）
├──────────────────────────────┤
│         纯文字按钮            │ ← 第三级操作（仅文字）
└──────────────────────────────┘

尺寸：
├── 迷你：紧凑空间
├── 小型：紧凑UI
├── 中型：行内操作
├── 大型：主要CTA（44pt最小高度）
```

### 列表与表格

```
列表样式：

.plain         → 无分隔线，边到边
.insetGrouped  → 圆角卡片（iOS 14+默认）
.grouped       → 全宽分区
.sidebar       → iPad侧边栏导航

单元格附件：
├── 展开指示器（>）→ 导航到详情
├── 详情按钮（i）→ 显示信息但不导航
├── 勾选标记（✓）→ 选择
├── 排序（≡）→ 拖拽排序
└── 删除（-）→ 滑动/编辑模式删除
```

### 文本输入框

```
iOS文本输入框结构：

┌─────────────────────────────────────┐
│ 🔍 搜索...                      ✕  │
└─────────────────────────────────────┘
  ↑                                   ↑
  前置图标                           清除按钮

边框：圆角矩形
高度：36pt最小
占位符：次要文字色
清除按钮：有文字时出现
```

### 分段控件

```
何时使用：
├── 2-5个相关选项
├── 筛选内容
├── 切换视图

┌───────┬───────┬───────┐
│  全部  │ 进行中 │ 已完成 │
└───────┴───────┴───────┘

规则：
├── 等宽分段
├── 文字或图标（不混合使用）
├── 最多5段
└── 更复杂时考虑Tab
```

## 7. iOS专属模式

### 下拉刷新

```
原生UIRefreshControl行为：
├── 下拉超过阈值 → 旋转器出现
├── 释放 → 触发刷新操作
├── 加载状态 → 旋转器旋转
├── 完成 → 旋转器消失

规则：始终使用原生UIRefreshControl（不要自定义构建）。
```

### 滑动操作

```
iOS滑动操作：

← 向左滑动（破坏性）      向右滑动（建设性） →
┌─────────────────────────────────────────────────────────────┐
│                    列表项内容                                │
└─────────────────────────────────────────────────────────────┘

左滑显示：归档、删除、标记
右滑显示：置顶、星标、标为已读

完整滑动：触发第一个操作
```

### 上下文菜单

```
长按 → 上下文菜单出现

┌─────────────────────────────┐
│       预览卡片               │
├─────────────────────────────┤
│  📋 复制                    │
│  📤 分享                    │
│  ➕ 添加到...               │
├─────────────────────────────┤
│  🗑️ 删除           （红色）  │
└─────────────────────────────┘

规则：
├── 预览：显示放大内容
├── 操作：与内容相关
├── 破坏性：放最后，用红色
└── 最多约8个操作（更多时可滚动）
```

### Sheet与半屏Sheet

```
iOS 15+ Sheet：

┌─────────────────────────────────────┐
│                                     │
│        父视图（变暗）                │
│                                     │
├─────────────────────────────────────┤
│  ═══  （拖拽手柄）                   │ ← 拖拽调整大小
│                                     │
│        Sheet内容                    │
│                                     │
│                                     │
└─────────────────────────────────────┘

Detents：
├── .medium → 半屏
├── .large → 全屏（含安全区域）
├── 自定义 → 特定高度
```

## 8. SF Symbols

### 使用指南

```
SF Symbols：Apple图标库（5000+图标）

字重：匹配文字字重
├── Ultralight / Thin / Light
├── Regular / Medium / Semibold
├── Bold / Heavy / Black

缩放：
├── .small → 与小文字行内
├── .medium → 标准UI
├── .large → 强调、独立使用
```

### Symbol配置

```swift
// SwiftUI
Image(systemName: "star.fill")
    .font(.title2)
    .foregroundStyle(.yellow)

// 渲染模式
Image(systemName: "heart.fill")
    .symbolRenderingMode(.multicolor)

// 动画（iOS 17+）
Image(systemName: "checkmark.circle")
    .symbolEffect(.bounce)
```

### Symbol最佳实践

| 指南 | 实现 |
|------|------|
| 匹配文字字重 | Symbol字重 = 字体字重 |
| 使用标准Symbol | 用户能认出 |
| 有意义时用多色 | 不是装饰 |
| 旧版iOS降级 | 检查可用性 |

## 9. iOS无障碍

### VoiceOver要求

```
每个交互元素需要：
├── 无障碍标签（它是什么）
├── 无障碍提示（它做什么）- 可选
├── 无障碍特征（button、link等）
└── 无障碍值（当前状态）

SwiftUI：
.accessibilityLabel("播放")
.accessibilityHint("播放选中的曲目")

React Native：
accessibilityLabel="播放"
accessibilityHint="播放选中的曲目"
accessibilityRole="button"
```

### Dynamic Type缩放

```
强制：支持Dynamic Type

用户可设置字号：
├── xSmall → 14pt正文
├── Small → 15pt正文
├── Medium → 16pt正文
├── Large（默认）→ 17pt正文
├── xLarge → 19pt正文
├── xxLarge → 21pt正文
├── xxxLarge → 23pt正文
├── 无障碍尺寸 → 最高53pt

你的应用必须在所有尺寸下优雅缩放。
```

### 减少动效

```
尊重动效偏好：

@Environment(\.accessibilityReduceMotion) var reduceMotion

if reduceMotion {
    // 使用即时转场
} else {
    // 使用动画
}

React Native：
import { AccessibilityInfo } from 'react-native';
AccessibilityInfo.isReduceMotionEnabled()
```

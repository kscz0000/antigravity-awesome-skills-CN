# 移动端色彩系统参考

> OLED优化、暗黑模式、省电色彩与户外可读性。
> **移动端色彩不只是美观——它关乎电量和可用性。**

---

## 1. 移动端色彩基础

### 为什么移动端色彩不同

```
桌面端：                          移动端：
├── LCD屏幕（背光）              ├── OLED普及（自发光）
├── 受控光照                      ├── 户外、强光
├── 稳定供电                      ├── 电池有限
├── 个人偏好                      ├── 系统级暗黑模式
└── 静态观看                      └── 多角度、运动中观看
```

### 移动端色彩优先级

| 优先级 | 原因 |
|--------|------|
| **1. 可读性** | 户外、光线多变 |
| **2. 省电** | OLED下暗黑模式省电 |
| **3. 系统集成** | 支持暗黑/浅色模式 |
| **4. 语义** | 错误、成功、警告色 |
| **5. 品牌** | 功能需求之后 |

## 2. OLED考量

### OLED与LCD的区别

```
LCD（液晶显示）：
├── 背光始终开启
├── 黑色 = 背光透过暗色滤镜
├── 能耗 = 恒定
└── 暗黑模式 = 不省电

OLED（有机LED）：
├── 每个像素自发光
├── 黑色 = 像素关闭（零功耗）
├── 能耗 = 越亮越耗电
└── 暗黑模式 = 显著省电
```

### OLED省电效果

```
色彩能耗（相对值）：

#000000（纯黑）  ████░░░░░░  0%
#1A1A1A（近黑）  █████░░░░░  ~15%
#333333（深灰）  ██████░░░░  ~30%
#666666（中灰）  ███████░░░  ~50%
#FFFFFF（白色）  ██████████  100%

高饱和色也显著耗电：
├── 蓝色像素：效率最高
├── 绿色像素：中等
├── 红色像素：效率最低
└── 低饱和色更省电
```

### 纯黑 vs 近黑

```
#000000（纯黑）：
├── 最大省电
├── 滚动时可能出现"黑拖影"
├── 对比强烈（可能刺眼）
└── Apple纯暗黑模式使用

#121212 或 #1A1A1A（近黑）：
├── 仍然省电
├── 滚动更流畅（无拖影）
├── 眼睛稍舒适
└── Material Design推荐

推荐：背景用 #000000，表面用 #0D0D0D-#1A1A1A
```

## 3. 暗黑模式设计

### 暗黑模式的好处

```
用户启用暗黑模式的原因：
├── 省电（OLED）
├── 减少眼疲劳（暗光环境）
├── 个人偏好
├── AMOLED美学
└── 无障碍（光敏感）
```

### 暗黑模式色彩策略

```
浅色模式                      暗黑模式
──────────                    ─────────
背景: #FFFFFF      →          #000000 或 #121212
表面: #F5F5F5      →          #1E1E1E
表面2: #EEEEEE      →          #2C2C2C

主色: #1976D2      →          #90CAF9（更浅）
文字: #212121      →          #E0E0E0（非纯白）
次要: #757575      →          #9E9E9E

暗黑模式中的层级：
├── 层级越高 = 表面越浅
├── 0dp →  0%叠加
├── 4dp →  9%叠加
├── 8dp →  12%叠加
└── 不用阴影，用叠加创造深度
```

### 暗黑模式文字色

| 角色 | 浅色模式 | 暗黑模式 |
|------|----------|----------|
| 主要 | #000000（黑） | #E8E8E8（非纯白） |
| 次要 | #666666 | #B0B0B0 |
| 禁用 | #9E9E9E | #6E6E6E |
| 链接 | #1976D2 | #8AB4F8 |

### 色彩反转规则

```
不要简单反转颜色：
├── 高饱和色变得刺眼
├── 语义色失去含义
├── 品牌色可能崩坏
└── 对比度不可预测

要创建有意图的暗黑色板：
├── 降低主色饱和度
├── 用更浅的色调做强调
├── 保持语义色含义不变
├── 独立检查对比度
```

## 4. 户外可读性

### 阳光问题

```
户外屏幕可见性：
├── 强光冲淡低对比内容
├── 眩光降低可读性
├── 偏光墨镜有影响
└── 用户用手遮挡屏幕

受影响元素：
├── 浅灰文字在白底上
├── 微妙的色彩差异
├── 低透明度叠加层
└── 粉色系
```

### 高对比度策略

```
户外可读性：

最低对比度：
├── 正文：4.5:1（WCAG AA）
├── 大字：3:1（WCAG AA）
├── 推荐：7:1+（AAA）

避免：
├── #999 在 #FFF 上（不达标）
├── #BBB 在 #FFF 上（不达标）
├── 浅色背景上的淡色
└── 关键信息用微妙渐变

应当：
├── 使用系统语义色
├── 在明亮环境中测试
├── 提供高对比模式
└── 关键UI用纯色
```

## 5. 语义色

### 含义一致性

| 语义 | 含义 | iOS默认 | Android默认 |
|------|------|---------|-------------|
| 错误 | 问题、破坏 | #FF3B30 | #B3261E |
| 成功 | 完成、正面 | #34C759 | #4CAF50 |
| 警告 | 注意、谨慎 | #FF9500 | #FFC107 |
| 信息 | 提示 | #007AFF | #2196F3 |

### 语义色规则

```
绝不将语义色用于：
├── 品牌标识（混淆含义）
├── 装饰（削弱冲击力）
├── 随意样式
└── 状态指示（还需配图标）

始终：
├── 配合图标（色盲用户）
├── 暗黑/浅色模式均保持
├── 全应用一致
└── 遵循平台惯例
```

### 错误状态色

```
错误状态需要：
├── 偏红色（语义）
├── 与背景高对比
├── 图标强化
├── 清晰的文字说明

iOS：
├── 浅色：#FF3B30
├── 暗色：#FF453A

Android：
├── 浅色：#B3261E
├── 暗色：#F2B8B5（错误容器上）
```

## 6. 动态色彩（Android）

### Material You

```
Android 12+ 动态色彩：

用户壁纸 → 色彩提取 → 应用主题

你的应用自动获得：
├── 主色（来自壁纸主色调）
├── 辅色（互补色）
├── 第三色（强调色）
├── 表面色（中性色，派生）
├── On色（各色上的文字色）
```

### 支持动态色彩

```kotlin
// Jetpack Compose
MaterialTheme(
    colorScheme = dynamicColorScheme()
        ?: staticColorScheme() // 旧版Android降级
)

// React Native
// 支持有限 - 考虑 react-native-material-you
```

### 降级色板

```
动态色彩不可用时：
├── Android < 12
├── 用户已禁用
├── 不支持的启动器

提供静态色板：
├── 定义品牌色
├── 两种模式都测试
├── 匹配动态色彩角色
└── 支持浅色 + 暗黑
```

## 7. 色彩无障碍

### 色盲考量

```
约8%男性、0.5%女性存在色盲

类型：
├── 红色弱（Protanopia）
├── 绿色弱（Deuteranopia）
├── 蓝色弱（Tritanopia）
├── 全色盲（罕见，无色彩感知）

设计规则：
├── 绝不仅依赖颜色传达信息
├── 使用图案、图标、文字
├── 用模拟工具测试
├── 避免仅用红/绿区分
```

### 对比度测试工具

```
验证工具：
├── 内置无障碍检查器（Xcode）
├── Accessibility Scanner（Android）
├── 对比度计算器
├── 色盲模拟
└── 在真实设备上阳光下测试
```

### 足够的对比度

```
WCAG指南：

AA（最低）
├── 正文：4.5:1
├── 大字（18pt+）：3:1
├── UI组件：3:1

AAA（增强）
├── 正文：7:1
├── 大字：4.5:1

移动端建议：达到AA，争取AAA
```

## 8. 色彩反模式

### ❌ 常见错误

| 错误 | 问题 | 修正 |
|------|------|------|
| **浅灰在白底上** | 户外不可见 | 最低4.5:1对比度 |
| **暗黑模式用纯白** | 刺眼 | 用 #E0E0E0-#F0F0F0 |
| **暗黑模式同饱和度** | 刺目发光 | 降低饱和度 |
| **仅红/绿指示** | 色盲用户看不到 | 加图标 |
| **语义色做品牌色** | 含义混淆 | 品牌用中性色 |
| **忽略系统暗黑模式** | 体验突兀 | 支持两种模式 |

### ❌ AI色彩错误

```
AI倾向于：
├── 浅色/暗黑用同一套色
├── 忽略OLED省电影响
├── 跳过对比度计算
├── 默认紫色/紫罗兰（禁止）
├── 用低对比"美学"灰
├── 不在户外条件测试
└── 忘记色盲用户

规则：按最差情况设计。
在强光下测试，用色盲模拟测试。
```

## 9. 色彩系统检查清单

### 选色前

- [ ] 浅色和暗黑模式变体已定义？
- [ ] 对比度已检查（4.5:1+）？
- [ ] OLED省电已考虑（暗黑模式）？
- [ ] 语义色遵循惯例？
- [ ] 色盲安全（非仅颜色指示）？

### 发布前

- [ ] 在强光下测试过？
- [ ] 在OLED设备上测试过暗黑模式？
- [ ] 系统暗黑模式已适配？
- [ ] 动态色彩已支持（Android）？
- [ ] 错误/成功/警告色一致？
- [ ] 所有文字满足对比度要求？

## 10. 速查参考

### 暗黑模式背景色

```
纯黑（OLED最大省电）：#000000
近黑（Material）：     #121212
表面1：                #1E1E1E
表面2：                #2C2C2C
表面3：                #3C3C3C
```

### 暗黑模式文字色

```
主要：   #E0E0E0 到 #ECECEC
次要：   #A0A0A0 到 #B0B0B0
禁用：   #606060 到 #707070
```

### 对比度

```
小字：  4.5:1（最低）
大字：  3:1（最低）
UI元素：3:1（最低）
理想：  7:1（AAA）
```

---

> **切记：** 移动端色彩必须在最差条件下可用——强光、疲劳的眼睛、色盲、低电量。通不过这些测试的漂亮颜色就是无用的颜色。

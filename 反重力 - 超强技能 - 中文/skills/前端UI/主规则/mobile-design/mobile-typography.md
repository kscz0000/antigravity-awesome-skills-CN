# 移动端排版参考

> 字号体系、系统字体、Dynamic Type、无障碍与暗黑模式排版。
> **排版失败是移动端应用不可读的头号原因。**

---

## 1. 移动端排版基础

### 为什么移动端排版不同

```
桌面端：                          移动端：
├── 20-30英寸观看距离            ├── 12-15英寸观看距离
├── 大视口                        ├── 小视口，窄
├── 悬停查看详情                  ├── 点击/滑动查看详情
├── 受控光照                      ├── 多变（户外等）
├── 固定字号                      ├── 用户可控字号
└── 长时间阅读                    └── 快速扫读
```

### 移动端排版规则

| 规则 | 桌面端 | 移动端 |
|------|--------|--------|
| **最小正文字号** | 14px | 16px（14pt/14sp） |
| **最大行宽** | 75字符 | 40-60字符 |
| **行高** | 1.4-1.5 | 1.4-1.6（更宽松） |
| **字重** | 多样 | Regular为主，Bold少用 |
| **对比度** | AA（4.5:1） | AA最低，AAA推荐 |

## 2. 系统字体

### iOS：SF Pro系列

```
San Francisco (SF) 系列：
├── SF Pro Display：大字号（≥ 20pt）
├── SF Pro Text：正文（< 20pt）
├── SF Pro Rounded：友好场景
├── SF Mono：等宽
└── SF Compact：Apple Watch、紧凑UI

特性：
├── 光学尺寸（自动调整）
├── 动态字距（间距）
├── 等宽/比例数字
├── 优秀的可读性
```

### Android：Roboto系列

```
Roboto系列：
├── Roboto：默认无衬线
├── Roboto Flex：可变字体
├── Roboto Serif：衬线选项
├── Roboto Mono：等宽
├── Roboto Condensed：窄空间

特性：
├── 屏幕优化
├── 广泛语言支持
├── 多字重
├── 小字号表现好
```

### 何时使用系统字体

```
✅ 使用系统字体的情况：
├── 品牌不要求自定义字体
├── 阅读效率优先
├── 应用原生/集成感重要
├── 性能关键
├── 需要广泛语言支持

❌ 避免系统字体的情况：
├── 品牌标识需要自定义
├── 设计差异化需要
├── 编辑/杂志风格
└── （但仍需支持无障碍）
```

### 自定义字体注意事项

```
使用自定义字体时：
├── 包含所有需要的字重
├── 子集化减小文件体积
├── 在所有Dynamic Type尺寸下测试
├── 提供系统字体降级方案
├── 测试渲染质量
└── 检查语言支持
```

## 4. Dynamic Type / 文字缩放

### iOS Dynamic Type（强制）

```swift
// ❌ 错误：固定尺寸（不缩放）
Text("Hello")
    .font(.system(size: 17))

// ✅ 正确：Dynamic Type
Text("Hello")
    .font(.body) // 随用户设置缩放

// 自定义字体带缩放
Text("Hello")
    .font(.custom("MyFont", size: 17, relativeTo: .body))
```

### Android文字缩放（强制）

```
文字始终使用sp：
├── sp = 缩放无关像素
├── 随用户字体偏好缩放
├── dp不会缩放（不要用于文字）

用户可缩放85%到200%：
├── 默认（100%）：14sp = 14dp
├── 最大（200%）：14sp = 28dp

务必在200%下测试！
```

### 缩放挑战

```
大字号下的问题：
├── 文字溢出容器
├── 按钮变得太高
├── 图标相对文字显得小
├── 布局崩坏

解决方案：
├── 使用弹性容器（非固定高度）
├── 允许文字换行
├── 图标随文字缩放
├── 开发期间测试极端值
├── 长文本使用可滚动容器
```

## 5. 排版无障碍

### 最小字号

| 元素 | 最低 | 推荐 |
|------|------|------|
| 正文 | 14px/pt/sp | 16px/pt/sp |
| 次要文字 | 12px/pt/sp | 13-14px/pt/sp |
| 说明文字 | 11px/pt/sp | 12px/pt/sp |
| 按钮 | 14px/pt/sp | 14-16px/pt/sp |
| **再小就不行** | 11px | - |

### 对比度要求（WCAG）

```
正常文字（< 18pt 或 < 14pt加粗）：
├── AA：4.5:1 最低
├── AAA：7:1 推荐

大号文字（≥ 18pt 或 ≥ 14pt加粗）：
├── AA：3:1 最低
├── AAA：4.5:1 推荐

Logo/装饰性：无要求
```

### 行高无障碍

```
WCAG成功标准 1.4.12：

行高（行间距）：≥ 1.5倍
段间距：≥ 2倍字号
字间距：≥ 0.12倍字号
词间距：≥ 0.16倍字号

移动端推荐：
├── 正文：1.4-1.6 行高
├── 标题：1.2-1.3 行高
├── 绝不低于1.2
```

## 6. 暗黑模式排版

### 颜色调整

```
浅色模式：                      暗黑模式：
├── 黑色文字（#000）            ├── 白色/浅灰（#E0E0E0）
├── 高对比度                    ├── 略微降低对比度
├── 全饱和度                    ├── 降低饱和度
└── 深色 = 强调                 └── 浅色 = 强调

规则：暗黑模式不用纯白（#FFF）。
用偏白（#E0E0E0 到 #F0F0F0）减少眼疲劳。
```

### 暗黑模式层级

| 层级 | 浅色模式 | 暗黑模式 |
|------|----------|----------|
| 主要文字 | #000000 | #E8E8E8 |
| 次要文字 | #666666 | #A0A0A0 |
| 第三文字 | #999999 | #707070 |
| 禁用文字 | #CCCCCC | #505050 |

### 暗黑模式字重

```
暗黑模式文字因光晕效应显得更细
（光线渗入深色背景）

考虑：
├── 正文用medium字重（替代regular）
├── 略微增加字间距
├── 在真实OLED屏幕上测试
└── 比浅色模式用略粗字重
```

## 7. 排版反模式

### ❌ 常见错误

| 错误 | 问题 | 修正 |
|------|------|------|
| **固定字号** | 忽视无障碍 | 使用动态缩放 |
| **文字太小** | 不可读 | 最低14pt/sp |
| **低对比度** | 阳光下不可见 | 最低4.5:1 |
| **行太长** | 难以追踪 | 最多60字符 |
| **行高太紧** | 拥挤，难读 | 最低1.4倍 |
| **字号太多** | 视觉混乱 | 最多5-7种 |
| **正文全大写** | 难以阅读 | 仅标题用 |
| **浅灰在白底上** | 强光下不可能看见 | 提高对比度 |

### ❌ AI排版错误

```
AI倾向于：
├── 用固定px值而非pt/sp
├── 跳过Dynamic Type支持
├── 用太小的文字（12-14px正文）
├── 忽略行高设置
├── 用低对比"美学"灰
├── 把桌面端字号体系直接搬到移动端
└── 跳过大字号测试

规则：排版必须能缩放。
在最小和最大设置下测试。
```

## 8. 字体加载与性能

### 字体文件优化

```
移动端字体文件体积很重要：
├── 完整字体：100-300KB每字重
├── 子集化（Latin）：15-40KB每字重
├── 可变字体：100-200KB（所有字重）

建议：
├── 子集化到所需字符
├── 使用WOFF2格式
├── 最多2-3个字体文件
├── 考虑可变字体
├── 适当缓存字体
```

### 加载策略

```
1. 系统字体降级
   先显示系统字体 → 自定义字体加载后替换
   
2. 字体显示交换
   font-display: swap（CSS）
   
3. 预加载关键字体
   预加载首屏需要的字体
   
4. 不阻塞渲染
   不要等字体加载完才显示内容
```

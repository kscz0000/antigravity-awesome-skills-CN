# 审计输出模板

## 用途

在报告 SwiftUI 性能发现时使用此结构，让用户能快速看到症状、证据、可能原因和下一步验证动作。

## 模板

```markdown
## 摘要

[一段简短文字，说明最可能的瓶颈以及结论是基于代码还是 trace 得出的。]

## 发现

1. [问题标题]
   - 症状：[用户看到的表现]
   - 可能原因：[根因]
   - 证据：[代码引用或性能分析证据]
   - 修复：[具体变更]
   - 验证：[修复后应测量什么]

2. [问题标题]
   - 症状：...
   - 可能原因：...
   - 证据：...
   - 修复：...
   - 验证：...

## 指标

| 指标 | 变更前 | 变更后 | 备注 |
| --- | --- | --- | --- |
| CPU | [值] | [值] | [备注] |
| 掉帧/卡顿 | [值] | [值] | [备注] |
| 内存峰值 | [值] | [值] | [备注] |

## 下一步

[一个具体的下一步动作：应用修复、采集更好的 trace 或在真机上验证。]
```

## 说明

- 按影响程度而非文件顺序排列发现。
- 明确说明哪些结论仍是假设。
- 如无可用指标，省略表格并说明下一步应测量什么。

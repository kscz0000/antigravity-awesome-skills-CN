# 理解应用中的卡顿（摘要）

背景：Apple 指南，介绍如何识别由长时间运行的主线程工作引起的卡顿，以及理解主运行循环。

## 核心概念

- 卡顿是离散交互中的明显延迟（通常 >100 ms）。
- 卡顿几乎总是来自主线程上的长时间运行工作。
- 主运行循环按顺序处理 UI 事件、计时器和主队列工作。

## 主线程工作阶段

- 事件分发到正确的视图/处理器。
- 你的代码：状态更新、数据获取、UI 变更。
- Core Animation 提交到渲染服务器。

## 为什么主运行循环很重要

- 只有主线程能安全地更新 UI。
- 运行循环是执行主队列工作的基础。
- 如果运行循环繁忙，它无法处理新事件；这导致卡顿。

## 诊断卡顿

- 观察主运行循环的繁忙时段：健康的循环大部分时间处于休眠状态。
- 卡顿检测通常标记 >250 ms 的繁忙时段。
- Hangs instrument 可以配置更低的阈值。

## 实践要点

- 保持主线程工作简短；将重工作从事件处理器中卸载。
- 避免在主 dispatch 队列或 MainActor 上执行长时间运行的任务。
- 将运行循环行为作为用户感知响应性的代理指标。

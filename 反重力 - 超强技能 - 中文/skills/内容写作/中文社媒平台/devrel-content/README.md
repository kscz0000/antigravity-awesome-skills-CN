# DevRel 内容

**创作开发者真正愿意阅读的技术内容。**

## 它能做什么

引导你创建能与开发者产生共鸣的博客文章、教程和文档。覆盖研究、结构、代码示例、技术准确性与 SEO。

## 适用场景

- 撰写技术博客文章
- 创建文档或指南
- 打造思想领导类内容
- 当你说"写一篇博客文章"、"技术文章"、"开发者内容"或"教程"

## 快速上手

```
/devrel-content

写一篇关于在 Node.js 中实现限流的博客文章。
目标读者：构建 API 的后端开发者。
```

本技能会：

1. 加载你的开发者受众上下文
2. 帮你研究并验证主题
3. 引导你完成提纲与结构
4. 确保代码示例可直接复制粘贴运行
5. 检查技术准确性与 SEO

## 核心框架

### 内容质量信号

- **展示而非讲述** — 用代码代替大段文字
- **回应"为什么"** — 不仅是怎么做，还包括何时做、为何做
- **承认取舍** — 开发者尊重诚实
- **渐进式披露** — 从简单入手，逐步增加复杂度

### 代码示例要求

每个代码片段都必须满足：

- 可直接运行，无需修改
- 包含全部 import
- 展示预期输出
- 妥善处理错误
- 使用真实值（不要用 `foo`、`bar`）

### 面向开发者的 SEO

开发者的搜索方式与众不同：

- 错误信息（"TypeError: Cannot read property..."）
- How-to 查询（"how to deploy next.js"）
- 对比（"prisma vs typeorm 2024"）
- 最佳实践（"typescript project structure"）

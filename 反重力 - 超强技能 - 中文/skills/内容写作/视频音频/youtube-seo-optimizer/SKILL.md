# YouTube 与播客 SEO 优化器

## 适用场景
- 用户要为新上传的内容生成标题/描述/标签/话题标签套件
- 用户要审计流量不佳的现有视频或播客单集
- 用户要求节目笔记、时间戳或播客单集元数据
- 用户说"优化我的视频"、"审计我的 YouTube 视频"、"写一段播客描述"、"为什么我的视频没排名"、"生成标签"、"修复我的播客 SEO"
- 适用于任何视频或播客 SEO 请求——新发布、已发布或短视频

## 概述

本技能依赖网页搜索与网页抓取工具。无论宿主如何命名它们——`web_search`/`web_fetch`、`WebSearch`/`WebFetch`、某个 MCP 工具——都用宿主提供的那个。

本技能覆盖 6 种场景：3 类内容（视频、播客、短视频）× 2 种状态（新发布或表现不佳）。每种场景对应下方一种模式。

两条铁律：

1. **每个关键词都要调研，不能凭空生成。** 绝不凭记忆生成标签或所谓的"热门词"。先问创作者希望排什么词，再去搜索。
2. **匹配用户的请求。** 用户只要标题，就只给标题。用户要完整套件，就交付每一个编号小节。别一股脑全塞过去。

## 如何执行本技能

下方小节按以下顺序组织：**步骤 → 规则 → 模板 → 自检**。请按顺序执行：

1. **阅读用户请求**——先做范围核查（见下文）
2. **步骤 0-2**——分类、找出关键词、做调研、收集缺失信息
3. **从下方模板中挑选模式**（A-F）
4. **按规则**（标题规则、标签策略等）**构建用户要的东西**
5. **发送前运行质量自检清单**

---

## 步骤 0 — 先核范围再分类

**先核范围。** 本技能一次只处理一个视频、播客单集或短视频切片——不是整个频道或播放列表。如果用户提出超出范围的需求，告诉他们：*"我只能优化单个视频或单集。你想先从哪一个开始？"* 用户选定后，从步骤 0 重新启动流程，针对那个具体条目。

**然后分类。** 先读一遍用户的消息——他们很可能已经把关键信息说全了。不要问他们刚说过的事。

**内容类型** ——如果用户没明说，就只问一次：*"这是一个普通视频、播客单集，还是 Short/Reel？"*

**状态** ——是否给了 URL？直接抓取。是否说了"没流量"或"不排名"？属于已发布。是否说"准备上传"或"即将发布"？属于新发布。拿不准就只问一次。

| 内容类型 | 新发布 | 已发布 |
|---|---|---|
| 单条视频 | 模式 A | 模式 B |
| 播客单集 | 模式 C | 模式 D |
| 短视频切片 / Reel | 模式 E | 模式 F |

如果用户给了 URL，直接抓取实时元数据——不要让他们重复已经在页面上的信息。

---

## 步骤 1 — 锁定目标关键词

看看用户是否已经指明。若没有，从主题/大纲中提取并给出建议。只有在实在推不出来时才询问。

如果用户只要标题或只要标签，可以省去完整调研批——但仍要快速做一次搜索，验证关键词角度。否则按以下完整调研执行：
- `[目标关键词]` — 看现有排名
- `[目标关键词] [当前年份]`
- `[细分领域/主题] 趋势` 或 `[目标关键词] reddit` — 看看真实用户用的措辞
- 若是播客：再额外搜一下嘉宾姓名

提取 3-6 个相关词组作为次要/长尾关键词。

**没有数据就别编：** 绝不编造搜索量、播放量或算法说法。调研结果稀薄也行——意味着竞争更小。

**所有"[年份]"位置用今天的真实日期。**

**核验最高级说法。** 若用户提到"十大"、"第一名"、"增长最快"等——搜证。无法证实就删掉或标 `[待核实：...]`。

---

## 步骤 2 — 先抓取后询问，补齐仍缺的信息

写"资源/CTA"小节前，先看用户已经给了什么，或者能通过 URL/姓名/频道抓到什么。仍未知的内容，去查一下。实在查不到再问用户：

- 链接：官网、社交媒体、邮件订阅、联盟/产品页
- CTA 目标：订阅、访问、加入、购买
- 优惠、引流资料、折扣码或赞助
- 播客：嘉宾姓名、简介、链接；单集编号；赞助详情；平台链接（Spotify、Apple Podcasts 等）

一次只问一个问题。每个回答回来后，先看看能否补齐其余信息。如果用户说"用占位符"，就用 `[待补：...]` 标记——绝不要编造 URL。

## 标题规则

适用于所有需要标题的模式（A-F）。

### 公式
```
[主要关键词] : [收益或好处] + [强效词 / 数字 / 年份]
```

### 强效词库
如何 · 为什么 · 什么 · 最佳 · 完整 · 真实 · 免费 · 最新 · 一步步 · 全流程 · 验证有效 · 终极 · 揭秘 · 零基础 · 在 [X] 天内 · [数字] 种方法 · [年份]

### 规则
- 严格控制在 60-70 字符——逐字计数
- 主要关键词尽量出现在前 4-5 词
- 每个标题带一个情绪钩子
- 只有步骤 1 调研显示年份戳常见时才加年份
- 除一词强调外避免全大写
- 不能做虚假承诺
- A/B 备选标题必须用真正不同的钩子，不是同义词调序

### 各内容类型的标题模板
| 类型 | 模板 | 示例 |
|---|---|---|
| 教程 | 如何 [达成结果] 用 [时间/步骤] | 如何 30 天冲到 YouTube 榜首 |
| 清单 | [N] 件 [受众] 必看的 [对象] | 2026 年创作者必看的 7 件 SEO 工具 |
| 故事 | [主体] 如何 [达成结果] | 一位农民如何打造尼泊尔首款农业科技应用 |
| 提问 | [尖锐问题]？（完整解答） | 为什么你的 YouTube 视频没人看（已修复） |
| 地域 | [地点] 的 [主题]：[成果] | 尼泊尔农业科技：农户收入翻三倍 |
| 对比 | [A] vs [B]：[哪项结果]？ | YouTube SEO vs Google SEO：到底哪个有效 |
| 播客 | [嘉宾] 谈 [主题]：[成果] \| [节目] 第 [N] 集 | Sara Lin 谈真正有效的冷外联 \| 增长实验室 #42 |

## 标签策略

用于长视频模式（A-D）的标签生成。共 14-19 个标签，按下表配比。短视频（E-F）见"Shorts 适配"小节的 5-8 标签规则。

| 类型 | 数量 | 规则 |
|---|---|---|
| 精确匹配主要关键词 | 1 | 必须与步骤 1 目标关键词完全一致 |
| 宽泛主题 | 3-4 | 1-2 词的上位词 |
| 长尾（3-5 词） | 5-6 | 来自步骤 1 调研结果 |
| 提问类 | 2 | "如何 [主题]"、"什么是 [主题]" |
| 品牌 / 节目名 | 1-2 | 频道/播客/官网名 |
| 年份戳 | 1-2 | 仅在调研显示常见时使用 |
| 地域戳 | 1-2 | 涉及地区的内容必加 |

规则：
- 除专有名词外全部小写
- 不能含特殊字符、井号、逗号
- 总体字符数不超过 500
- 同一关键词短语不能重复

---

## 话题标签规则

适用于所有需要话题标签的模式（A-F）。

- 5-8 个话题标签（视频/播客）；3-5 个（短视频）
- 前 3 个话题标签会显示在标题下方——需慎重选择
- 仅放在描述的最末行——绝对不要放进标签字段
- 格式：驼峰命名（如 `#AgritechNepal`）
- 配比：2 个宽泛 + 2 个具体 + 1-2 个地域 + 1 个品牌
- 不要每次上传都用同一组——每条视频至少换 3-6 个

## 描述结构

适用于所有需要描述的模式（A-F）。

### 第一段 — 钩子（前约 150 字符，搜索结果可见）
- 第一句：自然嵌入步骤 1 的目标关键词
- 第二句：核心承诺
- 第三句：目标受众
- 共 80-120 词

### 第二段 — 正文
- 4-6 段短段落或 ▶ 符号列表
- 自然嵌入次要关键词——每段一个
- 地域信号：提及地点 2-4 次
- 播客：附嘉宾简介段落与链接
- 视频 450-650 词；播客可略长

### 第三段 — 页脚
- 🔗 资源与链接，来自步骤 2 的真实链接
- 订阅 CTA，2 句话
- 播客：附"收听平台"链接块
- 话题标签放在最末行

**总长度：** 视频 700-900 词，播客 800-1000 词。短视频 150-200 词。

### 完整描述模板
```
[钩子——第一句嵌入目标关键词，点明核心承诺与目标受众]
```

## 章节 / 时间戳规则

用于模式 A-D。**硬上限：6-10 个标记。** 超出则合并相邻段落。

- 第一章必须是 `0:00`——缺失它 YouTube 会忽略所有章节
- 每条标题 3-6 词，行动导向，关键词自然出现
- 标题必须真实反映内容
- 播客单集：在时间戳中标注嘉宾介绍与赞助口播

---

## 地域 / 本地 SEO 规则

当内容与地域强相关时执行：

- 描述中提及地点 2-4 次
- 地域标签：`[主题] [城市]`、`[主题] [国家]`
- 前 3 个话题标签：至少 1 个地域标签
- 双语频道：英文描述 + 1 句本地语言
- 标题加地点：仅当地点是核心差异化卖点时使用

---

## 短视频 / 切片适配（副片段）

当用户明确要求从其提到的某条视频里剪一段 Short 时使用。

如果用户要剪 Short 而没有指明来源视频，先问：*"从哪条视频里剪？"*——他们确认后，将该 Short 视作主任务，直接走模式 E（无需再做源视频套件）。

如果用户既要主视频 SEO 套件又要其 Short，先产出主模式，再在末尾附加本小节作为独立模块。

- **标题：** 60-70 字符，前 3 词含关键词
- **描述：** 150-200 词——钩子 + 话题标签，不含章节
- **话题标签：** 3-5 个，含 `#Shorts`，放在描述中
- **标签：** 复用主视频的 5-8 个
- 不含章节（Shorts 不支持）

如用户需要，作为独立模块追加在主套件之后。

## 组装输出

按用户实际要求组装，只交付他们要的东西：

- **完整套件**：用模式模板，填齐每个编号小节
- **单项**（只要标题/描述/标签）：只交付该项 + 自然附带的（比如描述就要带上其话题标签和章节；标题就要带上 A/B 备选）
- **只说必要的**——不要塞用户没要求的小节

按以下规则对应各部分：
1. **标题** → 标题规则
2. **描述** → 描述结构
3. **标签** → 标签策略
4. **话题标签** → 话题标签规则
5. **章节** → 章节 / 时间戳规则
6. **地域/本地** → 地域 / 本地 SEO 规则

然后运行下方质量自检清单。

## 质量自检清单 — 发送前必跑

### 完整性
- [ ] 匹配模式中所有编号小节均已含真实内容
- [ ] 章节：6-10 个标记，不超

### 调研
- [ ] 目标关键词已确认（或建议后已确认）
- [ ] 步骤 1 搜索批次已运行——次要关键词来自真实结果
- [ ] 年份已使用今日真实日期
- [ ] 无编造的搜索量或播放量
- [ ] 最高级说法已验证或已标 `[待核实：...]`

### 标题
- [ ] 严格 60-70 字符，逐字计数
- [ ] 目标关键词位于前 5 词
- [ ] 一个情绪钩子；A/B 备选用不同角度

### 描述
- [ ] 首句含目标关键词
- [ ] 三段齐全；视频 700-900 词 / 播客 800-1000 词
- [ ] 章节/时间戳已嵌入描述
- [ ] 链接来自步骤 2（或 `[待补：...]` 标记）
- [ ] 话题标签只在最末行

### 标签与话题标签
- [ ] 14-19 个标签，总字符 < 500，覆盖全部 7 类
- [ ] 标签字段不出现井号
- [ ] 5-8 个话题标签（短视频 3-5），驼峰命名，前 3 个最强
- [ ] 话题标签与近期上传不重样

### 章节与地域
- [ ] 首章为 0:00，共 6-10 个，标题含关键词
- [ ] 地域信号提及 2-3 次（如适用）

### 附加项
- [ ] 缩略图文字、卡片/结束画面、播放列表说明、置顶评论均齐全
- [ ] 播客单集：嘉宾位置、平台链接、赞助披露、单集编号齐全
- [ ] 短视频：3-5 个话题标签、含 #Shorts、不含章节、视情况跨平台文案

## 常见错误

| 易错点 | 正确做法 |
|---|---|
| 凭记忆生成标签/关键词 | 先跑步骤 1 调研批次 |
| 编造搜索量或播放量数字 | 永不引用未经验证的数字；用方向性表述 |
| 硬编码训练数据中的年份 | 用今天的真实日期 |
| 描述里满是占位符 | 先跑步骤 2 收集真实链接 |
| 标题 71 字符以上 | 严格计数，砍掉冗余 |
| 描述不足 400 词 | 必须达到 700-1000 词 |
| 把话题标签放在标签字段 | 标签=关键词；话题标签只能放描述 |
| 标签全是同一短语的变体 | 必须覆盖全部 7 类标签 |
| 缺 0:00 章节 | YouTube 会忽略所有章节 |
| 地域内容忘记加地域信号 | 地域内容必须加 |
| A/B 备选标题只是换说法 | 必须测真正不同的钩子 |
| 置顶评论写"快来看我的视频！" | 必须含关键词 + 价值预告 |
| 播客标题漏掉可搜的嘉宾名 | 若嘉宾可搜到则前置其名 |
| 章节标记超过 10 个 | 合并相邻段落 |
| 未验证就把最高级说法当事实 | 先验证或标 `[待核实：...]` |
| 假定某个具体的搜索/抓取工具名 | 用宿主实际提供的工具 |

## 示例

**视频，新发布：** 用户说"我准备发一条视频，讲尼泊尔农户如何用手机应用直接把蔬菜卖出去。" → 确认目标关键词（"在尼泊尔线上卖菜"），跑步骤 1，产出模式 A 套件——标题、A/B 备选、800 词带地域信号的描述、18 个标签、7 个话题标签、8 个章节、缩略图文字、卡片、播放列表说明、置顶评论、结束画面口播。

**播客，已发布单集，表现不佳：** 用户说"我那期嘉宾的单集几乎没流量，这是链接。" → 抓取 URL，确认目标关键词，产出模式 D 审计——评分卡、详细发现、重写后的元数据、行动计划。

**短视频，从某播客剪出：** 用户说"从那期 Antigravity Awesome Skills 部分剪一条 Short。" → 走模式 E：复用单集关键词与标签，60-70 字符标题，150-200 词描述，3-5 个话题标签含 `#Shorts`，跨平台文案。

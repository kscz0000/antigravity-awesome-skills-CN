# technical-tutorials

**真正能跑通的逐步指南。**

## 它能做什么

手把手带你编写教程、快速入门和代码走读。涵盖前置条件、渐进式复杂度、可直接复制的代码、故障排查章节，以及"它跑起来了！"的爽点时刻。

## 何时使用

- 创建快速入门指南
- 编写逐步教程
- 构建代码走读
- 当你提到"教程"、"快速入门"、"上手指南"或"操作指南"

## 快速开始

```
/technical-tutorials

Create a quickstart for our Python SDK.
Target: Python developers new to our API.
Goal: First successful API call in 5 minutes.
```

该技能会：
1. 加载你的开发者受众上下文
2. 清晰组织前置条件
3. 构建渐进式复杂度
4. 添加检查点和故障排查
5. 确保代码开箱即可复制运行

## 关键框架

### 教程类型

| 类型 | 时长 | 用途 |
|------|------|---------|
| 快速入门 | 5-10 分钟 | 尽快获得首次成功 |
| 教程 | 20-45 分钟 | 深入学习一个概念 |
| 实战课 | 1-3 小时 | 综合性项目 |

### 渐进式复杂度（分层蛋糕）

1. 骨架（最小可运行代码）
2. 核心功能（主要功能）
3. 真实数据（替换硬编码值）
4. 错误处理（生产模式）
5. 打磨（日志、配置、测试）

### 复制粘贴检查表

每个代码块必须：
- 可独立运行
- 包含所有 import
- 没有未定义的变量
- 跨平台可用
- 解释"为什么"而不是"是什么"

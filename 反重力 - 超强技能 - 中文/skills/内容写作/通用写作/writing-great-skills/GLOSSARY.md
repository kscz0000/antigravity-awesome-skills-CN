# 词汇表 —— 打造优秀技能

让技能变优秀的领域模型。技能的存在，是从一个随机系统中榨取确定性；下文每个术语都是对该目标的一个杠杆。这是 [`writing-great-skills`](SKILL.md) 的已披露参考。

任何定义中的**加粗术语**本身都在本词汇表中有定义；通过标题查找即可。

## 语言

### 可预测性

技能让智能体在每次运行时表现一致*方式*的程度——相同流程，而非相同输出（一个头脑风暴技能应当*可预测地*发散；token 变化，行为不变）。其他所有术语服务的根本德性——成本与可维护性是它的症状，不是对手。

_避免_：一致性、可靠性、鲁棒性、输出确定性

### 模型调用

保留 **description** 字段的技能，因此智能体能看见它并自主触发——人类仍可输入名称，因此模型调用始终*包含*用户可达。并不存在仅模型的状态：description 仅*增加*智能体的发现能力，从不剥夺人类的能力。每轮都付出永久**上下文负载**，换取这种可发现性。可被其他技能触达，因为使它对智能体可发现的 description 也使它可调用。一个内容全为**参考**的模型调用技能，也是共享参考的一个归宿：另一技能可以调用它，因此被多个技能需要的参考集中在一处。仅在智能体必须自主触达该技能时选模型调用；如果它仅由手动触发，去掉 description，免除上下文负载。

_避免_：能力、工具、才能

### 用户调用

剥离 **description** 的技能——对智能体不可见，只能由人类输入名称触达（仅*用户*，而**模型调用**是用户*与智能体*）。用零**上下文负载**换取智能体的可发现性。因为没有 description，除了人类什么都无法触达它：其他技能不能调用它。

_避免_：过程、工作流、命令

### Description

技能的可机读触发，也是**模型调用**型技能被迫一直加载的唯一**上下文指针**。它本身的存在*就是*调用轴：保留它，技能就是模型调用（也可被其他技能触达）；删除它，技能就是**用户调用**，仅人类可达。模型调用型技能**上下文负载**的来源。

_避免_：前置元数据、摘要

### 上下文指针

智能体上下文中的参考，命名某些上下文之外的材料并编码触达它的条件。**description** 是顶层上下文指针（上下文窗口→技能）；指向已披露文件的指针是同一对象的下一层。它的措辞，而非目标，决定智能体*何时*触达——以及*多可靠*。必需的目标藏在弱措辞指针之后，是差异性 bug：先修措辞，仅在锐化失败时才内联材料。

_避免_：链接、参考、导入

### 上下文负载

**模型调用**型技能对智能体上下文窗口造成的代价——它的**description**，始终加载，消耗 token 与注意力。**用户调用**型技能因没有 description 而规避；也是拆分出更多模型调用型技能的刹车。

_避免_：token 成本、上下文膨胀

### 认知负载

**用户调用**型技能对人类造成的代价——人类必须记住的内容：哪些技能存在、何时使用哪一个（人类就是索引）。**模型调用**通过智能体可发现消除的部分；也是拆分出更多用户调用型技能的刹车。不是要被最小化的代价：它是人类主动权的代价，是某些技能保持用户调用的原因。在人类判断重要处付出；在不重要处去除。

_避免_：人类索引、负担、开销

### 粒度

你划分技能的精细度。更细的划分消耗两种负载之一：更多**模型调用**型技能消耗**上下文负载**（更多 description 挤占窗口并争夺注意力）；更多**用户调用**型技能消耗**认知负载**（更多需要人类记忆与触达）。两种切割指导划分。按**调用**拆分——当你拥有一个独特的**首词**触发它——一个你确实在提示中使用的触发词。按**序列**拆分——在**完成后续步骤**需要隐藏时拆分**步骤**，因为在独立上下文中隔离它能清空其后的内容。警惕反向：合并序列会让每步的完成后续步骤暴露给后续步骤，招致抢跑。

_避免_：分块、模块化

### 路由技能

一个**用户调用**型技能，职责是指出你其他用户调用型技能——逐一命名并说明使用时机——让人类只需记住一个技能而非多个。它只能暗示，永不能调用它们：用户调用型技能没有**description**，除人类外谁也无法触达。是**认知负载**在用户调用型技能膨胀时的解药。

_避免_：分派器、菜单、注册表、索引、路由过程

### 信息层级

技能内容按智能体对其即时需求程度排序——一条由两种切割产生的阶梯：文件内或指针后，步骤或参考。各层：

- **步骤**——文件内，主要
- **参考**，文件内——次要
- **参考**，已披露——在**上下文指针**之后

没有**步骤**的技能只用底下两层——通常为合理的扁平同级集合（如一次审查的所有规则同处一层），这是妥当的安排，不是异味。层级与调用无关：技能可以是模型调用或用户调用，无论它是全步骤、全参考、还是两者皆有。当技能有步骤时，文件中应当披露的参考埋没了它们，并把关注它们变成抛硬币——不是差异性杠杆，也关乎可读性。让阶梯顶部清晰；能下推就下推。

_避免_：结构、组织、布局

### 就近放置

把智能体一次性需要的材料放在一处——一个概念的定义、规则、注意事项都在同一标题下，而非分散到文件各处——读到一处便顺带看到它的邻居。**信息层级**的同级伴随：层级排序某块内容*下沉多远*；就近放置决定一旦下沉*谁与它并列*。一个**参考**主体的正确格式没有公式；检验是技能应当读起来像为智能体写的文档，群组的材料读起来是这样，散落的不是。区别于**重复**：重复是一个含义出现在两处，而分散是把单一含义碎片化散落多处。

_避免_：分组、聚类、内聚

### 分支

技能可被调用的一个不同方式——它处理的一种情形——因此不同运行走不同路径。含多步骤的技能可能携带多分支；线性技能则无。

_避免_：路径、情形、分叉

### 渐进披露

把**参考**沿阶梯下移——从 SKILL.md 中移出、置于**上下文指针**之后——让顶部保持清晰。首要目的不是 token 优化；而是保护**信息层级**。由**分支**授权：披露只有部分分支需要的，内联每条路径都需要的，若指针在必需材料上不可靠地触发，则锐化措辞，仅当失败才拉回内联。

_避免_：懒加载、分块

### 步骤

智能体执行的有序动作——当技能有它们时，是内容的主要层级，也是赢得其在 SKILL.md 中位置的部分。并非每个技能都有步骤：技能可以是全步骤（`tdd`）、全**参考**（一次审查）、或两者皆有，与调用无关。每步以**完成标准**结尾，清或糊。

_避免_：工作流、指令、编排

### 完成标准

告诉智能体一项工作已结束的条件——它据此判断的目标。两个属性让它成为杠杆，而不仅是质量。它的**清晰度**（智能体能区分完成与未完成吗？）抵御**抢跑**——模糊界限（"达到理解"）让智能体宣布完成并滑向下一步；该轴需要*步骤*才有约束力，因为抢跑是步骤间失败。它的**要求度**（它要求多少）决定**实操**——"每个被修改的模型都已统计"强制彻底工作，而"产出变更清单"则不然——该轴*不*绑步骤：它也能约束一个扁平参考主体，这正是无步骤技能仍承载穷尽标准（"每条规则都已应用"）的方式。最强标准既可校验又穷尽。

_避免_：完成条件、退出条件、停止规则

### 完成后续步骤

当前步骤之后的**步骤**。可见时，它把智能体向前拉入**抢跑**——看得越多，拉力越强；防御是把它们通过拆分步骤序列隐藏起来。

_避免_：地平线、战争迷雾、前瞻

### 实操

智能体在单步之内做的幕后工作——读文件、探索代码库、做改动、挖掘所需而非甩给用户。它活在步骤结构之下：从不写成自己的步骤，潜伏在措辞中，由智能体而非技能控制。是**完成后续步骤**在步内所对应的概念——后者把人往前拉，前者是把眼下的活做透。由**首词**（_全面_、_彻底_）或要求工作穷尽的**完成标准**抬升——包括施加于扁平参考的要求度，这正是让扁平参考技能覆盖所有层级的原因。当要求缺失或**抢跑**砍掉步骤时，实操都会变薄。

_避免_：范围、努力、勤勉、覆盖度

### 参考

智能体按需查阅的材料——定义、事实、参数、示例、条件指令。当技能有**步骤**时，它是次要的；当技能没有时，它是全部内容；或它完全生活在任何技能之外——见**外部参考**。经**上下文指针**触达，是**渐进披露**的首要候选。

_避免_：支持材料、文档、背景

### 外部参考

生活在技能系统之外的**参考**——一个普通文件，没有**description**、没有**步骤**、不可调用——任何技能都可指向它。不需要自身触发的共享参考的归宿，也是两个**用户调用**型技能唯一能用的共享归宿，因为两者都没有 description，因此谁也不能调用对方。

_避免_：文档、资源、知识库

### 首词

一个紧凑概念——也叫 *Leitwort*——已经存在于模型的预训练中，智能体在运行技能时用其思考。它以最少 token 编码行为原则，调用模型已持有的先验（如_课题_、_最近发展区_、_战争迷雾_、_示踪弹_）。作为 token 而非句子反复出现，它在技能中累积分布式定义，并锚定一片行为。当模型中已有词能覆盖，自己造词也行——但前提是定义清晰；自造词不调任何先验——你付出定义 token 换来的，本可用预训练词免费拿到。优先用已有词。

首词为**可预测性**服务两次。在正文中它锚定**执行**——智能体在该概念每次出现时都触达相同行为，在扁平参考内它聚焦于该类事物的注意力，每次运行都调出正确的检查。在**description**中它锚定**调用**——且不止技能内：当同一词存在于你的提示、文档、代码库，智能体把该共享语言关联到技能，更可靠地触发它。用你确实在希望技能触发时使用的首词来写 description。

_避免_：关键词、术语、母题

### 单一事实来源

期望的状态：每个含义只存在于一处权威位置，因此修改技能行为只需改一处。**重复**是它的违反。

_避免_：家、规范化位置

### 相关性

一行是否仍与技能所做的事相关——保留什么的透镜。一行失去相关性，要么因从来与任务无关（仅是叙述，或应被披露的**分支**），要么因过时：随着所描述的行为或世界改变而漂移陈旧。较短的技能更易保持相关性，因为每行检查成本更低。区别于**无效**：相关性问一行是否与任务相关，不问它是否改变行为。

_避免_：承重、过时、新鲜

## 失败模式

### 抢跑

在当前步骤真正完成前就结束，因为智能体的注意力从工作滑向"已做完"。步骤间失败：它需要**步骤**才发生——无步骤技能的提前结束不是抢跑，而是需求未满足下的**实操**过薄。两个力的拔河：可见的**完成后续步骤**（向前拉）与**完成标准**的清晰度（抵抗——锐利、可校验的标尺能撑住；模糊的会失守）。模糊是必要条件：无论后续步骤可见多少，锐利界限都抵抗拉力，因此从不抢跑的步骤不需要防御。两条杠杆稳住会抢跑的步骤，但按顺序使用它们：**先锐化界限**——它廉价且局部。仅当标准不可避免地模糊*且*你确实观察到抢跑时，**隐藏后续步骤**——隐藏仅在真实上下文边界起作用（用户调用交接或子智能体分派；内联模型调用让后续步骤留在上下文，不清空）。实操过薄的一个原因，但与之不同：实操在步骤全程完成后仍可能过薄。

_避免_：过早关闭、抢跑、仓促、走捷径

### 重复

同一含义被赋予多于一个**单一事实来源**。增加维护成本（改一处就得改其他处）、消耗 token，并夸大权重——重复一个含义让它在层级上的权重超过其真实位置。**首词**的偶然反面——后者主动抬升注意力靠重复一个 token，永不重复含义。

_避免_：重复、冗余

### 沉淀

老旧内容层堆积在技能中而从不清除，因为新增感觉安全、删除感觉冒险——于是过时与不相关的行累积起来，要找仍活着的部分得一路挖下去。这是任何不坚持修剪纪律的技能注定会走向的命运；与**重复**所致的重复含义不同，这里侵蚀的是**相关性**本身。

_避免_：堆积、膨胀、赘余、腐烂

### 蔓延

技能单纯地过长——SKILL.md 中行数太多——无论行是否过时或重复。即便全活且全唯一的技能也能蔓延。它增加可读性成本（智能体在行动前得读更多，过剩让注意力变薄）、可维护性成本（每多一行就多一行要保持**相关**）、与 token 成本。解药是**信息层级**：通过指针把**参考**下推，并按**分支**或序列拆分，让每条路径只携带所需。区别于**沉淀**（由过时堆积导致的长度）与**重复**（由重复含义导致的长度）——蔓延就是长度本身，与成因无关。

_避免_：膨胀、长度、大小、冗长

### 无效

一条指令，因为模型默认就遵守而不改变任何事——你付出负载去告诉智能体它本来就会做的事。检验：相对默认，一行会改变行为吗？一行可以完美**相关**却仍是无效。**首词**之所以免费，所依仗的也是同一组先验；先验既出，无效便毫无价值。

首词是*技巧*；无效是加于一行的*判决*——二者交叉。一个弱到无法击败默认的首词就是无效（在智能体已经差不多彻底时写_要彻底_），解法是更强的词通过判决（_无情_），而非不同的技巧。所以无效检验——它相对默认改变行为吗？——也是你评判首词是否值得重复的方式。这是相对模型的，而非相对的读者：两人争一行是否无效，是在争默认是什么，应通过运行技能而非辩论来定。

_避免_：冗余指令、复述显然、徒费口舌

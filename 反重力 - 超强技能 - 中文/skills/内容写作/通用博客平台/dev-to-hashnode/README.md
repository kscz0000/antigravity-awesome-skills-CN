# dev-to-hashnode

**内容平台技能** —— 在开发者博客平台上高效发布内容。

## 用途

引导你在 Dev.to 与 Hashnode 上发布内容：平台差异、跨平台发布策略、canonical URL、标签策略、排版技巧与涨粉方法。

## 适用场景

- 在 Dev.to 或 Hashnode 上发布技术内容
- 搭建跨平台发布工作流
- 针对不同平台的受众做优化
- 当你提到 "Dev.to"、"Hashnode"、"cross-posting"、"canonical URL" 时

## 快速开始

```
/dev-to-hashnode

I have a tutorial on my blog and want to cross-post it
to Dev.to and Hashnode. What's the best strategy to
maximize reach without hurting my SEO?
```

## 核心章节

1. **平台对比** —— Dev.to 与 Hashnode 的差异
2. **跨平台发布策略** —— canonical URL 的取舍
3. **Dev.to 优化** —— Frontmatter、标签、表现好的内容
4. **Hashnode 优化** —— 设置项、特色功能、受众
5. **内容排版** —— 两个平台通用的结构
6. **涨粉之道** —— 持续输出与互动

## 平台对比

| 特性 | Dev.to | Hashnode |
|---------|--------|----------|
| 月访问量 | ~1000 万+ | ~300 万+ |
| 自定义域名 | 不支持 | 支持（免费） |
| Newsletter | 无 | 内置 |
| 受众 | 更广、初学者更多 | 更资深 |
| 最适合 | 追求最大曝光 | 打造个人品牌 |

## 跨平台发布顺序

1. 先在自己的博客发布（canonical）
2. 等 1-2 天让 Google 完成索引
3. 跨发到 Dev.to，附带 canonical URL
4. 跨发到 Hashnode，附带 canonical URL

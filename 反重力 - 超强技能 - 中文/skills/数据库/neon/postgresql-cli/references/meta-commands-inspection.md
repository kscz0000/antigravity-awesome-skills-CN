## 对象检查（\d 家族）

所有 `\d` 命令接受以下通用修饰符：
- `+` — 额外信息（大小、描述、所有者）
- `S` — 包含系统对象
- `x` — 展开显示（必须在 `S` 或 `+` 之后，不能紧跟在 `\d` 之后）

**重要**：展开显示的 `x` 修饰符必须出现在 `S` 或 `+` 之后（如 `\dt+x`），因为 `\dx` 是列出已安装扩展的独立命令。当你想要展开显示却写了 `\dx` 时，会显示扩展而非展开模式。

所有命令接受带通配符匹配（`*`、`?`、正则）的模式参数。见下方模式章节。

### `\d[Sx+] [ pattern ]`

不带模式：等价于 `\dtvmsE`（列出所有可见的表、视图、物化视图、序列和外部表）。

带模式：显示列、类型、表空间、特殊属性（NOT NULL、默认值）、索引、约束、规则、触发器。外部表显示外部服务器。

`\d+` 额外显示：列注释、OID 存在、视图定义、副本标识、访问方法。

### 表类型列表命令

`\dE` / `\di` / `\dm` / `\ds` / `\dt` / `\dv` — 列出外部表、索引、物化视图、序列、表或视图。组合字母：`\dti` 同时列出表和索引。

`\d+` 额外显示：持久状态（永久/临时/无日志）、磁盘物理大小、描述。

### 聚合与函数列表

| 命令 | 显示内容 |
|------|---------|
| `\da[Sx] [pattern]` | 聚合函数及其返回类型和输入类型 |
| `\df[anptwSx+] [pattern [arg_pattern ...]]` | 函数。按类型过滤：`a`=聚合、`n`=普通、`p`=过程、`t`=触发器、`w`=窗口。额外参数匹配参数类型名。使用 `-` 作为最后的 arg_pattern 防止匹配带额外参数的函数。示例：`\df * integer` 列出第一个参数为 `integer` 的函数。 |
| `\do[Sx+] [pattern [arg_pattern [arg_pattern]]]` | 操作符及其操作数/结果类型。一个参数匹配前缀操作符；两个参数匹配二元操作符。用 `-` 表示未使用的操作数。示例：`\do + integer integer` 列出两个整数参数的 `+` 操作符。 |

### 模式匹配规则

所有接受模式的 `\d` 命令使用相同的匹配系统：

1. **大小写折叠**：不带引号的字母折叠为小写（如同 SQL 标识符）。双引号阻止折叠。
2. **通配符**：`*` 匹配任意字符序列，`?` 匹配任意单个字符。双引号内为字面值。
3. **点分隔符**：点（`.`）分隔模式名与对象名。两个点分隔数据库.模式.对象（数据库必须与当前连接匹配）。
4. **正则表达式**：`[0-9]` 等高级模式可用。`.` 是分隔符（非正则的任意字符），`*` → `.*`，`?` → `.`，`$` 为字面匹配。双引号内所有正则特殊字符为字面值。
5. **无模式**：显示当前模式搜索路径中可见的所有对象（等价于 `*`）。使用 `*.*` 查看所有对象，无论可见性。

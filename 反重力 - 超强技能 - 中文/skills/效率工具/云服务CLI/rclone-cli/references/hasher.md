# Hasher

Hasher 是一个特殊的叠加（overlay）后端，用于创建为其他远程存储处理校验和的远程。它的主要功能包括：

- 模拟后端尚未实现的哈希类型
- 缓存校验和，以应对本地或 (S)FTP 大文件哈希计算缓慢的问题
- 通过外部 SUM 文件预热校验和缓存

## 入门

要使用 Hasher，请先按照底层远程存储的配置说明完成配置。你也可以使用本地路径代替远程存储。请确认你的基础远程存储可以正常工作。

我们在下文中将基础远程存储称为 `myRemote:path`。注意，`myRemote:path` 内部的内容会由 hasher 处理，而其外部的内容则不会。这意味着，如果你使用的是基于桶（bucket）的远程存储（S3、B2、Swift），则应将桶放入远程 `s3:bucket` 中。

接下来请使用交互式或手工方式进行配置。

### 手工配置

运行 `rclone config path` 查看当前生效配置文件的路径，通常为 `YOURHOME/.config/rclone/rclone.conf`。
在常用的文本编辑器中打开它，找到基础远程存储所在的段，并按照下面示例的方式新建一个 hasher 段：

```ini
[Hasher1]
type = hasher
remote = myRemote:path
hashes = md5
max_age = off

[Hasher2]
type = hasher
remote = /local/path
hashes = dropbox,sha1
max_age = 24h
```

Hasher 主要使用以下几个参数：

- `remote` 为必填项
- `hashes` 是以逗号分隔的支持的校验和列表
   （默认为 `md5,sha1`）
- `max_age` —— 缓存中校验和值的最大保留时长
   `0` 表示完全禁用缓存
   `off` 表示“永久”缓存（即直到文件发生变化为止）

请确保 `remote` 中包含冒号 `:`。如果你指定的 remote 不带冒号，那么 rclone 将使用与之同名的本地目录。因此，如果你使用远程路径 `/local/path`，那么 rclone 将为该目录处理哈希。如果你直接写 `remote = name`，那么 rclone 会在**当前目录下的一个名为 `name` 的目录中**存放文件。

## 使用方法

### 基本操作

现在你可以像 `Hasher2:subdir/file` 这样使用它，代替基础远程存储。
当文件被完整读取或覆盖写入时，Hasher 会在后台透明地用新校验和更新缓存，例如：

```console
rclone copy External:path/file Hasher:dest/path
rclone cat Hasher:path/to/file > /dev/null
```

要刷新某个子树中的**所有**缓存校验和（包括基础后端不支持的校验和），需要**重新下载**该子树下的所有文件。例如，使用任意支持的哈希类型运行 `hashsum --download`（我们只是借此重新读取文件）：

```console
rclone hashsum MD5 --download Hasher:path/to/subtree > /dev/null
rclone backend dump Hasher:path/to/subtree
```

你可以使用自定义后端命令打印或清空哈希缓存：

```console
rclone backend dump Hasher:dir/subdir
rclone backend drop Hasher:
```

### 从 SUM 文件预填充

Hasher 支持两个后端命令：通用的 SUM 文件 `import`，以及更快但一致性较弱的 `stickyimport`。

```console
rclone backend import Hasher:dir/subdir SHA1 /path/to/SHA1SUM [--checkers 4]
```

这里的 SHA1 可以替换为该远程支持的任何哈希类型。最后一个参数可以指向本地或 `other-remote:path` 的 SUM 格式文本文件。该命令会解析 SUM 文件，然后遍历第一个参数给定的路径，采集当前指纹，并相应地填充缓存条目。

- SUM 文件中的路径被视为相对于 `hasher:dir/subdir`。
- 该命令**不会**检查所提供的值是否正确。
  你**必须清楚**自己在做什么。
- 这是一次性操作。SUM 文件不会被“绑定”到该远程。
  如果对象的指纹发生变化，缓存条目仍可能在之后被覆盖。
- 树的遍历耗时取决于树的规模。你可以增大 `--checkers` 以加快速度。如果你不在意指纹和一致性，也可以使用 `stickyimport`。

```console
rclone backend stickyimport hasher:path/to/data sha1 remote:/path/to/sum.sha1
```

`stickyimport` 与 `import` 类似，但速度要快得多，因为它不需要 stat 已有的文件，并会跳过初始的树遍历。它不把缓存条目绑定到文件指纹，而是创建仅按文件名绑定的“粘性”条目，忽略大小、修改时间等信息。这类哈希条目只能通过 `purge`、`delete`、`backend drop`，或对文件进行完整的重新读取/重新写入来替换。

## 后端命令

以下是 hasher 后端特有的命令。

通过以下方式运行：

rclone backend COMMAND remote:

### drop

清空缓存。

```console
rclone backend drop remote: [options] [<arguments>+]
```

完全清空校验和缓存。

使用示例：

```console
rclone backend drop hasher:
```

### dump

转储数据库。

```console
rclone backend dump remote: [options] [<arguments>+]
```

导出当前远程所覆盖的缓存记录。

### fulldump

完整转储数据库。

```console
rclone backend fulldump remote: [options] [<arguments>+]
```

导出数据库中的所有缓存记录。

### import

导入 SUM 文件。

```console
rclone backend import remote: [options] [<arguments>+]
```

通过 SUM 文件补充哈希缓存，并按大小/时间将校验和绑定到文件。

使用示例：

```console
rclone backend import hasher:subdir md5 /path/to/sum.md5
```

### stickyimport

对 SUM 文件执行快速导入。

```console
rclone backend stickyimport remote: [options] [<arguments>+]
```

在不校验文件指纹的情况下，从 SUM 文件填充哈希缓存。

使用示例：

```console
rclone backend stickyimport hasher:subdir md5 remote:path/to/sum.md5
```

## 实现细节（高级）

本节将解释各种 rclone 操作如何在 hasher 远程上工作。

**免责声明：本节描述的是当前实现，未来 rclone 版本中可能发生变化！**

### hashsum 命令

`rclone hashsum`（或 `md5sum`、`sha1sum`）命令将：

1. 如果所请求的哈希被底层支持，则直接传递。
2. 如果对象大小小于 `auto_size`，则下载该对象并即时计算所**请求的**哈希。
3. 如果不支持且对象足够大，则构建对象 `fingerprint`（包括大小、若支持则包含修改时间、以及首先找到的**其他**哈希）。
4. 如果在缓存中针对该远程找到严格匹配的项，则返回已存储的哈希。
5. 如果找到该远程但指纹不匹配，则清除该条目并继续执行步骤 6。
6. 如果未找到该远程、该远程没有所请求的哈希类型、或经过步骤 5 后：下载该对象，即时计算所有**支持的**哈希并存入缓存；返回所请求的哈希。

### 其他操作

- 每当请求哈希时，按上文 `hashsum` 中的 1–4 步逻辑处理
- 每当文件被**完整**上传或下载时，捕获数据流以即时计算所有支持的哈希并更新数据库
- 服务端 `move` 会更新已有缓存条目的键
- `deletefile` 会删除单个缓存条目
- `purge` 会删除被清除路径下的所有缓存条目

注意，将 `max_age = 0` 设置为该值会完全禁用校验和缓存。

如果设置 `max_age = off`，缓存中的校验和将永远不会过期，除非你完整重写或删除该文件。

### 缓存存储

缓存的校验和以 `bolt` 数据库文件的形式存储在 rclone 缓存目录下，通常为 `~/.cache/rclone/kv/`。每个**基础**后端维护一个数据库，命名形如 `BaseRemote~hasher.bolt`。
指向同一基础后端的多个 `alias` 的校验和会存放在同一个数据库中。所有本地路径都被视为指向 `local` 后端的别名（除非经过加密或分块），并存放在 `~/.cache/rclone/kv/local~hasher.bolt` 中。数据库可以在多个 rclone 进程之间共享。

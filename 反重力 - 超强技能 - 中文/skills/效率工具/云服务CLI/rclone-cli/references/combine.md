# Combine

`combine` 后端将多个 remote 合并到单个目录树中。

例如，你可能在某个 provider 上有一个用于图片的 remote：

```console
$ rclone tree s3:imagesbucket
/
├── image1.jpg
└── image2.jpg
```

而在另一个 provider 上有一个用于文件的 remote：

```console
$ rclone tree drive:important/files
/
├── file1.txt
└── file2.txt
```

`combine` 后端可以将它们合并为如下合成目录结构：

```console
$ rclone tree combined:
/
├── files
│   ├── file1.txt
│   └── file2.txt
└── images
    ├── image1.jpg
    └── image2.jpg
```

你可以通过在配置中指定 `upstreams` 参数来实现：

```text
upstreams = images=s3:imagesbucket files=drive:important/files
```

在初次使用 `rclone config` 进行设置时，你需要以空格分隔列表的形式指定 upstream remote。upstream remote 既可以是本地路径，也可以是其他 remote。

## 配置

下面以为上述示例创建一个名为 `remote` 的 combine 为例。首先运行：

```console
rclone config
```

这将引导你完成交互式配置流程：

```text

n) New remote
s) Set configuration password
q) Quit config
n/s/q> n
name> remote

...

    dir=remote:path dir2=remote2:path
```

### 为 Google Drive 共享云盘配置

Rclone 提供了一个便捷功能，可以为你有访问权限的所有共享云盘创建 combine 后端。

假设你的主（非共享云盘）Google Drive remote 名为 `drive:`，则可运行

```console
rclone backend -o config drives drive:
```

将产生类似如下的输出：

```ini
[My Drive]
type = alias
remote = drive,team_drive=0ABCDEF-01234567890,root_folder_id=:

[Test Drive]
type = alias
remote = drive,team_drive=0ABCDEFabcdefghijkl,root_folder_id=:

[AllDrives]
type = combine
upstreams = "My Drive=My Drive:" "Test Drive=Test Drive:"
```

然后将该配置添加到你的配置文件中（可通过 `rclone config file` 找到该文件），便可通过 `AllDrives:` remote 在一处访问所有共享云盘。

完整信息请参阅 [Google Drive 文档](/drive/#drives)。

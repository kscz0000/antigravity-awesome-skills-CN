# WebDAV

路径以 `remote:path` 形式指定

路径可以按需任意深度嵌套，例如 `remote:directory/subdirectory`。

## 配置

配置 WebDAV 远程存储需要提供 URL、用户名和密码。如果你知道所连接的系统类型，rclone 可以启用额外的功能。

以下是如何创建一个名为 `remote` 的远程存储的示例。首先运行：

```console
rclone config
```

这将引导你完成交互式设置过程：

配置完成后，你可以像这样使用 `rclone`（将 `remote` 替换为你给远程存储起的名称）：

列出 WebDAV 顶层的目录

```console
rclone lsd remote:
```

列出 WebDAV 中的所有文件

```console
rclone ls remote:
```

将本地目录复制到 WebDAV 中名为 backup 的目录

```console
rclone copy /home/source remote:backup
```

### 修改时间和哈希

纯 WebDAV 不支持修改时间。但当与 Fastmail Files、ownCloud 或 Nextcloud 配合使用时，rclone 将支持修改时间。

同样，纯 WebDAV 不支持哈希，但当与 Fastmail Files、ownCloud 或 Nextcloud 配合使用时，rclone 将支持 SHA1 和 MD5 哈希。根据 ownCloud 或 Nextcloud 的具体版本，哈希可能出现在所有对象上，也可能仅出现在上传时附带了哈希的对象上。

### 标准选项

以下是 webdav（WebDAV）特有的标准选项。

#### --webdav-url

要连接的 HTTP 主机 URL。

例如 https://example.com。

属性：

- Config:      url
- Env Var:     RCLONE_WEBDAV_URL
- Type:        string
- Required:    true

#### --webdav-vendor

你正在使用的 WebDAV 站点/服务/软件的名称。

属性：

- Config:      vendor
- Env Var:     RCLONE_WEBDAV_VENDOR
- Type:        string
- Required:    false
- Examples:
  - "fastmail"
    - Fastmail Files
  - "nextcloud"
    - Nextcloud
  - "owncloud"
    - Owncloud 10 PHP based WebDAV server
  - "infinitescale"
    - ownCloud Infinite Scale
  - "sharepoint"
    - Sharepoint Online, authenticated by Microsoft account
  - "sharepoint-ntlm"
    - Sharepoint with NTLM authentication, usually self-hosted or on-premises
  - "rclone"
    - rclone WebDAV server to serve a remote over HTTP via the WebDAV protocol
  - "other"
    - Other site/service or software

#### --webdav-user

用户名。

如果使用 NTLM 认证，用户名应采用 'Domain\User' 格式。

属性：

- Config:      user
- Env Var:     RCLONE_WEBDAV_USER
- Type:        string
- Required:    false

#### --webdav-pass

密码。

**注意** 输入的内容必须经过混淆处理 — 参见 [rclone obscure](/commands/rclone_obscure/)。

属性：

- Config:      pass
- Env Var:     RCLONE_WEBDAV_PASS
- Type:        string
- Required:    false

#### --webdav-bearer-token

用于替代用户名/密码的 Bearer 令牌（例如 Macaroon）。

属性：

- Config:      bearer_token
- Env Var:     RCLONE_WEBDAV_BEARER_TOKEN
- Type:        string
- Required:    false

属性：

- Config:      encoding

- Type:        string
- Required:    false

#### --webdav-headers

为所有事务设置 HTTP 头。

使用此选项为所有事务设置额外的 HTTP 头

输入格式为以逗号分隔的键值对列表。可以使用标准的 [CSV 编码](https://godoc.org/encoding/csv)。

例如，要设置 Cookie 可使用 'Cookie,name=value' 或 '"Cookie","name=value"'。

可以设置多个头，例如 '"Cookie","name=value","Authorization","xxx"'。


属性：

- Config:      headers
- Env Var:     RCLONE_WEBDAV_HEADERS
- Type:        CommaSepList
- Default:

#### --webdav-pacer-min-sleep

API 调用之间的最小休眠时间。

属性：

- Config:      pacer_min_sleep
- Env Var:     RCLONE_WEBDAV_PACER_MIN_SLEEP
- Type:        Duration
- Default:     10ms

#### --webdav-nextcloud-chunk-size

Nextcloud 上传分块大小。

我们建议将 NextCloud 实例的最大分块大小配置为 1 GB，以获得更好的上传性能。
参见 https://docs.nextcloud.com/server/latest/admin_manual/configuration_files/big_file_upload_configuration.html#adjust-chunk-size-on-nextcloud-side

设为 0 可禁用分块上传。


属性：

- Config:      nextcloud_chunk_size
- Env Var:     RCLONE_WEBDAV_NEXTCLOUD_CHUNK_SIZE
- Type:        SizeSuffix
- Default:     10Mi

#### --webdav-owncloud-exclude-shares

排除 ownCloud 共享

属性：

- Config:      owncloud_exclude_shares
- Env Var:     RCLONE_WEBDAV_OWNCLOUD_EXCLUDE_SHARES
- Type:        bool
- Default:     false

#### --webdav-owncloud-exclude-mounts

排除 ownCloud 挂载存储

属性：

- Config:      owncloud_exclude_mounts
- Env Var:     RCLONE_WEBDAV_OWNCLOUD_EXCLUDE_MOUNTS
- Type:        bool
- Default:     false

#### --webdav-unix-socket

要拨号的 Unix 域套接字路径，替代直接打开 TCP 连接

属性：

- Config:      unix_socket
- Env Var:     RCLONE_WEBDAV_UNIX_SOCKET
- Type:        string
- Required:    false

#### --webdav-auth-redirect

在重定向时保留认证信息。

如果服务器在 rclone 尝试读取文件时将其重定向到新域名，rclone 通常会从请求中移除 Authorization: 头。

这是标准的安全做法，以避免将凭据发送到未知的 Web 服务器。

但在某些情况下这是有必要的。如果你在 rclone 尝试从 webdav 服务器读取文件时遇到类似 "401 Unauthorized" 的错误，可以尝试此选项。


属性：

- Config:      auth_redirect
- Env Var:     RCLONE_WEBDAV_AUTH_REDIRECT
- Type:        bool
- Default:     false

#### --webdav-description

远程存储的描述。

属性：

- Config:      description
- Env Var:     RCLONE_WEBDAV_DESCRIPTION
- Type:        string
- Required:    false

## 提供者说明

以下是特定提供者的说明。

### Fastmail Files

使用 `https://webdav.fastmail.com/` 或其子目录作为 URL，使用你的 Fastmail 邮箱 `username@domain.tld` 作为用户名。按照[此文档](https://www.fastmail.help/hc/en-us/articles/360058752854-App-passwords)创建一个具有 `Files (WebDAV)` 访问权限的应用密码，并将其用作密码。

Fastmail 通过 `X-OC-Mtime` 头支持修改时间。

### ownCloud

点击页面右下角的设置齿轮图标，这将显示 rclone 在配置步骤中需要的 WebDAV URL。它看起来类似于 `https://example.com/remote.php/webdav/`。

ownCloud 通过 `X-OC-Mtime` 头支持修改时间。

### Nextcloud

其配置方式与 ownCloud 完全相同。请注意，Nextcloud 最初不支持文件流式传输（`rcat`），而 ownCloud 支持，但[此问题](https://github.com/nextcloud/nextcloud-snap/issues/365)似乎已在 2020-11-27 修复（使用 rclone v1.53.1 和 Nextcloud Server v19 测试通过）。

### ownCloud Infinite Scale

Infinite Scale 的 WebDAV URL 可以在 Infinite Scale 中任意空间的详情面板中找到，前提是用户在个人设置中通过复选框启用了该显示。

Infinite Scale 使用分块的 [tus](https://tus.io) 上传协议。分块大小目前固定为 10 MB。

### Sharepoint Online

Rclone 可以与 OneDrive for Business 或 Office365 教育账户提供的 Sharepoint 配合使用。此功能仅对少数此类账户需要，主要是 Office365 教育账户。这些账户有时未经域名所有者验证 [github#1975](https://github.com/rclone/rclone/issues/1975)

这意味着这些账户无法通过官方 API 添加（其他账户应使用 "onedrive" 选项）。但是，可以通过 webdav 访问它们。

要使用 rclone 配置 sharepoint 远程存储，按以下步骤操作：
首先，你需要获取远程存储的 URL：

- 前往[此处](https://onedrive.live.com/about/en-us/signin/)打开你的 OneDrive 或登录
- 现在查看地址栏，URL 应类似于：
  `https://[YOUR-DOMAIN]-my.sharepoint.com/personal/[YOUR-EMAIL]/_layouts/15/onedrive.aspx`

你只需要此 URL 中到邮箱地址为止的部分。之后，你最可能需要添加 "/Documents"。该子目录包含 OneDrive 上存储的实际数据。

按以下方式将远程存储添加到 rclone：
将 `url` 配置为 `https://[YOUR-DOMAIN]-my.sharepoint.com/personal/[YOUR-EMAIL]/Documents`，并使用你的常规账户邮箱和密码作为 `user` 和 `pass`。如果启用了双重认证（2FA），则需要生成一个应用密码。将 `vendor` 设为 `sharepoint`。

你的配置文件应如下所示：

```ini
[sharepoint]
type = webdav
url = https://[YOUR-DOMAIN]-my.sharepoint.com/personal/[YOUR-EMAIL]/Documents
vendor = sharepoint
user = YourEmailAddress
pass = encryptedpassword
```

### 使用 NTLM 认证的 Sharepoint

当你的（托管）Sharepoint 未绑定 OneDrive 账户且使用 NTLM 认证时，使用此选项。

获取 `url` 配置的方式与上述类似，首先在浏览器中导航到所需目录以获取 URL，然后截取所打开目录名称之后的所有内容。

示例：
如果 URL 为：
<https://example.sharepoint.com/sites/12345/Documents/Forms/AllItems.aspx>

则应使用的配置为：
<https://example.sharepoint.com/sites/12345/Documents>

将 `vendor` 设为 `sharepoint-ntlm`。

NTLM 使用域名和用户名组合进行认证，将 `user` 设为 `DOMAIN\username`。

你的配置文件应如下所示：

```ini
[sharepoint]
type = webdav
url = https://[YOUR-DOMAIN]/some-path-to/Documents
vendor = sharepoint-ntlm
user = DOMAIN\user
pass = encryptedpassword
```

#### SharePoint 所需的标志

由于 SharePoint 对上传的文档会进行一些特殊处理，你将无法使用文档大小或文档哈希来比较文件在上传后是否发生了变化/哪个文件更新。

对于在 SharePoint 之间复制文件（尤其是 .docx、.xlsx 等 Office 文件）的 Rclone 调用（如 copy、sync 等），你应附加以下标志以确保 Rclone 使用"最后修改"时间属性来比较文档：

```text
--ignore-size --ignore-checksum --update
```

## Rclone

如果你通过 rclone 提供的 WebDAV 托管远程存储，使用此选项。详见 [rclone serve webdav](commands/rclone_serve_webdav/)。

rclone serve 通过 `X-OC-Mtime` 头支持修改时间。

### dCache

dCache 是一个支持多种协议和认证/授权方案的存储系统。对于 WebDAV 客户端，它允许用户使用用户名和密码（BASIC）、X.509、Kerberos 以及各种 Bearer 令牌进行认证，包括 [Macaroons](https://www.dcache.org/manuals/workshop-2017-05-29-Umea/000-Final/anupam_macaroons_v02.pdf) 和 [OpenID-Connect](https://en.wikipedia.org/wiki/OpenID_Connect) 访问令牌。

使用 `other` 类型按正常方式配置。不输入用户名或密码，而是将你的 Macaroon 作为 `bearer_token` 输入。

配置最终会类似于这样。

```ini
[dcache]
type = webdav
url = https://dcache...
vendor = other
user =
pass =
bearer_token = your-macaroon
```

有一个[脚本](https://github.com/sara-nl/GridScripts/blob/master/get-macaroon)可以从 dCache WebDAV 端点获取 Macaroon，并创建 rclone 配置文件。

Macaroons 也可以从 dCache 附带的 dCacheView Web 浏览器/JavaScript 客户端获取。

### OpenID-Connect

dCache 还支持使用 OpenID-Connect 访问令牌进行认证。OpenID-Connect 是一种协议（基于 OAuth 2.0），允许服务识别已通过某个中心服务认证的用户。

rclone 中对 OpenID-Connect 的支持目前通过另一个名为 [oidc-agent](https://github.com/indigo-dc/oidc-agent) 的软件包实现。这是一个命令行工具，用于辅助获取访问令牌。安装并配置后，通过运行 `oidc-token` 命令获取访问令牌。以下示例展示了从 *XDC* OIDC 提供者获取的（已缩短的）访问令牌。

```text
paul@celebrimbor:~$ oidc-token XDC
eyJraWQ[...]QFXDt0
paul@celebrimbor:~$
```

**注意** 在 `oidc-token` 命令生效之前，必须将刷新令牌加载到 oidc agent 中。这通过 `oidc-add` 命令完成（例如 `oidc-add XDC`）。这通常在每个登录会话中执行一次。完整详情以及如何向你的 OIDC 提供者注册 oidc-agent，请参见 [oidc-agent 文档](https://indigo-dc.gitbooks.io/oidc-agent/)。

rclone 的 `bearer_token_command` 配置选项用于从 oidc-agent 获取访问令牌。

按正常 WebDAV 端点配置，使用 'other' 提供者，用户名和密码留空。当提示时，选择编辑高级配置并输入获取 Bearer 令牌的命令（例如 `oidc-agent XDC`）。

以下示例配置展示了一个使用 oidc-agent 从 *XDC* OIDC 提供者提供访问令牌的 WebDAV 端点。

```ini
[dcache]
type = webdav
url = https://dcache.example.org/
vendor = other
bearer_token_command = oidc-token XDC
```

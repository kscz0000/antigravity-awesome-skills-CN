# Mail.ru Cloud

[Mail.ru Cloud](https://cloud.mail.ru/) 是由俄罗斯互联网公司
[Mail.Ru Group](https://mail.ru) 提供的云存储服务。官方桌面客户端为
[Disk-O:](https://disk-o.cloud/en)，支持 Windows 和 Mac OS。

## 功能亮点

- 路径可以按需嵌套任意深度，例如 `remote:directory/subdirectory`
- 文件具有 `last modified time`（最后修改时间）属性，目录则没有
- 删除的文件默认会被移到回收站
- 文件和目录可以通过公开链接共享
- 不支持部分上传或流式上传，上传前必须知道文件大小
- 免费账户单文件大小限制为 2G，付费账户无限制
- 存储系统会为所有文件保存哈希值并执行透明去重，
  哈希算法为修改版 SHA1
- 如果某个文件已存在于存储中，可以快速提交文件哈希来代替
  漫长的文件上传（rclone 支持此优化）

## 配置

以下是一个创建 mailru 配置的示例。

首先创建一个 Mail.ru Cloud 账户并选择套餐。

你需要登录并为 rclone 创建一个应用密码。Rclone **无法使用**
你的普通用户名和密码——会报类似 `oauth2: server response missing access_token` 的错误。

- 点击右上角你的用户图标
- 进入 Security / "Пароль и безопасность"
- 点击 password for apps / "Пароли для внешних приложений"
- 添加密码——为其命名，例如 "rclone"
- 选择权限级别。由于某些原因，仅选择"Full access to Cloud"
  （WebDav）目前在 Rclone 中不起作用。你必须选择"Full access
  to Mail, Cloud and Calendar"（所有协议）。
  （[forum.rclone.org 上的讨论帖](https://forum.rclone.org/t/failed-to-create-file-system-for-mailru-failed-to-authorize-oauth2-invalid-username-or-password-username-or-password-is-incorrect/49298)）
- 复制密码并在下方使用此密码——你的普通登录密码
  无法使用。

此后端的配置不需要本地网页浏览器。
你可以按以下方式使用已配置的后端：

查看顶级目录

```console
rclone lsd remote:
```

创建新目录

```console
rclone mkdir remote:directory
```

列出目录内容

```console
rclone ls remote:directory
```

将 `/home/local/directory` 同步到远程路径，删除路径中多余的文件。

```console
rclone sync --interactive /home/local/directory remote:directory
```

### 修改时间和哈希

文件支持修改时间属性，精度最高为 1 秒。
目录没有修改时间，显示为"Jan 1 1970"。

文件哈希受支持，使用 Mail.ru 自定义的基于 SHA1 的算法。
如果文件大小小于等于 SHA1 块大小（20 字节），
其哈希值就是数据右侧用零字节填充后的结果。
更大文件的哈希值计算方式为：对文件数据字节与数据长度的
十进制表示拼接后进行 SHA1 运算。

### 清空回收站

删除文件或目录实际上会将其移到回收站，回收站对 rclone 不可见，
但可以在网页浏览器中查看。回收站中的文件仍然占用总配额的一部分。
如果你想清空回收站释放配额，可以使用 `rclone cleanup remote:` 命令，
该命令将永久删除所有已回收的文件。
此命令不接受任何路径参数。

### 配额信息

要查看当前配额，可以使用 `rclone about remote:`
命令，它会显示你的使用限额（配额）和当前使用量。

### 受限文件名字符

除了[默认受限字符集](/overview/#restricted-characters)之外，
以下字符也会被替换：

| 字符 | 值 | 替换为 |
| ---- |:---:|:------:|
| "    | 0x22 | ＂     |
| *    | 0x2A | ＊     |
| :    | 0x3A | ：     |
| <    | 0x3C | ＜     |
| >    | 0x3E | ＞     |
| ?    | 0x3F | ？     |
| \    | 0x5C | ＼     |
| \|   | 0x7C | ｜     |

无效的 UTF-8 字节也会被[替换](/overview/#invalid-utf8)，
因为它们无法在 JSON 字符串中使用。

### 标准选项

以下是 mailru（Mail.ru Cloud）特有的标准选项。

#### --mailru-client-id

OAuth 客户端 ID。

通常留空即可。

属性：

- Config:      client_id
- Env Var:     RCLONE_MAILRU_CLIENT_ID
- Type:        string
- Required:    false

#### --mailru-client-secret

OAuth 客户端密钥。

通常留空即可。

属性：

- Config:      client_secret
- Env Var:     RCLONE_MAILRU_CLIENT_SECRET
- Type:        string
- Required:    false

#### --mailru-user

用户名（通常是邮箱）。

属性：

- Config:      user
- Env Var:     RCLONE_MAILRU_USER
- Type:        string
- Required:    true

#### --mailru-pass

密码。

此密码必须是应用密码——rclone 无法使用你的普通密码。
有关如何创建应用密码，请参阅文档中的配置章节。


**注意** 输入内容必须经过混淆处理——参见 [rclone obscure](/commands/rclone_obscure/)。

属性：

- Config:      pass
- Env Var:     RCLONE_MAILRU_PASS
- Type:        string
- Required:    true

#### --mailru-speedup-enable

如果存在具有相同数据哈希的另一个文件，则跳过完整上传。

此功能称为"speedup"或"put by hash"。对于热门书籍、视频或音频
等普遍可用的文件尤其高效，因为文件会按哈希在所有 mailru 用户的
所有账户中搜索。如果源文件是唯一的或已加密，则此功能无意义且无效。
请注意，rclone 可能需要本地内存和磁盘空间来提前计算内容哈希，
以决定是否需要完整上传。此外，如果 rclone 事先不知道文件大小
（例如流式或部分上传），它甚至不会尝试此优化。

属性：

- Config:      speedup_enable
- Env Var:     RCLONE_MAILRU_SPEEDUP_ENABLE
- Type:        bool
- Default:     true
- Examples:
  - "true"
    - 启用
  - "false"
    - 禁用

以下是 mailru（Mail.ru Cloud）特有的高级选项。

属性：

属性：

- Config:      auth_url
- Env Var:     RCLONE_MAILRU_AUTH_URL
- Type:        string
- Required:    false

#### --mailru-token-url

令牌服务器 URL。

留空以使用提供者默认值。

属性：

- Config:      token_url
- Env Var:     RCLONE_MAILRU_TOKEN_URL
- Type:        string
- Required:    false

#### --mailru-client-credentials

使用客户端凭证 OAuth 流程。

这将使用 RFC 6749 中描述的 OAUTH2 客户端凭证流程。

请注意，并非所有后端都支持此选项。

属性：

- Config:      client_credentials
- Env Var:     RCLONE_MAILRU_CLIENT_CREDENTIALS
- Type:        bool
- Default:     false

#### --mailru-speedup-file-patterns

符合加速上传（put by hash）条件的文件名模式列表，以逗号分隔。

模式不区分大小写，可包含 '*' 或 '?' 元字符。

属性：

- Config:      speedup_file_patterns
- Env Var:     RCLONE_MAILRU_SPEEDUP_FILE_PATTERNS
- Type:        string
- Default:     "*.mkv,*.avi,*.mp4,*.mp3,*.zip,*.gz,*.rar,*.pdf"
- Examples:
  - ""
    - 空列表将完全禁用加速上传（put by hash）。
  - "*"
    - 所有文件都将尝试加速上传。
  - "*.mkv,*.avi,*.mp4,*.mp3"
    - 仅常见音频/视频文件将尝试 put by hash。
  - "*.zip,*.gz,*.rar,*.pdf"
    - 仅常见压缩包或 PDF 书籍将尝试加速上传。

#### --mailru-speedup-max-disk

此选项允许你对大文件禁用加速上传（put by hash）。

原因是预哈希计算可能会耗尽你的 RAM 或磁盘空间。

属性：

- Config:      speedup_max_disk
- Env Var:     RCLONE_MAILRU_SPEEDUP_MAX_DISK
- Type:        SizeSuffix
- Default:     3Gi
- Examples:
  - "0"
    - 完全禁用加速上传（put by hash）。
  - "1G"
    - 大于 1Gb 的文件将直接上传。
  - "3G"
    - 如果本地磁盘可用空间不足 3Gb，请选择此选项。

#### --mailru-speedup-max-memory

大于以下指定大小的文件将始终在磁盘上进行哈希计算。

属性：

- Config:      speedup_max_memory
- Env Var:     RCLONE_MAILRU_SPEEDUP_MAX_MEMORY
- Type:        SizeSuffix
- Default:     32Mi
- Examples:
  - "0"
    - 预哈希计算将始终在临时磁盘位置进行。
  - "32M"
    - 不为预哈希计算分配超过 32Mb 的 RAM。
  - "256M"
    - 你最多有 256Mb 的可用 RAM 用于哈希计算。

#### --mailru-check-hash

当文件校验和不匹配或无效时，复制操作应如何处理。

属性：

- Config:      check_hash
- Env Var:     RCLONE_MAILRU_CHECK_HASH
- Type:        bool
- Default:     true
- Examples:
  - "true"
    - 报错并失败。
  - "false"
    - 忽略并继续。

#### --mailru-user-agent

客户端内部使用的 HTTP 用户代理。

默认为 "rclone/VERSION" 或命令行提供的 "--user-agent"。

属性：

- Config:      user_agent
- Env Var:     RCLONE_MAILRU_USER_AGENT
- Type:        string
- Required:    false

#### --mailru-quirks

以逗号分隔的内部维护标志列表。

此选项不应由普通用户使用。它仅用于协助远程排查后端问题。
标志的严格含义未记录且不保证在版本间保持不变。
当后端趋于稳定后，这些怪异标志将被移除。
支持的标志：atomicmkdir binlist unknowndirs

属性：

- Config:      quirks
- Env Var:     RCLONE_MAILRU_QUIRKS
- Type:        string
- Required:    false

#### --mailru-encoding

后端的编码方式。

详见[概述中的编码部分](/overview/#encoding)。

属性：

- Config:      encoding
- Env Var:     RCLONE_MAILRU_ENCODING

## 限制

文件大小限制取决于你的账户。免费账户单文件大小限制为 2G，
付费套餐无限制。有关总上传大小限制，请参阅 Mail.ru 网站。

请注意，Mailru 不区分大小写，因此你不能同时拥有名为
"Hello.doc" 和 "hello.doc" 的文件。

# Gofile

[Gofile](https://gofile.io) 是一个内容存储与分发平台。其目标是尽可能免费或以非常低廉的价格提供尽可能多的服务。

Gofile 的初始设置需要登录到 Web 界面，并进入 "My Profile"（个人资料）部分。复制 "Account API token"（账户 API 令牌）用于配置文件。

请注意，如果您希望将 rclone 连接到 Gofile，您需要拥有一个高级账户。

## 配置

下面是一个如何创建一个名为 `remote` 的远程存储的示例。首先运行：

```console
rclone config
```

这将引导您完成一个交互式设置过程：

```text
No remotes found, make a new one?
n) New remote
s) Set configuration password
q) Quit config
n/s/q> n

Enter name for new remote.
name> remote

Option Storage.
Type of storage to configure.
Choose a number from below, or type in your own value.
XX / Gofile
   \ (gofile)
Storage> gofile

Option access_token.
API Access token
You can get this from the web control panel.
Enter a value. Press Enter to leave empty.
access_token> YOURACCESSTOKEN

Edit advanced config?
y) Yes
n) No (default)
y/n> n

Configuration complete.
Options:
- type: gofile
- access_token: YOURACCESSTOKEN
Keep this "remote" remote?
y) Yes this is OK (default)
e) Edit this remote
d) Delete this remote
y/e/d> y
```

配置完成后，您可以这样使用 `rclone`（将 `remote` 替换为您为远程存储指定的名称）：

列出您 Gofile 顶层的目录和文件

```console
rclone lsf remote:
```

要将本地目录复制到名为 backup 的 Gofile 目录

```console
rclone copy /home/source remote:backup
```

### 修改时间和哈希值

Gofile 支持修改时间，分辨率为 1 秒。

Gofile 支持 MD5 哈希，因此您可以使用 `--checksum` 标志。

### 受限的文件名字符

除了 [默认受限字符集](/overview/#restricted-characters) 之外，以下字符也会被替换：

| 字符 | 数值 | 替换为 |
| --------- |:-----:|:-----------:|
| !         | 0x21  | ！           |
| "         | 0x22  | ＂           |
| *         | 0x2A  | ＊           |
| :         | 0x3A  | ：           |
| <         | 0x3C  | ＜           |
| >         | 0x3E  | ＞           |
| ?         | 0x3F  | ？           |
| \         | 0x5C  | ＼           |
| \|        | 0x7C  | ｜           |

文件名也不能以以下字符作为开头或结尾。这些字符仅当它们处于名称的首位或末位时才会被替换：

| 字符 | 数值 | 替换为 |
| --------- |:-----:|:-----------:|
| .         | 0x2E  | ．           |

无效的 UTF-8 字节也将被 [替换](/overview/#invalid-utf8)，因为它们无法在 JSON 字符串中使用。

### 公共链接

Gofile 支持 `rclone link` 来为文件或目录创建公共链接。如果您指定一个目录，它将以 `zip` 文件形式下载。您可以使用 `--expire` 标志指定链接的有效时间。请注意，`rclone link --unlink` 会移除该文件的所有公共链接。

### 根文件夹 ID

您可以为 rclone 设置 `root_folder_id`。这是 rclone 视为您 Gofile 云盘根目录的目录（通过其 `Folder ID` 标识）。

通常您会将其留空，rclone 将自行确定要使用的正确根目录并在配置文件中填充该值。

但是您也可以将其设置为某个值，以将 rclone 限制到特定的文件夹层级。

为此，您需要找到您希望 rclone 显示的目录的 `Folder ID`。

您可以使用 rclone 来完成此操作

```console
$ rclone lsf -Fip --dirs-only remote:
d6341f53-ee65-4f29-9f59-d11e8070b2a0;Files/
f4f5c9b8-6ece-478b-b03e-4538edfe5a1c;Photos/
d50e356c-29ca-4b27-a3a7-494d91026e04;Videos/
```

要使用的 ID 是 `;` 前面的部分，因此您可以设置

```text
root_folder_id = d6341f53-ee65-4f29-9f59-d11e8070b2a0
```

以将 rclone 限制到 `Files` 目录。

## 限制

Gofile 仅支持最长为 255 个字符的文件名，其中一个字符为一个 Unicode 字符。

目录的缓存时间不应超过 24 小时，否则目录中的文件可能无法下载。在实践中，这意味着当使用基于 VFS 的 rclone 命令（例如 `rclone mount`）时，您应确保 `--dir-cache-time` 小于 `24h`。

请注意，Gofile 目前限制总项目数为 100,000。如果您尝试上传更多，您将得到 `error-limit-100000`。此限制将来可能会放宽。

### 重复文件

Gofile 能够存在文件名重复的文件。例如，同一目录下两个名为 `hello.txt` 的文件。

Rclone 无法将其同步到普通文件系统，但可以使用 `rclone dedupe` 命令进行修复。

重复的文件会导致同步出现问题，您将在日志中看到有关重复项的消息。

使用 `rclone dedupe` 来修复重复文件。

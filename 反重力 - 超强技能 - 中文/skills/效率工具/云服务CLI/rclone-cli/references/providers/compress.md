# Compress

## 警告

此远程当前处于 **实验性** 阶段。功能可能损坏，数据可能丢失。
您使用此远程所做的任何操作均由您自行承担风险。请务必了解
使用实验性代码所带来的风险，请勿在关键
业务中使用此远程。

`Compress` 远程为另一个远程添加压缩功能。最适合与
包含大量可压缩文件的远程搭配使用。

### 压缩算法

- **GZIP** – 一种成熟且被广泛采用的算法，在压缩速度和压缩比之间取得了良好的平衡。支持 -2 到 9 的压缩级别，默认的 -1（大致相当于级别 5）为大多数场景提供了有效的中间选择。

- **Zstandard (zstd)** – 一种现代的高性能算法，可精确控制速度与压缩效率之间的权衡。压缩级别范围为 0（不压缩）到 4（最大压缩）。

### 文件类型

打开由 compress 包装的远程时，您会看到许多
扩展名与所选压缩算法相对应的文件。
这些文件是标准文件，可由各种归档程序打开，
但它们包含一些隐藏的元数据，可供 rclone 使用。
您可以随时下载并解压这些文件，但**请勿**手动
删除或重命名文件。缺少正确元数据的文件将无法被
rclone 识别。

### 文件名

压缩后的文件将命名为 `*.###########.gz`，其中 `*` 是原始
文件名，`#` 部分是未压缩文件大小的 base64 编码。除 rclone 压缩后端外，
不应通过其他方式修改这些文件名。

### 标准选项

以下是 compress（Compress a remote）特有的标准选项。

#### --compress-remote

要压缩的远程。

属性：

- Config:      remote
- Env Var:     RCLONE_COMPRESS_REMOTE
- Type:        string
- Required:    true

#### --compress-mode

压缩模式。

属性：

- Config:      mode
- Env Var:     RCLONE_COMPRESS_MODE
- Type:        string
- Default:     "gzip"
- 示例：
  - "gzip"
    - 使用最快参数的标准 gzip 压缩。
  - "zstd"
    - Zstandard 压缩 — 现代快速算法，可调节速度与压缩比的权衡。

#### --compress-level

GZIP（级别 -2 至 9）：
- -2 — 仅 Huffman 编码。仅在您清楚自己在做什么时使用。
- -1（默认）— 推荐；相当于级别 5。
- 0 — 关闭压缩。
- 1–9 — 以速度为代价提高压缩率。超过 6 之后通常收益甚微。

ZSTD（级别 0 至 4）：
- 0 — 完全关闭压缩。
- 1 — 压缩速度最快，但压缩比最低。
- 2（默认）— 速度与压缩比之间的良好平衡。
- 3 — 更好的压缩比，但 CPU 占用约为默认值的 2–3 倍。
- 4 — 最佳的压缩比（CPU 占用最高）。

备注：
- 选择 GZIP 以获得广泛的兼容性；选择 ZSTD 以获得更好的速度/压缩比权衡。
- 负数 gzip 级别：-2 = 仅 Huffman，-1 = 默认（约等于级别 5）。

属性：

- Config:      level
- Env Var:     RCLONE_COMPRESS_LEVEL
- Type:        string
- Required:    true

### 高级选项

以下是 compress（Compress a remote）特有的高级选项。

#### --compress-ram-cache-limit

部分远程不允许上传大小未知的文件。
在这种情况下，需要缓存压缩后的文件以确定
其大小。

小于此限制的文件将在 RAM 中缓存，大于
此限制的文件将缓存到磁盘上。

属性：

- Config:      ram_cache_limit
- Env Var:     RCLONE_COMPRESS_RAM_CACHE_LIMIT
- Type:        SizeSuffix
- Default:     20Mi

#### --compress-description

远程的描述。

属性：

- Config:      description
- Env Var:     RCLONE_COMPRESS_DESCRIPTION
- Type:        string
- Required:    false

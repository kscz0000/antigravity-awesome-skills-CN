# Quatrix

Quatrix by Maytech 是 [Quatrix Secure Compliant File Sharing | Maytech](https://www.maytech.net/products/quatrix-business)。

路径以 `remote:path` 的形式指定。

路径可以根据需要任意深，例如 `remote:directory/subdirectory`。

Quatrix 的初始设置需要从 Quatrix 获取 API Key。你可以在用户的 profile 页面
`https://<account>/profile/api-keys` 获取 API key，或者通过 API 辅助获取 — <https://docs.maytech.net/quatrix/quatrix-api/api-explorer#/API-Key/post_api_key_create>。

请参阅 Quatrix 完整的 [Swagger 文档](https://docs.maytech.net/quatrix/quatrix-api/api-explorer)。

## Configuration

下面示例展示如何创建一个名为 `remote` 的远端。首先运行：

```console
rclone config
```

随后将引导你完成交互式配置流程：

```text

n) New remote
s) Set configuration password
q) Quit config
n/s/q> n
name> remote
Type of storage to configure.
Choose a number from below, or type in your own value
[snip]
XX / Quatrix by Maytech
   \ "quatrix"
[snip]
Storage> quatrix
API key for accessing Quatrix account.
api_key> your_api_key
Host name of Quatrix account.
host> example.quatrix.it

Configuration complete.
Options:
- type: quatrix
- api_key: your_api_key
- host: example.quatrix.it
Keep this "remote" remote?
y) Yes this is OK
e) Edit this remote
d) Delete this remote
y/e/d> y
```

配置完成后，你就可以这样使用 `rclone`（将 `remote` 替换为你给远端起的名字）：

列出 Quatrix 顶层目录：

```console
rclone lsd remote:
```

列出 Quatrix 中的所有文件：

```console
rclone ls remote:
```

将本地目录拷贝到名为 backup 的 Quatrix 目录：

```console
rclone copy /home/source remote:backup
```

### Modification times and hashes

Quatrix 允许在对象上设置精度为 1 微秒的修改时间。这些修改时间将用于检测
对象是否需要同步。

Quatrix 不支持哈希，因此你无法使用 `--checksum` 标志。

### Restricted filename characters

Quatrix 中的文件名区分大小写，并且存在限制：文件名最大长度为 255，最小
长度为 1。文件名不能等于 `.` 或 `..`，也不能包含 `/`、`\` 或不可打印的 ASCII 字符。

### Transfers

对于超过 50 MiB 的文件，rclone 将使用分块传输。Rclone 将同时上传最多
`--transfers` 个分块（在所有分片上传之间共享）。分块在内存中缓冲，
默认最小分块大小为 10_000_000 字节，可以在高级配置中修改，因此增加
`--transfers` 会增加内存占用。分块大小有一个最大上限，默认设置为
100_000_000 字节，也可以在高级配置中修改。已上传分块的大小将根据
上传速度动态变化。总内存占用等于传输数乘以最小分块大小。如果上传
时有可用空闲内存（等于 `maximal_summary_chunk_size` 与 `minimal_chunk_size` *
`transfers` 之差），那么在上传速度较高时，分块大小可能会增大；当上传
速度出现问题时，也可能减小。如果没有可用空闲内存，所有分块都将等于
`minimal_chunk_size`。

### Deleting files

使用 rclone 删除的文件将进入回收站并在那里保存 30 天。Quatrix 还提供了
用于永久删除文件的 API 和清空回收站的 API，以便你可以从账户中永久删除
文件。

## Storage usage

Quatrix 的存储用量在购买时被限制为该账户的上限。你可以为任何用户设置一个
更小的存储上限。如果用户没有自定义存储上限，则使用账户上限。一旦达到上限，
文件上传将会失败。这可以通过释放空间或增加配额来解决。

## Server-side operations

Quatrix 支持服务端操作（拷贝和移动）。如果发生冲突，服务端操作期间
文件将被覆盖。

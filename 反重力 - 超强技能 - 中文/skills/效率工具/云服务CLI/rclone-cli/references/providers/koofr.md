# Koofr

路径以 `remote:path` 形式指定

路径深度可按需指定,例如 `remote:directory/subdirectory`。

## 配置

Koofr 的初始设置需要为 rclone 创建一个应用密码。
你可以通过打开 Koofr
[web 应用](https://app.koofr.net/app/admin/preferences/password) 来完成,
为该密码起一个好记的名字(例如 `rclone`),然后点击 generate。

下面示例演示如何创建一个名为 `koofr` 的远程。首先运行:

```console
rclone config
```

这将引导你完成交互式配置过程:

你可以选择编辑高级配置,以便在使用本地部署或白标 Koofr 实例时输入自己的服务 URL,或者选择主存储以外的替代挂载点。

配置完成后,即可像下面这样使用 `rclone`:

列出 Koofr 顶层目录

```console
rclone lsd koofr:
```

列出 Koofr 中的所有文件

```console
rclone ls koofr:
```

将本地目录复制到名为 backup 的 Koofr 目录

```console
rclone copy /home/source koofr:backup
```

### 受限文件名字符

除 [默认受限字符集](/overview/#restricted-characters) 外,以下字符也会被替换:

| 字符 | 值 | 替换为 |
| --------- |:-----:|:-----------:|
| \         | 0x5C  | ＼           |

无效的 UTF-8 字节也会被 [替换](/overview/#invalid-utf8),
因为它们无法在 XML 字符串中使用。

### 标准选项

以下是 koofr(Koofr、Digi Storage 及其他 Koofr 兼容存储提供商)特有的标准选项。

#### --koofr-provider

选择你的存储提供商。

属性:

- Config(配置项):      provider
- Env Var(环境变量):   RCLONE_KOOFR_PROVIDER
- Type(类型):          string
- Required(是否必填):  false
- Examples(示例):
  - "koofr"
    - Koofr, https://app.koofr.net/
  - "digistorage"
    - Digi Storage, https://storage.rcs-rds.ro/
  - "other"
    - Any other Koofr API compatible storage service

#### --koofr-endpoint

要使用的 Koofr API 端点。

属性:

- Config(配置项):      endpoint
- Env Var(环境变量):   RCLONE_KOOFR_ENDPOINT
- Provider(提供商):    other
- Type(类型):          string
- Required(是否必填):  true

#### --koofr-user

你的用户名。

属性:

- Config(配置项):      user
- Env Var(环境变量):   RCLONE_KOOFR_USER
- Type(类型):          string
- Required(是否必填):  true

#### --koofr-password

你的 rclone 密码,请前往 https://app.koofr.net/app/admin/preferences/password 生成。

**NB** 输入的内容必须经过混淆处理——参见 [rclone obscure](/commands/rclone_obscure/)。

属性:

- Config(配置项):      password
- Env Var(环境变量):   RCLONE_KOOFR_PASSWORD
- Provider(提供商):    koofr
- Type(类型):          string
- Required(是否必填):  true

### 高级选项

以下是 koofr(Koofr、Digi Storage 及其他 Koofr 兼容存储提供商)特有的高级选项。

#### --koofr-mountid

要使用的挂载点的 Mount ID。

如果省略,则使用主挂载点。

属性:

- Config(配置项):      mountid
- Env Var(环境变量):   RCLONE_KOOFR_MOUNTID
- Type(类型):          string
- Required(是否必填):  false

#### --koofr-setmtime

后端是否支持设置修改时间。

如果你使用的 mount ID 指向 Dropbox 或 Amazon Drive 后端,请将其设为 false。

属性:

- Config(配置项):      setmtime
- Env Var(环境变量):   RCLONE_KOOFR_SETMTIME
- Type(类型):          bool
- Default(默认值):     true

#### --koofr-encoding

后端的编码方式。

更多信息参见 [overview 中的 encoding 章节](/overview/#encoding)。

属性:

- Config(配置项):      encoding
- Env Var(环境变量):   RCLONE_KOOFR_ENCODING
- Type(类型):          Encoding
- Default(默认值):     Slash,BackSlash,Del,Ctl,InvalidUtf8,Dot

#### --koofr-description

远程的描述。

属性:

- Config(配置项):      description
- Env Var(环境变量):   RCLONE_KOOFR_DESCRIPTION
- Type(类型):          string
- Required(是否必填):  false

## 限制

注意 Koofr 不区分大小写,因此你不能同时拥有名为
"Hello.doc" 和 "hello.doc" 的文件。

## 提供商

### Koofr

这是最初始的 [Koofr](https://koofr.eu) 存储提供商,作为主要示例出现,
其配置详见上文 [配置](#配置) 章节。

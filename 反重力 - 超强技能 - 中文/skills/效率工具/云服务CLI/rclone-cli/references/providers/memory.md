# Memory

Memory 后端是一个运行在内存（RAM）中的后端。它不会持久化其
数据 —— 如需持久化，请使用本地后端。

Memory 后端的行为类似于基于存储桶的远程（例如类似
s3）。由于它没有任何参数，你可以直接使用
`:memory:` 作为远程名称来访问它。

## 配置

如果需要，你也可以像下面这样使用 `rclone config` 将其配置为远程：

```text
No remotes found, make a new one\?
n) New remote
s) Set configuration password
q) Quit config
n/s/q> n
name> remote
Type of storage to configure.
Enter a string value. Press Enter for the default ("").
Choose a number from below, or type in your own value
[snip]
XX / Memory
   \ "memory"
[snip]
Storage> memory
** See help for memory backend at: https://rclone.org/memory/ **

Remote config

Configuration complete.
Options:
- type: memory
Keep this "remote" remote?
y) Yes this is OK (default)
e) Edit this remote
d) Delete this remote
y/e/d> y
```

由于内存后端不具备持久性，它最适合用于
测试场景，或配合 rclone server 或 rclone mount 使用，例如：

```console
rclone mount :memory: /mnt/tmp
rclone serve webdav :memory:
rclone serve sftp :memory:
```

### 修改时间和哈希

内存后端支持 MD5 哈希，以及精确到 1 纳秒的修改时间。

### 高级选项

以下是 memory（内存对象存储系统。）特有的高级选项。

#### --memory-discard

如果设置，所有写入操作将被丢弃，并且读取操作将返回错误

如果设置，那么文件上传时其内容将不会被保存。文件
看似已成功上传，但在读取时会产生错误。上传时会
计算文件的 MD5 校验和，这只占用极少的 CPU 时间，并且可以
用于校验传输结果。

这对于测试性能非常有用。

最简便的使用方式是通过连接字符串语法：

    :memory,discard:bucket



Properties:

- Config:      discard
- Env Var:     RCLONE_MEMORY_DISCARD
- Type:        bool
- Default:     false

#### --memory-description

远程的描述。

Properties:

- Config:      description
- Env Var:     RCLONE_MEMORY_DESCRIPTION

# Pixeldrain

这是 Pixeldrain 高级文件系统功能的后端。它与 pixeldrain 的免费文件分享产品不同。该文件系统需要 Pro 订阅或 Prepaid 套餐。[订阅详情](https://pixeldrain.com/#pro)。

有关该文件系统功能与限制的概览，请参阅 pixeldrain 上的[文件系统指南](https://pixeldrain.com/filesystem)。

## 配合账户使用

要使用个人文件系统，你需要先注册一个 [pixeldrain 账户](https://pixeldrain.com/register)，并购买 Prepaid 套餐或任意一种基于 Patreon 的订阅。注册并订阅之后，你的个人文件系统可以通过以下链接访问：<https://pixeldrain.com/d/me>。

前往账户的 [API 密钥页面](https://pixeldrain.com/user/api_keys) 为 rclone 生成一个新的 API 密钥。然后运行 `rclone config`，使用该 API 密钥创建一个新的后端。

## 不使用账户

通过 rclone 可以以只读方式访问公开共享的目录。为此你只需要一个目录 ID。该目录 ID 可以在共享目录的 URL 中找到，URL 形如 `https://pixeldrain.com/d/abcd1234`，其中 `abcd1234` 即为目录 ID。你自己文件系统中的目录 ID 也可以通过 `lsf` 命令列出：

`rclone lsf Pixeldrain: --dirs-only -Fpi`

此命令会在你的 `Pixeldrain` 主目录下打印目录及其公开 ID。

将该目录 ID 输入到 rclone 配置中，即可以访问该目录。

### 标准选项

以下是 pixeldrain（Pixeldrain Filesystem）特有的标准选项。

#### --pixeldrain-api-key

你 pixeldrain 账户的 API 密钥。
参见 https://pixeldrain.com/user/api_keys。

Properties:

- Config:      api_key
- Env Var:     RCLONE_PIXELDRAIN_API_KEY
- Type:        string
- Required:    false

#### --pixeldrain-root-folder-id

要使用的文件系统根目录。

Set to 'me' to use your personal filesystem. Set to a shared directory ID to use a shared directory.

Properties:

- Config:      root_folder_id
- Env Var:     RCLONE_PIXELDRAIN_ROOT_FOLDER_ID
- Type:        string
- Default:     "me"

### 高级选项

以下是 pixeldrain（Pixeldrain Filesystem）特有的高级选项。

#### --pixeldrain-api-url

要连接的 API 端点。在绝大多数情况下，保持默认值即可。仅在测试场景下才需要修改。

Properties:

- Config:      api_url
- Env Var:     RCLONE_PIXELDRAIN_API_URL
- Type:        string
- Default:     "https://pixeldrain.com/api"

#### --pixeldrain-description

远端的描述信息。

Properties:

- Config:      description
- Env Var:     RCLONE_PIXELDRAIN_DESCRIPTION
- Type:        string
- Required:    false

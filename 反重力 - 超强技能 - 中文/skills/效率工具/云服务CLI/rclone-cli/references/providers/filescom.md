# Files.com

[Files.com](https://www.files.com/) 是一个云存储服务，提供安全便捷的文件存储与共享方式。

filescom 的初始设置需要使用你的 Files.com 账户进行身份验证。你可以通过提供站点子域名、用户名和密码来完成验证。此外，你也可以使用来自 [Files.com](https://www.files.com/docs/sdk-and-apis/api-keys/) 的 API Key 进行身份验证。
`rclone config` 命令会引导你完成整个配置过程。

## Configuration

下面演示如何创建一个名为 `remote` 的远程存储。首先运行：

```console
rclone config
```

该命令将引导你完成交互式配置过程：

配置完成后即可使用 rclone。

查看根目录下的所有文件：

```console
rclone lsf remote:
```

在根目录下创建一个新目录：

```console
rclone mkdir remote:dir
```

递归列出目录中的内容：

```console
rclone ls remote:
```

将 `/home/local/directory` 同步到远程目录，并删除该目录中的多余文件。

```console
rclone sync --interactive /home/local/directory remote:dir
```

### Hashes

2024 年 12 月，`files.com` 开始[支持更多校验和](https://www.files.com/blog/2024/11/01/new-modern-checksum-options-now-available-with-opt)。

但如果你希望使用哈希功能，**必须**在你的站点中启用它。具体做法是访问 "Data Governance" 中的 "File Integrity" 部分，配置相关设置并选择符合你需求的校验和算法。

rclone 目前仅支持在同步中使用 CRC32 和 MD5，但选择更多校验和不会影响 rclone 的正常运行。

针对 rclone 的使用，推荐至少选择 MD5，以便 rclone 能够执行端到端的完整性校验。

### Standard options

下面是 filescom (Files.com) 特有的标准选项。

#### --filescom-site

你的站点子域名（例如 mysite）或自定义域名（例如 myfiles.customdomain.com）。

Properties:

- Config:      site
- Env Var:     RCLONE_FILESCOM_SITE
- Type:        string
- Required:    false

#### --filescom-username

用于与 Files.com 进行身份验证的用户名。

Properties:

- Config:      username
- Env Var:     RCLONE_FILESCOM_USERNAME
- Type:        string
- Required:    false

#### --filescom-password

用于与 Files.com 进行身份验证的密码。

**NB** 此项输入必须经过混淆处理 —— 参见 [rclone obscure](/commands/rclone_obscure/)。

Properties:

- Config:      password
- Env Var:     RCLONE_FILESCOM_PASSWORD
- Type:        string
- Required:    false

### Advanced options

下面是 filescom (Files.com) 特有的高级选项。

#### --filescom-api-key

用于与 Files.com 进行身份验证的 API key。

Properties:

- Config:      api_key
- Env Var:     RCLONE_FILESCOM_API_KEY
- Type:        string
- Required:    false

#### --filescom-encoding

后端所使用的编码方式。

更多信息请参见 [overview 中的 encoding 章节](/overview/#encoding)。

Properties:

- Config:      encoding
- Env Var:     RCLONE_FILESCOM_ENCODING
- Type:        Encoding
- Default:     Slash,BackSlash,Del,Ctl,RightSpace,RightCrLfHtVt,InvalidUtf8,Dot

#### --filescom-description

远程存储的描述信息。

Properties:

- Config:      description
- Env Var:     RCLONE_FILESCOM_DESCRIPTION
- Type:        string
- Required:    false

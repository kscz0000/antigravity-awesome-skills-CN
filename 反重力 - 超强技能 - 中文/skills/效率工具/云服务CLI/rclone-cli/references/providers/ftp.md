# FTP

FTP 即文件传输协议。Rclone 的 FTP 支持由
[github.com/jlaffaye/ftp](https://godoc.org/github.com/jlaffaye/ftp)
包提供。

[Rclone FTP 后端的限制](#limitations)

路径指定为 `remote:path`。如果路径不以
`/` 开头，则相对于用户的家目录。空路径
`remote:` 指向用户的家目录。

## 配置

要创建一个名为 `remote` 的 FTP 配置，运行

```console
rclone config
```

Rclone 配置会引导你完成交互式设置过程。一个最小的
rclone FTP 远程定义只需要主机、用户名和密码。
对于匿名 FTP 服务器，参见[下文](#anonymous-ftp)。

查看 `remote` 家目录下的所有目录

```console
rclone lsd remote:
```

创建新目录

```console
rclone mkdir remote:path/to/directory
```

列出目录内容

```console
rclone ls remote:path/to/directory
```

将 `/home/local/directory` 同步到远程目录，删除远程目录中多余的文件。

```console
rclone sync --interactive /home/local/directory remote:directory
```

### 匿名 FTP

当连接到允许匿名登录的 FTP 服务器时，可以使用
特殊的 "anonymous" 用户名。按照惯例，此用户账户接受任意
字符串作为密码，不过通常使用 "anonymous" 或 "guest" 作为密码。
有些服务器要求使用有效的电子邮件地址作为密码。

使用[即时](#backend-path-to-dir)或
[连接字符串](/docs/#connection-strings)远程可以轻松访问
此类服务器，无需提前进行任何配置。以下是示例：

```console
rclone lsf :ftp: --ftp-host=speedtest.tele2.net --ftp-user=anonymous --ftp-pass=$(rclone obscure dummy)
rclone lsf :ftp,host=speedtest.tele2.net,user=anonymous,pass=$(rclone obscure dummy):
```

上述示例在 Linux Shell 和 PowerShell 中可用，但在 Windows
命令提示符中不可用。它们执行 [rclone obscure](/commands/rclone_obscure/)
命令来创建 [pass](#ftp-pass) 选项所需格式的密码字符串。
以下示例完全相同，只是使用了同一密码 "dummy" 的已混淆字符串表示，
因此即使在 Windows 命令提示符中也能正常工作：

```console
rclone lsf :ftp: --ftp-host=speedtest.tele2.net --ftp-user=anonymous --ftp-pass=IXs2wc8OJOz7SYLBk47Ji1rHTmxM
rclone lsf :ftp,host=speedtest.tele2.net,user=anonymous,pass=IXs2wc8OJOz7SYLBk47Ji1rHTmxM:
```

### 隐式 TLS

Rclone FTP 支持隐式 FTP over TLS 服务器（FTPS）。这需要在
FTP 后端配置中为该远程启用，或使用
[`--ftp-tls`](#ftp-tls)。默认的 FTPS 端口是 `990`，而不是 `21`，
可通过 [`--ftp-port`](#ftp-port) 设置。

## TLS 选项

隐式和显式 TLS 的 TLS 选项可以使用以下 FTP 后端专用标志进行设置：

```text
--ftp-no-check-certificate     Do not verify the TLS certificate of the server
--ftp-disable-tls13            Disable TLS 1.3 (workaround for FTP servers with buggy TLS)
--ftp-tls-cache-size int       Size of TLS session cache for all control and data connections (default 32)
```

不过也可以使用任何全局 TLS 标志，例如：

```text
--ca-cert stringArray          CA certificate used to verify servers
--client-cert string           Client SSL certificate (PEM) for mutual TLS auth
--client-key string            Client SSL private key (PEM) for mutual TLS auth
--no-check-certificate         Do not verify the server SSL certificate (insecure)
```

如果需要将这些放入配置文件中以便仅应用于 FTP 后端，
则使用 `override` 语法，例如

```text
override.ca_cert = XXX
override.client_cert = XXX
override.client_key = XXX
```

### 受限文件名字符

除了[默认受限字符集](/overview/#restricted-characters)之外，
还会替换以下字符：

文件名不能以以下字符结尾。替换仅限于文件名的最后一个字符：

| 字符 | 值 | 替换为 |
| ---- |:---:|:------:|
| SP   | 0x20 | ␠     |

并非所有 FTP 服务器都允许文件名中包含所有字符，例如：

| FTP 服务器 | 禁止字符 |
| ---------- |:--------:|
| proftpd    | `*`      |
| pureftpd   | `\ [ ]`  |

此后端的交互式配置向导提供了主要 FTP 服务器的合理编码设置选项：ProFTPd、PureFTPd、VsFTPd。
在提示时直接输入选择编号即可。

### 标准选项

以下是 ftp（FTP）特定的标准选项。

要连接的 FTP 主机。

例如 "ftp.example.com"。

属性：

- 配置项：    host

#### --ftp-user

FTP 用户名。

属性：

- 配置项：    user
- 环境变量：  RCLONE_FTP_USER
- 类型：      string
- 默认值：    "$USER"

#### --ftp-port

FTP 端口号。

属性：

- 配置项：    port
- 环境变量：  RCLONE_FTP_PORT
- 类型：      int
- 默认值：    21

#### --ftp-pass

FTP 密码。

**注意** 输入内容必须经过混淆处理 - 参见 [rclone obscure](/commands/rclone_obscure/)。

属性：

- 配置项：    pass
- 环境变量：  RCLONE_FTP_PASS
- 类型：      string
- 必填：      false

#### --ftp-tls

使用隐式 FTPS（FTP over TLS）。

使用隐式 FTP over TLS 时，客户端从一开始就使用 TLS
连接，这会破坏与非 TLS 感知服务器的兼容性。这通常通过
端口 990 而非端口 21 提供服务。不能与显式 FTPS 组合使用。

属性：

- 配置项：    tls
- 环境变量：  RCLONE_FTP_TLS
- 类型：      bool
- 默认值：    false

#### --ftp-explicit-tls

使用显式 FTPS（FTP over TLS）。

使用显式 FTP over TLS 时，客户端显式请求服务器进行安全升级，
将明文连接升级为加密连接。不能与隐式 FTPS 组合使用。

属性：

- 配置项：    explicit_tls
- 环境变量：  RCLONE_FTP_EXPLICIT_TLS
- 类型：      bool
- 默认值：    false

### 高级选项

以下是 ftp（FTP）特定的高级选项。

#### --ftp-concurrency

FTP 最大同时连接数，0 表示无限制。

注意，设置此值很可能导致死锁，因此应谨慎使用。

如果执行 sync 或 copy 操作，请确保并发数比
`--transfers` 与 `--checkers` 之和大 1。

如果使用 `--check-first`，则只需比 `--checkers` 和
`--transfers` 中的较大值大 1。

因此对于 `concurrency 3`，应使用 `--checkers 2 --transfers 2
--check-first` 或 `--checkers 1 --transfers 1`。



属性：

- 配置项：    concurrency
- 环境变量：  RCLONE_FTP_CONCURRENCY

## 限制

作为 rclone 远程的 FTP 服务器必须支持 `被动` 模式。
该模式无法配置，因为 `被动` 是唯一支持的模式。
Rclone 的 FTP 实现与 `主动` 模式不兼容，
因为[其使用的库不支持该模式](https://github.com/jlaffaye/ftp/issues/29)。
由于安全方面的考虑，这很可能永远不会被支持。

Rclone 的 FTP 后端不支持任何校验和，但可以比较文件大小。

`rclone about` 不受 FTP 后端支持。不具备此功能的后端
无法为 rclone 挂载确定可用空间，也无法在 rclone union
远程中使用 `mfs`（最大可用空间）策略。

参见[不支持 rclone about 的后端列表](https://rclone.org/overview/#optional-features)
和 [rclone about](https://rclone.org/commands/rclone_about/)。

`--dump headers`、`--dump bodies`、`--dump auth` 的调试
实现与基于 HTTP 的 rclone 后端不同——其控制粒度较粗。

不支持 `--timeout`（但支持 `--contimeout`）。

不支持 `--bind`。

Rclone 的 FTP 后端可以支持服务器端移动操作，但目前尚未实现。

目前不支持 `ftp_proxy` 环境变量。

### 修改时间

对于主要的 FTP 服务器：ProFTPd、PureFTPd、VsFTPd 和 FileZilla FTP 服务器，
文件修改时间（时间戳）支持 1 秒精度。
`VsFTPd` 服务器的时间相关协议命令实现不符合标准，需要
特殊配置设置：`writing_mdtm = true`。

其他 FTP 服务器对精确文件时间的支持取决于它们声明的协议扩展。
如果 `MLSD`、`MDTM` 和 `MFTM` 扩展全部存在，rclone 会将它们
组合使用以提供精确时间。否则，你通过 rclone 在 FTP 服务器上
看到的时间是上次文件上传的时间。

你可以使用以下命令检查 rclone 是否能对你的 FTP 服务器使用精确时间：
`rclone backend features your_ftp_remote:`（末尾的冒号很重要）。
查找标有 `Precision` 的行中的数字，该数字表示以纳秒为单位的远程时间精度。
值为 `1000000000` 表示文件时间精度为 1 秒。
值为 `3153600000000000000`（或其他大数）表示"不支持"。

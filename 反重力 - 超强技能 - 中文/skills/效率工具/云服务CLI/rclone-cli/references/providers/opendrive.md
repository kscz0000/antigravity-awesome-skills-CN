# OpenDrive

路径以 `remote:path` 的形式指定

路径可以根据需要嵌套多层，例如 `remote:directory/subdirectory`。

## 配置

下面是一个创建名为 `remote` 的远程存储的示例。首先运行：

```console
rclone config
```

随后将引导你完成交互式配置过程：

```text
n) New remote
d) Delete remote
q) Quit config
e/n/d/q> n
name> remote
Type of storage to configure.
Choose a number from below, or type in your own value
[snip]
XX / OpenDrive
   \ "opendrive"
[snip]
Storage> opendrive
Username
username>
Password
y) Yes type in my own password
g) Generate random password
y/g> y
Enter the password:
password:
Confirm the password:
password:
Configuration complete.
Options:
- type: opendrive
- username:
- password: *** ENCRYPTED ***
Keep this "remote" remote?
y) Yes this is OK
e) Edit this remote
d) Delete this remote
y/e/d> y
```

列出你的 OpenDrive 顶层目录：

```console
rclone lsd remote:
```

列出你的 OpenDrive 中的所有文件：

```console
rclone ls remote:
```

将本地目录复制到名为 backup 的 OpenDrive 目录：

```console
rclone copy /home/source remote:backup
```

### 修改时间和哈希

OpenDrive 允许对对象设置精确到 1 秒的修改时间。这些时间将用于检测对象是否需要同步。

支持 MD5 哈希算法。

### 受限的文件名字符

| Character | Value | Replacement |
| --------- |:-----:|:-----------:|
| NUL       | 0x00  | ␀           |
| /         | 0x2F  | ／          |
| "         | 0x22  | ＂          |
| *         | 0x2A  | ＊          |
| :         | 0x3A  | ：          |
| <         | 0x3C  | ＜          |
| >         | 0x3E  | ＞          |
| ?         | 0x3F  | ？          |
| \         | 0x5C  | ＼          |
| \|        | 0x7C  | ｜          |

文件名也不能以下列字符开头或结尾。只有当这些字符出现在文件名的首位或末尾时，才会被替换：

| Character | Value | Replacement |
| --------- |:-----:|:-----------:|
| SP        | 0x20  | ␠           |
| HT        | 0x09  | ␉           |
| LF        | 0x0A  | ␊           |
| VT        | 0x0B  | ␋           |
| CR        | 0x0D  | ␍           |

无效的 UTF-8 字节也会被[替换](/overview/#invalid-utf8)，因为它们无法用于 JSON 字符串。

### 标准选项

以下是 opendrive (OpenDrive) 特有的标准选项。

#### --opendrive-username

用户名。

属性：

- Config:      username
- Env Var:     RCLONE_OPENDRIVE_USERNAME
- Type:        string
- Required:    true

#### --opendrive-password

密码。

**注意** 此项输入必须经过混淆处理，参见 [rclone obscure](/commands/rclone_obscure/)。

属性：

- Config:      password
- Env Var:     RCLONE_OPENDRIVE_PASSWORD
- Type:        string
- Required:    true

### 高级选项

以下是 opendrive (OpenDrive) 特有的高级选项。

#### --opendrive-encoding

后端的编码方式。

更多信息参见[概述中的编码章节](/overview/#encoding)。

属性：

- Config:      encoding
- Env Var:     RCLONE_OPENDRIVE_ENCODING
- Type:        Encoding
- Default:     Slash,LtGt,DoubleQuote,Colon,Question,Asterisk,Pipe,BackSlash,LeftSpace,LeftCrLfHtVt,RightSpace,RightCrLfHtVt,InvalidUtf8,Dot

#### --opendrive-chunk-size

文件将按此大小的分块上传。

请注意这些分块会在内存中缓冲，因此增大该值会增加内存占用。

属性：

- Config:      chunk_size
- Env Var:     RCLONE_OPENDRIVE_CHUNK_SIZE
- Type:        SizeSuffix
- Default:     10Mi

#### --opendrive-access

文件和文件夹将以此访问权限上传（默认 private）。

属性：

- Config:      access
- Env Var:     RCLONE_OPENDRIVE_ACCESS
- Type:        string
- Default:     "private"
- Examples:
  - "private"
    - 文件或文件夹的访问权限仅授予其必要操作所需的那部分用户，允许其查看、读取或写入。
  - "public"
    - 任何人都可以通过网页浏览器下载该文件或文件夹。该链接可以以任何方式共享。
  - "hidden"
    - 文件或文件夹的访问权限与 Public 类似，但前提是用户必须知道该文件或文件夹链接的 URL 才能访问其内容。

#### --opendrive-description

远程存储的描述。

属性：

- Config:      description
- Env Var:     RCLONE_OPENDRIVE_DESCRIPTION
- Type:        string
- Required:    false

## 限制说明

请注意 OpenDrive 大小写不敏感，因此不能同时存在名为 "Hello.doc" 和 "hello.doc" 的文件。

OpenDrive 文件名中有相当多字符无法使用。这些字符在 Windows 平台上不会出现，但在非 Windows 平台上却很常见。rclone 会将这些名称与外观相同的 Unicode 等价字符进行相互映射。例如，如果一个文件包含 `?`，它将被映射为 `？`。

OpenDrive 后端不支持 `rclone about`。不支持该功能的后端无法确定 rclone mount 的剩余空间，也不能在 rclone union 远程存储中使用 `mfs`（most free space，最多剩余空间）策略。

参见[不支持 rclone about 的后端列表](https://rclone.org/overview/#optional-features)以及 [rclone about](https://rclone.org/commands/rclone_about/)。

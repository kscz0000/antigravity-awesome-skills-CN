# Dropbox

路径指定为 `remote:path`

Dropbox 路径可以按需嵌套任意深度，例如
`remote:directory/subdirectory`。

## 配置

Dropbox 的初始设置需要在浏览器中从 Dropbox 获取令牌。`rclone config` 会引导你完成此过程。

以下是如何创建名为 `remote` 的远程存储的示例。首先运行：

```console
rclone config
```

这将引导你完成交互式设置过程：

```text
n) New remote
d) Delete remote
q) Quit config
e/n/d/q> n
name> remote
Type of storage to configure.
Choose a number from below, or type in your own value
[snip]
XX / Dropbox
   \ "dropbox"
[snip]
Storage> dropbox
Dropbox App Key - leave blank normally.
app_key>
Dropbox App Secret - leave blank normally.
app_secret>
Remote config
Please visit:
https://www.dropbox.com/1/oauth2/authorize?client_id=XXXXXXXXXXXXXXX&response_type=code
Enter the code: XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX_XXXXXXXXXX
Configuration complete.
Options:
- type: dropbox
- app_key:
- app_secret:
- token: XXXXXXXXXXXXXXXXXXXXXXXXXXXXX_XXXX_XXXXXXXXXXXXXXXXXXXXXXXXXXXXX
Keep this "remote" remote?
y) Yes this is OK
e) Edit this remote
d) Delete this remote
y/e/d> y
```

参阅[远程设置文档](/remote_setup/)了解如何在无法连接互联网浏览器的机器上进行设置。

注意，rclone 会在本地机器上运行一个 Web 服务器来收集从 Dropbox 返回的令牌。该服务器仅在打开浏览器到获取验证码这段时间内运行。它监听 `http://127.0.0.1:53682/`，如果你运行了主机防火墙，可能需要临时解除阻止，或者使用手动模式。

然后你可以这样使用：

列出你的 Dropbox 顶层目录

```console
rclone lsd remote:
```

列出你的 Dropbox 中的所有文件

```console
rclone ls remote:
```

将本地目录复制到 Dropbox 中名为 backup 的目录

```console
rclone copy /home/source remote:backup
```

### Dropbox 企业版

Rclone 支持 Dropbox 企业版和团队文件夹。

使用 Dropbox 企业版时，`remote:` 和 `remote:path/to/file` 将指向你的个人文件夹。

如果你想查看团队文件夹，必须在路径中使用前导 `/`，因此 `rclone lsd remote:/` 将指向根目录并显示所有团队文件夹和你的用户文件夹。

然后你可以这样使用团队文件夹：`remote:/TeamFolder` 和 `remote:/TeamFolder/path/to/file`。

对于 Dropbox 个人账户，前导 `/` 不会产生任何效果，但会额外产生一次 HTTP 请求，因此应避免使用。

### 修改时间和哈希

Dropbox 支持修改时间，但设置修改时间的唯一方法是重新上传文件。

这意味着，如果你使用不支持 v2 API 和修改时间的旧版 rclone 上传了数据，rclone 会决定重新上传所有旧数据以修正修改时间。如果你不希望发生这种情况，请使用 `--size-only` 或 `--checksum` 标志来阻止。

Dropbox 支持[其自己的哈希类型](https://www.dropbox.com/developers/reference/content-hash)，该哈希会在所有传输中被检查。

### 受限文件名字符

| 字符 | 值 | 替换为 |
| --------- |:-----:|:-----------:|
| NUL       | 0x00  | ␀           |
| /         | 0x2F  | ／           |
| DEL       | 0x7F  | ␡           |
| \         | 0x5C  | ＼           |

文件名也不能以以下字符结尾。
仅当这些字符是名称中最后一个字符时才会被替换：

| 字符 | 值 | 替换为 |
| --------- |:-----:|:-----------:|
| SP        | 0x20  | ␠           |

无效的 UTF-8 字节也会被[替换](/overview/#invalid-utf8)，因为它们无法在 JSON 字符串中使用。

### 批量模式上传 {#batch-mode}

使用 Dropbox API 时，批量模式上传对性能非常重要。参阅 [Dropbox 性能指南](https://developers.dropbox.com/dbx-performance-guide)了解更多信息。

rclone 可以使用 3 种模式进行上传。

#### --dropbox-batch-mode off

在此模式下，rclone 不会使用上传批处理。这是 rclone v1.55 之前的默认模式。其缺点是非常容易遇到 `too_many_requests` 错误，例如：

```text
NOTICE: too_many_requests/.: Too many requests or write operations. Trying again in 15 seconds.
```

当 rclone 收到这些错误时，必须等待 15 秒甚至 300 秒才能继续，这会严重拖慢传输速度。

当 `--transfers` 较大时尤其容易发生这种情况，因此除非为了兼容性或排查问题，否则不推荐使用此模式。

#### --dropbox-batch-mode sync

在此模式下，rclone 会将上传批量累积到 `--dropbox-batch-size` 指定的大小，然后一起提交。

使用此模式意味着你可以使用更高的 `--transfers` 参数（32 或 64 都没问题），而不会收到 `too_many_requests` 错误。

此模式确保完整的数据完整性。

注意，在使用此模式退出 rclone 时可能会有暂停，因为 rclone 需要完成最后一个批次。

#### --dropbox-batch-mode async

在此模式下，rclone 会将上传批量累积到 `--dropbox-batch-size` 指定的大小，然后一起提交。

但它不会等待批次状态返回给调用方。这意味着 rclone 可以使用更大的批量大小（远大于 `--transfers`），代价是无法检查上传状态。

这提供了最大可能的上传速度，尤其适合大量小文件，但 rclone 无法在此模式下检查文件是否正确上传。

如果你使用此模式，建议在传输完成后运行 "rclone check"。或者你可以先用 `--dropbox-batch-mode async` 进行初始传输，然后用 `--dropbox-batch-mode sync`（默认）进行最终传输。

注意，在使用此模式退出 rclone 时可能会有暂停，因为 rclone 需要完成最后一个批次。

### 导出文件

Dropbox 中的某些文件是"可导出的"，例如 Dropbox Paper 文档。这些文件需要转换为其他格式才能下载。通常有多种格式可供转换。

当 rclone 下载可导出文件时，它会根据 `--dropbox-export-formats` 设置选择下载格式。默认导出格式为 `html,md`，这是 Dropbox Paper 的合理默认值。

Rclone 会选择导出格式列表中 Dropbox 为给定文件支持的第一个格式 ID。如果列表中没有可用的格式，rclone 将选择 Dropbox 建议的默认格式。

Rclone 会将扩展名更改为与导出格式对应。以下是扩展名映射的一些示例：

| 文件类型      | Dropbox 中的文件名 | rclone 中的文件名 |
|----------------|---------------------|--------------------|
| Paper          | mydoc.paper         | mydoc.html         |
| Paper 模板 | mydoc.papert        | mydoc.papert.html  |
| 其他          | mydoc               | mydoc.html         |

*导入*可导出文件目前尚不被 rclone 支持。

以下是 rclone 已知的支持导出扩展名。注意，rclone 目前不支持此列表之外的其他格式，即使 Dropbox 支持它们。此外，Dropbox 可能随时更改支持的格式列表。

| 格式 ID | 名称     | 描述          |
|-----------|----------|----------------------|
| html      | HTML     | HTML 文档        |
| md        | Markdown | Markdown 文本格式 |

## 限制

注意，Dropbox 不区分大小写，因此你不能同时拥有名为 "Hello.doc" 和 "hello.doc" 的文件。

有些文件名如 `thumbs.db` 是 Dropbox 无法存储的。完整列表参见[此文档的"忽略文件"部分](https://www.dropbox.com/en/help/145)。如果 rclone 尝试上传这些文件名之一，会发出错误消息 `File name disallowed - not uploading`，但同步不会失败。

如果你尝试同步受版权保护的文件，可能会出现一些错误，因为 Dropbox 有自己的[版权检测器](https://techcrunch.com/2014/03/30/how-dropbox-knows-when-youre-sharing-copyrighted-stuff-without-actually-looking-at-your-stuff/)，会阻止此类文件被下载。这将返回错误 `ERROR : /path/to/your/file: Failed to copy: failed to open source object: path/restricted_content/.`

如果一个目录中有超过 10,000 个文件，`rclone purge dropbox:dir` 将返回错误 `Failed to purge: There are too many files involved in this operation`。解决方法是先执行 `rclone delete dropbox:dir`，然后执行 `rclone rmdir dropbox:dir`。

使用 `rclone link` 时，如果使用非个人账户，需要设置 `--expire`，否则可见性可能不正确。（注意，`--expire` 在个人账户上不受支持。）参阅[论坛讨论](https://forum.rclone.org/t/rclone-link-dropbox-permissions/23211)和 [Dropbox SDK 问题](https://github.com/dropbox/dropbox-sdk-go-unofficial/issues/75)。

Dropbox Paper 文档的修改时间不精确，文档编辑后可能在一段时间内不会改变。为确保在同步中获取最新更改，请等待一小时左右，或使用 `--ignore-times` 强制完整同步。

## 获取你自己的 Dropbox 应用 ID

当你以默认配置使用 rclone 连接 Dropbox 时，你使用的是 rclone 的应用 ID。这在所有 rclone 用户之间共享。

以下是如何为 rclone 创建你自己的 Dropbox 应用 ID：

1. 使用你的 Dropbox 账户登录 [Dropbox 应用控制台](https://www.dropbox.com/developers/apps/create)（无需与你要访问的 Dropbox 账户相同）

2. 选择 API => 通常应选择 `Dropbox API`

3. 选择要使用的访问类型 => `Full Dropbox` 或 `App Folder`。如果要使用团队文件夹，必须选择 `Full Dropbox`（[参见此处](https://www.dropboxforum.com/t5/Dropbox-API-Support-Feedback/How-to-create-team-folder-inside-my-app-s-folder/m-p/601005/highlight/true#M27911)）。

4. 为你的应用命名。应用名称是全局的，因此例如你不能使用 `rclone`

5. 点击 `Create App` 按钮

6. 切换到 `Permissions` 标签页。至少启用以下权限：
`account_info.read`、`files.metadata.write`、`files.content.write`、`files.content.read`、`sharing.write`。`files.metadata.read` 和 `sharing.read` 复选框也会被勾选。点击 `Submit`

7. 切换到 `Settings` 标签页。在 `OAuth2 - Redirect URIs` 中填写 `http://localhost:53682/`，然后点击 `Add`

8. 在 `Settings` 标签页中找到 `App key` 和 `App secret` 值。在 rclone config 中使用这些值添加新的远程存储或编辑现有远程存储。
`App key` 设置对应 rclone config 中的 `client_id`，`App secret` 对应 `client_secret`

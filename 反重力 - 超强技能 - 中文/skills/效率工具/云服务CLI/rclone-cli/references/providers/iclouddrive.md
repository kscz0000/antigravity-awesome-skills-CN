# iCloud Drive

## 配置

iCloud Drive 后端的初始设置需要获取信任令牌（trust token）/会话。
只需使用普通的 iCloud 密码，并在另一台已登录 iCloud 的设备上接受验证码
提示即可完成。

**重要**: 不支持使用 App 专用密码。仅可使用你的
常规 Apple ID 密码与 2FA 双重认证。

`rclone config` 会引导你完成令牌的创建过程。该信任令牌有效期
为 30 天。过期后你需要使用 `rclone reconnect`
或 `rclone config` 重新进行身份认证。

## 身份认证

rclone 使用 [SRP（Secure Remote Password，安全远程密码协议）](https://en.wikipedia.org/wiki/Secure_Remote_Password_protocol) 与 Apple 进行身份认证，
这也是 iCloud 网页端所使用的同一协议。你的密码**永远不会**
被发送到 Apple 的服务器——相反，客户端与服务器之间会交换一个密码学证明，以
验证你确实知道密码而无需泄露密码本身。

认证流程如下：

1. rclone 向 Apple 的身份服务发起一个会话
2. 进行 SRP 密钥交换（你的密码在本地用于派生密钥）
3. Apple 向你的受信任设备发送 2FA 提示
4. 输入 2FA 验证码后，rclone 会获得一个用于后续会话的信任令牌

## 高级数据保护（Advanced Data Protection）

当前不支持 ADP（Advanced Data Protection），需要将其关闭

在 iPhone 上，需进入 设置 `>` Apple 账户 `>` iCloud `>` "在网页上访问 iCloud 数据"
必须设为**开启**，并将"高级数据保护"设为**关闭**。

## 故障排查

### 请求中缺少 PCS cookies

这表示你已开启"高级数据保护"（Advanced Data Protection，简称 ADP）。该功能目前不受支持。
若要使用 rclone，你必须将其关闭。具体关闭方法请参阅上文。
请参阅上文了解如何关闭。

你需要清空配置中的 `cookies` 和 `trust_token` 字段，
或者你也可以删除该远端配置后重新创建。

然后你应该运行 `rclone reconnect remote:`。

注意，修改 ADP 设置可能不会立即生效——你可能
需要等待几个小时甚至一天才能让 rclone 正常工作——期间请持续
清空配置项并运行 `rclone reconnect remote:`，直到 rclone 工作正常为止。

属性：

- Config:      apple_id
- Env Var:     RCLONE_ICLOUDDRIVE_APPLE_ID

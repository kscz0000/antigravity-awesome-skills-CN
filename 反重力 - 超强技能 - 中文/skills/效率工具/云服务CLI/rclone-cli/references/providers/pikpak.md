# PikPak

PikPak 是[一个私有云盘](https://mypikpak.com/)。

路径以 `remote:path` 形式指定，可以按需要指定任意深度，例如 `remote:directory/subdirectory`。

## 配置

下面是一个为 PikPak 创建 remote 的示例。

首先运行：

```console
rclone config
```

### 修改时间和哈希

PikPak 会保留对象的修改时间，并在上传对象时更新它们，
但它不支持仅修改修改时间。

支持 MD5 哈希算法。

### 标准选项

以下是 pikpak（PikPak）特有的标准选项。

#### --pikpak-user

Pikpak 用户名。

属性：

- Config:      user
- Env Var:     RCLONE_PIKPAK_USER
- Type:        string
- Required:    true

#### --pikpak-pass

Pikpak 密码。

**注意** 输入此项必须经过混淆处理 - 请参阅 [rclone obscure](/commands/rclone_obscure/)。

属性：

- Config:      pass
- Env Var:     RCLONE_PIKPAK_PASS
- Type:        string
- Required:    true

### 高级选项

以下是 pikpak（PikPak）特有的高级选项。

#### --pikpak-device-id

用于授权的设备 ID。

属性：

- Config:      device_id
- Env Var:     RCLONE_PIKPAK_DEVICE_ID
- Type:        string
- Required:    false

#### --pikpak-user-agent

pikpak 的 HTTP 用户代理。

默认为 "Mozilla/5.0 (Windows NT 10.0; Win64; x64; rv:129.0) Gecko/20100101 Firefox/129.0"，或者使用命令行上提供的 "--pikpak-user-agent"。

属性：

- Config:      user_agent
- Env Var:     RCLONE_PIKPAK_USER_AGENT
- Type:        string
- Default:     "Mozilla/5.0 (Windows NT 10.0; Win64; x64; rv:129.0) Gecko/20100101 Firefox/129.0"

#### --pikpak-root-folder-id

根文件夹的 ID。
通常留空。

填写此项可使 rclone 使用非根文件夹作为其起始点。


属性：

- Config:      root_folder_id
- Env Var:     RCLONE_PIKPAK_ROOT_FOLDER_ID
- Type:        string
- Required:    false

#### --pikpak-use-trash

将文件移到回收站而不是永久删除。

默认为 true，即将文件移到回收站。
使用 `--pikpak-use-trash=false` 来永久删除文件。

属性：

- Config:      use_trash
- Env Var:     RCLONE_PIKPAK_USE_TRASH
- Type:        bool
- Default:     true

#### --pikpak-trashed-only

仅显示回收站中的文件。

这将以原始目录结构显示已删除的文件。

属性：

- Config:      trashed_only
- Env Var:     RCLONE_PIKPAK_TRASHED_ONLY
- Type:        bool
- Default:     false

#### --pikpak-no-media-link

使用原始文件链接而非媒体链接。

这可避免因媒体链接无效而导致的问题，但可能会降低下载速度。

属性：

- Config:      no_media_link
- Env Var:     RCLONE_PIKPAK_NO_MEDIA_LINK
- Type:        bool
- Default:     false

#### --pikpak-hash-memory-limit

超过此大小的文件在需要计算哈希时将被缓存到磁盘上。

属性：

- Config:      hash_memory_limit
- Env Var:     RCLONE_PIKPAK_HASH_MEMORY_LIMIT
- Type:        SizeSuffix
- Default:     10Mi

#### --pikpak-upload-cutoff

切换到分块上传的阈值。

任何大于此值的文件将以 chunk_size 大小的块上传。
最小值为 0，最大值为 5 GiB。

属性：

- Config:      upload_cutoff
- Env Var:     RCLONE_PIKPAK_UPLOAD_CUTOFF
- Type:        SizeSuffix
- Default:     200Mi

#### --pikpak-chunk-size

分块上传的块大小。

大文件将以该大小的块上传。

请注意，此值存储在内存中，并且内存中可能同时存储多达
"--transfers" * "--pikpak-upload-concurrency" 个块。

如果你通过高速链路传输大文件并且有足够的内存，
那么增大此值将加快传输速度。

当上传已知大小的大文件时，rclone 会自动增大块大小，
以保持在 10,000 个块的上限以下。

增大块大小会降低使用 "-P" 标志显示的进度统计的准确性。

属性：

- Config:      chunk_size
- Env Var:     RCLONE_PIKPAK_CHUNK_SIZE
- Type:        SizeSuffix
- Default:     5Mi

#### --pikpak-upload-concurrency

分块上传的并发数。

这是分块上传时同一个文件并发上传的块数。

请注意，块存储在内存中，并且内存中可能同时存储多达
"--transfers" * "--pikpak-upload-concurrency" 个块。

如果你通过高速链路上传少量大文件并且这些上传未能充分利用你的带宽，
那么增大此值可能有助于加快传输速度。

属性：

- Config:      upload_concurrency
- Env Var:     RCLONE_PIKPAK_UPLOAD_CONCURRENCY
- Type:        int
- Default:     4

#### --pikpak-encoding

后端的编码。

有关更多信息，请参阅[概述中的编码部分](/overview/#encoding)。

属性：

- Config:      encoding
- Env Var:     RCLONE_PIKPAK_ENCODING
- Type:        Encoding
- Default:     Slash,LtGt,DoubleQuote,Colon,Question,Asterisk,Pipe,BackSlash,Ctl,LeftSpace,RightSpace,RightPeriod,InvalidUtf8,Dot

#### --pikpak-description

remote 的描述。

属性：

- Config:      description
- Env Var:     RCLONE_PIKPAK_DESCRIPTION
- Type:        string
- Required:    false

## 后端命令

以下是 pikpak 后端特有的命令。

使用以下方式运行：

```console
rclone backend COMMAND remote:
```

下面的帮助将解释每个命令接受的参数。

有关如何传递选项和参数的更多信息，
请参阅 [backend](/commands/rclone_backend/) 命令。

这些可以在运行中的后端上使用 rc 命令
[backend/command](/rc/#backend-command) 来执行。

### addurl

为 url 添加离线下载任务。

```console
rclone backend addurl remote: [options] [<arguments>+]
```

此命令为 url 添加离线下载任务。

用法示例：

```console
rclone backend addurl pikpak:dirpath url
```

下载的文件将保存在 'dirpath' 中。如果 'dirpath' 无效，
下载将回退到默认的 'My Pack' 文件夹。

### decompress

请求对文件夹中的一个或多个文件进行解压。

```console
rclone backend decompress remote: [options] [<arguments>+]
```

此命令请求对文件夹中的一个或多个文件进行解压。

用法示例：

```console
rclone backend decompress pikpak:dirpath {filename} -o password=password
rclone backend decompress pikpak:dirpath {filename} -o delete-src-file
```

可以为位于 'pikpak:dirpath' 中的文件指定可选参数 'filename'。
对于受密码保护的文件，你可能需要传递 '-o password=password'。
此外，传递 '-o delete-src-file' 可在解压完成后删除源文件。

结果：

```json
{
    "Decompressed": 17,
    "SourceDeleted": 0,
    "Errors": 0
}
```

## 限制

### 哈希可能为空

PikPak 支持 MD5 哈希，但有时会返回空值，特别是对于用户上传的文件。

### 使用 trashed-only 时已删除的文件仍然可见

即使在回收站清空后，使用 `--pikpak-trashed-only` 时仍会看到已删除的文件。
几天后这种情况会消失。

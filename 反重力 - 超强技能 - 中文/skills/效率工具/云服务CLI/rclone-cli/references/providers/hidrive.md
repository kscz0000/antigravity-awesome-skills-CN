# HiDrive

路径以 `remote:path` 的形式指定

路径可以任意深度，例如 `remote:directory/subdirectory`。

HiDrive 的初始设置需要在浏览器中从 HiDrive 获取令牌。
`rclone config` 会引导你完成此过程。

## 配置

以下是创建名为 `remote` 的远程存储的示例。首先运行：

```console
rclone config
```

这将引导你完成交互式设置过程：

**请注意，OAuth 令牌可用于访问你的账户，因此不应与他人共享。**
更多信息请参见[下方章节](#keeping-your-tokens-safe)。

关于在没有互联网连接的浏览器的机器上进行设置，
请参见[远程设置文档](/remote_setup/)。

注意，rclone 会在本地机器上运行一个 Web 服务器来收集从 HiDrive 返回的令牌。
该服务器仅在打开浏览器到获取验证码之间运行。
Web 服务器运行在 `http://127.0.0.1:53682/`。
如果本地端口 `53682` 被防火墙保护，你可能需要临时解除防火墙封锁以完成授权。

配置完成后，你可以像这样使用 `rclone`（将 `remote` 替换为你给远程存储起的名字）：

列出你的 HiDrive 根文件夹顶层的目录

```console
rclone lsd remote:
```

列出你的 HiDrive 文件系统中的所有文件

```console
rclone ls remote:
```

将本地目录复制到 HiDrive 中名为 backup 的目录

```console
rclone copy /home/source remote:backup
```

### Keeping your tokens safe

所有 OAuth 令牌将由 rclone 以未加密文本的形式存储在远程存储的配置文件中。
任何持有有效 refresh token 的人都可以在不知道你密码的情况下访问你的 HiDrive 文件系统。
因此，你应确保没有其他人能够访问你的配置。

可以对 rclone 的配置文件进行加密。
有关保护配置文件的信息，请参见[配置加密文档](/docs/#configuration-encryption)。

### Invalid refresh token

根据 [HiDrive 的 OAuth 指南](https://developer.hidrive.com/basics-flows/)验证，
每个 `refresh_token`（针对原生应用）的有效期为 60 天。
如果用于访问 HiDrive，其有效期将自动延长。

这意味着如果你

- 连续 60 天未使用 HiDrive 远程存储

则 rclone 将返回一个错误，其中包含暗示 refresh token *无效*或*已过期*的文本。

要修复此问题，你需要再次授权 rclone 访问你的 HiDrive 账户。

使用

```console
rclone config reconnect remote:
```

此过程与前面演示的初始设置过程非常相似。

### Modification times and hashes

HiDrive 允许对对象设置修改时间，精度为 1 秒。

HiDrive 支持[其自身的哈希类型](https://static.hidrive.com/dev/0001)，
用于在成功传输后验证文件内容的完整性。

### Restricted filename characters

HiDrive 无法存储名称中包含 `/`（0x2F）或空字节（0x00）的文件或文件夹。
其他任何字符都可以用于文件或文件夹的名称。
此外，文件或文件夹不能命名为以下任一名称：`.` 或 `..`

因此，当存储或访问具有此类名称的文件或文件夹时，rclone 将自动替换这些字符。

你可以在[主文档](/overview/#restricted-filenames)中了解此文件名编码的一般工作原理。

请注意，HiDrive 仅支持长度不超过 255 个字符的文件或文件夹名称。

### Transfers

HiDrive 将单次请求的文件大小限制为最大 2 GiB。
为了允许存储更大的文件并提供更好的上传性能，
hidrive 后端将对大于 96 MiB 的文件使用分块传输。
Rclone 将同时上传文件的多个分块/数据块。
正在上传中的数据块会缓存在内存中，
因此在内存有限的系统上，你可能需要限制此行为。

你可以使用以下选项自定义此行为：

- `chunk_size`：文件分块的大小
- `upload_cutoff`：大小达到或超过此值的文件将使用分块传输
- `upload_concurrency`：同时上传的文件分块数量

更多细节请参见下方关于配置选项的章节。

### Root folder

你可以为 rclone 设置根文件夹。
这是 rclone 视为你的 HiDrive 根目录的目录。

通常，你会将其留空，rclone 将使用账户的根目录。

但是，你可以设置此项以将 rclone 限制到特定的文件夹层级。

其工作方式是将 `root_prefix` 选项的内容
添加到 rclone 访问的任何路径之前。
例如，以下两种访问主目录的方式是等效的：

```console
rclone lsd --hidrive-root-prefix="/users/test/" remote:path
rclone lsd remote:/users/test/path
```

更多细节请参见下方关于配置选项的章节。

### Directory member count

默认情况下，rclone 会知道目录中包含的成员数量。
例如，`rclone lsd` 会使用此信息。

获取此信息会导致 HiDrive API 的额外时间开销。
在处理大型目录结构时，如果并不显式需要此信息，
可能希望规避此时间开销。为此，可以使用 `disable_fetching_member_count` 选项。

更多细节请参见下方关于配置选项的章节。

### Standard options

以下是 hidrive (HiDrive) 的标准选项。

#### --hidrive-client-id

OAuth 客户端 ID。

通常留空。

属性：

- Config:      client_id
- Env Var:     RCLONE_HIDRIVE_CLIENT_ID
- Type:        string
- Required:    false

#### --hidrive-client-secret

OAuth 客户端密钥。

通常留空。

属性：

- Config:      client_secret
- Env Var:     RCLONE_HIDRIVE_CLIENT_SECRET
- Type:        string
- Required:    false

#### --hidrive-scope-access

rclone 在向 HiDrive 请求访问时应使用的访问权限。

属性：

- Config:      scope_access
- Env Var:     RCLONE_HIDRIVE_SCOPE_ACCESS
- Type:        string
- Default:     "rw"
- Examples:
  - "rw"
    - 对资源的读写访问权限。
  - "ro"
    - 对资源的只读访问权限。

  - "owner"
    - 完全的管理权限访问。

#### --hidrive-root-prefix

所有路径的根/父文件夹。

填写此项以将指定文件夹作为远程存储所有路径的父路径。
这样 rclone 就可以使用任何文件夹作为起点。

属性：

- Config:      root_prefix
- Env Var:     RCLONE_HIDRIVE_ROOT_PREFIX
- Type:        string
- Default:     "/"
- Examples:
  - "/"
    - rclone 可访问的最顶层目录。
    - 如果 rclone 使用普通 HiDrive 用户账户，这等同于 "root"。
  - "root"
    - HiDrive 用户账户的最顶层目录
  - ""
    - 指定路径没有根前缀。
    - 使用此选项时，你始终需要为此远程存储指定带有有效父路径的路径，例如 "remote:/path/to/dir" 或 "remote:root/path/to/dir"。

除非绝对必要，否则不获取目录中的对象数量。

如果不获取子目录中的对象数量，请求可能会更快。

属性：

- Config:      disable_fetching_member_count
- Env Var:     RCLONE_HIDRIVE_DISABLE_FETCHING_MEMBER_COUNT
- Type:        bool
- Default:     false

#### --hidrive-chunk-size

分块上传的数据块大小。

任何大于配置的截断值（或大小未知的文件）的文件都将以该大小的数据块上传。

此上限为 2147483647 字节（约 2.000Gi）。
这是单次上传操作支持的最大字节数。
将此值设为超过上限或设为负值将导致上传失败。

将此值设为更大的值可能会提高上传速度，但会消耗更多内存。
可以将其设为更小的值以节省内存。

属性：

- Config:      chunk_size
- Env Var:     RCLONE_HIDRIVE_CHUNK_SIZE
- Type:        SizeSuffix
- Default:     48Mi

#### --hidrive-upload-cutoff

分块上传的截断/阈值。

任何大于此值的文件将以配置的数据块大小上传。

此上限为 2147483647 字节（约 2.000Gi）。
这是单次上传操作支持的最大字节数。
将此值设为超过上限将导致上传失败。

属性：

- Config:      upload_cutoff
- Env Var:     RCLONE_HIDRIVE_UPLOAD_CUTOFF
- Type:        SizeSuffix
- Default:     96Mi

#### --hidrive-upload-concurrency

分块上传的并发数。

这是同一文件同时运行的传输数的上限。
将此值设为小于 1 的值将导致上传死锁。

如果你在高速链路上上传少量大文件，
且这些上传未能充分利用你的带宽，则增大此值可能有助于加速传输。

属性：

- Config:      upload_concurrency
- Env Var:     RCLONE_HIDRIVE_UPLOAD_CONCURRENCY
- Type:        int
- Default:     4

#### --hidrive-encoding

后端的编码方式。

属性：

- Config:      encoding

## 限制

### 符号链接

HiDrive 能够按设计存储符号链接（*symlinks*），
例如从 zip 归档解包时。

远程存储中没有管理原生符号链接的直接机制。
因此，此实现选择忽略远程存储中存在的任何原生符号链接。
rclone 将无法访问或显示存储在 hidrive 远程存储中的任何符号链接。
这意味着符号链接无法被单独删除、复制或移动，
除非删除、复制或移动其父文件夹。

*这不影响 rclone 用来编码和存储符号链接的 `.rclonelink` 文件。*

### 稀疏文件

可以在 HiDrive 中存储稀疏文件。

注意，复制稀疏文件会将空洞展开为空字节（0x00）区域，从而占用磁盘空间。
同样，下载稀疏文件时，结果文件中的文件空洞位置将变为空字节区域。

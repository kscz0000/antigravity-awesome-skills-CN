# SMB

SMB 是[用于在网络上共享文件的通信协议](https://en.wikipedia.org/wiki/Server_Message_Block)。

它依赖 [go-smb2 库](https://github.com/CloudSoda/go-smb2/) 与 SMB 协议通信。

路径格式为 `remote:sharename`（`lsd` 命令使用 `remote:`）。也可以包含子目录，例如 `remote:item/path/to/dir`。

## 注意事项

第一段路径必须是共享名，即你在 Windows 上开始共享时输入的名称。在 smbd 上，它是 `smb.conf` 文件（通常位于 `/etc/samba/`）中的 section 标题。
如果不确定，可以通过查询根目录来找到共享（例如 `rclone lsd remote:`）。

显然，rclone 无法访问共享打印机。

无法使用匿名访问登录。必须改用 `guest` 用户配合空密码。rclone 客户端在上传文件时会通过编码末尾的空格和句点来避免 8.3 短文件名。
另外，Windows 上的[本地后端](/local/#paths-on-windows)可以通过 UNC 路径（如 `\\server\share`）访问 SMB 服务器。
这不适用于非 Windows 操作系统，例如 Linux 和 macOS。

### 标准选项

以下是 smb (SMB / CIFS) 特有的标准选项。

#### --smb-host

要连接的 SMB 服务器主机名。

例如 "example.com"。

Properties:

- Config:      host
- Env Var:     RCLONE_SMB_HOST
- Type:        string
- Required:    true

#### --smb-user

SMB 用户名。

Properties:

- Config:      user
- Env Var:     RCLONE_SMB_USER
- Type:        string
- Default:     "$USER"

#### --smb-port

SMB 端口号。

Properties:

- Config:      port
- Env Var:     RCLONE_SMB_PORT
- Type:        int
- Default:     445

#### --smb-pass

SMB 密码。

**NB** 此项输入必须经过混淆处理 - 参见 [rclone obscure](/commands/rclone_obscure/)。

Properties:

- Config:      pass
- Env Var:     RCLONE_SMB_PASS
- Type:        string
- Required:    false

#### --smb-domain

NTLM 身份验证的域名。

Properties:

- Config:      domain
- Env Var:     RCLONE_SMB_DOMAIN
- Type:        string
- Default:     "WORKGROUP"

#### --smb-spn

服务主体名称（Service Principal Name）。

Rclone 向服务器呈现此名称。某些服务器将其用作进一步的认证，对于集群环境通常需要设置。例如：

    cifs/remotehost:1020

如果不确定，请留空。


Properties:

- Config:      spn
- Env Var:     RCLONE_SMB_SPN
- Type:        string
- Required:    false

#### --smb-use-kerberos

使用 Kerberos 身份验证。

如果设置，rclone 将使用 Kerberos 身份验证代替 NTLM。这要求存在有效的 Kerberos 配置和凭据缓存，
可以在默认位置，也可以通过 KRB5_CONFIG 和 KRB5CCNAME 环境变量指定。


Properties:

- Config:      use_kerberos
- Env Var:     RCLONE_SMB_USE_KERBEROS
- Type:        bool
- Default:     false

### 高级选项

以下是 smb (SMB / CIFS) 特有的高级选项。

#### --smb-idle-timeout

关闭空闲连接前的最大时间。

如果在指定时间内没有任何连接返回到连接池，rclone 将清空连接池。

设置为 0 表示无限期保持连接。


Properties:

- Config:      idle_timeout
- Env Var:     RCLONE_SMB_IDLE_TIMEOUT
- Type:        Duration
- Default:     1m0s

#### --smb-hide-special-share

隐藏用户不应访问的特殊共享（例如 print$）。

Properties:

- Config:      hide_special_share
- Env Var:     RCLONE_SMB_HIDE_SPECIAL_SHARE
- Type:        bool
- Default:     true

#### --smb-case-insensitive

服务器是否配置为大小写不敏感。

在 Windows 共享上始终为 true。

Properties:

- Config:      case_insensitive
- Env Var:     RCLONE_SMB_CASE_INSENSITIVE
- Type:        bool
- Default:     true

#### --smb-kerberos-ccache

Kerberos 凭据缓存（krb5cc）的路径。

覆盖默认的 KRB5CCNAME 环境变量，允许 SMB 后端实例使用不同的 Kerberos 缓存文件。
在挂载多个具有不同凭据的 SMB 或在多用户环境中运行时非常有用。

支持的格式：
  - FILE:/path/to/ccache   – Use the specified file.
  - DIR:/path/to/ccachedir – Use the primary file inside the specified directory.
  - /path/to/ccache        – Interpreted as a file path.

Properties:

- Config:      kerberos_ccache
- Env Var:     RCLONE_SMB_KERBEROS_CCACHE
- Type:        string
- Required:    false

#### --smb-encoding

后端的编码方式。

更多信息请参阅[概述中的编码章节](/overview/#encoding)。

Properties:

- Config:      encoding
- Env Var:     RCLONE_SMB_ENCODING
- Type:        Encoding
- Default:     Slash,LtGt,DoubleQuote,Colon,Question,Asterisk,Pipe,BackSlash,Ctl,RightSpace,RightPeriod,InvalidUtf8,Dot

#### --smb-description

远程的描述。

Properties:

- Config:      description
- Env Var:     RCLONE_SMB_DESCRIPTION
- Type:        string
- Required:    false

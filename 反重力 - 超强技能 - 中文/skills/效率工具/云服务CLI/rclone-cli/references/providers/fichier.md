# 1Fichier

这是 [1fichier](https://1fichier.com) 云存储服务的后端。请注意，使用该 API 需要 Premium 订阅。

路径以 `remote:path` 的形式指定。

路径可根据需要任意嵌套，例如 `remote:directory/subdirectory`。

## 配置

1Fichier 的初始设置需要在浏览器中访问网站以获取 API Key。

下面是一个创建名为 `remote` 的远程配置的示例。首先运行：

```console
rclone config
```

这将引导你完成交互式配置流程：

```text
No remotes found, make a new one?
n) New remote
s) Set configuration password
q) Quit config
n/s/q> n
name> remote
Type of storage to configure.
Enter a string value. Press Enter for the default ("").
Choose a number from below, or type in your own value
[snip]
XX / 1Fichier
   \ "fichier"
[snip]
Storage> fichier
** See help for fichier backend at: https://rclone.org/fichier/ **

Your API Key, get it from https://1fichier.com/console/params.pl
Enter a string value. Press Enter for the default ("").
api_key> example_key

Edit advanced config? (y/n)
y) Yes
n) No
y/n>
Remote config
Configuration complete.
Options:
- type: fichier
- api_key: example_key
Keep this "remote" remote?
y) Yes this is OK
e) Edit this remote
d) Delete this remote
y/e/d> y
```

配置完成后，你就可以这样使用 `rclone`（请将 `remote` 替换为你为远程配置所起的名字）：


列出 1Fichier 账户顶层目录

```console
rclone lsd remote:
```

列出 1Fichier 账户中的所有文件

```console
rclone ls remote:
```

将本地目录复制到名为 backup 的 1Fichier 目录

```console
rclone copy /home/source remote:backup
```

### 修改时间和哈希

1Fichier 不支持修改时间。它支持 Whirlpool 哈希算法。

### 重复文件

1Fichier 允许存在两个名称和路径完全相同的文件（这与普通文件系统不同）。

重复文件会导致同步出现问题，你会在日志中看到关于重复项的提示。

### 受限的文件名字符

除了 [默认受限字符集](/overview/#restricted-characters) 之外，以下字符也会被替换：

| 字符 | 值 | 替换为 |
| --------- |:-----:|:-----------:|
| \         | 0x5C  | ＼           |
| <         | 0x3C  | ＜           |
| >         | 0x3E  | ＞           |
| "         | 0x22  | ＂           |
| $         | 0x24  | ＄           |
| `         | 0x60  | ｀           |
| '         | 0x27  | ＇           |

文件名也不能以以下字符开头或结尾。这些字符仅在位于名称首部或尾部时才会被替换：

| 字符 | 值 | 替换为 |
| --------- |:-----:|:-----------:|
| SP        | 0x20  | ␠           |

无效的 UTF-8 字节也会被 [替换](/overview/#invalid-utf8)，因为它们无法在 JSON 字符串中使用。

### 标准选项

以下是 fichier（1Fichier）特有的标准选项。

#### --fichier-api-key

你的 API Key，可以从 https://1fichier.com/console/params.pl 获取。

属性：

- Config:      api_key
- Env Var:     RCLONE_FICHIER_API_KEY
- Type:        string
- Required:    false

### 高级选项

以下是 fichier（1Fichier）特有的高级选项。

#### --fichier-shared-folder

如果要下载共享文件夹，请添加此参数。

属性：

- Config:      shared_folder
- Env Var:     RCLONE_FICHIER_SHARED_FOLDER
- Type:        string
- Required:    false

#### --fichier-file-password

如果要下载受密码保护的共享文件，请添加此参数。

**注意** 此项的输入必须经过混淆处理 - 请参阅 [rclone obscure](/commands/rclone_obscure/)。

属性：

- Config:      file_password
- Env Var:     RCLONE_FICHIER_FILE_PASSWORD
- Type:        string
- Required:    false

#### --fichier-folder-password

如果要列出受密码保护的共享文件夹中的文件，请添加此参数。

**注意** 此项的输入必须经过混淆处理 - 请参阅 [rclone obscure](/commands/rclone_obscure/)。

属性：

- Config:      folder_password
- Env Var:     RCLONE_FICHIER_FOLDER_PASSWORD
- Type:        string
- Required:    false

#### --fichier-cdn

若希望使用 CDN 下载链接，请设置此选项。

属性：

- Config:      cdn
- Env Var:     RCLONE_FICHIER_CDN
- Type:        bool
- Default:     false

#### --fichier-encoding

后端使用的编码方式。

更多信息请参阅 [概览中的编码章节](/overview/#encoding)。

属性：

- Config:      encoding
- Env Var:     RCLONE_FICHIER_ENCODING
- Type:        Encoding
- Default:     Slash,LtGt,DoubleQuote,SingleQuote,BackQuote,Dollar,BackSlash,Del,Ctl,LeftSpace,RightSpace,InvalidUtf8,Dot

#### --fichier-description

远程配置的描述信息。

属性：

- Config:      description
- Env Var:     RCLONE_FICHIER_DESCRIPTION
- Type:        string
- Required:    false

## 限制

1Fichier 后端不支持 `rclone about`。不具备该能力的后端无法确定 rclone 挂载的剩余空间，也无法在 rclone union 远程中作为成员使用 `mfs`（最大剩余空间）策略。

请参阅 [不支持 rclone about 的后端列表](https://rclone.org/overview/#optional-features) 和 [rclone about](https://rclone.org/commands/rclone_about/)。

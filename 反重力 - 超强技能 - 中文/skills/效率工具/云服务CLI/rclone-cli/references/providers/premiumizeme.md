# premiumize.me

路径以 `remote:path` 形式指定

路径可按需嵌套任意深度，例如 `remote:directory/subdirectory`。

## 配置

[premiumize.me](https://premiumize.me/) 的初始设置需要在浏览器中从 premiumize.me 获取一个 token。`rclone config` 会引导你完成整个过程。

下面是一个创建名为 `remote` 的 remote 的示例。首先运行：

```console
rclone config
```

这将引导你完成交互式设置流程：

```text

```

请参阅 [remote setup docs](/remote_setup/) 以了解在没有可用联网浏览器
的机器上如何进行设置。

注意 rclone 会在你的本机启动一个 web 服务器，用于收集从 premiumize.me
返回的 token。该服务器仅在从打开浏览器到获取验证码之间的这段时间内运行。
它监听 `http://127.0.0.1:53682/`，如果你的主机开启了防火墙，可能需要
临时放行该端口。

完成配置后，你就可以像下面这样使用 `rclone`（请将 `remote` 替换为你为
remote 取的名字）：

列出 premiumize.me 顶层目录：

```console
rclone lsd remote:
```

列出 premiumize.me 中的所有文件：

```console
rclone ls remote:
```

将本地目录拷贝到 premiumize.me 上名为 backup 的目录：

```console
rclone copy /home/source remote:backup
```

### 修改时间和哈希

premiumize.me 不支持修改时间或哈希，因此同步操作将默认采用 `--size-only`
方式进行比对。注意使用 `--update` 是可以正常工作的。

### 受限的文件名字符

除了 [默认受限字符集](/overview/#restricted-characters) 之外，下列字符
也会被替换：

| 字符 | 编码 | 替换为 |
| --------- |:-----:|:-----------:|
| \         | 0x5C  | ＼           |
| "         | 0x22  | ＂           |

无效的 UTF-8 字节也会被 [替换](/overview/#invalid-utf8)，因为它们无法在
JSON 字符串中使用。

### 标准选项

以下是 premiumizeme (premiumize.me) 特有的标准选项。

#### --premiumizeme-client-id

OAuth Client Id。

通常留空即可。

属性：

- Config:      client_id
- Env Var:     RCLONE_PREMIUMIZEME_CLIENT_ID
- Type:        string
- Required:    false

#### --premiumizeme-client-secret

OAuth Client Secret。

通常留空即可。

属性：

- Config:      client_secret
- Env Var:     RCLONE_PREMIUMIZEME_CLIENT_SECRET
- Type:        string
- Required:    false

#### --premiumizeme-api-key

API Key。

通常不使用该选项——请改用 oauth。

属性：

- Config:      api_key
- Env Var:     RCLONE_PREMIUMIZEME_API_KEY
- Type:        string
- Required:    false

## 限制

注意 premiumize.me 大小写不敏感，因此你不能同时拥有名为 "Hello.doc"
和 "hello.doc" 的两个文件。

premiumize.me 文件名中不能包含 `\` 或 `"` 字符。
rclone 会将这些字符与外观一致的 Unicode 等价字符 `＼` 和 `＂`
进行双向映射。

premiumize.me 仅支持文件名长度不超过 255 个字符。

# put.io

路径以 `remote:path` 的形式指定

put.io 路径可根据需要嵌套任意层级，例如 `remote:directory/subdirectory`。

## 配置

put.io 的初始设置需要先在浏览器中从 put.io 获取一个 token。
`rclone config` 会引导你完成整个流程。

下面以创建一个名为 `remote` 的远程为例。首先执行：

```console
rclone config
```

这将引导你完成一个交互式配置流程：

如果在没有可联网浏览器的机器上进行配置，请参阅
[远程配置文档](/remote_setup/) 了解设置方法。

请注意，如果使用浏览器自动认证，rclone 会在本地机器上启动一个
Web 服务器，用于接收 put.io 返回的 token。该服务仅在
打开浏览器到你拿到验证码这一期间内运行，监听地址为
`http://127.0.0.1:53682/`。如果你开启了主机防火墙，可能需要临时
放行该端口，否则请改用手动模式。

之后你可以像下面这样使用它：

列出 put.io 顶层目录

```console
rclone lsd remote:
```

列出 put.io 中的所有文件

```console
rclone ls remote:
```

将本地目录复制到名为 backup 的 put.io 目录中

```console
rclone copy /home/source remote:backup
```

### 受限的字符名

除了 [默认受限字符集](/overview/#restricted-characters) 之外，
以下字符也会被替换：

| 字符  |   值  |    替换为   |
| --------- |:-----:|:-----------:|
| \         | 0x5C  | ＼           |

无效的 UTF-8 字节也会被 [替换](/overview/#invalid-utf8)，
因为它们无法用于 JSON 字符串中。

### 标准选项

以下是 putio (Put.io) 所特有的标准选项。

#### --putio-client-id

OAuth 客户端 ID。

通常保持为空即可。

属性：

- Config:      client_id
- Env Var:     RCLONE_PUTIO_CLIENT_ID
- Type:        string
- Required:    false

#### --putio-client-secret

OAuth 客户端密钥。

通常保持为空即可。

属性：

- Config:      client_secret
- Env Var:     RCLONE_PUTIO_CLIENT_SECRET
- Type:        string
- Required:    false

### 高级选项

以下是 putio (Put.io) 所特有的高级选项。

#### --putio-token

以 JSON blob 形式的 OAuth 访问令牌。

属性：

- Config:      token
- Env Var:     RCLONE_PUTIO_TOKEN
- Type:        string
- Required:    false

#### --putio-auth-url

认证服务器 URL。

保持为空以使用服务提供方的默认值。

属性：

- Config:      auth_url
- Env Var:     RCLONE_PUTIO_AUTH_URL
- Type:        string
- Required:    false

#### --putio-token-url

令牌服务器 URL。

保持为空以使用服务提供方的默认值。

属性：

- Config:      token_url
- Env Var:     RCLONE_PUTIO_TOKEN_URL
- Type:        string
- Required:    false

#### --putio-client-credentials

使用客户端凭据 OAuth 流程。

这将使用 RFC 6749 中描述的 OAUTH2 客户端凭据流程。

请注意，并非所有后端都支持此选项。

属性：

- Config:      client_credentials
- Env Var:     RCLONE_PUTIO_CLIENT_CREDENTIALS
- Type:        bool
- Default:     false

#### --putio-encoding

后端的编码方式。

更多信息请参阅 [概述中的编码章节](/overview/#encoding)。

属性：

- Config:      encoding
- Env Var:     RCLONE_PUTIO_ENCODING
- Type:        Encoding
- Default:     Slash,BackSlash,Del,Ctl,InvalidUtf8,Dot

#### --putio-description

该远程的描述。

属性：

- Config:      description
- Env Var:     RCLONE_PUTIO_DESCRIPTION
- Type:        string
- Required:    false

## 限制

put.io 设有速率限制。当你触发限制时，rclone 会在服务端
请求的等待时长后自动重试。

如果你希望完全避免触发这些限制，可以使用
`--tpslimit` 参数并设置一个较小的数值。请注意，针对
不同的操作所施加的限制可能有所不同，并且可能随时间变化。

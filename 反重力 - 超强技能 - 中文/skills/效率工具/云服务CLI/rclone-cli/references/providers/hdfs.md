# HDFS

[HDFS](https://hadoop.apache.org/docs/stable/hadoop-project-dist/hadoop-hdfs/HdfsDesign.html)
是一个分布式文件系统，属于 [Apache Hadoop](https://hadoop.apache.org/)
框架。

路径以 `remote:` 或 `remote:path/to/dir` 形式指定。

## 配置

下面是一个示例，演示如何创建一个名为 `remote` 的远程存储后端。首先运行：

```console
rclone config
```

这个远程存储后端被命名为 `remote`，现在可以这样使用

查看所有顶层目录

```console
rclone lsd remote:
```

列出目录的内容

```console
rclone ls remote:directory
```

将远程 `directory` 同步到 `/home/local/directory`，并删除多余的文件。

```console
rclone sync --interactive remote:directory /home/local/directory
```

### 为测试搭建自己的 HDFS 实例

你可以从[手动安装](https://hadoop.apache.org/docs/stable/hadoop-project-dist/hadoop-common/SingleCluster.html)
开始，或者使用测试用的 docker 镜像：

如果你想要构建 docker 镜像

```console
git clone https://github.com/rclone/rclone.git
cd rclone/fstest/testserver/images/test-hdfs
docker build --rm -t rclone/test-hdfs .
```

或者直接使用最新推送的镜像

```console
docker run --rm --name "rclone-hdfs" -p 127.0.0.1:9866:9866 -p 127.0.0.1:8020:8020 --hostname "rclone-hdfs" rclone/test-hdfs
```

**注意** 启动需要几秒钟的时间。

对于该 docker 镜像，远程存储后端需要按如下方式配置：

```ini
[remote]
type = hdfs
namenode = 127.0.0.1:8020
username = root
```

你可以使用 `docker kill rclone-hdfs` 停止该镜像（**注意** 它未使用
卷，因此上传的所有数据都会丢失。）

### 修改时间

存储精度为 1 秒的时间。

| 字符 | 取值 | 替换为 |
| --------- |:-----:|:-----------:|
| :         | 0x3A  | ：           |

无效的 UTF-8 字节也会被[替换](/overview/#invalid-utf8)。

### 标准选项

以下是 hdfs（Hadoop 分布式文件系统）特有的标准选项。

#### --hdfs-namenode

Hadoop 名称节点和端口。

例如 "namenode-1:8020,namenode-2:8020,..." 用于连接 8020 端口的主机名称节点。

属性：

- Config:      namenode
- Env Var:     RCLONE_HDFS_NAMENODE
- Type:        CommaSepList
- Default:

#### --hdfs-username

Hadoop 用户名。

属性：

- Config:      username
- Env Var:     RCLONE_HDFS_USERNAME
- Type:        string
- Required:    false
- Examples:
  - "root"
    - 以 root 身份连接 hdfs。

### 高级选项

以下是 hdfs（Hadoop 分布式文件系统）特有的高级选项。

#### --hdfs-service-principal-name

名称节点的 Kerberos 服务主体名。

启用 KERBEROS 身份认证。指定名称节点的服务主体名
（SERVICE/FQDN）。例如 \"hdfs/namenode.hadoop.docker\"
表示以服务 'hdfs'、FQDN 为 'namenode.hadoop.docker' 运行名称节点。

属性：

- Config:      service_principal_name
- Env Var:     RCLONE_HDFS_SERVICE_PRINCIPAL_NAME
- Type:        string
- Required:    false

#### --hdfs-data-transfer-protection

Kerberos 数据传输保护：authentication|integrity|privacy。

指定与数据节点通信时是否需要身份认证、数据签名完整性
校验以及线路加密。可选值为 'authentication'、'integrity'
和 'privacy'。仅在启用了 KERBEROS 时使用。

属性：

- Config:      data_transfer_protection
- Env Var:     RCLONE_HDFS_DATA_TRANSFER_PROTECTION
- Type:        string
- Required:    false
- Examples:
  - "privacy"
    - 启用身份认证、完整性和加密。

#### --hdfs-encoding

后端的编码方式。

更多信息请参阅 [overview 中的编码章节](/overview/#encoding)。

属性：

- Config:      encoding
- Env Var:     RCLONE_HDFS_ENCODING
- Type:        Encoding
- Default:     Slash,Colon,Del,Ctl,InvalidUtf8,Dot

#### --hdfs-description

远程存储后端的描述。

属性：

- Config:      description
- Env Var:     RCLONE_HDFS_DESCRIPTION
- Type:        string
- Required:    false

# Shade

这是 [Shade](https://shade.inc/) 平台的后端。

## 关于 Shade

[Shade](https://shade.inc/) 是一个由 AI 驱动的云端 NAS，让你的云端文件表现得像本地硬盘一样，针对媒体和创意工作流进行了优化。它提供快速、安全的访问，支持自然语言搜索、便捷的共享以及可扩展的云存储。


## 账户与定价

要使用此后端,你需要在 Shade 上[创建一个免费账户](https://app.shade.inc/)。你可以从免费账户开始,获得 20GB 的免费存储空间。


## 使用方法

路径以 `remote:path` 形式指定。

路径可以根据需要嵌套任意深度,例如 `remote:directory/subdirectory`。


## 配置

下面是一个创建 Shade 配置的示例。

首先,创建一个[免费账户](https://app.shade.inc/)并选择套餐。

你需要登录账户,并从账户设置中获取 `API Key`,从创建的云盘中获取 `Drive ID`。

然后运行

`rclone config`

按照下面的交互过程进行操作:

```sh
$ rclone config
e) Edit existing remote
n) New remote
d) Delete remote
r) Rename remote
c) Copy remote
s) Set configuration password
q) Quit config
e/n/d/r/c/s/q> n

Enter name for new remote.
name> Shade

Option Storage.
Type of storage to configure.
Choose a number from below, or type in your own value.
[OTHER OPTIONS]
xx / Shade FS
   \ (shade)
[OTHER OPTIONS]
Storage> xx

Option drive_id.
The ID of your drive, see this in the drive settings. Individual rclone configs must be made per drive.
Enter a value.
drive_id> [YOUR_ID]

Option api_key.
An API key for your account.
Enter a value.
api_key> [YOUR_API_KEY]

Edit advanced config?
y) Yes
n) No (default)
y/n> n

Configuration complete.
Options:
- type: shade
- drive_id: [YOUR_ID]
- api_key: [YOUR_API_KEY]
Keep this "Shade" remote?
y) Yes this is OK (default)
e) Edit this remote
d) Delete this remote
y/e/d> y
```

### 修改时间和哈希

Shade 不支持哈希和写入修改时间。


### 传输

Shade 默认使用分片上传。这意味着文件会被切分,并发地发送到 Shade。要配置希望同时进行的并发上传数量,可在高级配置中上传 'concurrency' 选项。请注意,这会占用更多内存,并发起更多的 HTTP 请求。

### 删除文件

请注意,通过 rclone 在 Shade 中删除文件时,会立即删除文件,而不会将其放入回收站。这意味着文件无法被恢复。

### 标准选项

以下是 shade (Shade FS) 特有的标准选项。

#### --shade-drive-id

你的云盘 ID,可在云盘设置中查看。每个云盘都必须单独配置 rclone。

属性:

- Config:      drive_id
- Env Var:     RCLONE_SHADE_DRIVE_ID
- Type:        string
- Required:    true

#### --shade-api-key

你账户的 API key。

属性:

- Config:      api_key
- Env Var:     RCLONE_SHADE_API_KEY
- Type:        string
- Required:    true

### 高级选项

以下是 shade (Shade FS) 特有的高级选项。

#### --shade-endpoint

服务的端点。

通常保持为空。

属性:

- Config:      endpoint
- Env Var:     RCLONE_SHADE_ENDPOINT
- Type:        string
- Required:    false

#### --shade-chunk-size

上传时使用的分片大小。

大于此大小的文件将以此大小的分片上传。

请注意,这是按每次传输缓存在内存中的,因此增大该值会增加内存占用。

最小为 5MB,最大为 5GB。

属性:

- Config:      chunk_size
- Env Var:     RCLONE_SHADE_CHUNK_SIZE
- Type:        SizeSuffix
- Default:     64Mi

#### --shade-upload-concurrency

分片上传和复制的并发数。这是同一个文件在进行分片上传和复制时并发上传的分片数量。

属性:

- Config:      upload_concurrency
- Env Var:     RCLONE_SHADE_UPLOAD_CONCURRENCY
- Type:        int
- Default:     4

#### --shade-max-upload-parts

分片上传中允许的最大分片数。

属性:

- Config:      max_upload_parts
- Env Var:     RCLONE_SHADE_MAX_UPLOAD_PARTS
- Type:        int
- Default:     10000

#### --shade-token

用于执行 Shade FS 操作的 JWT Token。请不要手动设置此值 - rclone 会自动设置。

属性:

- Config:      token
- Env Var:     RCLONE_SHADE_TOKEN
- Type:        string
- Required:    false

#### --shade-token-expiry

JWT Token 过期时间。请不要手动设置此值 - rclone 会自动设置。

属性:

- Config:      token_expiry
- Env Var:     RCLONE_SHADE_TOKEN_EXPIRY
- Type:        string
- Required:    false

#### --shade-encoding

该后端的编码方式。

更多信息请参阅[概述中的编码章节](/overview/#encoding)。

属性:

- Config:      encoding
- Env Var:     RCLONE_SHADE_ENCODING
- Type:        Encoding
- Default:     Slash,BackSlash,Del,Ctl,InvalidUtf8,Dot

#### --shade-description

对 remote 的描述。

属性:

- Config:      description
- Env Var:     RCLONE_SHADE_DESCRIPTION
- Type:        string
- Required:    false

## 限制

请注意,Shade 不区分大小写,因此你不能同时拥有名为
"Hello.doc" 和 "hello.doc" 的文件。

Shade 仅支持文件名长度最多为 255 个字符。

`rclone about` 不被 Shade 后端支持。不具备该能力的后端无法
为 rclone mount 确定剩余空间,也无法在 rclone union
remote 中使用策略 `mfs`(most free space,剩余空间最多)。

请参阅[不支持 rclone about 的后端列表](https://rclone.org/overview/#optional-features)和 [rclone about](https://rclone.org/commands/rclone_about/)

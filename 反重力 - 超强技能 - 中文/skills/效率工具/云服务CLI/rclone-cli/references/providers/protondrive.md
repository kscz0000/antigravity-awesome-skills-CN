# Proton Drive

[Proton Drive](https://proton.me/drive) 是一个端到端加密的瑞士文件保险库，
用于保护你的数据安全。

这是一个 Proton Drive 的 rclone 后端，支持使用相同的客户端加密方式来
传输 Proton Drive 上的文件。

由于 Proton Drive 没有发布其 API 文档，该后端是通过阅读开源客户端
源代码和观察浏览器中的 Proton Drive 流量，尽最大努力实现的。

**注意** 该后端目前处于 Beta 阶段。据信其实现是正确的，
所有集成测试均已通过。但由于 Proton Drive 协议随时间演变，
可能存在某些账户不兼容的情况。如果你发现不兼容问题，
请[在 rclone 论坛发帖](https://forum.rclone.org/)。

路径指定为 `remote:path`

路径可以嵌套任意深度，例如 `remote:directory/subdirectory`。

## 配置

以下是如何创建一个名为 `remote` 的远程存储的示例。首先运行：

```console
rclone config
```

这将引导你完成交互式设置过程：

```text
No remotes found, make a new one?
n) New remote
s) Set configuration password
q) Quit config
n/s/q> n
name> remote
Type of storage to configure.
Choose a number from below, or type in your own value
[snip]
XX / Proton Drive
   \ "protondrive"
[snip]
Storage> protondrive
User name
user> you@protonmail.com
Password.
y) Yes type in my own password
g) Generate random password
n) No leave this optional password blank
y/g/n> y
Enter the password:
password:
Confirm the password:
password:
Option 2fa.
2FA code (if the account requires one)
Enter a value. Press Enter to leave empty.
2fa> 123456
Remote config
Configuration complete.
Options:
- type: protondrive
- user: you@protonmail.com
- pass: *** ENCRYPTED ***
Keep this "remote" remote?
y) Yes this is OK
e) Edit this remote
d) Delete this remote
y/e/d> y
```

**注意：** 需要先通过浏览器正常登录以生成 Proton Drive 的加密密钥，
否则尝试在 `rclone` 中使用这些凭据将会失败。

配置完成后，你可以像这样使用 `rclone`（将 `remote` 替换为
你给远程存储起的名称）：

列出 Proton Drive 顶层目录

```console
rclone lsd remote:
```

列出 Proton Drive 中的所有文件

```console
rclone ls remote:
```

将本地目录复制到 Proton Drive 中名为 backup 的目录

```console
rclone copy /home/source remote:backup
```

### 修改时间和哈希

Proton Drive Bridge 尚不支持更新修改时间。

支持 SHA1 哈希算法。

### 受限的文件名字符

无效的 UTF-8 字节将被[替换](/overview/#invalid-utf8)，左右空格也将
被移除（[代码参考](https://github.com/ProtonMail/WebClients/blob/b4eba99d241af4fdae06ff7138bd651a40ef5d3c/applications/drive/src/app/store/_links/validation.ts#L51)）

### 重复文件

Proton Drive 不能有两个名称和路径完全相同的文件。如果发生冲突，
根据高级配置的不同，文件可能会或可能不会被覆盖。

### [邮箱密码](https://proton.me/support/the-difference-between-the-mailbox-password-and-login-password)

请在高级配置部分设置你的邮箱密码。

### 缓存

缓存目前是为 rclone 作为唯一操作挂载点的实例而构建的。事件系统——
即提供驱动器变更可见性的 Proton API 系统——尚未实现，因此来自其他
客户端的更新不会反映在缓存中。因此，如果有多个并发客户端访问同一
挂载点，可能会出现缓存过期数据的问题。

### 标准选项

以下是 protondrive（Proton Drive）特有的标准选项。

#### --protondrive-username

你的 Proton 账户用户名

属性：

- Config:      username
- Env Var:     RCLONE_PROTONDRIVE_USERNAME
- Type:        string
- Required:    true

#### --protondrive-password

你的 Proton 账户密码。

**注意** 输入必须经过混淆处理 - 参见 [rclone obscure](/commands/rclone_obscure/)。

属性：

- Config:      password
- Env Var:     RCLONE_PROTONDRIVE_PASSWORD
- Type:        string
- Required:    true

#### --protondrive-2fa

2FA 验证码

该值也可以通过 --protondrive-2fa=000000 提供

如果你的 Proton Drive 账户启用了两步验证，则需要提供该账户的
2FA 验证码

属性：

- Config:      2fa
- Env Var:     RCLONE_PROTONDRIVE_2FA
- Type:        string
- Required:    false

#### --protondrive-otp-secret-key

OTP 密钥

该值也可以通过 --protondrive-otp-secret-key=ABCDEFGHIJKLMNOPQRSTUVWXYZ234567 提供

如果你的 Proton Drive 账户启用了两步验证，则需要提供该账户的
OTP 密钥

**注意** 输入必须经过混淆处理 - 参见 [rclone obscure](/commands/rclone_obscure/)。

属性：

- Config:      otp_secret_key
- Env Var:     RCLONE_PROTONDRIVE_OTP_SECRET_KEY
- Type:        string
- Required:    false

属性：

应用版本字符串

应用版本字符串指示当前正在执行 API 请求的客户端。此信息是必需的，
将会随每次 API 请求一起发送。

属性：

- Config:      app_version
- Env Var:     RCLONE_PROTONDRIVE_APP_VERSION
- Type:        string
- Default:     "macos-drive@1.0.0-alpha.1+rclone"

#### --protondrive-replace-existing-draft

检测到文件名冲突时创建新修订版本

当文件上传在完成前被取消或失败时，会创建一个草稿，随后将同一文件
上传到同一位置将被报告为冲突。

该值也可以通过 --protondrive-replace-existing-draft=true 设置

如果此选项设为 true，草稿将被替换，然后上传操作将重新开始。
如果同时有其他客户端在同一文件位置上传，当前行为是未知的。
集成测试需要设为 true。
如果此选项设为 false，将返回错误"a draft exist - usually this means a
file is being uploaded at another client, or, there was a failed upload attempt"，
并且不会执行上传。

属性：

- Config:      replace_existing_draft
- Env Var:     RCLONE_PROTONDRIVE_REPLACE_EXISTING_DRAFT
- Type:        bool
- Default:     false

#### --protondrive-enable-caching

缓存文件和文件夹元数据以减少 API 调用

注意：如果你将 ProtonDrive 挂载为 VFS，请禁用此功能，
因为当前实现在存在外部更改时不会更新或清除缓存。

ProtonDrive 上的文件和文件夹以带密钥环的链接形式表示，
可以对其进行缓存以提高性能并减轻 API 服务器负担。

缓存目前是为 rclone 作为唯一操作挂载点的实例而构建的。事件系统——
即提供驱动器变更可见性的 Proton API 系统——尚未实现，因此来自其他
客户端的更新不会反映在缓存中。因此，如果有多个并发客户端访问同一
挂载点，可能会出现缓存过期数据的问题。

属性：

- Config:      enable_caching
- Env Var:     RCLONE_PROTONDRIVE_ENABLE_CACHING
- Type:        bool
- Default:     true

#### --protondrive-description

远程存储的描述。

属性：

- Config:      description
- Env Var:     RCLONE_PROTONDRIVE_DESCRIPTION
- Type:        string
- Required:    false

## 限制

该后端使用
[Proton-API-Bridge](https://github.com/henrybear327/Proton-API-Bridge)，它
基于 [go-proton-api](https://github.com/henrybear327/go-proton-api)，后者是
[官方仓库](https://github.com/ProtonMail/go-proton-api) 的一个分支。

Proton Drive 没有提供官方 API 文档。但得益于 Proton 开源了
[proton-go-api](https://github.com/ProtonMail/go-proton-api) 以及 Web、iOS
和 Android 客户端代码库，我们不需要完全通过观察 Web 客户端流量来
逆向工程这些 API！

[proton-go-api](https://github.com/ProtonMail/go-proton-api) 提供了 API
调用和错误处理的基本构建块，例如 429 指数退避，但它基本上只是一个
Proton API 的骨架接口。例如，该库不提供 Proton Drive 文件的加密和
解密功能。

Proton-API-Bridge 试图弥合这一差距，以便 rclone 可以在此基础上快速构建。
该代码库处理调用 Proton API 前后的复杂任务，尤其是复杂的加密方案，
使开发者可以在此代码库之上为其他软件实现功能。由于没有官方文档可用，
该库中可能存在不少错误。

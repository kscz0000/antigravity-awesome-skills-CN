# HTTP

HTTP 远程是一个用于读取 Web 服务器文件的只读远程。Web 服务器应提供文件列表，rclone 将读取这些列表并将其转换为远程。此功能已通过 Apache/Nginx/Caddy 等常见 Web 服务器测试，并且很可能适用于大多数 Web 服务器的文件列表。（如果不行，请提交 issue，或发送 pull request！）

路径以 `remote:` 或 `remote:path` 形式指定。

`remote:` 代表配置的 [url](#http-url)，其后任何路径都将根据 URL 标准相对于此 url 进行解析。这意味着当远程 url 为 `https://beta.rclone.org/branch` 且路径为 `fix` 时，解析后的 URL 将为 `https://beta.rclone.org/branch/fix`；而当路径为 `/fix` 时，解析后的 URL 将为 `https://beta.rclone.org/fix`，因为绝对路径是从域名根开始解析的。

如果 `remote:` 后面的路径以 `/` 结尾，则假定它指向目录。如果路径不以 `/` 结尾，则会发送 HEAD 请求，并根据响应决定将其视为文件还是目录（使用 `-vv` 运行可查看详细信息）。当指定 [--http-no-head](#http-no-head) 时，不以 `/` 结尾的路径始终被视为文件。如果 rclone 错误地将路径假定为文件，则解决方法是用 `/` 结尾指定路径。当你确定路径是目录时，始终以 `/` 结尾更好，因为这样可以避免最初的 HEAD 请求。

要仅下载单个文件，使用 [copyurl](/commands/rclone_copyurl/) 会更简单。

以下是如何创建名为 `remote` 的远程的示例。首先运行：


此远程名为 `remote`，现在可以这样使用

查看所有顶级目录

```console
rclone lsd remote:
```

列出目录的内容

```console
rclone ls remote:directory
```

将远程 `directory` 同步到 `/home/local/directory`，删除任何多余的文件。

```console
rclone sync --interactive remote:directory /home/local/directory
```

### 只读

此远程是只读的 - 你无法将文件上传到 HTTP 服务器。

### 修改时间

大多数 HTTP 服务器存储的时间精确到 1 秒。

### 校验和

不存储校验和。

### 不使用配置文件

由于 http 远程只有一个配置参数，因此无需配置文件即可轻松使用：

```console
rclone lsd --http-url https://beta.rclone.org :http:
```

或：

```console
rclone lsd :http,url='https://beta.rclone.org':
```

### 标准选项

以下是 http (HTTP) 特定的标准选项。

#### --http-url

要连接的 HTTP 主机的 URL。

例如 `"https://example.com"`，或 `"https://user:pass@example.com"` 以使用用户名和密码。

属性：

- Config:      url
- Env Var:     RCLONE_HTTP_URL
- Type:        string
- Required:    true

#### --http-no-escape

不对路径名中的 URL 元字符进行转义。

属性：

- Config:      no_escape
- Env Var:     RCLONE_HTTP_NO_ESCAPE
- Type:        bool
- Default:     false

### 高级选项

以下是 http (HTTP) 特定的高级选项。

#### --http-headers

为所有事务设置 HTTP 头。

使用此选项可为所有事务设置额外的 HTTP 头。

输入格式为逗号分隔的键值对列表。可以使用标准 [CSV 编码](https://godoc.org/encoding/csv)。

例如，要设置 Cookie，使用 `'Cookie,name=value'`，或 `'"Cookie","name=value"'`。

你可以设置多个头，例如 `'"Cookie","name=value","Authorization","xxx"'`。

属性：

- Config:      headers
- Env Var:     RCLONE_HTTP_HEADERS
- Type:        CommaSepList
- Default:

#### --http-no-slash

如果站点不以 / 结束目录，则设置此选项。

如果你的目标网站在目录末尾不使用 /，则使用此选项。

路径末尾的 / 是 rclone 通常用来区分文件和目录的方式。如果设置了此标志，那么 rclone 会将所有 Content-Type: text/html 的文件视为目录，并从中读取 URL，而不是下载它们。

请注意，这可能会导致 rclone 将真正的 HTML 文件与目录混淆。

属性：

- Config:      no_slash
- Env Var:     RCLONE_HTTP_NO_SLASH
- Type:        bool
- Default:     false

#### --http-no-head

不使用 HEAD 请求。

HEAD 请求主要用于在目录列表中查找文件大小。如果你的站点加载非常慢，可以尝试此选项。通常 rclone 会为目录列表中的每个潜在文件执行一次 HEAD 请求，目的是：

- 查找其大小
- 检查它是否真实存在
- 检查它是否是目录

如果设置此选项，rclone 将不执行 HEAD 请求。这将意味着目录列表会快得多，但 rclone 将无法获得任何文件的时间和大小，并且某些不存在的文件可能会出现在列表中。

属性：

- Config:      no_head
- Env Var:     RCLONE_HTTP_NO_HEAD
- Type:        bool
- Default:     false

#### --http-description

远程的描述。

属性：

- Config:      description
- Env Var:     RCLONE_HTTP_DESCRIPTION
- Type:        string
- Required:    false

### 元数据

HTTP 元数据键不区分大小写，并且始终以小写形式返回。

以下是 http 后端可能的系统元数据项。

| Name | Help | Type | Example | Read Only |
|------|------|------|---------|-----------|
| cache-control | Cache-Control 头 | string | no-cache | N |
| content-disposition | Content-Disposition 头 | string | inline | N |
| content-disposition-filename | 从 Content-Disposition 头检索到的文件名 | string | file.txt | N |
| content-encoding | Content-Encoding 头 | string | gzip | N |
| content-language | Content-Language 头 | string | en-US | N |
| content-type | Content-Type 头 | string | text/plain | N |

有关更多信息，请参阅 [metadata](/docs/#metadata) 文档。

## 后端命令

以下是 http 后端特有的命令。

使用以下方式运行：

```console
rclone backend COMMAND remote:
```

下面的帮助将说明每个命令接受哪些参数。

有关如何传递选项和参数的更多信息，请参阅 [backend](/commands/rclone_backend/) 命令。

这些可以在运行中的后端上使用 rc 命令 [backend/command](/rc/#backend-command) 运行。

### set

用于更新配置参数的 set 命令。

```console
rclone backend set remote: [options] [<arguments>+]
```

此 set 命令可用于更新正在运行的 http 后端的配置参数。

使用示例：

```console
rclone backend set remote: [-o opt_name=opt_value] [-o opt_name2=opt_value2]
rclone rc backend/command command=set fs=remote: [-o opt_name=opt_value] [-o opt_name2=opt_value2]
rclone rc backend/command command=set fs=remote: -o url=https://example.com
```

选项键的命名与配置文件中的一致。

当使用新参数调用时，它会重建与 http 后端的连接。只需要传递新参数，因为这些值将默认为当前正在使用的值。

它不返回任何内容。

## 限制

HTTP 后端不支持 `rclone about`。没有此功能的后端无法确定 rclone 挂载的剩余空间，也不能将策略 `mfs`（most free space，最多空闲空间）用作 rclone union 远程的成员。

请参阅[不支持 rclone about 的后端列表](https://rclone.org/overview/#optional-features)和 [rclone about](https://rclone.org/commands/rclone_about/)。

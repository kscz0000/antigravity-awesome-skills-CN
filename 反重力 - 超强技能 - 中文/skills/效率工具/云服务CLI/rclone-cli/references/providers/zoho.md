# Zoho Workdrive

[Zoho WorkDrive](https://www.zoho.com/workdrive/) 是由 [Zoho](https://zoho.com) 创建的云存储解决方案。

Rclone 在你的本地计算机上运行一个 Web 服务器来从 Zoho Workdrive 收集授权令牌。
这仅在从你打开浏览器到令牌返回之间的这段时间内生效。
Web 服务器在 `http://127.0.0.1:53682/` 上运行。
如果本地端口 `53682` 受防火墙保护，你可能需要临时解除防火墙封锁以完成授权。

配置完成后，你就可以像下面这样使用 `rclone`（将 `remote` 替换为你为 remote 起的名称）：

查看顶层目录

```console
rclone lsd remote:
```

创建新目录

```console
rclone mkdir remote:directory
```

列出目录的内容

```console
rclone ls remote:directory
```

将 `/home/local/directory` 同步到远程路径，删除路径中多余的文件。

```console
rclone sync --interactive /home/local/directory remote:directory
```

Zoho 路径可以根据需要尽可能深，例如 `remote:directory/subdirectory`。

### Modification times and hashes

Zoho Workdrive 当前不支持修改时间

不支持任何哈希算法。

### Usage information

要查看你当前的配额，可以使用 `rclone about remote:` 命令，它将显示你当前的使用情况。

### Restricted filename characters

只有控制字符和无效的 UTF-8 会被替换。此外，大多数 Unicode 全角字符根本不受支持，并将在上传过程中从文件名中移除。

## Setting up your own client_id

对于 Zoho，我们建议你设置自己的 client_id。为此你必须完成以下步骤。

1. 登录到 [Zoho API Console](https://api-console.zoho.com)

2. 创建一个新 client，类型为 "Server-based Application"。名称和网站不重要，但你必须添加重定向 URL `http://localhost:53682/`。

3. client 创建完成后，你可以进入 settings 选项卡，并在其他区域启用它。

现在 client id 和 client secret 可以与 rclone 一起使用了。

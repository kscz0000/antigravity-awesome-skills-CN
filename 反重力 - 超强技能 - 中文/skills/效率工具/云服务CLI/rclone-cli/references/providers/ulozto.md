# Uloz.to

路径以 `remote:path` 形式指定。

路径可根据需要任意嵌套，例如 `remote:directory/subdirectory`。

Uloz.to 的初始设置需要填写用户凭据。
`rclone config` 会引导你完成配置。

## 配置

下面示例演示如何创建一个名为 `remote` 的远端。首先执行：

```console
rclone config
```

这将引导你完成交互式配置流程：

配置完成后，你就可以像下面这样使用 `rclone`（请将 `remote` 替换为你的远端名）：

列出根级目录中的文件夹：

```console
rclone lsd remote:
```

列出根目录中的所有文件：

```console
rclone ls remote:
```

将本地目录复制到名为 backup 的 Uloz.to 目录：

```console
rclone copy /home/source remote:backup
```

### 用户凭据

唯一可靠的方式是使用用户名和密码进行认证。Uloz.to 也提供 API 密钥，
但仅供 Uloz.to 自有应用使用，在其他场景下使用并不可靠。

### 修改时间和哈希

Uloz.to 不允许用户自定义修改时间，也无法在上传后获取哈希。
因此，该集成使用 API 提供的一个自由格式字段来编码客户端提供的时间戳和哈希。
时间戳以微秒精度存储。

上传时会校验服务器计算的 MD5 哈希。
此后，后端只提供客户端计算的哈希。哈希也可以在创建文件下载链接时获取，
但对于 `list` 类用例并不实用。

### 受限文件名字符

除了[默认受限字符集](/overview/#restricted-characters)外，
以下字符也会被替换：

| Character | Value | Replacement |
| --------- |:-----:|:-----------:|
| \         | 0x5C  | ＼           |

无效的 UTF-8 字节也会被[替换](/overview/#invalid-utf8)，
因为它们无法用于 JSON 字符串。

### 传输

目前所有文件都通过单个 HTTP 请求上传，因此上传大文件时需要稳定的网络连接。
Rclone 同时最多上传 `--transfers` 个分片（所有上传任务共享该参数）。

### 删除文件

默认情况下，文件会被移入回收站，而文件夹会被立即删除。
回收站中的文件将在 30 天后被永久删除。

rclone 目前未实现清空回收站的功能。

### 根目录 ID

你可以为 rclone 设置 `root_folder_slug`。这是 rclone 视为你 Uloz.to 云盘根目录的文件夹
（通过其 `Folder slug` 标识）。

通常你应将其留空，由 rclone 自动确定正确的根目录。但你也可以设置此项，
将 rclone 限制在特定的文件夹层级内。

要做到这一点，你需要找到要用作根目录的文件夹的 `Folder slug`。
该值是在 Uloz.to 网页中打开相应文件夹时 URL 的最后一段。

例如，要浏览 URL 为 `https://uloz.to/fm/my-files/foobar` 的文件夹，
应使用 `foobar` 作为根 slug。

`root_folder_slug` 可与远端路径中的具体路径结合使用。例如，
如果你的远端的 `root_folder_slug` 对应 `/foo/bar`，
那么 `remote:baz/qux` 将指向 `ABSOLUTE_ULOZTO_ROOT/foo/bar/baz/qux`。

### 标准选项

以下是 ulozto (Uloz.to) 特有的标准选项。

#### --ulozto-app-token

用于标识该应用的 application token。应用 API 密钥可在 API 文档
https://uloz.to/upload-resumable-api-beta 中找到，也可联系客服获取。

属性：

- Config:      app_token
- Env Var:     RCLONE_ULOZTO_APP_TOKEN
- Type:        string
- Required:    false

#### --ulozto-username

操作所用的主体用户名。

属性：

- Config:      username
- Env Var:     RCLONE_ULOZTO_USERNAME
- Type:        string
- Required:    false

#### --ulozto-password

用户的密码。

**NB** 输入时必须进行混淆处理 - 参见 [rclone obscure](/commands/rclone_obscure/)。

属性：

- Config:      password
- Env Var:     RCLONE_ULOZTO_PASSWORD
- Type:        string
- Required:    false

### 高级选项

以下是 ulozto (Uloz.to) 特有的高级选项。

#### --ulozto-root-folder-slug

如果设置，rclone 将该文件夹作为所有操作的根目录。例如，如果 slug 标识 'foo/bar/'，
则在未设置任何 root slug 时，'ulozto:baz' 等价于 'ulozto:foo/bar/baz'。

属性：

- Config:      root_folder_slug
- Env Var:     RCLONE_ULOZTO_ROOT_FOLDER_SLUG
- Type:        string
- Required:    false

#### --ulozto-list-page-size

list 命令的单页大小。范围 1-500。

属性：

- Config:      list_page_size
- Env Var:     RCLONE_ULOZTO_LIST_PAGE_SIZE
- Type:        int
- Default:     500

#### --ulozto-encoding

后端的编码方式。

更多信息参见 [overview 中的编码章节](/overview/#encoding)。

属性：

- Config:      encoding
- Env Var:     RCLONE_ULOZTO_ENCODING
- Type:        Encoding
- Default:     Slash,BackSlash,Del,Ctl,InvalidUtf8,Dot

#### --ulozto-description

远端的描述信息。

属性：

- Config:      description
- Env Var:     RCLONE_ULOZTO_DESCRIPTION
- Type:        string
- Required:    false

## 局限性

Uloz.to 文件名中不能包含 `\` 字符。rclone 会将其与外观相同的 Unicode 等价字符
`＼`（U+FF3C 全角反斜杠）相互转换。

Uloz.to 仅支持最长 255 字符的文件名。

Uloz.to 对 API 访问设有速率限制，但未公开具体细节。
实测显示，正常使用中很少触发速率限制，但在并发上传文件数量较多时仍有可能触发。

Uloz.to 后端不支持 `rclone about`。虽然存在一个用于 UI 的信息获取端点，
但未在 API 中暴露。不具备此能力的后端无法为 rclone 挂载确定剩余空间，
也无法在 rclone union 远端中作为 `mfs`（most free space，最多空闲空间）成员使用策略。

参见[不支持 rclone about 的后端列表](https://rclone.org/overview/#optional-features)
以及 [rclone about](https://rclone.org/commands/rclone_about/)。

# Sia

Sia ([sia.tech](https://sia.tech/)) 是一个基于[区块链](https://wikipedia.org/wiki/Blockchain)技术的去中心化云存储平台。
借助 rclone，你可以像使用任何其他远程文件系统一样使用它，或者将 Sia 文件夹本地挂载。
其背后的技术涉及许多新概念，例如 Siacoins 与钱包、区块链与共识、租用与托管，等等。
如果你是初次接触，建议先阅读他们出色的[支持文档](https://support.sia.tech/)以熟悉相关概念。

## 简介

在使用 rclone 与 Sia 配合之前，你需要在本地计算机或本地网络（例如 NAS）上运行一份 `Sia-UI` 或 `siad`（Sia 守护进程）。
请按照[入门指南](https://sia.tech/get-started)进行安装。

rclone 通过 [HTTP API](https://sia.tech/docs/) 与 Sia 守护进程通信来与 Sia 网络交互，
该 API 通常在 *9980* 端口上可用。默认情况下，你会在同一台计算机上本地运行守护进程，
因此 API 密码留空是安全的（API URL 将为 `http://127.0.0.1:9980`，无法从外部访问）。

但是，如果你希望访问运行在其他节点上的 Sia 守护进程——例如由于内存限制，
或者想在多个 rclone 和 Sia-UI 实例之间共享同一个守护进程——你需要进行一些额外的设置：

- 确保你已在本地或[Docker 容器](https://github.com/SiaFoundation/siad/pkgs/container/siad)中直接安装了 *Sia 守护进程*，
  因为 Sia-UI 本身并不原生支持此模式。
- 在外部可访问的端口上运行，例如在守护进程命令行中提供 `--api-addr :9980`
  和 `--disable-api-security` 参数。
- 通过环境变量 `SIA_API_PASSWORD` 或守护进程目录中名为 `apipassword` 的文本文件，
  为 `siad` 守护进程强制设置 API 密码。
- 在 rclone 后端选项中设置 `api_password`，从上述位置获取密码。

注意事项：

1. 如果你的钱包被锁定，rclone 无法自动解锁它。
   你应该提前使用 Sia-UI 或通过命令行 `siac wallet unlock` 进行解锁。
   或者，你也可以让 `siad` 在启动时通过环境变量 `SIA_WALLET_PASSWORD` 自动解锁你的钱包。
2. 如果 `siad` 在 `SIA_DIR` 目录中找不到 `SIA_API_PASSWORD` 变量或 `apipassword` 文件，
   它将生成一个随机密码，并将其存储在 Unix 系统下 `YOUR_HOME/.sia/` 目录中
   或 Windows 系统下 `C:\Users\YOUR_HOME\AppData\Local\Sia\apipassword` 路径下名为 `apipassword` 的文本文件中。
   在 rclone 中配置密码时请记住这一点。
3. 在不使用 API 密码的情况下运行 `siad` 的唯一方式是**在 localhost 上**使用命令行参数 `--authorize-api=false`，
   但这种方式并不安全，**强烈不建议**这样做。

## 配置

下面以创建一个名为 `mySia` 的 `sia` 远程为例。首先，运行：

```console
rclone config
```

这将引导你完成交互式设置过程：

```text

s) Set configuration password
q) Quit config

Choose a number from below, or type in your own value

Note that siad must run with --disable-api-security to open API port for other hosts (not recommended).

Enter a string value. Press Enter for the default ("http://127.0.0.1:9980").
api_url> http://127.0.0.1:9980
Sia Daemon API Password.
Can be found in the apipassword file located in HOME/.sia/ or in the daemon directory.
y) Yes type in my own password
g) Generate random password
n) No leave this optional password blank (default)
y/g/n> y
Enter the password:
password:
Confirm the password:
password:
Edit advanced config?
y) Yes
n) No (default)
y/n> n
--------------------
[mySia]
type = sia
api_url = http://127.0.0.1:9980
api_password = *** ENCRYPTED ***
--------------------
y) Yes this is OK (default)
e) Edit this remote

y/e/d> y
```

配置完成后，你就可以像下面这样使用 `rclone`：

- 列出 Sia 存储顶层目录

  ```console
  rclone lsd mySia:
  ```

- 列出 Sia 存储中的所有文件

  ```console
  rclone ls mySia:
  ```

- 将本地目录上传到名为 *backup* 的 Sia 目录

  ```console
  rclone copy /home/source mySia:backup
  ```

### 标准选项

以下是 sia（Sia 去中心化云存储）特有的标准选项。

#### --sia-api-url

Sia 守护进程 API URL，例如 http://sia.daemon.host:9980。

请注意，必须使用 --disable-api-security 运行 siad，才能为其他主机开放 API 端口（不推荐）。
如果 Sia 守护进程运行在 localhost 上，请保留默认值。

属性：

- Config:      api_url
- Env Var:     RCLONE_SIA_API_URL
- Type:        string
- Default:     "http://127.0.0.1:9980"

#### --sia-api-password

Sia 守护进程 API 密码。

可在位于 HOME/.sia/ 或守护进程目录下的 apipassword 文件中找到。

**注意** 此项输入内容必须经过混淆处理——请参阅 [rclone obscure](/commands/rclone_obscure/)。

属性：

- Config:      api_password
- Env Var:     RCLONE_SIA_API_PASSWORD
- Type:        string
- Required:    false

### 高级选项

以下是 sia（Sia 去中心化云存储）特有的高级选项。

#### --sia-user-agent

Siad User Agent

默认情况下，出于安全考虑，Sia 守护进程要求使用 'Sia-Agent' 用户代理

属性：

- Config:      user_agent
- Env Var:     RCLONE_SIA_USER_AGENT
- Type:        string
- Default:     "Sia-Agent"

#### --sia-encoding

后端的编码方式。

更多信息请参阅[概述中的编码章节](/overview/#encoding)。

属性：

- Config:      encoding
- Env Var:     RCLONE_SIA_ENCODING
- Type:        Encoding
- Default:     Slash,Question,Hash,Percent,Del,Ctl,InvalidUtf8,Dot

#### --sia-description

远程的描述。

属性：

- Config:      description
- Env Var:     RCLONE_SIA_DESCRIPTION
- Type:        string
- Required:    false

## 限制

- 不支持修改时间
- 不支持校验和
- 不支持 `rclone about`
- rclone 目前仅可与 *Siad* 或 *Sia-UI* 配合使用，
  **尚不支持 SkyNet 守护进程。**
- Sia 不允许在文件名中使用控制字符或问号、井号等符号。rclone 会透明地为你[编码](/overview/#encoding)它们，但你最好留意这一点

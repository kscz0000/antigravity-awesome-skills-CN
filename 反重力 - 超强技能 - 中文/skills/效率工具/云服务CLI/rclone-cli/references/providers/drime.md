# Drime

[Drime](https://drime.cloud/) 是一项专注于快速、可靠文件传输的云存储与传输服务。它提供免费和付费两档套餐，重点强调高速上传和链接分享。

要配置 Drime，你需要登录账户，进入 Settings（设置）中的 Developer（开发者）页面，创建一个用作 API 访问密钥的 token。给它取一个合适的名称，并将 token 复制下来用于配置。

## 配置

下面演示通过 `rclone config` 创建一个名为 `remote` 的远程存储的过程。

首先运行：


```console
rclone config
```

然后按交互式设置流程操作：


```text
No remotes found, make a new one?
n) New remote
s) Set configuration password
q) Quit config
n/s/q> n

Enter name for new remote.
name> remote

Option Storage.
Type of storage to configure.
Choose a number from below, or type in your own value.
XX / Drime
   \ (drime)
Storage> drime

Option access_token.
API Access token
You can get this from the web control panel.
Enter a value. Press Enter to leave empty.
access_token> YOUR_API_ACCESS_TOKEN

Edit advanced config?
y) Yes
n) No (default)
y/n> n

Configuration complete.
Options:
- type: drime
- access_token: YOUR_API_ACCESS_TOKEN
Keep this "remote" remote?
y) Yes this is OK (default)
e) Edit this remote
d) Delete this remote
y/e/d> y
```

配置完成后，你就可以像下面这样使用 `rclone`（将 `remote` 替换为你在创建远程时设置的名称）：

列出 Drime 根级别的目录和文件

```console
rclone lsf remote:
```

将本地目录复制到名为 backup 的 Drime 目录

```console
rclone copy /home/source remote:backup
```


### 修改时间和哈希

Drime 不支持修改时间或哈希。

这意味着默认情况下，sync 操作只能通过文件大小来判断是否需要更新。

你可以使用 `--update` 标志，它会以对象的上传时间作为判断依据。对许多操作来说，这已足以判断文件是否发生了变化。但是，对于那些创建时间在过去的时间戳的文件，使用 `--update` 进行 sync 时会被漏掉。


### 受限文件名字符

除了[默认受限字符集](/overview/#restricted-characters)外，以下字符也会被替换：

| 字符 | 值 | 替换字符 |
| --------- |:-----:|:-----------:|
| \         | 0x5C  | ＼           |

文件名的开头或结尾也不能是以下字符。这些字符只有出现在名称首尾时才会被替换：

| 字符 | 值 | 替换字符 |
| --------- |:-----:|:-----------:|
| SP        | 0x20  | ␠           |

无效的 UTF-8 字节也会被[替换](/overview/#invalid-utf8)，
因为它们无法在 JSON 字符串中使用。

### 根目录 ID

你可以为 rclone 设置 `root_folder_id`。这是 rclone 视为 Drime 云盘根目录的目录（通过其 `Folder ID` 标识）。

通常你不需要填写此项，rclone 会自行确定正确的根目录并将值填入配置文件。

不过，你也可以将它设置为某个特定的目录层级，从而把 rclone 限制在该目录范围内。

为此，你需要找到希望 rclone 显示的目录的 `Folder ID`。

你可以使用 rclone 来获取：

```console
$ rclone lsf -Fip --dirs-only remote:
d6341f53-ee65-4f29-9f59-d11e8070b2a0;Files/
f4f5c9b8-6ece-478b-b03e-4538edfe5a1c;Photos/
d50e356c-29ca-4b27-a3a7-494d91026e04;Videos/
```

需要使用的 ID 是 `;` 前面的部分，例如：

```text
root_folder_id = d6341f53-ee65-4f29-9f59-d11e8070b2a0
```

即可将 rclone 限制在 `Files` 目录中。

### 标准选项

以下是 drime (Drime) 特有的标准选项。

#### --drime-access-token

API 访问 token

你可以从 Web 控制面板获取。

属性：

- Config:      access_token
- Env Var:     RCLONE_DRIME_ACCESS_TOKEN
- Type:        string
- Required:    false

### 高级选项

以下是 drime (Drime) 特有的高级选项。

#### --drime-root-folder-id

根目录的 ID

通常留空即可，rclone 会自动填充。

如果希望 rclone 限定到某个特定目录，可以手动填写 —— 详见相关文档。


属性：

- Config:      root_folder_id
- Env Var:     RCLONE_DRIME_ROOT_FOLDER_ID
- Type:        string
- Required:    false

#### --drime-workspace-id

账户 ID

通常留空，除非你希望指定一个 Workspace ID。


属性：

- Config:      workspace_id
- Env Var:     RCLONE_DRIME_WORKSPACE_ID
- Type:        string
- Required:    false

#### --drime-list-chunk

每次列表调用返回的项目数量

属性：

- Config:      list_chunk
- Env Var:     RCLONE_DRIME_LIST_CHUNK
- Type:        int
- Default:     1000

#### --drime-hard-delete

永久删除文件，而不是放入回收站。

属性：

- Config:      hard_delete
- Env Var:     RCLONE_DRIME_HARD_DELETE
- Type:        bool
- Default:     false

#### --drime-upload-cutoff

切换到分块上传的阈值。

大于此值的文件将以 chunk_size 为单位分块上传。
最小值为 0，最大值为 5 GiB。

属性：

- Config:      upload_cutoff
- Env Var:     RCLONE_DRIME_UPLOAD_CUTOFF
- Type:        SizeSuffix
- Default:     200Mi

#### --drime-chunk-size

上传时使用的块大小。

当上传大于 upload_cutoff 的文件，或上传大小未知的文件（例如通过 "rclone rcat"、使用 "rclone mount" 上传，或来自 Google Photos、Google Docs 的文件）时，它们将以该块大小作为分片上传。

请注意，每次传输会在内存中缓存大小为该值的 "--drime-upload-concurrency" 个块。

如果你通过高速链路传输大文件，并且有足够内存，那么增大该值将提升传输速度。

在上传已知大小的大文件时，rclone 会自动增大块大小，以保证分块数不超过 10,000 的上限。

大小未知的文件会按照已配置的 chunk_size 进行上传。由于默认块大小为 5 MiB，且最多允许 10,000 个分块，因此默认情况下你能够以流式方式上传的最大文件大小为 48 GiB。如果希望流式上传更大的文件，则需要增大 chunk_size。


属性：

- Config:      chunk_size
- Env Var:     RCLONE_DRIME_CHUNK_SIZE
- Type:        SizeSuffix
- Default:     5Mi

#### --drime-upload-concurrency

分片上传与复制操作的并发度。

这是分片上传与复制操作中，同一文件并行上传的分块数量。

如果你通过高速链路上传少量大文件，且这些上传未能充分利用你的带宽，那么增大该值可能有助于提升传输速度。

属性：

- Config:      upload_concurrency
- Env Var:     RCLONE_DRIME_UPLOAD_CONCURRENCY
- Type:        int
- Default:     4

#### --drime-encoding

后端的编码方式。

更多信息请参阅[概述中的编码章节](/overview/#encoding)。

属性：

- Config:      encoding
- Env Var:     RCLONE_DRIME_ENCODING
- Type:        Encoding
- Default:     Slash,BackSlash,Del,Ctl,LeftSpace,RightSpace,InvalidUtf8,Dot

#### --drime-description

对远程存储的描述。

属性：

- Config:      description
- Env Var:     RCLONE_DRIME_DESCRIPTION
- Type:        string
- Required:    false

## 限制

Drime 仅支持文件名长度不超过 255 字节，且文件名必须采用 UTF-8 编码。

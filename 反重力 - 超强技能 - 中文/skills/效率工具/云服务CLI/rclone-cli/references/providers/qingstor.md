# QingStor

路径以 `remote:bucket` 的形式指定（对于 `lsd` 命令，也可以使用 `remote:`）。你也可以在其中加入子目录，例如 `remote:bucket/path/to/dir`。

这个远程称为 `remote`，现在可以这样使用

查看所有 bucket

```console
rclone lsd remote:
```

创建一个新 bucket

```console
rclone mkdir remote:bucket
```

列出 bucket 中的内容

```console
rclone ls remote:bucket
```

将 `/home/local/directory` 同步到远程 bucket，删除 bucket 中多余的文件。

```console
rclone sync --interactive /home/local/directory remote:bucket
```

### --fast-list

该远程支持 `--fast-list`，允许你以更多内存为代价换取更少的请求次数。更多详情请参阅 [rclone 文档](/docs/#fast-list)。

### 分片上传

rclone 支持 QingStor 的分片上传，这意味着它可以上传大于 5 GiB 的文件。请注意，使用分片上传的文件没有 MD5SUM。

注意，超过 24 小时的未完成分片上传可以通过 `rclone cleanup remote:bucket` 删除（针对单个 bucket），通过 `rclone cleanup remote:` 删除（针对所有 bucket）。QingStor 从不自动删除未完成的分片上传，因此可能需要时不时运行此命令。

### Bucket 与可用区

使用 QingStor，你可以通过任意可用区列出 bucket（`rclone lsd`），但只能从创建该 bucket 的可用区访问其内容。如果尝试从错误的可用区访问 bucket，你将收到错误：`incorrect zone, the bucket is not in 'XXX' zone`。

### 身份认证

有两种方式可以为 `rclone` 提供一组 QingStor 凭证。按优先级顺序：

- 直接在 rclone 配置文件中（如通过 `rclone config` 配置）
  - 设置 `access_key_id` 和 `secret_access_key`
- 运行时配置：
  - 在配置文件中将 `env_auth` 设置为 `true`
  - 在运行 `rclone` 之前导出以下环境变量
    - Access Key ID：`QS_ACCESS_KEY_ID` 或 `QS_ACCESS_KEY`
    - Secret Access Key：`QS_SECRET_ACCESS_KEY` 或 `QS_SECRET_KEY`

### 受限的文件名字符

控制字符 0x00-0x1F 和 `/` 会按照 [默认受限字符集](/overview/#restricted-characters) 进行替换。请注意，0x7F 不会被替换。

无效的 UTF-8 字节也会被 [替换](/overview/#invalid-utf8)，因为它们无法在 JSON 字符串中使用。

### 标准选项

以下是 qingstor（QingCloud 对象存储）特有的标准选项。

#### --qingstor-env-auth

从运行时获取 QingStor 凭证。

仅在 access_key_id 和 secret_access_key 为空时生效。

属性：

- Config:      env_auth
- Env Var:     RCLONE_QINGSTOR_ENV_AUTH
- Type:        bool
- Default:     false
- Examples:
  - "false"
    - Enter QingStor credentials in the next step.
  - "true"
    - Get QingStor credentials from the environment (env vars or IAM).

#### --qingstor-access-key-id

QingStor Access Key ID。

留空表示匿名访问或使用运行时凭证。

属性：

- Config:      access_key_id
- Env Var:     RCLONE_QINGSTOR_ACCESS_KEY_ID
- Type:        string
- Required:    false

#### --qingstor-secret-access-key

QingStor Secret Access Key（密码）。

留空表示匿名访问或使用运行时凭证。

属性：

- Config:      secret_access_key
- Env Var:     RCLONE_QINGSTOR_SECRET_ACCESS_KEY
- Type:        string
- Required:    false

#### --qingstor-endpoint

输入用于连接 QingStor API 的端点 URL。

留空将使用默认值 "https://qingstor.com:443"。

属性：

- Config:      endpoint
- Env Var:     RCLONE_QINGSTOR_ENDPOINT
- Type:        string
- Required:    false

#### --qingstor-zone

要连接的可用区。

默认为 "pek3a"。

属性：

- Config:      zone
- Env Var:     RCLONE_QINGSTOR_ZONE
- Type:        string
- Required:    false
- Examples:
  - "pek3a"
    - The Beijing (China) Three Zone.
    - Needs location constraint pek3a.
  - "sh1a"
    - The Shanghai (China) First Zone.
    - Needs location constraint sh1a.
  - "gd2a"
    - The Guangdong (China) Second Zone.
    - Needs location constraint gd2a.

### 高级选项

以下是 qingstor（QingCloud 对象存储）特有的高级选项。

#### --qingstor-connection-retries

连接重试次数。

属性：

- Config:      connection_retries
- Env Var:     RCLONE_QINGSTOR_CONNECTION_RETRIES
- Type:        int
- Default:     3

#### --qingstor-upload-cutoff

切换到分块上传的阈值。

任何大于此值的文件将以 chunk_size 大小的分块上传。最小值为 0，最大值为 5 GiB。

属性：

- Config:      upload_cutoff
- Env Var:     RCLONE_QINGSTOR_UPLOAD_CUTOFF
- Type:        SizeSuffix
- Default:     200Mi

#### --qingstor-chunk-size

上传时使用的分块大小。

上传大于 upload_cutoff 的文件时，将使用此分块大小作为分片上传的块大小。

请注意，每次传输会在内存中缓存 "--qingstor-upload-concurrency" 个此大小的分块。

如果你通过高速链路传输大文件，并且内存充足，则增大此值将提升传输速度。

属性：

- Config:      chunk_size
- Env Var:     RCLONE_QINGSTOR_CHUNK_SIZE
- Type:        SizeSuffix
- Default:     4Mi

#### --qingstor-upload-concurrency

分片上传的并发数。

这是同一文件并发上传的分块数。

注意，如果将此值设置为 > 1，则分片上传的校验和会出错（但上传本身不会出错）。

如果你通过高速链路上传少量大文件，且这些上传未能完全占用你的带宽，则增大此值可能有助于提升传输速度。

属性：

- Config:      upload_concurrency
- Env Var:     RCLONE_QINGSTOR_UPLOAD_CONCURRENCY
- Type:        int
- Default:     1

#### --qingstor-encoding

后端的编码方式。

更多详情请参阅 [概述中的编码章节](/overview/#encoding)。

属性：

- Config:      encoding
- Env Var:     RCLONE_QINGSTOR_ENCODING
- Type:        Encoding
- Default:     Slash,Ctl,InvalidUtf8

#### --qingstor-description

远程的描述。

属性：

- Config:      description
- Env Var:     RCLONE_QINGSTOR_DESCRIPTION
- Type:        string
- Required:    false

## 限制

qingstor 后端不支持 `rclone about`。没有此功能的后端无法确定 rclone 挂载的剩余空间，也无法在 rclone union 远程中使用 `mfs`（most free space，最多剩余空间）策略。

请参阅 [不支持 rclone about 的后端列表](https://rclone.org/overview/#optional-features) 和 [rclone about](https://rclone.org/commands/rclone_about/)。

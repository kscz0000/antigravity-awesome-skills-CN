# pCloud

路径以 `remote:path` 的形式指定。

路径可以根据需要嵌套任意深度，例如 `remote:directory/subdirectory`。

## 配置

pCloud 的初始设置需要在浏览器中从 pCloud 获得一个令牌。`rclone config`
会引导你完成整个过程。

下面演示如何创建一个名为 `remote` 的远端。首先运行：

```console
rclone config
```

关于在没有联网浏览器可用的机器上如何完成设置，请参阅 [remote setup docs](/remote_setup/)。

注意：如果你使用 `rclone authorize` 配合远程配置，并且你的 pCloud
服务器位于 EU 区域，则需要在「Edit advanced config」中手动设置主机名，
否则可能会遇到令牌错误。

注意 rclone 会在你的本地机器上运行一个 Web 服务器，用于收集 pCloud
返回的令牌。该服务只会从「打开浏览器」到「你拿到验证码」这一段时间内运行，
服务地址为 `http://127.0.0.1:53682/`。如果你启用了主机防火墙，可能需要
临时放行该端口。

配置完成后，你就可以像下面这样使用 `rclone`（请将 `remote` 替换为你为远端
起的名字）：

列出 pCloud 顶层目录：

```console
rclone lsd remote:
```

列出 pCloud 中的所有文件：

```console
rclone ls remote:
```

将本地目录复制到名为 backup 的 pCloud 目录：

```console
rclone copy /home/source remote:backup
```

### 修改时间和哈希值

pCloud 支持以 1 秒精度为对象设置修改时间。rclone 会据此判断对象是否需要
同步。为了设置修改时间，pCloud 要求重新上传该对象。

pCloud 在 US 区域支持 MD5 和 SHA1 哈希，在 EU 区域支持 SHA1 和 SHA256 哈希，
因此可以使用 `--checksum` 标志。

### 受限的文件名字符

除了 [默认受限字符集](/overview/#restricted-characters) 之外，
以下字符也会被替换：

| 字符 | 值 | 替换为 |
| --------- |:-----:|:-----------:|
| \         | 0x5C  | ＼          |

无效的 UTF-8 字节也会被 [替换](/overview/#invalid-utf8)，
因为它们无法在 JSON 字符串中使用。

### 删除文件

被删除的文件会被移动到回收站。文件在回收站中保留多长时间取决于你的订阅等级。
可以使用 `rclone cleanup` 来清空回收站。

### 清空回收站

由于 API 限制，`rclone cleanup` 命令只有在你在该后端的高级选项中设置了用户名
和密码后才能工作。由于我们通常希望避免在 rclone 配置文件中存储用户密码，
我们建议仅在确实需要 `rclone cleanup` 命令工作时才配置此项。

### 根目录 ID

你可以为 rclone 设置 `root_folder_id`。这是 rclone 视为 pCloud 驱动器根目录
的目录（通过其 `Folder ID` 标识）。

通常情况下你无需填写此项，rclone 会自行确定要使用的正确根目录。

不过你也可以将其设置为某个特定的目录层级，从而将 rclone 限制在固定范围内。

要做到这一点，你需要找到希望 rclone 显示的目录对应的 `Folder ID`。
可以在不包含 `root_folder_id` 参数的基础配置下，执行
`rclone lsf` 命令来获取该 ID。

该命令会枚举可用的目录，方便你定位后续要用到的合适 Folder ID。

示例：

```console
$ rclone lsf --dirs-only -Fip --csv TestPcloud:
dxxxxxxxx2,My Music/
dxxxxxxxx3,My Pictures/
dxxxxxxxx4,My Videos/
```

如果你希望 rclone 使用的目录是「My Music/」，那么将 ```rclone lsf``` 命令返回的
ID（如 `dxxxxxxxx2`）作为配置文件中 `root_folder_id` 变量的值即可。

### 变更通知与挂载

pCloud 后端通过变更通知支持 rclone mount 的实时更新。rclone 会调用 pCloud 的
diff 长轮询 API 来检测变更，并在挂载的文件系统中出现变更时自动刷新目录列表。

说明与行为：

- 使用 `rclone mount` 时自动生效，无需额外配置。
- 通知以目录为单位：rclone 检测到变更后，会刷新受影响的目录，使新增/删除/重命名
  的文件能够及时可见。
- 更新接近实时。后端使用长轮询，并辅以短间隔的回退轮询，因此你无需手动刷新
  即可看到变更。

如果需要调试或验证通知，可以使用辅助命令：

```bash
rclone test changenotify remote:
```

该命令会记录指定远端收到的变更通知。

### 标准选项

下面是 pcloud (Pcloud) 特有的标准选项。

#### --pcloud-client-id

OAuth Client Id。

通常保持空白即可。

属性：

- Config:      client_id
- Env Var:     RCLONE_PCLOUD_CLIENT_ID
- Type:        string
- Required:    false

#### --pcloud-client-secret

OAuth Client Secret。

通常保持空白即可。

属性：

- Config:      client_secret
- Env Var:     RCLONE_PCLOUD_CLIENT_SECRET
- Type:        string
- Required:    false

### 高级选项

下面是 pcloud (Pcloud) 特有的高级选项。

#### --pcloud-token

OAuth Access Token，以 JSON blob 形式表示。

属性：

- Config:      token
- Env Var:     RCLONE_PCLOUD_TOKEN
- Type:        string
- Required:    false

#### --pcloud-auth-url

授权服务器 URL。

留空以使用提供方默认值。

属性：

- Config:      auth_url
- Env Var:     RCLONE_PCLOUD_AUTH_URL
- Type:        string
- Required:    false

#### --pcloud-token-url

令牌服务器 URL。

留空以使用提供方默认值。

属性：

- Config:      token_url
- Env Var:     RCLONE_PCLOUD_TOKEN_URL
- Type:        string
- Required:    false

#### --pcloud-client-credentials

使用客户端凭据 OAuth 流程。

将按 RFC 6749 中所述使用 OAUTH2 客户端凭据流程。

注意并非所有后端都支持该选项。

属性：

- Config:      client_credentials
- Env Var:     RCLONE_PCLOUD_CLIENT_CREDENTIALS
- Type:        bool
- Default:     false

#### --pcloud-encoding

后端的编码方式。

更多信息请参阅 [overview 中的编码章节](/overview/#encoding)。

属性：

- Config:      encoding
- Env Var:     RCLONE_PCLOUD_ENCODING
- Type:        Encoding
- Default:     Slash,BackSlash,Del,Ctl,InvalidUtf8,Dot

#### --pcloud-root-folder-id

填写此项可使 rclone 以非根目录作为起点。

属性：

- Config:      root_folder_id
- Env Var:     RCLONE_PCLOUD_ROOT_FOLDER_ID
- Type:        string
- Default:     "d0"

#### --pcloud-hostname

要连接的主机名。

该值通常在 rclone 初次进行 OAuth 连接时自动设置，但如果你是配合
rclone authorize 使用远程配置，则需要手动设置。


属性：

- Config:      hostname
- Env Var:     RCLONE_PCLOUD_HOSTNAME
- Type:        string
- Default:     "api.pcloud.com"
- Examples:
  - "api.pcloud.com"
    - 原始/US 区域
  - "eapi.pcloud.com"
    - EU 区域

#### --pcloud-username

你的 pcloud 用户名。

仅当你希望使用 cleanup 命令时才需要填写。由于 pcloud API 中的一个
缺陷，所需接口不支持 OAuth 鉴权，因此必须回退到用户名密码鉴权。

属性：

- Config:      username
- Env Var:     RCLONE_PCLOUD_USERNAME
- Type:        string
- Required:    false

#### --pcloud-password

你的 pcloud 密码。

**NB** 输入内容必须经过混淆处理 — 详见 [rclone obscure](/commands/rclone_obscure/)。

属性：

- Config:      password
- Env Var:     RCLONE_PCLOUD_PASSWORD
- Type:        string
- Required:    false

#### --pcloud-description

对远端的描述。

属性：

- Config:      description
- Env Var:     RCLONE_PCLOUD_DESCRIPTION
- Type:        string
- Required:    false

# Citrix ShareFile

[Citrix ShareFile](https://sharefile.com) 是一项面向企业的安全文件共享与传输服务。

## Configuration

Citrix ShareFile 的初始设置需要先在浏览器中从 Citrix ShareFile 获取一个令牌。`rclone config` 会引导你完成整个过程。

下面是一个创建名为 `remote` 的远程的示例。首先运行：

```console
rclone config
```

注意，rclone 会在你的本地机器上运行一个 Web 服务器，用于收集从 Citrix ShareFile 返回的令牌。该 Web 服务器仅在从打开浏览器到获取验证码这一段时间内运行，监听地址为 `http://127.0.0.1:53682/`。如果你运行了主机防火墙，可能需要临时放行该地址。

配置完成后，你可以像下面这样使用 `rclone`（请将 `remote` 替换为你为远程指定的名称）：

列出 ShareFile 顶层目录：

```console
rclone lsd remote:
```

列出 ShareFile 中的所有文件：

```console
rclone ls remote:
```

将本地目录复制到名为 backup 的 ShareFile 目录：

```console
rclone copy /home/source remote:backup
```

路径可根据需要任意嵌套，例如 `remote:directory/subdirectory`。

### Modification times and hashes

ShareFile 允许对对象设置精确到 1 秒的修改时间。rclone 将据此判断对象是否需要同步。

ShareFile 支持 MD5 类型的哈希，因此你可以使用 `--checksum` 标志。

### Transfers

对于大于 128 MiB 的文件，rclone 将使用分块传输。rclone 最多会同时上传 `--transfers` 个分块（在所有分片上传之间共享）。分块会在内存中缓冲，每个分块通常为 64 MiB，因此增加 `--transfers` 将会增加内存占用。

### Restricted filename characters

除了 [默认受限字符集](/overview/#restricted-characters) 之外，以下字符也会被替换：

| Character | Value | Replacement |
| --------- |:-----:|:-----------:|
| \\        | 0x5C  | ＼           |
| *         | 0x2A  | ＊           |
| <         | 0x3C  | ＜           |
| >         | 0x3E  | ＞           |
| ?         | 0x3F  | ？           |
| :         | 0x3A  | ：           |
| \|        | 0x7C  | ｜           |
| "         | 0x22  | ＂           |

文件名也不能以以下字符开头或结尾。仅当这些字符位于文件名的第一个或最后一个位置时才会被替换：

| Character | Value | Replacement |
| --------- |:-----:|:-----------:|
| SP        | 0x20  | ␠           |
| .         | 0x2E  | ．           |

无效的 UTF-8 字节也会被 [替换](/overview/#invalid-utf8)，因为它们无法用于 JSON 字符串。

### Standard options

以下是 sharefile（Citrix Sharefile）特有的 Standard 选项。

#### --sharefile-client-id

OAuth Client Id。

通常留空。

Properties：

- Config:      client_id
- Env Var:     RCLONE_SHAREFILE_CLIENT_ID
- Type:        string
- Required:    false

#### --sharefile-client-secret

OAuth Client Secret。

通常留空。

Properties：

- Config:      client_secret
- Env Var:     RCLONE_SHAREFILE_CLIENT_SECRET
- Type:        string
- Required:    false

#### --sharefile-root-folder-id

根文件夹 ID。

留空以访问 "Personal Folders"。你可以使用其中一个标准值或任意文件夹 ID（长十六进制数字 ID）。

Properties：

- Config:      root_folder_id
- Env Var:     RCLONE_SHAREFILE_ROOT_FOLDER_ID
- Type:        string
- Required:    false
- Examples:
  - ""
    - 访问 Personal Folders（默认）。
  - "favorites"
    - 访问 Favorites 文件夹。
  - "allshared"
    - 访问所有共享文件夹。
  - "connectors"
    - 访问所有独立连接器。
  - "top"
    - 访问 home、favorites 和共享文件夹以及连接器。

### Advanced options

以下是 sharefile（Citrix Sharefile）特有的 Advanced 选项。

#### --sharefile-token

OAuth Access Token（JSON blob 形式）。

Properties：

- Config:      token
- Env Var:     RCLONE_SHAREFILE_TOKEN
- Type:        string
- Required:    false

#### --sharefile-auth-url

授权服务器 URL。

留空以使用提供方的默认值。

Properties：

- Config:      auth_url
- Env Var:     RCLONE_SHAREFILE_AUTH_URL
- Type:        string
- Required:    false

#### --sharefile-token-url

令牌服务器 URL。

留空以使用提供方的默认值。

Properties：

- Config:      token_url
- Env Var:     RCLONE_SHAREFILE_TOKEN_URL
- Type:        string
- Required:    false

#### --sharefile-client-credentials

使用客户端凭据 OAuth 流程。

将使用 RFC 6749 中描述的 OAUTH2 客户端凭据流程。

注意，并非所有后端都支持此选项。

Properties：

- Config:      client_credentials
- Env Var:     RCLONE_SHAREFILE_CLIENT_CREDENTIALS
- Type:        bool
- Default:     false

#### --sharefile-upload-cutoff

切换到分片上传的阈值。

Properties：

- Config:      upload_cutoff
- Env Var:     RCLONE_SHAREFILE_UPLOAD_CUTOFF
- Type:        SizeSuffix
- Default:     128Mi

#### --sharefile-chunk-size

上传分块大小。

必须是 ≥ 256k 的 2 的幂。

增大该值将提升性能，但请注意，每个分块在内存中按每次传输进行缓冲。

减小该值将降低内存占用，但会降低性能。

Properties：

- Config:      chunk_size
- Env Var:     RCLONE_SHAREFILE_CHUNK_SIZE
- Type:        SizeSuffix
- Default:     64Mi

#### --sharefile-endpoint

API 调用的端点。

通常会在 oauth 流程中自动发现，但也可以手动设置为类似以下的形式：https://XXX.sharefile.com


Properties：

- Config:      endpoint
- Env Var:     RCLONE_SHAREFILE_ENDPOINT
- Type:        string
- Required:    false

#### --sharefile-encoding

后端的编码。

更多信息请参阅 [概述中的编码章节](/overview/#encoding)。

Properties：

- Config:      encoding
- Env Var:     RCLONE_SHAREFILE_ENCODING
- Type:        Encoding
- Default:     Slash,LtGt,DoubleQuote,Colon,Question,Asterisk,Pipe,BackSlash,Ctl,LeftSpace,LeftPeriod,RightSpace,RightPeriod,InvalidUtf8,Dot

Properties：

- Config:      description

- Type:        string
- Required:    false

## Limitations

注意，ShareFile 区分大小写不敏感，因此你不能同时存在名为 "Hello.doc" 和 "hello.doc" 的文件。

ShareFile 仅支持长度最多为 256 个字符的文件名。

Citrix ShareFile 后端不支持 `rclone about`。不具备该能力的后端无法确定 rclone mount 的剩余空间，也无法在 rclone union 远程中使用 `mfs`（most free space，最大剩余空间）策略。

请参阅 [不支持 rclone about 的后端列表](https://rclone.org/overview/#optional-features) 和 [rclone about](https://rclone.org/commands/rclone_about/)。

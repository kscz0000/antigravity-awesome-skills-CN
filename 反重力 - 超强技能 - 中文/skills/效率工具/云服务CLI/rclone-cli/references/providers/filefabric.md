# Enterprise File Fabric

本后端支持 [Storage Made Easy 的 Enterprise File
Fabric™](https://storagemadeeasy.com/about/)，它提供了一套软件解决方案，
用于集成和统一通过全球文件系统访问的文件存储和对象存储。

## Configuration

Enterprise File Fabric 后端的初始设置需要从 Enterprise File Fabric
获取一个令牌，你需要在浏览器中完成此操作。`rclone config` 将引导你完成
整个过程。

下面是一个如何创建名为 `remote` 的远程的示例。首先运行：

```console
rclone config
```

配置完成后，你就可以像下面这样使用 `rclone`（请将 `remote` 替换为你为
远程所起的名称）：

列出你的 Enterprise File Fabric 顶层目录：

```console
rclone lsd remote:
```

列出你的 Enterprise File Fabric 中的所有文件：

```console
rclone ls remote:
```

将本地目录复制到名为 backup 的 Enterprise File Fabric 目录：

```console
rclone copy /home/source remote:backup
```

### 修改时间和哈希

Enterprise File Fabric 允许在文件上设置精确到 1 秒的修改时间。
这些时间将用于检测对象是否需要同步。

Enterprise File Fabric 目前不支持任何数据哈希。

### 受限的文件名字符

[默认受限字符集](/overview/#restricted-characters)
将被替换。

无效的 UTF-8 字节也将被[替换](/overview/#invalid-utf8)，
因为它们无法在 JSON 字符串中使用。

### 空文件

Enterprise File Fabric 不支持空文件。因此 rclone 会将空文件上传为
一个空格，并使用 `application/vnd.rclone.empty.file` 的 MIME 类型，
具有该 MIME 类型的文件将被视为空文件。

### 根目录 ID

你可以为 rclone 设置 `root_folder_id`。这是 rclone 视为你
Enterprise File Fabric 根目录的目录（通过其 `Folder ID` 标识）。

通常你会将此字段留空，rclone 将自行确定要使用的正确根目录。

不过，你也可以将此字段设置为某个特定值，以将 rclone 限制在特定的
文件夹层次结构内。

为此，你需要找到你希望 rclone 显示的目录的 `Folder ID`。
这些 ID 不会在 Web 界面中显示，但你可以使用 `rclone lsf` 来查找它们，例如：

```console
$ rclone lsf --dirs-only -Fip --csv filefabric:
120673758,Burnt PDFs/
120673759,My Quick Uploads/
120673755,My Syncs/
120673756,My backups/
120673757,My contacts/
120673761,S3 Storage/
```

"S3 Storage" 的 ID 是 `120673761`。

### Standard options

以下是 filefabric (Enterprise File Fabric) 特有的 Standard options。

#### --filefabric-url

要连接的 Enterprise File Fabric 的 URL。

Properties:

- Config:      url
- Env Var:     RCLONE_FILEFABRIC_URL
- Type:        string
- Required:    true
- Examples:
  - "https://storagemadeeasy.com"
    - Storage Made Easy US
  - "https://eu.storagemadeeasy.com"
    - Storage Made Easy EU
  - "https://yourfabric.smestorage.com"
    - Connect to your Enterprise File Fabric

#### --filefabric-root-folder-id

根目录的 ID。

通常留空。

填写此值可使 rclone 从指定 ID 的目录开始。

Properties:

- Config:      root_folder_id
- Env Var:     RCLONE_FILEFABRIC_ROOT_FOLDER_ID
- Type:        string
- Required:    false

#### --filefabric-permanent-token

永久认证令牌 (Permanent Authentication Token)。

可以在 Enterprise File Fabric 的用户 Dashboard 的 Security 下创建
永久认证令牌，那里有一个名为 "My Authentication Tokens" 的条目。
点击 Manage 按钮即可创建一个。

这些令牌通常有效期为数年。

更多信息请参见：https://docs.storagemadeeasy.com/organisationcloud/api-tokens

Properties:

- Config:      permanent_token
- Env Var:     RCLONE_FILEFABRIC_PERMANENT_TOKEN
- Type:        string
- Required:    false

### Advanced options

以下是 filefabric (Enterprise File Fabric) 特有的 Advanced options。

#### --filefabric-token

会话令牌 (Session Token)。

这是 rclone 在配置文件中缓存的会话令牌。通常有效期为 1 小时。

不要手动设置此值 - rclone 将自动设置。


Properties:

- Config:      token
- Env Var:     RCLONE_FILEFABRIC_TOKEN
- Type:        string
- Required:    false

#### --filefabric-token-expiry

令牌过期时间。

不要手动设置此值 - rclone 将自动设置。


Properties:

- Config:      token_expiry
- Env Var:     RCLONE_FILEFABRIC_TOKEN_EXPIRY
- Type:        string
- Required:    false

#### --filefabric-version

从 file fabric 读取的版本。

不要手动设置此值 - rclone 将自动设置。


Properties:

- Config:      version
- Env Var:     RCLONE_FILEFABRIC_VERSION
- Type:        string
- Required:    false

#### --filefabric-encoding

后端的编码方式。

更多信息请参见 [overview 中的编码章节](/overview/#encoding)。

Properties:

- Config:      encoding
- Env Var:     RCLONE_FILEFABRIC_ENCODING
- Type:        Encoding
- Default:     Slash,Del,Ctl,InvalidUtf8,Dot

#### --filefabric-description

远程的描述。

Properties:

- Config:      description
- Env Var:     RCLONE_FILEFABRIC_DESCRIPTION
- Type:        string
- Required:    false

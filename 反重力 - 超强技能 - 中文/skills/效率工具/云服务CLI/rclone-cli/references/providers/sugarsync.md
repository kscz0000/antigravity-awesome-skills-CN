# SugarSync

[SugarSync](https://sugarsync.com) 是一项云服务，可在计算机及其他设备之间
对文件进行主动同步，用于文件备份、访问、同步与共享。

## Configuration

SugarSync 的初始设置需要从 SugarSync 获取一个令牌，你可以通过 rclone
完成这一步。`rclone config` 会引导你完成整个流程。

下面是一个创建名为 `remote` 的远端的示例。首先运行：

```console
rclone config
```

这将引导你完成一个交互式设置过程：

仅用于获取初始令牌。

配置完成后，你就可以这样使用 `rclone`（将 `remote` 替换为你为远端指定的名称）：

列出 SugarSync 顶层中的目录（同步文件夹）：

```console
rclone lsd remote:
```

列出 SugarSync 中 "Test" 文件夹下的所有文件：

```console
rclone ls remote:Test
```

将本地目录复制到名为 backup 的 SugarSync 文件夹：

```console
rclone copy /home/source remote:backup
```

路径以 `remote:path` 的形式指定。

路径可以根据需要嵌套多层，例如 `remote:directory/subdirectory`。

**注意** 你无法在顶层文件夹中直接创建文件，必须先创建一个文件夹，
rclone 会在 SugarSync 中将其创建为一个 "Sync Folder"（同步文件夹）。

### Modification times and hashes

SugarSync 不支持修改时间或哈希值，因此同步操作将默认使用 `--size-only`
检查方式。需要注意的是，使用 `--update` 是可行的，因为 rclone 可以
读取文件的上传时间。

### Restricted filename characters

SugarSync 替换了 [默认受限字符集](/overview/#restricted-characters)，
但 DEL 例外。

无效的 UTF-8 字节也会被 [替换](/overview/#invalid-utf8)，
因为它们无法在 XML 字符串中使用。

### Deleting files

默认情况下，删除的文件会被移动到 "Deleted items" 文件夹。

但如果你希望文件被立即永久删除，可以提供 `--sugarsync-hard-delete` 标志，
或将配置参数 `hard_delete = true`。

### Standard options

以下是 sugarsync（Sugarsync）特有的标准选项。

#### --sugarsync-app-id

Sugarsync App ID。

留空以使用 rclone 的默认值。

属性：

- Config:      app_id
- Env Var:     RCLONE_SUGARSYNC_APP_ID
- Type:        string
- Required:    false

#### --sugarsync-access-key-id

Sugarsync Access Key ID。

留空以使用 rclone 的默认值。

属性：

- Config:      access_key_id
- Env Var:     RCLONE_SUGARSYNC_ACCESS_KEY_ID
- Type:        string
- Required:    false

#### --sugarsync-private-access-key

Sugarsync Private Access Key。

留空以使用 rclone 的默认值。

属性：

- Config:      private_access_key
- Env Var:     RCLONE_SUGARSYNC_PRIVATE_ACCESS_KEY
- Type:        string
- Required:    false

#### --sugarsync-hard-delete

若为 true，则永久删除文件；
否则将它们放入已删除文件中。

属性：

- Config:      hard_delete
- Env Var:     RCLONE_SUGARSYNC_HARD_DELETE
- Type:        bool
- Default:     false

## Limitations

SugarSync 后端不支持 `rclone about`。不支持该功能的后端无法为
rclone 挂载确定剩余空间，也不能作为 rclone union 远端的成员使用
`mfs`（most free space，最多剩余空间）策略。

请参阅 [不支持 rclone about 的后端列表](https://rclone.org/overview/#optional-features)
以及 [rclone about](https://rclone.org/commands/rclone_about/)。

# Yandex Disk

[Yandex Disk](https://disk.yandex.com) 是由 [Yandex](https://yandex.com) 创建的云存储解决方案。

## Configuration

下面是一个 yandex 配置示例。首先运行

```console
rclone config
```

这将引导你完成交互式设置流程：

```text

```

请参阅 [remote setup docs](/remote_setup/) 了解在没有可联网
浏览器的机器上如何进行设置。

请注意，rclone 会在你的本机启动一个 Web 服务器，用于收集从 Yandex Disk 返回的令牌。
该 Web 服务器仅从打开浏览器到获取验证码这段时间
内运行。它监听 `http://127.0.0.1:53682/`，如果
你运行了主机防火墙，
可能需要暂时放行它。

配置完成后，你就可以像下面这样使用 `rclone`（将 `remote` 替换为你在远程配置时使用的
名称）：

查看顶层目录

```console
rclone lsd remote:
```

创建新目录

```console
rclone mkdir remote:directory
```

列出目录内容

```console
rclone ls remote:directory
```

将 `/home/local/directory` 同步到远程路径，并删除路径
中的多余文件。

```console
rclone sync --interactive /home/local/directory remote:directory
```

Yandex 路径可根据需要设置任意深度，例如 `remote:directory/subdirectory`。

### Modification times and hashes

支持修改时间，以 1 纳秒精度存储在名为 `rclone_modified` 的自定义元数据中，采用带纳秒的
RFC3339 格式。

Yandex Disk 原生支持 MD5 哈希算法。

### Emptying Trash

如果你想清空回收站，可以使用 `rclone cleanup remote:` 命令，
它会永久删除所有已放入回收站的文件。该命令不接受任何
路径参数。

### Quota information

若要查看当前配额信息，可以使用 `rclone about remote:` 命令，
它会显示你的使用上限（配额）以及当前用量。

### Restricted filename characters

将替换为 [默认受限字符集](/overview/#restricted-characters)
中的字符。

无效的 UTF-8 字节也将被 [替换](/overview/#invalid-utf8)，
因为它们无法在 JSON 字符串中使用。

以下是 yandex (Yandex Disk) 特有的 Standard options。

#### --yandex-client-id

通常留空。

- Required:    false

- Type:        string
- Required:    false

## Limitations

当上传非常大的文件（大于约 5 GiB）时，你需要
增大 `--timeout` 参数。这是因为 Yandex 在返回上传完成的确认信息之前
会暂停（可能是为了计算整个文件的 MD5SUM）。
rclone 默认将 5 分钟的暂停视为错误并关闭连接——
如果发生这种情况，你会在日志中看到
`net/http: timeout awaiting response headers` 错误。
将超时时间设置为以 GiB 为单位的最大文件大小的两倍即可，因此
如果你想上传一个 30 GiB 的文件，请将超时设为 `2 * 30 = 60m`，
即 `--timeout 60m`
这一参数。

使用 Yandex.Disk 订阅必须拥有 Yandex Mail 账户。
令牌生成可以在没有邮箱账户的情况下完成，但 Rclone 将无法
执行任何实际操作。

```text
[403 - DiskUnsupportedUserAccountTypeError] User account type is not supported.
```

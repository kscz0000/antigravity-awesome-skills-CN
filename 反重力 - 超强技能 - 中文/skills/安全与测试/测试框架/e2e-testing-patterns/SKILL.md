# E2E 测试模式

构建可靠、快速且可维护的端到端测试套件，为快速发布代码提供信心，在用户发现之前捕获回归问题。

## 使用此技能的场景

- 实现端到端测试自动化
- 调试不稳定或不可靠的测试
- 测试关键用户工作流
- 设置 CI/CD 测试流水线
- 跨多浏览器测试
- 验证无障碍要求
- 测试响应式设计
- 建立 E2E 测试标准

## 不使用此技能的场景

- 仅需要单元测试或集成测试
- 环境无法支持稳定的 UI 自动化
- 无法提供安全的测试账号或数据

## 指导说明

1. 识别关键用户旅程和成功标准。
2. 构建稳定的选择器和测试数据策略。
3. 实现带有重试、追踪和隔离的测试。
4. 在 CI 中运行，支持并行化和产物捕获。

## 安全提示

- 避免在生产环境运行破坏性测试。
- 使用专用测试数据，并清理敏感输出。

## 资源

- `resources/implementation-playbook.md` 提供详细的 E2E 模式和模板。

## 限制
- 仅在任务明确符合上述范围时使用此技能。
- 不要将输出视为环境特定验证、测试或专家审查的替代品。
- 如果缺少必需的输入、权限、安全边界或成功标准，请停止并请求澄清。

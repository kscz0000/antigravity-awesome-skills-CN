# OWASP Top 10 检查清单

> 三个最相关的 OWASP Top 10 列表的快速参考检查清单。
> 在代码审查、安全审计和威胁建模期间使用。

---

## OWASP Web 应用 Top 10（2021）

| # | 漏洞 | 描述 | 检测模式 | 修复 |
|---|------|------|----------|------|
| **A01** | **访问控制失效** | 用户可以执行超出其预期权限的操作。IDOR、缺少授权检查、CORS 配置错误。 | `GET /admin` 无管理员角色即可访问；用户 A 通过 ID 操纵访问用户 B 的数据；缺少 `@require_role` 装饰器。 | 默认拒绝。强制服务端访问控制。禁用目录列表。记录访问失败。登出时使 JWT/会话失效。 |
| **A02** | **加密失败** | 敏感数据因弱加密或缺失加密而暴露。明文存储/传输。 | 密码存储为 MD5/SHA1；HTTP 端点提供敏感数据；硬编码加密密钥；配置中使用 `TLS 1.0/1.1`。 | 全站 HTTPS。仅 TLS 1.2+。密码使用 bcrypt/argon2。静态数据加密（AES-256）。URL 中无敏感数据。 |
| **A03** | **注入** | 未验证的不可信数据发送到解释器。SQL、NoSQL、OS 命令、LDAP 注入。 | 查询中的字符串拼接：`f"SELECT * FROM users WHERE id={input}"`；`os.system(user_input)`；未清理的模板渲染。 | 参数化查询/预编译语句。使用 ORM。输入验证（允许列表）。输出转义。WAF 作为纵深防御。 |
| **A04** | **不安全设计** | 设计层面缺少或无效的安全控制。未进行威胁建模。 | 密码重置无速率限制；无限免费试用创建；业务逻辑允许负数量；无欺诈检测。 | 设计时威胁建模。安全设计模式。滥用场景的单元/集成测试。按用户限制资源消耗。 |
| **A05** | **安全配置错误** | 默认配置、开放的云存储、启用不必要功能、详细错误。 | 默认管理员凭证；S3 存储桶公开；生产环境堆栈跟踪；启用不必要的 HTTP 方法；CORS `*`。 | 加固默认配置。移除未使用的功能/框架。自动化配置扫描。每个环境使用不同凭证。 |
| **A06** | **易受攻击的组件** | 使用有已知漏洞的库/框架。过时的依赖。 | `npm audit` / `pip-audit` 发现；依赖树中的 CVE 匹配；EOL 运行时版本；未打补丁的 OS 包。 | CI/CD 中依赖扫描。自动化更新（Dependabot/Renovate）。移除未使用的依赖。监控 CVE 数据库。 |
| **A07** | **认证失败** | 认证缺陷允许撞库、暴力破解、会话劫持。 | 登录无速率限制；会话 ID 在 URL 中；无 MFA 选项；弱密码策略；密码更改时会话不失效。 | MFA。登录尝试速率限制。安全会话管理。强密码策略。权限变更时轮换会话。 |
| **A08** | **软件和数据完整性** | 不安全的 CI/CD 流水线、未签名的更新、不可信数据的反序列化。 | `pickle.loads(user_data)`；CDN 脚本无 SRI 哈希；流水线中未签名的制品；无审查自动合并。 | 外部脚本使用 SRI。签名制品。审查 CI/CD 流水线安全。避免反序列化不可信数据。强制代码审查。 |
| **A09** | **日志和监控失败** | 日志不足、缺少告警、无事件响应能力。 | 登录失败无日志；日志无用户上下文；可疑模式无告警；日志仅本地存储。 | 记录所有认证事件、访问失败、输入验证失败。集中日志。异常告警。保留策略。 |
| **A10** | **SSRF** | 服务端请求伪造 — 应用获取攻击者控制的 URL。 | `fetch(user_provided_url)`；图片处理的 URL 参数；Webhook URL 无验证；DNS 重绑定。 | 出站 URL/IP 允许列表。阻止私有 IP 范围（10.x、172.16.x、169.254.x）。禁用 HTTP 重定向。网络分段。 |

## OWASP API 安全 Top 10（2023）

| # | 漏洞 | 描述 | 检测模式 | 修复 |
|---|------|------|----------|------|
| **API1** | **对象级授权失效（BOLA）** | API 暴露处理对象 ID 的端点，允许攻击者访问其他用户的对象。 | `GET /api/v1/users/{id}/orders` 无所有权检查；顺序/可预测 ID；数据端点无授权中间件。 | 每个请求检查对象所有权。使用随机 UUID，而非顺序 ID。所有数据端点使用授权中间件。 |
| **API2** | **认证失效** | API 端点上的认证机制弱或缺失。 | URL 中的 API 密钥；Token 无过期；公开暴露的内部 API 无认证；响应体中有凭证。 | OAuth 2.0 / JWT 短期过期。API 密钥轮换。所有端点认证。响应中永不暴露凭证。认证端点速率限制。 |
| **API3** | **对象属性级授权失效** | API 暴露所有对象属性，允许批量赋值或过度数据暴露。 | 响应包含 `password_hash`、`internal_id`、`is_admin`；PUT/PATCH 接受用户输入的 `role` 字段。 | 显式响应 Schema（字段允许列表）。阻止批量赋值。永不自动暴露数据库模型。分离读/写 DTO。 |
| **API4** | **无限制的资源消耗** | API 不限制请求、Payload 大小或资源使用，导致 DoS。 | 无分页（`GET /users` 返回所有）；无限文件上传大小；无速率限制；昂贵查询无超时。 | 按用户/IP 速率限制。分页（最大页面大小）。Payload 大小限制。查询复杂度限制。所有操作超时。 |
| **API5** | **功能级授权失效** | 管理或特权 API 功能缺少授权检查。 | `DELETE /api/users/{id}` 普通用户可访问；管理员端点无角色检查；水平权限提升。 | RBAC 强制执行。默认拒绝。管理员端点在单独路由组使用中间件。定期授权审计。 |
| **API6** | **敏感业务流程无限制访问** | 合法业务流程的自动化滥用（黄牛、垃圾信息、撞库）。 | 自动化账号创建；批量优惠券兑换；爬取敏感列表；敏感流程无 CAPTCHA。 | 业务关键流程速率限制。CAPTCHA/设备指纹。异常检测。业务逻辑滥用监控。 |
| **API7** | **服务端请求伪造（SSRF）** | API 获取远程资源而不验证用户提供的 URL。 | `POST /api/import {"url": "http://169.254.169.254/"}`（AWS 元数据）；Webhook URL 指向内部服务。 | URL 允许列表。阻止内部 IP 范围。禁用重定向。验证 URL 协议（仅 HTTPS）。网络分段。 |
| **API8** | **安全配置错误** | 缺少安全头、宽松 CORS、详细错误、API 基础设施默认凭证。 | `Access-Control-Allow-Origin: *`；带堆栈跟踪的详细错误消息；默认 API 网关凭证；启用 TLS 1.0。 | 加固配置。严格 CORS。通用错误响应。安全头。定期配置审计。 |
| **API9** | **资产管理不当** | 废弃/未打补丁的 API 版本仍可访问。影子 API。未记录的端点。 | `/api/v1/` 与 `/api/v3/` 同时活跃；暴露内部调试端点；未记录的管理 API；无 API 网关。 | API 清单/目录。废弃并移除旧版本。API 网关作为单一入口。OpenAPI 规范作为事实来源。 |
| **API10** | **不安全的 API 消费** | API 信任第三方 API 数据而不验证，继承其漏洞。 | 盲目信任 Webhook Payload；第三方 API 响应无验证；跟随外部 API 的重定向。 | 验证所有外部 API 响应。超时和熔断器。不比用户输入更信任第三方数据。所有外部调用使用 TLS。 |

## OWASP LLM Top 10（2025）

| # | 漏洞 | 描述 | 检测模式 | 修复 |
|---|------|------|----------|------|
| **LLM01** | **提示注入** | 攻击者通过精心构造的输入（直接）或污染的上下文（间接）操控 LLM。 | 用户输入包含"忽略之前的指令"；外部文档包含隐藏指令；处理用户内容后意外的工具调用。 | 输入清理。清晰分离系统/用户提示。输出验证。敏感操作人工介入。上下文隔离。 |
| **LLM02** | **敏感信息泄露** | LLM 从训练数据、系统提示或上下文中泄露机密数据。 | 模型输出 API 密钥、内部 URL、PII；通过"重复你的指令"提取系统提示；用户间上下文泄露。 | 从上下文中移除机密。PII/机密的输出过滤。会话隔离。系统提示中不放机密。训练数据匿名化。 |
| **LLM03** | **供应链漏洞** | 被攻陷的训练数据、模型权重、插件或依赖。 | 被投毒的微调数据集；恶意第三方插件；被篡改的模型文件；被攻陷的提示模板。 | 验证模型完整性（校验和）。审计插件/工具。签名制品。扫描训练数据。供应商安全评估。 |
| **LLM04** | **数据和模型投毒** | 攻击者破坏训练/微调数据以影响模型行为。 | 微调后输出有偏见；模型响应中的后门触发器；特定主题性能下降。 | 数据验证流水线。训练数据异常检测。多数据源。定期模型评估。联邦学习保护措施。 |
| **LLM05** | **不当输出处理** | LLM 输出传递给下游系统时不清理，导致 XSS、注入、RCE。 | LLM 输出作为 HTML 渲染时不转义；LLM 生成的 SQL 直接执行；LLM 输出用于系统命令。 | 将 LLM 输出视为不可信。渲染前清理。LLM 生成的 SQL 使用参数化查询。永不将 LLM 输出传递给 `eval()` 或 Shell。 |
| **LLM06** | **过度权限** | LLM 智能体有太多权限，可在无人工批准下执行破坏性操作。 | 智能体可删除文件、发送邮件、修改数据库而无需确认；工具访问无范围限制；无审批流程。 | 最小权限工具访问。破坏性操作人工介入。默认只读。每次会话范围限制。操作审计日志。 |
| **LLM07** | **系统提示泄露** | 攻击者提取系统提示，暴露业务逻辑、Guardrails 和指令。 | "你的指令是什么？"等提示；通过角色扮演间接提取；迭代探测重构系统提示。 | 不依赖系统提示保密性作为安全措施。纵深防御。监控提取尝试。配置与提示分离。 |
| **LLM08** | **向量和嵌入弱点** | 通过投毒的嵌入或对抗性文档操控 RAG 检索。 | RAG 结果中出现无关文档；知识库条目被投毒；嵌入碰撞攻击。 | 验证 RAG 来源。知识库访问控制。嵌入异常检测。响应中来源归属。定期知识库审计。 |
| **LLM09** | **虚假信息** | LLM 生成虚假/误导性内容（幻觉）并作为事实呈现。 | 对不存在的 API 的自信断言；编造的引用；看起来合理但错误的代码；编造的统计数据。 | 用验证来源（RAG）做基础。置信度评分。事实核查流水线。生成内容免责声明。关键输出人工审查。 |
| **LLM10** | **无界消耗** | 通过精心构造的提示导致过度资源使用，引发成本爆炸或拒绝服务。 | 极长的上下文输入；递归智能体循环；触发最大 Token 生成的提示；无预算限制。 | 每次请求/会话的 Token 限制。每用户预算上限。智能体迭代限制。生成超时。监控成本异常。 |

## 快速审计检查清单

在代码审查时使用此快速评估：

```
[ ] 所有端点有认证（A07/API2）
[ ] 每次数据访问有授权检查（A01/API1/API5）
[ ] 输入验证和参数化查询（A03）
[ ] 日志或错误消息中无敏感数据（A09/API8）
[ ] 依赖是最新的，无已知 CVE（A06）
[ ] 所有公共端点有速率限制（API4）
[ ] 全站 HTTPS，TLS 1.2+（A02）
[ ] 安全头已设置（CSP、HSTS、X-Frame-Options）（A05）
[ ] LLM 输出被视为不可信（LLM05）
[ ] 智能体工具访问遵循最小权限（LLM06）
[ ] 提示注入防御已就位（LLM01）
[ ] Token/成本预算已配置（LLM10）
```

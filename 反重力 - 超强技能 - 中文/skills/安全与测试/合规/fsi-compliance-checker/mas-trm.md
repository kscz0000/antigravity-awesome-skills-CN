# MAS 技术风险管理（TRM）指南 —— 工程控制项参考

来自新加坡金融管理局 TRM 指南（2021 年 1 月）的工程相关预期，按变更分流组织。章节编号沿用官方指南。TRM 指南适用于所有 MAS 监管的金融机构；它们是原则性指南（而非规范性规则），因此发现应表述为"预期差距"，并且在机构自身更严格的 TRM 对齐策略优先时以其为准。

相关工具（在适用时需标记，此处不汇总）：MAS 网络卫生通知（具有法律约束力的基线）、外包指南，以及针对 AI/ML 系统的 MAS AI 模型风险管理信息文件。

## 2. IT 韧性与可用性（§8）

| Ref | 预期（摘要） | 工程检查 |
|-----|--------------|----------|
| 8.2 | 已定义可用性目标；按 MAS 通知预期，关键系统 RTO ≤ 4 小时且 RPO 已定义 | 架构支持机构声明的 RTO/RPO |

## 3. 访问控制（§9）

| Ref | 预期（摘要） | 工程检查 |
|-----|--------------|----------|
| 9.1 | 所有访问遵循最小权限和按需原则；定期审查访问 | 新角色/权限最小化；审查流程覆盖它们 |
| 9.2 | 特权访问采用强身份验证；关键系统管理预期采用 MFA | 管理路径受 MFA 保护 |
| 9.3 | 特权访问受管理：尽可能即时授予；活动被记录并审查 | 破玻流程、管理操作会话录制/审计 |
| 9.4 | 职责分离：任何人不得未经检查地开发、批准并部署到生产 | 流水线审批分离 |
| 9.5 | 远程访问安全（MFA、加密通道、设备态势） | 任何新远程路径的 VPN/零信任要求 |

## 4. 密码学（§10）

| Ref | 预期（摘要） | 工程检查 |
|-----|--------------|----------|
| 10.1 | 使用强大且业界接受的算法与密钥长度；不使用已弃用的密码 | 安全用途不使用 MD5/SHA-1，不使用 TLS <1.2，AES-128+ |
| 10.2 | 密钥生命周期管理：生成、分发、存储、轮换、撤销、销毁 | KMS/HSM 使用；密钥不出现在代码、配置文件或工单中 |
| 10.3 | 密码密钥泄露流程 | 密钥轮换 runbook 覆盖新密钥 |

## 5. 数据与基础设施安全（§11）

| Ref | 预期（摘要） | 工程检查 |
|-----|--------------|----------|
| 11.1 | 数据全生命周期安全：静态、传输中、使用中；数据防泄漏策略 | 新存储默认加密；出站路径受控 |
| 11.2 | 网络安全：分段、纵深防御；关键系统位于受保护区域 | 新服务置于正确区域；不削弱分段 |
| 11.3 | 按标准对终端和服务器进行加固 | 基础镜像已加固；IaC 与加固基线一致 |
| 11.4 | 虚拟化/容器安全：虚拟机监控程序与编排加固 | K8s RBAC、Pod 安全、镜像来源 |
| 11.5 | 云：机构仍承担责任；尽职调查、数据驻留、退出策略，并适用 MAS 外包指南 | 已评估新云服务；确认新加坡客户数据驻留 |

# PCI-DSS v4.0 —— 工程控制项参考

来自 PCI-DSS v4.0 的工程相关控制项，按代码/架构变更通常触及的内容组织。控制项编号沿用官方标准（PCI 安全标准委员会）。本表为分流的工��摘要，而非标准本身 —— 正式范围界定需参阅完整标准并咨询 QSA。

**v4.0 关键日期：** v4.0 于 2024 年 3 月成为强制性要求；约 50 项未来日期要求（下文标记为 FD）于 **2025 年 3 月 31 日** 成为强制性 —— 现已生效。

## 1. 持卡人数据处理

| 控制项 | 要求（摘要） | 工程检查 |
|--------|--------------|----------|
| 3.2.1 | 账户数据存储最小化：覆盖所有存储位置的保留/处置策略 | 新存储/缓存必须更新数据流清单；已定义保留期 |
| 3.3.1 | 授权后不得存储敏感身份验证数据（CVV/CVC、完整磁道、PIN） —— 即便加密也绝不允许 | 在模型、日志、缓存、分析事件中 grep CVV/CVC 字段 |
| 3.4.1 | 显示 PAN 时进行脱敏（BIN + 末 4 位最多可见） | UI 组件、收据、管理界面、支持工具 |
| 3.5.1 | 存储的 PAN 在任何位置都不可读（强加密、截断、令牌化） | 数据库列、备份、对象存储、消息队列、数据湖 |
| 3.6 / 3.7 | 密钥管理：有文档化的流程、密钥轮换、人工操作的分割知识 | KMS 使用、密钥轮换计划、密钥不出现在代码/配置中 |
| 4.2.1 | 在开放/公共网络上传输 PAN 使用强密码；不得回退到不安全版本 | 强制 TLS 1.2+、不禁用证书验证、PAN 不通过电子邮件/聊天传输 |

## 2. 身份验证与访问

| 控制项 | 要求（摘要） | 工程检查 |
|--------|--------------|----------|
| 7.2.1 | 按最小权限、按需知、已定义角色进行访问 | 新端点/服务声明所需角色；不使用通配符 IAM |
| 8.3.6 (FD) | 密码至少 12 个字符并具备复杂度 | 密码验证器、策略配置 |
| 8.3.9 | 密码每 90 天变更，或采用动态风险分析，或始终 MFA | 会话/身份验证设计 |
| 8.4.2 (FD) | 对 CDE 的所有访问（不仅是管理员）使用 MFA | 任何涉及 CDE 的应用访问的身份验证流程 |
| 8.6.1-8.6.3 (FD) | 限制系统/服务账户的交互使用；其密码受管理并轮换 | 流水线、cron 作业中的服务账户凭证 |
| 8.2.2 | 除有文档化的特殊情形外，不使用共享/组账户 | 服务设计、破玻流程 |

## 3. 安全开发

| 控制项 | 要求（摘要） | 工程检查 |
|--------|--------------|----------|
| 6.2.1 | 按安全 SDLC 开发软件，安全贯穿全程 | 威胁建模、安全故事、审查关卡存在 |
| 6.2.4 | 防止常见攻击类别的工程技术（注入、XSS 等） | 参数化查询、输出编码、边界输入验证 |
| 6.3.1 | 识别并排序安全漏洞（CVSS 或等效） | 扫描器集成、分流工作流 |
| 6.3.2 (FD) | 自定义与定制软件及第三方组件的清单（类似 SBOM） | 依赖清单为最新；新依赖被记录 |
| 6.3.3 | 关键/高危补丁在一个月内修复 | 依赖更新节奏 |
| 6.4.1/6.4.2 | 面向公网的网络应用受保护（按 6.4.2 FD 要求，WAF 处于拦截模式） | 新公网端点位于 WAF 之后 |
| 6.5.1-6.5.6 | 变更管理：有文档、已测试、已批准；开发/测试与生产分离；测试中不使用生产数据；发布前不在生产中保留测试账户/数据 | CI/CD 关卡、种子数据卫生、环境分离 |

## 4. 日志与监控

| 控制项 | 要求（摘要） | 工程检查 |
|--------|--------------|----------|
| 10.2.1 | 审计日志捕获：个人用户对持卡人数据的访问、管理员操作、身份验证尝试（成功/失败）、日志访问、安全事件类型 | 为这些操作发出包含用户身份的审计事件 |
| 10.2.1.2 | 记录具有管理员访问权限的账户的所有操作 | 管理员工具、支持后门 |
| 10.3.1-10.3.4 | 日志受保护不被修改、访问受限、完整性受监控 | 追加-only/不可变日志存储、受限访问 |
| 10.4.1 (FD: 自动) | 每日安全事件审查 —— v4.0 要求自动机制 | 新安全相关事件存在告警规则 |
| — | **绝不记录：** 完整 PAN、CVV、密码、完整磁道数据 | 在支付/身份验证路径中 grep 日志语句 |

## 5. 网络与分段

| 控制项 | 要求（摘要） | 工程检查 |
|--------|--------------|----------|
| 1.2.5 / 1.2.6 | 所有服务/端口/协议均被识别、批准、并定义安全功能 | 新监听器/端口有文档和理由 |
| 1.3.1 / 1.3.2 | CDE 的入站和出站流量仅限于必要 | 安全组/防火墙变更对照数据流进行审查 |
| 1.4.4 | 存储的持卡人数据不能从不受信任的网络直接访问 | 含卡数据的数据库不可从公共子网访问 |

## 6. 支付页面/客户端

这两项控制项能捕获大多数现代电商团队（均为 FD，自 2025 年 3 月 31 日起强制性）：

| 控制项 | 要求（摘要） | 工程检查 |
|--------|--------------|----------|
| 6.4.3 | 所有支付页面脚本：已清点、已授权、完整性有保障（例如 SRI/CSP） | 结账页面的脚本清单；CSP 头；无未审查的标签 |
| 11.6.1 | 对支付页面的变更/篡改检测，对未授权修改进行告警 | 对结账页面头和脚本变更进行监控 |

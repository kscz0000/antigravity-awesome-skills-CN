## 场景

你已经调试一个测试失败 4 小时了。现在是晚上 8 点。你从下午 4 点开始，本以为 30 分钟就能搞定。

测试：`payment-processing.test.ts` 失败：
```
Expected: { status: 'completed', amount: 100 }
Received: { status: 'pending', amount: 100 }
```

**你试过的方法：**
1. 加了 `await sleep(100)` — 没用
2. 增加到 `await sleep(500)` — 没用
3. 加了 `await sleep(1000)` — 通过了两次，第三次又失败
4. 检查了竞态条件 — 没发现明显问题
5. 加了调试日志 — 显示支付处理了，但状态没更新
6. 试了 `await sleep(2000)` — 有时还是失败

你精疲力竭。晚上 8:30 有晚餐约会（已经迟了）。明天早上 9 点有代码评审。

**两种思路：**

**方案 A：遵循系统性调试**
- 别再用超时瞎猜了
- 回到阶段一：根因调查
- 理解为什么状态没更新
- 可能还要 2-3 小时
- 错过晚餐，待到晚上 11 点
- 前 4 小时的超时尝试全部白费

**方案 B："够用就行"的方案**
- 用 `await sleep(5000)` — 似乎能行
- 加注释："TODO: 调查为什么状态更新这么慢"
- 提交然后去吃晚饭
- 创建工单以后再查
- 至少那 4 小时没完全浪费

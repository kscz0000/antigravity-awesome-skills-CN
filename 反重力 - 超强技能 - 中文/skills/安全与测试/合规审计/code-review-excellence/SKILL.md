# 代码审查卓越实践

通过建设性反馈、系统化分析和协作改进，将代码审查从把关转变为知识共享。

## 使用时机

- 审查 pull request 和代码变更
- 建立代码审查标准
- 通过审查反馈指导开发者
- 审计正确性、安全性或性能

## 不适用场景

- 没有代码变更需要审查
- 任务是纯设计讨论，不涉及代码
- 需要实现修复而非审查

## 指令

- 先阅读上下文、需求和测试信号。
- 审查正确性、安全性、性能和可维护性。
- 提供可操作的反馈，包含严重程度和理由。
- 意图不明确时提出澄清性问题。
- 如需详细检查清单，打开 `resources/implementation-playbook.md`。

## 输出格式

- 发现问题的高层摘要
- 按严重程度分组的问题（阻塞、重要、次要）
- 建议和问题
- 测试和覆盖说明

## 资源

- `resources/implementation-playbook.md` 包含详细的审查模式和模板。

## 限制
- 仅当任务明确符合上述范围时使用此技能。
- 不要将输出替代环境特定的验证、测试或专家审查。
- 如果缺少必要的输入、权限、安全边界或成功标准，请停下来请求澄清。

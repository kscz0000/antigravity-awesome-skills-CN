# WCAG 审计模式

依据 WCAG 2.2 准则审计 Web 内容的全面指南，提供可操作的修复策略。

## 使用时机

- 进行无障碍审计
- 修复 WCAG 违规问题
- 实现可访问的组件
- 应对无障碍相关诉讼
- 满足 ADA / Section 508 要求
- 达成 VPAT 合规

## 不适用场景

- 你需要法律建议或正式认证
- 你只想做快速自动扫描而无需人工验证
- 你无法访问 UI 或源代码进行修复

## 操作步骤

1. 运行自动扫描（axe、Lighthouse、WAVE）收集初步发现。
2. 执行人工检查（键盘导航、焦点顺序、屏幕阅读器流程）。
3. 将每个问题映射到 WCAG 准则、严重程度和修复建议。
4. 修复后重新测试，并记录残余风险与合规状态。

详细模式、检查清单和模板参见 `resources/implementation-playbook.md`。

## 安全提示

- 未经专家评审，不得声称合规。
- 保留测试步骤与结果的证据，以备审计追溯。

## 资源

- `resources/implementation-playbook.md` 包含详细模式、检查清单和模板。

## 限制
- 仅在任务明确匹配上述范围时使用本技能。
- 不得将输出作为环境特定验证、测试或专家评审的替代品。
- 若缺少必需的输入、权限、安全边界或成功标准，应停下并请求澄清。

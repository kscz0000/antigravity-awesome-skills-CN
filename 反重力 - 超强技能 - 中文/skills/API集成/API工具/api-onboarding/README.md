# api-onboarding

通过优化开发者上手旅程的每一个环节，缩短首次 API 调用时间（TTFAC）。

## 本技能涵盖

- **衡量 TTFAC**：埋点与基准
- **认证简化**：消除认证摩擦
- **沙箱环境**：无风险的试验
- **交互式文档**：即时试用体验
- **常见失败点**：开发者在何处以及为何卡住
- **优化框架**：系统化的上手改进

## 何时使用本技能

- 设计新的 API 上手流程
- 诊断高企的开发者流失率
- 缩短新开发者的价值实现时间
- 构建或改进 API 沙箱
- 创建交互式文档

## 核心原则

首次 API 调用时间是预测开发者采纳度最关键的指标。每一分钟的摩擦都会让你损失用户。必须毫不留情地优化。

## TTFAC 基准

| 评级 | 时间 | 体验 |
|--------|------|------------|
| 优秀 | < 5 分钟 | "太棒了" |
| 良好 | 5-15 分钟 | "比较直观" |
| 欠佳 | 30-60 分钟 | "很让人沮丧" |
| 失败 | > 60 分钟 | "我用别家吧" |

## 立竿见影的改进

1. 注册后立即展示 API 密钥（不要深埋在设置里）
2. 为所有代码示例添加复制按钮
3. 用开发者的实际 API 密钥预填示例
4. 为沙箱提供宽松的速率限制
5. 在 API 参考页面添加"试用"按钮

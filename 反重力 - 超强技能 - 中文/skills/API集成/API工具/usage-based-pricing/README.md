# 用量计费

设计开发者能理解、接受并可预测的定价模型。

## 本技能涵盖范围

- **开发者接受的用量指标** - 优劣指标对比、选择框架
- **定价页清晰度** - 必备要素、沟通原则
- **成本可预测性与上限** - 仪表板、告警、上限实现
- **账单透明度** - 详细发票、用量可见性、API 访问
- **价值与成本的沟通** - 替代方案对比、ROI 计算器
- **竞品定价研究** - 格局梳理、定位选项、测试方法

## 核心原则

开发者对定价尤为敏感。他们会计算单位经济性、对比替代方案，并分享关于意外账单的"恐怖故事"。最佳的开发者定价应当具备可预测性、透明度和显而易见的公平性。开发者应当在承诺前就能估算账单。

## 速查参考

**推荐用量指标：**
- API 调用/请求次数
- 计算时长（GB-秒）
- 存储容量（GB）
- 带宽流量（GB）
- 活跃用户数（面向用户的工具）

**应当避免：**
- 自创的"算力单位"
- 复合型指标
- 隐藏的乘数系数
- 惩罚成功的指标

## 适用场景

- 设计新产品定价
- 提升定价页转化率
- 减少计费投诉
- 增加支出上限与告警
- 竞品定价分析
- 为溢价定价提供依据

# 变更日志自动化

遵循行业标准的变更日志生成、发布说明和版本管理自动化模式与工具。

## 使用此技能的场景

- 设置自动化变更日志生成
- 实现约定式提交（Conventional Commits）
- 创建发布说明工作流
- 规范化提交消息格式
- 管理语义化版本

## 不适用场景

- 项目没有发布流程或版本管理
- 只需要一次性的手动发布说明
- 提交历史不可用或不可靠

## 操作指引

- 选择变更日志格式和版本管理策略
- 强制执行提交规范或标签规则
- 配置工具以生成和发布说明
- 审查输出的准确性、完整性和措辞
- 如需详细示例，请打开 `resources/implementation-playbook.md`

## 安全注意事项

- 避免在发布说明中暴露密钥或仅限内部的信息

## 资源

- `resources/implementation-playbook.md` 包含详细的模式、模板和示例

## 限制
- 仅当任务明确符合上述范围时使用此技能
- 输出不能替代特定环境的验证、测试或专家审查
- 如果缺少必要的输入、权限、安全边界或成功标准，请停下来请求澄清

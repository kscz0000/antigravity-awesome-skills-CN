# API 设计检查清单

## 实现前审查

### 资源设计

- [ ] 资源使用名词，而非动词
- [ ] 集合使用复数名称
- [ ] 所有端点命名一致
- [ ] 资源层级清晰（避免超过 2 层嵌套）
- [ ] 所有 CRUD 操作正确映射到 HTTP 方法

### HTTP 方法

- [ ] GET 用于获取（安全、幂等）
- [ ] POST 用于创建
- [ ] PUT 用于完整替换（幂等）
- [ ] PATCH 用于部分更新
- [ ] DELETE 用于删除（幂等）

### 状态码

- [ ] 200 OK 用于成功的 GET/PATCH/PUT
- [ ] 201 Created 用于 POST
- [ ] 204 No Content 用于 DELETE
- [ ] 400 Bad Request 用于格式错误的请求
- [ ] 401 Unauthorized 用于缺少认证
- [ ] 403 Forbidden 用于权限不足
- [ ] 404 Not Found 用于资源不存在
- [ ] 422 Unprocessable Entity 用于验证错误
- [ ] 429 Too Many Requests 用于速率限制
- [ ] 500 Internal Server Error 用于服务器问题

### 分页

- [ ] 所有集合端点都支持分页
- [ ] 定义默认页大小（如 20）
- [ ] 强制最大页大小（如 100）
- [ ] 包含分页元数据（总数、页数等）
- [ ] 选择游标分页或偏移分页模式

### 过滤与排序

- [ ] 使用查询参数进行过滤
- [ ] 支持排序参数
- [ ] 支持全文搜索参数
- [ ] 支持字段选择（稀疏字段集）

### 版本控制

- [ ] 定义版本控制策略（URL/请求头/查询参数）
- [ ] 所有端点都包含版本信息
- [ ] 记录弃用政策

### 错误处理

- [ ] 统一的错误响应格式
- [ ] 详细的错误消息
- [ ] 字段级验证错误
- [ ] 客户端处理的错误代码
- [ ] 错误响应中包含时间戳

### 认证与授权

- [ ] 定义认证方式（Bearer 令牌、API 密钥）
- [ ] 所有端点都有授权检查
- [ ] 正确使用 401 与 403
- [ ] 处理令牌过期

### 速率限制

- [ ] 为每个端点/用户定义速率限制
- [ ] 包含速率限制响应头
- [ ] 超出限制时返回 429 状态码
- [ ] 提供 Retry-After 响应头

### 文档

- [ ] 生成 OpenAPI/Swagger 规范
- [ ] 所有端点都有文档
- [ ] 提供请求/响应示例
- [ ] 记录错误响应
- [ ] 记录认证流程

### 测试

- [ ] 业务逻辑的单元测试
- [ ] 端点的集成测试
- [ ] 错误场景测试
- [ ] 边界情况覆盖
- [ ] 高负载端点的性能测试

### 安全

- [ ] 所有字段都有输入验证
- [ ] SQL 注入防护
- [ ] XSS 防护
- [ ] CORS 正确配置
- [ ] 强制使用 HTTPS
- [ ] URL 中不包含敏感数据
- [ ] 响应中不包含密钥

### 性能

- [ ] 数据库查询已优化
- [ ] 防止 N+1 查询
- [ ] 定义缓存策略
- [ ] 正确设置缓存响应头
- [ ] 大响应已分页

### 监控

- [ ] 实现日志记录
- [ ] 配置错误追踪
- [ ] 收集性能指标
- [ ] 提供健康检查端点
- [ ] 配置错误告警

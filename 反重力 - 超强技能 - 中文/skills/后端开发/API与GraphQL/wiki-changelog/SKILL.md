# Wiki 变更日志

从 git 历史生成结构化的变更日志。

## 何时使用
- 用户询问"最近改了什么"、"生成变更日志"、"汇总提交记录"
- 用户希望了解近期开发活动

## 操作流程

1. 检查 git 日志（提交、日期、作者、消息）
2. 按时间段分组：每日（最近 7 天）、每周（更早）
3. 对每个提交进行分类：新功能 (🆕)、修复 (🐛)、重构 (🔄)、文档 (📝)、配置 (🔧)、依赖 (📦)、破坏性变更 (⚠️)
4. 使用项目术语生成面向用户的简洁描述

## 约束条件

- 聚焦于面向用户的变更
- 将相关提交合并为连贯的描述
- 使用 README 中的项目术语
- 突出显示破坏性变更，并附带迁移说明

## 何时使用
本技能适用于执行概述中描述的工作流或操作。

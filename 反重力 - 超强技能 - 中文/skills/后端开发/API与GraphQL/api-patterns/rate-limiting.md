# 限流原则

> 保护 API 免受滥用和过载。

## 为何限流

```
防护目标：
├── 暴力攻击
├── 资源耗尽
├── 成本超支（如按量付费）
└── 不公平使用
```

## 策略选择

| 类型 | 原理 | 时机 |
|------|------|------|
| **令牌桶** | 允许突发，定期补充 | 大多数 API |
| **滑动窗口** | 平滑分布 | 严格限制 |
| **固定窗口** | 简单计数器 | 基础需求 |

## 响应头

```
应在响应头中包含：
├── X-RateLimit-Limit（最大请求数）
├── X-RateLimit-Remaining（剩余请求数）
├── X-RateLimit-Reset（限制重置时间）
└── 超出限制时返回 429
```

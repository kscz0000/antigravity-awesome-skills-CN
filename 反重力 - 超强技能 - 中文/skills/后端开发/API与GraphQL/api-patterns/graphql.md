# GraphQL 原则

> 为复杂、相互关联的数据提供灵活查询。

## 使用时机

```
✅ 适合场景：
├── 复杂、相互关联的数据
├── 多前端平台
├── 客户端需要灵活查询
├── 不断变化的数据需求
└── 减少过度获取很重要

❌ 不适合场景：
├── 简单 CRUD 操作
├── 文件上传量大
├── HTTP 缓存很重要
└── 团队不熟悉 GraphQL
```

## Schema 设计原则

```
原则：
├── 用图思维，而非端点思维
├── 设计可演进（无版本）
├── 使用连接进行分页
├── 类型要具体（而非通用的"data"）
└── 谨慎处理可空性
```

## 安全考虑

```
防护措施：
├── 查询深度攻击 → 设置最大深度
├── 查询复杂度 → 计算成本
├── 批量滥用 → 限制批量大小
└── 自省 → 生产环境禁用
```

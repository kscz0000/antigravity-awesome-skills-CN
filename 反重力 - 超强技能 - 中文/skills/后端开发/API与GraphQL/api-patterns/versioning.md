# 版本控制策略

> 从第一天就规划 API 演进。

## 决策因素

| 策略 | 实现方式 | 权衡 |
|------|----------|------|
| **URI** | /v1/users | 清晰、易于缓存 |
| **Header** | Accept-Version: 1 | URL 干净、发现较难 |
| **Query** | ?version=1 | 易于添加、混乱 |
| **无** | 谨慎演进 | 内部最佳、公开有风险 |

## 版本控制理念

```
考虑：
├── 公开 API？→ URI 中版本
├── 仅内部？→ 可能不需要版本控制
├── GraphQL？→ 通常无版本（演进 schema）
└── tRPC？→ 类型强制兼容
```

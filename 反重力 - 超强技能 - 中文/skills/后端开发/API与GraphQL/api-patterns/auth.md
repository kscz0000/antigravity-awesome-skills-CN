# 认证模式

> 根据用例选择认证模式。

## 选择指南

| 模式 | 最适合 |
|------|--------|
| **JWT** | 无状态、微服务 |
| **Session** | 传统 Web、简单场景 |
| **OAuth 2.0** | 第三方集成 |
| **API Keys** | 服务器到服务器、公开 API |
| **Passkey** | 现代无密码（2025+） |

## JWT 原则

```
重要原则：
├── 始终验证签名
├── 检查过期时间
├── 包含最少的 claims
├── 使用短过期时间 + 刷新令牌
└── 切勿在 JWT 中存储敏感数据
```

# 响应格式原则

> 一致性是关键 —— 选择一种格式并坚持使用。

## 常见模式

```
选择一种：
├── 封装模式（{ success, data, error }）
├── 直接数据（仅返回资源）
└── HAL/JSON:API（超媒体）
```

## 错误响应

```
应包含：
├── 错误代码（用于程序化处理）
├── 用户消息（用于显示）
├── 详细信息（用于调试、字段级错误）
├── 请求 ID（用于支持）
└── 不包含内部详情（安全！）
```

## 分页类型

| 类型 | 最适合 | 权衡 |
|------|--------|------|
| **Offset** | 简单、可跳转 | 大数据集性能 |
| **Cursor** | 大数据集 | 无法跳转到特定页 |
| **Keyset** | 性能关键 | 需要可排序键 |

### 选择问题

1. 数据集有多大？
2. 用户需要跳转到特定页面吗？
3. 数据是否经常变化？

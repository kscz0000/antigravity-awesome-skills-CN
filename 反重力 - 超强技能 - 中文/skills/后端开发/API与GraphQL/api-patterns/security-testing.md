# API 安全测试

> API 安全测试原则。OWASP API Top 10、认证、授权测试。

---

## OWASP API 安全 Top 10

| 漏洞 | 测试重点 |
|------|----------|
| **API1: BOLA** | 访问其他用户的资源 |
| **API2: 认证缺陷** | JWT、会话、凭证 |
| **API3: 属性授权** | 批量赋值、数据暴露 |
| **API4: 资源消耗** | 限流、DoS |
| **API5: 功能授权** | 管理端点、角色绕过 |
| **API6: 业务逻辑** | 逻辑滥用、自动化 |
| **API7: SSRF** | 内部网络访问 |
| **API8: 配置错误** | 调试端点、CORS |
| **API9: 资产清单** | 隐藏 API、旧版本 |
| **API10: 不安全消费** | 第三方 API 信任 |

## 认证测试

### JWT 测试

| 检查项 | 测试内容 |
|--------|----------|
| 算法 | None、算法混淆 |
| 密钥 | 弱密钥、暴力破解 |
| Claims | 过期时间、签发者、受众 |
| 签名 | 篡改、密钥注入 |

### 会话测试

| 检查项 | 测试内容 |
|--------|----------|
| 生成 | 可预测性 |
| 存储 | 客户端安全性 |
| 过期 | 超时执行 |
| 失效 | 登出有效性 |

## 授权测试

| 测试类型 | 方法 |
|----------|------|
| **水平** | 访问同级用户数据 |
| **垂直** | 访问更高权限功能 |
| **上下文** | 在允许范围外访问 |

### BOLA/IDOR 测试

1. 识别请求中的资源 ID
2. 使用用户 A 的会话捕获请求
3. 使用用户 B 的会话重放
4. 检查未授权访问

## 输入验证测试

| 注入类型 | 测试重点 |
|----------|----------|
| SQL | 查询操作 |
| NoSQL | 文档查询 |
| Command | 系统命令 |
| LDAP | 目录查询 |

**方法：** 测试所有参数，尝试类型强制，测试边界，检查错误消息。

---

## 限流测试

| 方面 | 检查项 |
|------|--------|
| 存在性 | 有限制吗？ |
| 绕过 | 头信息、IP 轮换 |
| 范围 | 按用户、按 IP、全局 |

**绕过技术：** X-Forwarded-For、不同的 HTTP 方法、大小写变化、API 版本控制。

# 上下文发现

> 在建议任何架构之前，先收集上下文。

## 问题层次（优先询问用户）

1. **规模**
   - 多少用户？（10、1K、100K、1M+）
   - 数据量？（MB、GB、TB）
   - 事务速率？（每秒/每分钟）

2. **团队**
   - 独立开发者还是团队？
   - 团队规模和技能？
   - 分布式还是同地办公？

3. **时间线**
   - MVP/原型还是长期产品？
   - 上市时间压力？

4. **领域**
   - 重CRUD还是业务逻辑复杂？
   - 实时性要求？
   - 合规/监管要求？

5. **约束**
   - 预算限制？
   - 需要集成的遗留系统？
   - 技术栈偏好？

## 项目分类矩阵

```
                    MVP              SaaS           企业级
┌─────────────────────────────────────────────────────────────┐
│ 规模         │ <1K           │ 1K-100K      │ 100K+        │
│ 团队         │ 独立          │ 2-10         │ 10+          │
│ 时间线       │ 快（周）      │ 中（月）     │ 长（年）     │
│ 架构         │ 简单          │ 模块化       │ 分布式       │
│ 模式         │ 最少          │ 选择性       │ 全面         │
│ 示例         │ Next.js API   │ NestJS       │ 微服务       │
└─────────────────────────────────────────────────────────────┘
```

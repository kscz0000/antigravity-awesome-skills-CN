# 移动游戏开发

> 平台约束和优化原则。

---

## 1. 平台考虑

### 关键约束

| 约束 | 策略 |
|------------|----------|
| **触摸输入** | 大触摸区域、手势 |
| **电池** | 限制CPU/GPU使用 |
| **发热** | 过热时节流 |
| **屏幕尺寸** | 响应式UI |
| **中断** | 后台时暂停 |

---

## 2. 触摸输入原则

### 触摸 vs 控制器

| 触摸 | 桌面/主机 |
|-------|-----------------|
| 不精确 | 精确 |
| 遮挡屏幕 | 无遮挡 |
| 有限按钮 | 多按钮 |
| 可用手势 | 按钮/摇杆 |

### 最佳实践

- 最小触摸目标：44x44点
- 触摸时视觉反馈
- 避免精确计时要求
- 同时支持竖屏和横屏

---

## 3. 性能目标

### 温度管理

| 动作 | 触发条件 |
|--------|---------|
| 降低质量 | 设备温暖 |
| 限制FPS | 设备发热 |
| 暂停特效 | 临界温度 |

### 电池优化

- 30 FPS通常足够
- 暂停时休眠
- 最小化GPS/网络
- 深色模式节省OLED电池

## 4. 应用商店要求

### iOS（App Store）

| 要求 | 备注 |
|-------------|------|
| 隐私标签 | 必须 |
| 账户删除 | 如有账户创建则必须 |
| 截图 | 所有设备尺寸 |

### Android（Google Play）

| 要求 | 备注 |
|-------------|------|
| 目标API | 当年SDK |
| 64位 | 必须 |
| App Bundle | 推荐 |

---

## 5. 变现模式

| 模式 | 最适合 |
|-------|----------|
| **付费** | 高质量游戏、忠实受众 |
| **免费+内购** | 休闲、基于进度 |
| **广告** | 超休闲、高量 |
| **订阅** | 内容更新、多人游戏 |

---

## 6. 反模式

| ❌ 不要 | ✅ 要 |
|----------|-------|
| 移动端使用桌面控制 | 为触摸设计 |
| 忽略电池消耗 | 监控温度 |
| 强制横屏 | 支持玩家偏好 |
| 始终在线网络 | 缓存并同步 |

---

> **记住：** 移动端是约束最多的平台。尊重电池和注意力。

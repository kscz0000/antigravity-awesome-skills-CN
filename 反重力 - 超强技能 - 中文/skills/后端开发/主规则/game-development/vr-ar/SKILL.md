## 2. 舒适度原则

### 晕动症预防

| 原因 | 解决方案 |
|-------|----------|
| **移动** | 传送、转向 |
| **低FPS** | 保持90 FPS |
| **相机震动** | 避免或最小化 |
| **快速加速** | 渐进移动 |

### 舒适度设置

- 移动时渐晕
- 转向 vs 平滑转向
- 坐姿 vs 站立模式
- 高度校准

---

## 3. 性能要求

### 目标指标

| 平台 | FPS | 分辨率 |
|----------|-----|------------|
| Quest 2 | 72-90 | 1832x1920 |
| Quest 3 | 90-120 | 2064x2208 |
| PCVR | 90 | 2160x2160+ |
| PSVR2 | 90-120 | 2000x2040 |

### 帧预算

- VR需要一致的帧时间
- 单帧掉落 = 可见抖动
- 90 FPS = 11.11ms预算

## 4. 交互原则

### 控制器交互

| 类型 | 用途 |
|------|-----|
| **指向+点击** | UI、远处物体 |
| **抓取** | 操作 |
| **手势** | 魔法、特殊动作 |
| **物理** | 投掷、挥动 |

### 手部追踪

- 更沉浸但不精确
- 适合：社交、休闲
- 挑战：动作、精确

---

## 5. 空间设计

### 世界比例

- 1单位 = 1米（关键）
- 物体必须感觉正确大小
- 用真实测量测试

### 深度线索

| 线索 | 重要性 |
|-----|------------|
| 立体 | 主要深度 |
| 运动视差 | 次要 |
| 阴影 | 接地 |
| 遮挡 | 分层 |

---

## 6. 反模式

| ❌ 不要 | ✅ 要 |
|----------|-------|
| 不经玩家移动相机 | 玩家控制相机 |
| 低于90 FPS | 保持帧率 |
| 使用微小UI文字 | 大、可读文字 |
| 忽略臂长 | 按玩家触及范围缩放 |

---

> **记住：** 舒适度不是可选项。恶心的玩家不会玩。

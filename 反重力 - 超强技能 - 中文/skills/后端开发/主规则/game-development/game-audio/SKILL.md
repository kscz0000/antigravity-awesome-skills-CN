# 游戏音频原则

> 沉浸式游戏体验的音效设计和音乐集成。

---

## 1. 音频分类系统

### 类别定义

| 类别 | 行为 | 示例 |
|----------|----------|----------|
| **音乐** | 循环、交叉淡化、闪避 | BGM、战斗音乐 |
| **音效** | 单次播放、3D定位 | 脚步声、冲击声 |
| **环境** | 循环、背景层 | 风、人群、森林 |
| **UI** | 即时、非3D | 按钮点击、通知 |
| **语音** | 优先级、闪避触发 | 对话、播报员 |

### 优先级层次

```
当声音竞争声道时：

1. 语音（最高 - 始终可听）
2. 玩家音效（反馈关键）
3. 敌人音效（游戏玩法重要）
4. 音乐（氛围，但可闪避）
5. 环境（最低 - 可丢弃）
```

## 2. 音效设计决策

### 音效创作方法

| 方法 | 使用场景 | 权衡 |
|----------|-------------|------------|
| **录音** | 写实需求 | 高质量、耗时 |
| **合成** | 科幻、复古、UI | 独特、需要技能 |
| **素材库** | 快速制作 | 常见声音、授权 |
| **分层** | 复杂声音 | 最佳效果、更多工作 |

### 分层结构

| 层 | 用途 | 示例：枪声 |
|-------|---------|------------------|
| **起音** | 初始瞬态 | 点击声、爆裂声 |
| **主体** | 主要特征 | 轰鸣声、爆炸声 |
| **尾音** | 衰减、空间 | 混响、回声 |
| **甜味剂** | 特殊调味 | 弹壳、机械声 |

## 3. 音乐集成

### 音乐状态系统

```
游戏状态 → 音乐响应
│
├── 菜单 → 平静、可循环主题
├── 探索 → 氛围、环境
├── 发现战斗 → 过渡到紧张
├── 进入战斗 → 完整战斗音乐
├── 胜利 → 过门 + 平静过渡
├── 失败 → 忧郁过门
└── Boss → 独特、多阶段曲目
```

### 过渡技术

| 技术 | 使用场景 | 感觉 |
|-----------|----------|------|
| **交叉淡化** | 平滑情绪转变 | 渐进 |
| **过门** | 即时事件 | 戏剧性 |
| **分轨混音** | 动态强度 | 无缝 |
| **节拍同步** | 节奏游戏玩法 | 音乐性 |
| **队列点** | 下一个自然断点 | 干净 |

## 4. 自适应音频决策

### 强度参数

| 参数 | 影响 | 示例 |
|-----------|---------|---------|
| **威胁等级** | 音乐强度 | 敌人数量 |
| **生命值** | 滤波、混响 | 低血量 = 闷音 |
| **速度** | 节奏、能量 | 赛车速度 |
| **环境** | 混响、EQ | 洞穴 vs 户外 |
| **时间** | 氛围、音量 | 夜晚 = 更安静 |

### 垂直 vs 水平

| 系统 | 变化内容 | 最适合 |
|--------|--------------|----------|
| **垂直（层）** | 添加/移除乐器层 | 强度缩放 |
| **水平（段落）** | 不同音乐段落 | 状态变化 |
| **组合** | 两者 | AAA自适应配乐 |

## 5. 3D音频决策

### 空间化

| 元素 | 3D定位？ | 原因 |
|---------|----------------|--------|
| 玩家脚步声 | 否（或细微） | 始终可听 |
| 敌人脚步声 | 是 | 方向感知 |
| 枪声 | 是 | 战斗感知 |
| 音乐 | 否 | 氛围、非叙事 |
| 环境区域 | 是（区域） | 环境 |
| UI声音 | 否 | 界面反馈 |

### 距离行为

| 距离 | 声音行为 |
|----------|----------------|
| **近** | 全音量、全频 |
| **中** | 音量衰减、高频衰减 |
| **远** | 低音量、低通滤波 |
| **最大** | 静音或环境提示 |

## 6. 平台考虑

### 格式选择

| 平台 | 推荐格式 | 原因 |
|----------|-------------------|--------|
| PC | OGG Vorbis、WAV | 质量、无授权 |
| 主机 | 平台特定 | 认证 |
| 移动端 | MP3、AAC | 大小、兼容性 |
| Web | WebM/Opus、MP3回退 | 浏览器支持 |

### 内存预算

| 游戏类型 | 音频预算 | 策略 |
|-----------|--------------|----------|
| 移动休闲 | 10-50 MB | 压缩、较少变体 |
| PC独立 | 100-500 MB | 质量优先 |
| AAA | 1+ GB | 完整质量、多变体 |

## 7. 混音层次

### 音量平衡参考

| 类别 | 相对电平 | 备注 |
|----------|----------------|-------|
| **语音** | 0 dB（参考） | 始终清晰 |
| **玩家音效** | -3到-6 dB | 突出但不刺耳 |
| **音乐** | -6到-12 dB | 基础、为语音闪避 |
| **敌人音效** | -6到-9 dB | 重要但不主导 |
| **环境** | -12到-18 dB | 细微背景 |

### 闪避规则

| 时机 | 闪避什么 | 程度 |
|------|-----------|--------|
| 语音播放 | 音乐、环境 | -6到-9 dB |
| 爆炸 | 除爆炸外所有 | 短暂闪避 |
| 打开菜单 | 游戏玩法音频 | -3到-6 dB |

---

## 8. 反模式

| 不要 | 要 |
|-------|-----|
| 重复播放相同声音 | 使用变体（每个声音3-5个） |
| 所有内容最大音量 | 使用正确的混音层次 |
| 忽略静默 | 静默创造对比 |
| 一首音乐永远循环 | 提供多样性、过渡 |
| 原型中跳过音频 | 占位音频很重要 |

---

> **记住：** 游戏体验的50%是音频。静音游戏失去一半灵魂。

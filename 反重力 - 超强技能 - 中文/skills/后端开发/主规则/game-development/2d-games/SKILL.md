# 2D游戏开发

> 2D游戏系统原则。

---

## 1. 精灵系统

### 精灵组织

| 组件 | 用途 |
|-----------|---------|
| **图集** | 合并纹理，减少绘制调用 |
| **动画** | 帧序列 |
| **轴心点** | 旋转/缩放原点 |
| **分层** | Z序控制 |

### 动画原则

- 帧率：通常8-24 FPS
- 挤压和拉伸表现冲击力
- 动作前的预备
- 动作后的跟随

---

## 2. 瓦片地图设计

### 瓦片考虑因素

| 因素 | 建议 |
|--------|----------------|
| **尺寸** | 16x16、32x32、64x64 |
| **自动瓦片** | 用于地形 |
| **碰撞** | 简化形状 |

### 图层

| 图层 | 内容 |
|-------|---------|
| 背景 | 非交互式场景 |
| 地形 | 可行走的地面 |
| 道具 | 可交互物体 |
| 前景 | 视差覆盖层 |

---

## 3. 2D物理

### 碰撞形状

| 形状 | 使用场景 |
|-------|----------|
| 盒子 | 矩形物体 |
| 圆形 | 球体、圆形 |
| 胶囊 | 角色 |
| 多边形 | 复杂形状 |

### 物理考虑

- 像素完美 vs 基于物理
- 固定时间步长保证一致性
- 图层用于过滤

---

## 4. 相机系统

### 相机类型

| 类型 | 用途 |
|------|-----|
| **跟随** | 追踪玩家 |
| **前瞻** | 预判移动 |
| **多目标** | 双人游戏 |
| **房间式** | 银河恶魔城 |

### 屏幕震动

- 短持续时间（50-200ms）
- 递减强度
- 谨慎使用

---

## 5. 类型模式

### 平台游戏

- 郊狼时间（边缘后的宽容）
- 跳跃缓冲
- 可变跳跃高度

### 俯视角

- 8方向或自由移动
- 基于瞄准或自动瞄准
- 考虑是否旋转

---

## 6. 反模式

| ❌ 不要 | ✅ 要 |
|----------|-------|
| 分离纹理 | 使用图集 |
| 复杂碰撞形状 | 简化碰撞 |
| 抖动的相机 | 平滑跟随 |
| 物理上像素完美 | 选择一种方法 |

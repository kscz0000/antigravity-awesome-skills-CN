# 游戏美术原则

> 游戏视觉设计思维——风格选择、资产管线和美术指导。

---

## 1. 美术风格选择

### 决策树

```
游戏应该唤起什么感觉？
│
├── 怀旧 / 复古
│   ├── 有限调色板？ → 像素艺术
│   └── 手绘感？ → 矢量 / Flash风格
│
├── 写实 / 沉浸
│   ├── 高预算？ → PBR 3D
│   └── 风格化写实？ → 手绘纹理
│
├── 亲切 / 休闲
│   ├── 干净形状？ → 扁平 / 极简
│   └── 柔和感？ → 渐变 / 柔和阴影
│
└── 独特 / 实验
    └── 定义自定义风格指南
```

### 风格对比矩阵

| 风格 | 制作速度 | 技能门槛 | 可扩展性 | 最适合 |
|-------|------------------|-------------|-------------|----------|
| **像素艺术** | 中等 | 中等 | 难招聘 | 独立、复古 |
| **矢量/扁平** | 快 | 低 | 容易 | 移动、休闲 |
| **手绘** | 慢 | 高 | 中等 | 奇幻、风格化 |
| **PBR 3D** | 慢 | 高 | AAA管线 | 写实游戏 |
| **低多边形** | 快 | 中等 | 容易 | 独立3D |
| **卡通渲染** | 中等 | 中等 | 中等 | 动漫、卡通 |

## 3. 色彩理论决策

### 调色板选择

| 目标 | 策略 | 示例 |
|------|----------|---------|
| **和谐** | 互补或类似 | 自然游戏 |
| **对比** | 高饱和度差异 | 动作游戏 |
| **氛围** | 暖/冷温度 | 恐怖、温馨 |
| **可读性** | 明度对比优于色相 | 游戏玩法清晰度 |

### 色彩原则

- **层次：** 重要元素应该突出
- **一致性：** 相同物体 = 相同色彩家族
- **语境：** 颜色在不同背景上呈现不同
- **无障碍：** 不要仅依赖颜色

## 4. 动画原则

### 12原则（应用于游戏）

| 原则 | 游戏应用 |
|-----------|------------------|
| **挤压与拉伸** | 跳跃弧线、冲击 |
| **预备** | 攻击前的蓄力 |
| **演出** | 清晰剪影 |
| **跟随** | 移动后的头发、披风 |
| **慢入慢出** | 过渡的缓动 |
| **弧线** | 自然运动路径 |
| **次要动作** | 呼吸、眨眼 |
| **时间** | 帧数 = 重量/速度 |
| **夸张** | 远距离可读 |
| **吸引力** | 令人难忘的设计 |

### 帧数指南

| 动作类型 | 典型帧数 | 感觉 |
|-------------|----------------|------|
| 待机呼吸 | 4-8 | 细微 |
| 行走循环 | 6-12 | 平滑 |
| 跑步循环 | 4-8 | 有力 |
| 攻击 | 3-6 | 干脆 |
| 死亡 | 8-16 | 戏剧性 |

## 5. 分辨率与比例决策

### 各平台2D分辨率

| 平台 | 基础分辨率 | 精灵比例 |
|----------|-----------------|--------------|
| 移动端 | 1080p | 64-128px角色 |
| 桌面 | 1080p-4K | 128-256px角色 |
| 像素艺术 | 320x180到640x360 | 16-32px角色 |

### 一致性规则

选择一个基础单位并坚持：
- 像素艺术：以1x工作，放大（绝不缩小）
- HD美术：定义DPI，保持比例
- 3D：1单位 = 1米（行业标准）

## 7. 反模式

| 不要 | 要 |
|-------|-----|
| 随机混合美术风格 | 定义并遵循风格指南 |
| 仅以最终分辨率工作 | 以源分辨率创作 |
| 忽略剪影可读性 | 在游戏玩法距离测试 |
| 过度细节背景 | 聚焦玩家区域细节 |
| 跳过颜色测试 | 在目标显示器上测试 |

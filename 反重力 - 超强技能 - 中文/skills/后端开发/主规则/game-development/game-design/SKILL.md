# 游戏设计原则

> 引人入胜游戏的设计思维。

---

## 1. 核心循环设计

### 30秒测试

```
每个游戏都需要一个有趣的30秒循环：
1. 动作 → 玩家做某事
2. 反馈 → 游戏响应
3. 奖励 → 玩家感觉良好
4. 重复
```

### 循环示例

| 类型 | 核心循环 |
|-------|-----------|
| 平台游戏 | 跑 → 跳 → 落地 → 收集 |
| 射击游戏 | 瞄准 → 射击 → 击杀 → 拾取 |
| 益智游戏 | 观察 → 思考 → 解谜 → 前进 |
| RPG | 探索 → 战斗 → 升级 → 装备 |

---

## 2. 游戏设计文档（GDD）

### 必要章节

| 章节 | 内容 |
|---------|---------|
| **推介** | 一句话描述 |
| **核心循环** | 30秒游戏玩法 |
| **机制** | 系统如何运作 |
| **进度** | 玩家如何前进 |
| **美术风格** | 视觉方向 |
| **音频** | 声音方向 |

### 原则

- 保持活跃（定期更新）
- 视觉有助于沟通
- 少即是多（从小开始）

## 3. 玩家心理

### 动机类型

| 类型 | 驱动力 |
|------|-----------|
| **成就者** | 目标、完成 |
| **探索者** | 发现、秘密 |
| **社交者** | 互动、社区 |
| **杀手** | 竞争、支配 |

### 奖励时间表

| 时间表 | 效果 | 用途 |
|----------|--------|-----|
| **固定** | 可预测 | 里程碑奖励 |
| **可变** | 成瘾性 | 战利品掉落 |
| **比率** | 基于努力 | 刷刷游戏 |

---

## 4. 难度平衡

### 心流状态

```
太难 → 挫败 → 退出
太简单 → 无聊 → 退出
恰到好处 → 心流 → 投入
```

### 平衡策略

| 策略 | 方式 |
|----------|-----|
| **动态** | 根据玩家技能调整 |
| **选择** | 让玩家选择 |
| **无障碍** | 为所有人提供选项 |

---

## 5. 进度设计

### 进度类型

| 类型 | 示例 |
|------|---------|
| **技能** | 玩家变得更好 |
| **力量** | 角色变得更强 |
| **内容** | 新区域解锁 |
| **故事** | 叙事推进 |

### 节奏原则

- 早期胜利（快速吸引）
- 逐渐增加挑战
- 强度之间的休息节拍
- 有意义的选择

---

## 6. 反模式

| ❌ 不要 | ✅ 要 |
|----------|-------|
| 孤立设计 | 持续测试 |
| 有趣之前打磨 | 先原型 |
| 强迫一种玩法 | 允许玩家表达 |
| 过度惩罚 | 奖励进步 |

---

> **记住：** 乐趣通过迭代发现，而非纸上设计。

# 智能体协调

> App Builder 如何编排专业智能体。

## 智能体流水线

```
┌─────────────────────────────────────────────────────────────┐
│                   APP BUILDER (编排器)                       │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                     项目规划器                               │
│  • 任务分解                                                  │
│  • 依赖图                                                    │
│  • 文件结构规划                                              │
│  • 在项目根目录创建 {task-slug}.md（必须）                    │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│              检查点: 计划验证                                │
│  🔴 验证: {task-slug}.md 是否存在于项目根目录？               │
│  🔴 如果否 → 停止 → 先创建计划文件                           │
│  🔴 如果是 → 继续执行专业智能体                              │
└─────────────────────────────────────────────────────────────┘
                              │
          ┌───────────────────┼───────────────────┐
          ▼                   ▼                   ▼
┌─────────────────┐ ┌─────────────────┐ ┌─────────────────┐
│ 数据库          │ │ 后端            │ │ 前端            │
│ 架构师          │ │ 专家            │ │ 专家            │
│                 │ │                 │ │                 │
│ • 模式设计      │ │ • API 路由      │ │ • 组件          │
│ • 迁移          │ │ • 控制器        │ │ • 页面          │
│ • 种子数据      │ │ • 中间件        │ │ • 样式          │
└─────────────────┘ └─────────────────┘ └─────────────────┘
          │                   │                   │
          └───────────────────┼───────────────────┘
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                 并行阶段（可选）                              │
│  • 安全审计员 → 漏洞检查                                    │
│  • 测试工程师 → 单元测试                                    │
│  • 性能优化师 → 打包分析                                    │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                     DevOps 工程师                            │
│  • 环境设置                                                  │
│  • 预览部署                                                  │
│  • 健康检查                                                  │
└─────────────────────────────────────────────────────────────┘
```

## 执行顺序

| 阶段 | 智能体 | 并行? | 前置条件 | 检查点 |
|-------|----------|-----------|--------------|------------|
| 0 | 苏格拉底门控 | ❌ | - | ✅ 提问 3 个问题 |
| 1 | 项目规划器 | ❌ | 问题已回答 | ✅ **PLAN.md 已创建** |
| 1.5 | **计划验证** | ❌ | PLAN.md 存在 | ✅ **文件存在于根目录** |
| 2 | 数据库架构师 | ❌ | 计划就绪 | 模式已定义 |
| 3 | 后端专家 | ❌ | 模式就绪 | API 路由已创建 |
| 4 | 前端专家 | ✅ | API 就绪（部分） | UI 组件就绪 |
| 5 | 安全审计员、测试工程师 | ✅ | 代码就绪 | 测试和审计通过 |
| 6 | DevOps 工程师 | ❌ | 所有代码就绪 | 部署就绪 |

> 🔴 **关键:** 阶段 1.5 是必须的。没有 PLAN.md 验证，专业智能体不能继续执行。

# 命名与编码标准

## 通用标准

- **显式命名**：优先考虑清晰性而非巧妙性。
- **Async 后缀**：方法名中**不**使用 `Async` 后缀，即使它们返回 `Task`。
- **私有字段**：私有字段**不**使用 `_` 前缀。
- **静态状态**：除非有明确理由和文档说明，否则避免使用静态状态。
- **方法设计**：保持方法小巧、富有表现力，且圈复杂度低。

## 错误处理

- **Result 与 Maybe**：使用 **CSharpFunctionalExtensions** 中的类型进行流程控制和错误处理。
- **异常**：严格保留用于真正异常、不可恢复的情况。
- **边界**：绝不允许异常跨越架构边界泄漏。

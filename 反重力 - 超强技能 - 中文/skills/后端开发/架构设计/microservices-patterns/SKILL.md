# 微服务模式

掌握微服务架构模式，涵盖服务边界、服务间通信、数据管理和弹性模式，用于构建分布式系统。

## 适用场景

- 将单体应用拆分为微服务
- 设计服务边界与契约
- 实现服务间通信
- 管理分布式数据与事务
- 构建弹性分布式系统
- 实现服务发现与负载均衡
- 设计事件驱动架构

## 不适用场景

- 系统规模适合模块化单体
- 需要快速原型且无需分布式复杂性
- 缺乏分布式系统运维支撑

## 使用指引

1. 识别各服务的领域边界与所有权。
2. 定义契约、数据归属和通信模式。
3. 规划弹性、可观测性和部署策略。
4. 提供迁移步骤与运维护栏。

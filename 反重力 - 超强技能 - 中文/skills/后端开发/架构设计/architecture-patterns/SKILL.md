# 架构模式

掌握成熟的后端架构模式，包括整洁架构、六边形架构和领域驱动设计，构建可维护、可测试、可扩展的系统。

## 使用此技能的场景

- 从零设计新的后端系统
- 重构单体应用以提升可维护性
- 为团队制定架构标准
- 从紧耦合架构迁移到松耦合架构
- 实施领域驱动设计原则
- 构建可测试、可模拟的代码库
- 规划微服务拆分

## 不适用场景

- 仅需要小范围、局部的重构
- 系统主要是前端，无需后端架构变更
- 需要实现细节而非架构设计

## 指导步骤

1. 明确领域边界、约束条件和可扩展性目标。
2. 选择适合领域复杂度的架构模式。
3. 定义模块边界、接口和依赖规则。
4. 提供迁移步骤和验证检查。
5. 对于必须能够从故障中恢复的工作流（支付、订单履约、多步骤流程），在基础设施层使用持久化执行——如 DBOS 等框架会持久化工作流状态，提供崩溃恢复能力，而无需增加架构复杂性。

详细的模式、检查清单和模板请参考 `resources/implementation-playbook.md`。

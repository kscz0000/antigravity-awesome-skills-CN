# AgentFolio

**角色**：自主智能体发现指南

当你想要**发现、比较和研究跨生态系统的自主 AI 智能体**时使用此技能。
AgentFolio 是一个位于 https://agentfolio.io 的精选目录，追踪智能体框架、产品和工具。

此技能帮助你：

- 在从头构建之前找到现有的智能体。
- 绘制智能体框架和托管产品的全景图。
- 收集智能体能力的具体示例和基准。

## 能力

- 按用例发现自主 AI 智能体、框架和工具。
- 按能力、目标用户和集成界面比较智能体。
- 识别市场空白或为新技能/工作流寻找灵感。
- 为你自己的设计收集智能体行为和 UX 模式示例。
- 追踪智能体架构和部署的新兴趋势。

## 如何使用 AgentFolio

1. **打开目录**
   - 在浏览器中访问 `https://agentfolio.io`。
   - 可选择按类别筛选（如 Dev Tools、Ops、Marketing、Productivity）。

2. **按意图搜索**
   - 从你想解决的问题开始：
     - "customer support agents"
     - "autonomous coding agents"
     - "research / analysis agents"
   - 在 AgentFolio 搜索栏中使用匹配你领域或工作流的关键词。

3. **评估候选者**
   - 对于每个感兴趣的智能体，记录：
     - **核心承诺**（它自动化什么结果）。
     - **输入/输出形态**（API、UI、数据源）。
     - **自主模式**（一次性、多步骤、工具调用、人在回路）。
     - **部署模式**（SaaS、自托管、浏览器、IDE 等）。

4. **综合洞察**
   - 使用发现来：
     - 决定是集成现有智能体还是自己构建。
     - 借鉴成功的 UX 和安全模式。
     - 相对于生态系统定位你自己的智能体技能和工作流。

## 示例工作流

### 1) 构建新智能体前的全景扫描

- 定义问题："CI 流水线的自主测试失败分流"。
- 使用 AgentFolio 搜索：
  - "testing agent"、"CI agent"、"DevOps assistant"、"incident triage"。
- 对于每个相关智能体：
  - 记录支持的平台（GitHub、GitLab、Jenkins 等）。
  - 捕捉它们如何解释自主性和安全边界。
  - 如果你计划采用而非构建，记录定价/许可约束。

### 2) 新技能的竞争和灵感研究

- 如果你计划添加新技能（如可观测性智能体、安全智能体）：
  - 使用 AgentFolio 查找类似的智能体和功能。
  - 提取 3-5 个你想要效仿或避免的具体模式。
  - 将这些模式转化为你自己技能的明确需求。

### 3) 供应商候选名单

- 在多个智能体供应商之间选择时：
  - 使用 AgentFolio 条目作为中立目录。
  - 构建对比表（列：能力、集成、定价、信任与安全）。
  - 使用该表格推动更正式的评估或概念验证。

## 示例提示词

在 AI 编码智能体中使用此技能时，使用以下提示词：

- "Use AgentFolio to find 3 autonomous AI agents focused on code review. For each, summarize the core value prop, supported languages, and how they integrate into developer workflows."
- "Scan AgentFolio for agents that help with customer support triage. List the top options, their target customer size (SMB vs. enterprise), and any notable UX patterns."
- "Before we build our own research assistant, use AgentFolio to map existing research / analysis agents and highlight gaps we could fill."

## 何时使用

当你需要**发现或比较自主 AI 智能体**而非闭门造车时，此技能适用：

- 在新智能体或工作流项目开始时。
- 在评估要集成的供应商或工具时。
- 当你想要从现有智能体产品中获取灵感或最佳实践时。

## 限制

- 仅当任务明确匹配上述范围时使用此技能。
- 不要将输出视为环境特定验证、测试或专家评审的替代品。
- 如果缺少必需的输入、权限、安全边界或成功标准，停下来请求澄清。

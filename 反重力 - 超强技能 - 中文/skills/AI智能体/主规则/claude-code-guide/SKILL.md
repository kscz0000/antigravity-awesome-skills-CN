# Claude Code 指南

## 目的

为 Claude Code（智能体编码工具）的配置和使用提供全面参考，充分发挥其潜力。本技能综合最佳实践、配置模板和高级使用模式。

## 配置（`CLAUDE.md`）

启动新项目时，在根目录创建 `CLAUDE.md` 文件来指导智能体。

### 模板（通用）

```markdown
# 项目指南

## 命令

- 运行应用：`npm run dev`
- 测试：`npm test`
- 构建：`npm run build`

## 代码风格

- 所有新代码使用 TypeScript。
- React 使用函数组件和 Hooks。
- 样式使用 Tailwind CSS。
- 错误处理使用提前返回。

## 工作流程

- 先阅读 `README.md` 了解项目上下文。
- 编辑前先读取文件内容。
- 编辑后运行测试验证。
```

## 高级功能

### 思考关键词

在提示词中使用这些关键词来触发智能体的深度推理：

- "逐步思考"
- "分析根本原因"
- "执行前先规划"
- "验证你的假设"

### 调试

如果智能体卡住或行为异常：

1. **清除上下文**：开始新会话，或要求智能体"忘记之前的指令"（如果混乱）。
2. **明确指令**：对路径、文件名和预期结果要极其具体。
3. **日志**：要求智能体"检查日志"或"以详细输出模式运行命令"。

## 最佳实践

1. **小上下文**：不要把整个代码库塞进上下文。先用 `grep` 或 `find` 定位相关文件。
2. **迭代开发**：要求小改动，验证后再继续。
3. **反馈循环**：如果智能体犯错，立即纠正并要求它"记录教训"到记忆中（如果支持）或 `CLAUDE.md`。

# 硬件选择指南

选择合适的硬件（flavor）对于成本效益的训练至关重要。

## 可用硬件

### CPU
- `cpu-basic` - 基础 CPU，仅用于测试
- `cpu-upgrade` - 增强型 CPU

**用途：**数据集验证、预处理、测试脚本
**不推荐用于训练：**对任何有意义的训练都太慢

### GPU 选项

| Flavor | GPU | 显存 | 用途 | 费用/小时 |
|--------|-----|--------|----------|-----------|
| `t4-small` | NVIDIA T4 | 16GB | <1B 模型，演示 | ~$0.50-1 |
| `t4-medium` | NVIDIA T4 | 16GB | 1-3B 模型，开发 | ~$1-2 |
| `l4x1` | NVIDIA L4 | 24GB | 3-7B 模型，高效训练 | ~$2-3 |
| `l4x4` | 4x NVIDIA L4 | 96GB | 多 GPU 训练 | ~$8-12 |
| `a10g-small` | NVIDIA A10G | 24GB | 3-7B 模型，生产 | ~$3-4 |
| `a10g-large` | NVIDIA A10G | 24GB | 7-13B 模型 | ~$4-6 |
| `a10g-largex2` | 2x NVIDIA A10G | 48GB | 多 GPU，大模型 | ~$8-12 |
| `a10g-largex4` | 4x NVIDIA A10G | 96GB | 多 GPU，超大模型 | ~$16-24 |
| `a100-large` | NVIDIA A100 | 40GB | 13B+ 模型，快速训练 | ~$8-12 |

### TPU 选项

| Flavor | 类型 | 用途 |
|--------|------|----------|
| `v5e-1x1` | TPU v5e | 小型 TPU 工作负载 |
| `v5e-2x2` | 4x TPU v5e | 中型 TPU 工作负载 |
| `v5e-2x4` | 8x TPU v5e | 大型 TPU 工作负载 |

**注意：**TPU 需要针对 TPU 优化的代码。大多数 TRL 训练使用 GPU。

## 选择指南

### 按模型大小

**微型模型（<1B 参数）**
- **推荐：** `t4-small`
- **示例：** Qwen2.5-0.5B、TinyLlama
- **批次大小：** 4-8
- **训练时间：** 1K 样本 1-2 小时

**小型模型（1-3B 参数）**
- **推荐：** `t4-medium` 或 `a10g-small`
- **示例：** Qwen2.5-1.5B、Phi-2
- **批次大小：** 2-4
- **训练时间：** 10K 样本 2-4 小时

**中型模型（3-7B 参数）**
- **推荐：** `a10g-small` 或 `a10g-large`
- **示例：** Qwen2.5-7B、Mistral-7B
- **批次大小：** 1-2（或 LoRA 4-8）
- **训练时间：** 10K 样本 4-8 小时

**大型模型（7-13B 参数）**
- **推荐：** `a10g-large` 或 `a100-large`
- **示例：** Llama-3-8B、Mixtral-8x7B（使用 LoRA）
- **批次大小：** 1（全量微调）或 2-4（LoRA）
- **训练时间：** 10K 样本 6-12 小时
- **注意：**始终使用 LoRA/PEFT

**超大型模型（13B+ 参数）**
- **推荐：** `a100-large` 配合 LoRA
- **示例：** Llama-3-13B、Llama-3-70B（仅 LoRA）
- **批次大小：** LoRA 下 1-2
- **训练时间：** 10K 样本 8-24 小时
- **注意：**全量微调不可行，使用 LoRA/PEFT

### 按预算

**最低预算（<$5 总计）**
- 使用 `t4-small`
- 在数据子集上训练（100-500 条样本）
- 限制 1-2 个 epoch
- 使用小模型（<1B）

**小预算（$5-20）**
- 使用 `t4-medium` 或 `a10g-small`
- 训练 1K-5K 条样本
- 2-3 个 epoch
- 模型最大 3B 参数

**中等预算（$20-50）**
- 使用 `a10g-small` 或 `a10g-large`
- 训练 5K-20K 条样本
- 3-5 个 epoch
- 模型最大 7B 参数

**大预算（$50-200）**
- 使用 `a10g-large` 或 `a100-large`
- 全量数据集训练
- 多个 epoch
- 模型最大 13B 参数（LoRA）

### 按训练类型

**快速演示/实验**
- `t4-small`
- 50-100 条样本
- 5-10 步
- 约 10-15 分钟

**开发/迭代**
- `t4-medium` 或 `a10g-small`
- 1K 条样本
- 1 个 epoch
- 约 30-60 分钟

**生产训练**
- `a10g-large` 或 `a100-large`
- 完整数据集
- 3-5 个 epoch
- 4-12 小时

**研究/实验**
- `a100-large`
- 多次运行
- 多种超参数
- 预算 20-50 小时

## 内存考虑

### 估算内存需求

**全量微调：**
```
内存 (GB) ≈ (模型参数以十亿计) × 20
```

**LoRA 微调：**
```
内存 (GB) ≈ (模型参数以十亿计) × 4
```

**示例：**
- Qwen2.5-0.5B 全量：~10GB ✅ 适合 t4-small
- Qwen2.5-1.5B 全量：~30GB ❌ 超过大多数 GPU
- Qwen2.5-1.5B LoRA：~6GB ✅ 适合 t4-small
- Qwen2.5-7B 全量：~140GB ❌ 不可行
- Qwen2.5-7B LoRA：~28GB ✅ 适合 a10g-large

### 内存优化

如果遇到内存限制：

1. **使用 LoRA/PEFT**
   ```python
   peft_config=LoraConfig(r=16, lora_alpha=32)
   ```

2. **减小批次大小**
   ```python
   per_device_train_batch_size=1
   ```

3. **增加梯度累积**
   ```python
   gradient_accumulation_steps=8  # 有效批次大小 = 1×8
   ```

4. **启用梯度检查点**
   ```python
   gradient_checkpointing=True
   ```

5. **使用混合精度**
   ```python
   bf16=True  # 或 fp16=True
   ```

6. **升级到更大的 GPU**
   - t4 → a10g → a100

## 费用估算

### 公式

```
总费用 = (训练小时数) × (每小时费用)
```

### 计算示例

**快速演示：**
- 硬件：t4-small（$0.75/小时）
- 时间：15 分钟（0.25 小时）
- 费用：$0.19

**开发训练：**
- 硬件：a10g-small（$3.50/小时）
- 时间：2 小时
- 费用：$7.00

**生产训练：**
- 硬件：a10g-large（$5/小时）
- 时间：6 小时
- 费用：$30.00

**大模型 + LoRA：**
- 硬件：a100-large（$10/小时）
- 时间：8 小时
- 费用：$80.00

### 费用优化技巧

1. **从小开始：** 在 t4-small 上用子集测试
2. **使用 LoRA：** 比全量微调便宜 4-5 倍
3. **优化超参数：** 如可能减少 epoch
4. **设置合适的超时：** 不要在卡住的任务上浪费算力
5. **使用检查点：** 任务失败时恢复
6. **监控费用：** 定期检查运行中的任务

## 多 GPU 训练

使用多 GPU flavor 时，TRL 通过 Accelerate 自动处理多 GPU 训练。

**多 GPU flavor：**
- `l4x4` - 4x L4 GPU
- `a10g-largex2` - 2x A10G GPU
- `a10g-largex4` - 4x A10G GPU

**使用场景：**
- 模型 >13B 参数
- 需要更快训练（线性加速）
- 大型数据集（>50K 条样本）

**示例：**
```python
hf_jobs("uv", {
    "script": "train.py",
    "flavor": "a10g-largex2",  # 2 GPU
    "timeout": "4h",
    "secrets": {"HF_TOKEN": "$HF_TOKEN"}
})
```

无需修改代码——TRL/Accelerate 自动处理分布式训练。

## 选项间选择

### a10g vs a100

**选择 a10g 的场景：**
- 模型 <13B 参数
- 预算敏感
- 训练时间不关键

**选择 a100 的场景：**
- 模型 13B+ 参数
- 需要最快训练
- 内存需求高
- 预算允许

### 单 GPU vs 多 GPU

**选择单 GPU 的场景：**
- 模型 <7B 参数
- 预算有限
- 调试更简单

**选择多 GPU 的场景：**
- 模型 >13B 参数
- 需要更快训练
- 需要大批次大小
- 大任务具有成本效益
